# 5.2 智能指针详解

> **目标**：掌握UE中非UObject对象的内存管理——TSharedPtr、TSharedRef、TUniquePtr、TWeakPtr的使用场景、原理和常见陷阱。

---

## 为什么需要智能指针？

在上一节我们学习了UE的GC，它专门管理UObject及其子类的生命周期。但UE项目中还有大量**非UObject的纯C++对象**需要管理：

```cpp
// ===== 这些对象不受GC管理！=====
// UE的GC只管理UObject体系的对象
// 下面的对象不会自动回收：

// 1. 纯C++数据结构
struct FPlayerStats  // F开头 = 纯C++结构体，非UObject
{
    float Health;
    float Mana;
    int32 Level;
    TArray<FString> Inventory;  // TArray本身也会动态分配内存
};

// 2. Slate UI对象
TSharedPtr<SWidget> MyWidget;  // Slate UI不是UObject

// 3. 网络/IO数据包
class FNetworkPacket { /* ... */ };  // 不是UObject

// 4. 临时计算数据
struct FPathfindingNode { /* ... */ };

// 5. 第三方库的对象
class ThirdPartyObject { /* ... */ };

// 这些对象如果用 new/delete 手动管理，又回到了C++老问题：
// 内存泄漏、重复删除、野指针...
```

> **关键记忆点**：**UObject的用GC（UPROPERTY保护），非UObject的用智能指针（TSharedPtr等）。** 这是UE内存管理的两条腿，缺一不可。

---

## 智能指针家族总览

UE提供了四种主要的智能指针：

| 智能指针        | 所有权     | 可为空 | 典型用途                 |
| --------------- | ---------- | ------ | ------------------------ |
| `TSharedPtr<T>` | 共享所有权 | ✅     | 最常用，多个地方共同持有 |
| `TSharedRef<T>` | 共享所有权 | ❌     | 保证非空，安全参数传递   |
| `TUniquePtr<T>` | 独占所有权 | ✅     | 单一所有者，所有权可转移 |
| `TWeakPtr<T>`   | 无所有权   | ✅     | 打破循环引用，观察对象   |

```cpp
// 所有智能指针的头文件：
#include "Templates/SharedPointer.h"  // 通常已通过CoreMinimal.h间接包含

// UE智能指针命名约定：
// T前缀 = Template（模板类）
// Shared/Unique/Weak = 所有权语义
// Ptr = 可以为空，Ref = 不能为空
```

## TSharedPtr —— 共享所有权的智能指针（最常用）

`TSharedPtr` 使用**引用计数**来管理对象的生命周期。每当你复制一个TSharedPtr，引用计数+1；每当一个TSharedPtr被销毁或重置，引用计数-1；引用计数归零时，对象被自动删除。

### 引用计数原理（图解）

```
创建：MakeShared<FMyData>()
┌──────────────┐
│  FMyData对象  │  ← 真实数据对象
├──────────────┤
│  引用计数 = 1 │  ← 内部计数器（RefCount）
└──────────────┘
       ↑
  ┌─────────┐
  │ Ptr1     │  持有者1
  └─────────┘

复制：Ptr2 = Ptr1
┌──────────────┐
│  FMyData对象  │
├──────────────┤
│  引用计数 = 2 │  ← 两个持有者
└──────────────┘
       ↑            ↑
  ┌─────────┐  ┌─────────┐
  │ Ptr1     │  │ Ptr2     │
  └─────────┘  └─────────┘

Ptr1.Reset()：
┌──────────────┐
│  FMyData对象  │
├──────────────┤
│  引用计数 = 1 │  ← 只剩下Ptr2持有
└──────────────┘
                    ↑
               ┌─────────┐
               │ Ptr2     │
               └─────────┘

Ptr2.Reset()：
引用计数 = 0 → 对象被自动删除！
FMyData的内存被释放
```

### 基础用法

```cpp
// ===== 头文件 =====
#include "Templates/SharedPointer.h"  // 通常CoreMinimal.h已包含

// ===== 创建方式1：MakeShared（推荐）=====
// 直接分配内存并构造对象
TSharedPtr<FPlayerStats> StatsPtr = MakeShared<FPlayerStats>();
// MakeShared 一次性分配引用计数块和数据块
// 效率高，推荐使用

// ===== 创建方式2：MakeShareable（从现有指针包装）=====
FPlayerStats* RawPtr = new FPlayerStats();  // 已经有一个原始指针
RawPtr->Health = 100.0f;
RawPtr->Mana = 50.0f;

TSharedPtr<FPlayerStats> StatsPtr2 = MakeShareable(RawPtr);
// MakeShareable 接管一个裸指针
// ⚠️ 注意：使用后不要再手动delete RawPtr！

// ===== 创建方式3：从另一个TSharedPtr复制 =====
TSharedPtr<FPlayerStats> StatsPtr3 = StatsPtr;  // 引用计数+1
// StatsPtr 和 StatsPtr3 指向同一个对象
// 各自销毁时引用计数-1，最后一个销毁的负责释放内存

// ===== 使用方式1：像正常指针一样操作 =====
if (StatsPtr.IsValid())  // 检查指针是否有效（不为空）
{
    StatsPtr->Health = 80.0f;     // -> 访问成员
    StatsPtr->Mana = 60.0f;
}

// ===== 使用方式2：解引用获取原始引用 =====
if (StatsPtr.IsValid())
{
    FPlayerStats& StatsRef = *StatsPtr;  // * 解引用
    StatsRef.Health = 80.0f;
}

// ===== 使用方式3：Get()获取原始指针 =====
FPlayerStats* RawStats = StatsPtr.Get();  // 获取原始指针
// ⚠️ 注意：不要delete这个原始指针！所有权仍在StatsPtr
// RawStats只在StatsPtr存活期间有效
```

### 每个成员函数详解

```cpp
TSharedPtr<FPlayerStats> Ptr = MakeShared<FPlayerStats>();

// ===== IsValid()：检查是否持有有效对象 =====
// TSharedPtr的IsValid() 不同于 UObject的IsValid()
// TSharedPtr的IsValid() 只检查指针是否为null
// 因为智能指针管理的不是UObject，没有PendingKill概念
if (Ptr.IsValid())
{
    // 安全使用
}

// 也可以用 bool 转换：
if (Ptr)  // 等同于 Ptr.IsValid()
{
    // ...
}

// ===== Reset()：释放对这个对象的持有 =====
Ptr.Reset();  // 引用计数-1，Ptr变为空
// 如果这是最后一个持有者，对象会被自动删除

// ===== Get()：获取原始指针（不影响引用计数）=====
FPlayerStats* RawPtr = Ptr.Get();
// ⚠️ 不要delete RawPtr！
// ⚠️ Ptr销毁后RawPtr变成野指针！

// ===== 获取引用计数 =====
int32 RefCount = Ptr.GetSharedReferenceCount();
// 注意：返回的是共享引用计数对象中的计数
// 仅在调试时使用，不依赖它做业务逻辑
UE_LOG(LogTemp, Warning, TEXT("当前引用计数: %d"), RefCount);

// ===== 检查是否唯一持有者 =====
if (Ptr.IsUnique())
{
    // 引用计数 = 1，只有我一个人持有
    // 可以安全地修改对象内容
}

// ===== 比较操作 =====
TSharedPtr<FPlayerStats> Ptr2 = Ptr;  // 指向同一对象
TSharedPtr<FPlayerStats> Ptr3 = MakeShared<FPlayerStats>();  // 不同对象

bool bSame = (Ptr == Ptr2);   // true → 指向同一个对象
bool bDiff = (Ptr != Ptr3);   // true → 指向不同对象
```

### 完整示例：游戏玩家数据管理

```cpp
// PlayerData.h
#pragma once

#include "CoreMinimal.h"

// 玩家数据：纯C++结构体，不用UObject
// 因为这是纯数据，不需要编辑器可见、不需要蓝图访问、不需要网络同步
struct FPlayerData
{
    // 基础信息
    FString PlayerName;       // 玩家名字
    int32 Level;              // 等级
    float Experience;         // 经验值

    // 属性
    float MaxHealth;          // 最大生命值
    float CurrentHealth;      // 当前生命值
    float MaxMana;            // 最大法力值
    float CurrentMana;        // 当前法力值

    // 构造函数
    FPlayerData()
        : PlayerName(TEXT("Unknown"))
        , Level(1)
        , Experience(0.0f)
        , MaxHealth(100.0f)
        , CurrentHealth(100.0f)
        , MaxMana(50.0f)
        , CurrentMana(50.0f)
    {
    }

    // 打印数据（调试用）
    void PrintDebugInfo() const
    {
        UE_LOG(LogTemp, Warning, TEXT("=== 玩家数据 ==="));
        UE_LOG(LogTemp, Warning, TEXT("名称: %s"), *PlayerName);
        UE_LOG(LogTemp, Warning, TEXT("等级: %d"), Level);
        UE_LOG(LogTemp, Warning, TEXT("HP: %.0f / %.0f"), CurrentHealth, MaxHealth);
        UE_LOG(LogTemp, Warning, TEXT("MP: %.0f / %.0f"), CurrentMana, MaxMana);
    }
};

// PlayerManager.h
#pragma once

#include "CoreMinimal.h"
#include "PlayerData.h"

// 玩家管理器：使用TSharedPtr管理PlayerData
class FPlayerManager
{
private:
    // 使用TSharedPtr存储玩家数据
    // 多个地方可能需要共享同一份数据
    TSharedPtr<FPlayerData> CurrentPlayerData;

public:
    FPlayerManager()
    {
        // 初始化为空
        // CurrentPlayerData = nullptr;  // 默认就是nullptr
    }

    // 创建新玩家数据
    void CreateNewPlayer(const FString& PlayerName)
    {
        // MakeShared: 在堆上创建对象并返回TSharedPtr
        CurrentPlayerData = MakeShared<FPlayerData>();

        // 设置初始值
        CurrentPlayerData->PlayerName = PlayerName;
        CurrentPlayerData->Level = 1;
        CurrentPlayerData->MaxHealth = 100.0f;
        CurrentPlayerData->CurrentHealth = 100.0f;

        UE_LOG(LogTemp, Warning, TEXT("创建新玩家: %s"), *PlayerName);
        UE_LOG(LogTemp, Warning, TEXT("引用计数: %d"),
               CurrentPlayerData.GetSharedReferenceCount());
        // 输出：引用计数: 1（只有CurrentPlayerData持有）
    }

    // 获取玩家数据的共享副本
    TSharedPtr<FPlayerData> GetPlayerData() const
    {
        return CurrentPlayerData;  // 返回拷贝，引用计数+1
        // 调用者获得一个TSharedPtr，引用计数变成2
        // 当调用者的TSharedPtr销毁时，引用计数变回1
    }

    // 保存到存档（演示TSharedPtr的传递）
    void SavePlayerData(const FString& SaveSlotName)
    {
        if (!CurrentPlayerData.IsValid())
        {
            UE_LOG(LogTemp, Error, TEXT("没有玩家数据可以保存！"));
            return;
        }

        // 演示：引用计数
        UE_LOG(LogTemp, Warning, TEXT("保存前引用计数: %d"),
               CurrentPlayerData.GetSharedReferenceCount());

        // 这里假设有一个外部保存函数
        SaveToDisk(SaveSlotName, CurrentPlayerData);

        UE_LOG(LogTemp, Warning, TEXT("保存后引用计数: %d"),
               CurrentPlayerData.GetSharedReferenceCount());
        // 如果SaveToDisk接收的是TSharedPtr（值拷贝），保存后引用计数跟保存前一样
    }

    // 重置玩家数据
    void ClearPlayerData()
    {
        if (CurrentPlayerData.IsValid())
        {
            FString OldName = CurrentPlayerData->PlayerName;

            // Reset 释放当前持有的引用
            // 如果引用计数变为0，对象被自动删除
            CurrentPlayerData.Reset();

            UE_LOG(LogTemp, Warning, TEXT("已清除玩家数据: %s"), *OldName);
        }
    }

private:
    void SaveToDisk(const FString& SlotName, TSharedPtr<FPlayerData> Data)
    {
        // Data是一个TSharedPtr的拷贝，引用计数+1
        // 函数结束时Data销毁，引用计数恢复
        UE_LOG(LogTemp, Warning, TEXT("保存到 %s..."), *SlotName);
        if (Data.IsValid())
        {
            Data->PrintDebugInfo();
        }
    }
};
```

## TSharedRef —— 不可为空的共享引用

`TSharedRef` 和 `TSharedPtr` 的核心区别：**TSharedRef 保证不为空**。

### TSharedPtr vs TSharedRef

```cpp
// ===== TSharedPtr：可以为空 =====
TSharedPtr<FPlayerData> NullablePtr;        // ✅ 可以持有空值
NullablePtr = nullptr;                      // ✅ 可以设为空
if (NullablePtr.IsValid()) { /* ... */ }    // ✅ 使用前检查

// ===== TSharedRef：不能为空 =====
// TSharedRef<FPlayerData> NullableRef;    // ❌ 编译错误！必须初始化
TSharedRef<FPlayerData> Ref = MakeShared<FPlayerData>();  // ✅ 必须初始化
// Ref = nullptr;                          // ❌ 编译错误！不能设为空
// 不需要检查IsValid()——它保证有效

// ===== 互相转换 =====
// TSharedPtr → TSharedRef（不安全，需要确认非空）
TSharedPtr<FPlayerData> Ptr = MakeShared<FPlayerData>();
TSharedRef<FPlayerData> Ref2 = Ptr.ToSharedRef();  // 如果Ptr为空 → 断言失败/崩溃！

// TSharedRef → TSharedPtr（安全，总是成功）
TSharedPtr<FPlayerData> Ptr2 = Ref;  // 隐式转换，总是成功
```

### 使用场景

```cpp
// ===== TSharedRef 作为函数参数：强制调用者提供有效对象 =====
class FDataProcessor
{
public:
    // ❌ 用TSharedPtr：调用者可能传空指针
    void ProcessData_Bad(TSharedPtr<FPlayerData> Data)
    {
        // 必须先检查
        if (!Data.IsValid())
        {
            return;  // 增加了一层检查逻辑
        }
        // ... 处理 ...
    }

    // ✅ 用TSharedRef：调用者必须提供有效数据
    // 语义清晰："这个函数需要数据，不能为空"
    void ProcessData_Good(const TSharedRef<FPlayerData>& Data)
    {
        // 不需要检查IsValid()——TSharedRef保证非空
        // 直接用！安全、简洁
        Data->PrintDebugInfo();
    }

    // ✅ 用 const TSharedRef<T>& 避免不必要的引用计数操作
    // & = 引用传递，不增加引用计数
    // const = 保证函数内不会修改这个TSharedRef本身
};

// ===== 使用对比 =====
void Example()
{
    FDataProcessor Processor;

    // TSharedPtr版本（容易出错）
    TSharedPtr<FPlayerData> NullData;  // 空的
    Processor.ProcessData_Bad(NullData);  // 编译通过，运行时静默跳过 → 可能引入Bug

    // TSharedRef版本（编译时就阻止错误）
    // Processor.ProcessData_Good(NullData);  // ❌ 编译错误！不能用nullptr构造TSharedRef

    TSharedPtr<FPlayerData> ValidData = MakeShared<FPlayerData>();
    Processor.ProcessData_Good(ValidData.ToSharedRef());  // ✅ 必须显式转换，表明你确认非空
}

// ===== TSharedRef 作为返回值：保证调用者能直接用 =====
class FPlayerFactory
{
public:
    // 返回值用TSharedRef："我保证返回有效对象"
    TSharedRef<FPlayerData> CreateDefaultPlayer()
    {
        TSharedRef<FPlayerData> NewPlayer = MakeShared<FPlayerData>();
        NewPlayer->PlayerName = TEXT("DefaultPlayer");
        NewPlayer->MaxHealth = 100.0f;
        return NewPlayer;  // 总是返回有效对象
    }

    // 如果用TSharedPtr做返回值 → 可能返回空 → 调用者每次都要检查
};
```

## TUniquePtr —— 独占所有权的智能指针

`TUniquePtr` 表示**独占所有权**——同一时间只能有一个指针持有对象。这防止了多个地方的意外共享。

### TUniquePtr vs TSharedPtr

```cpp
// ===== TSharedPtr：共享所有权 =====
TSharedPtr<FPlayerData> Shared1 = MakeShared<FPlayerData>();
TSharedPtr<FPlayerData> Shared2 = Shared1;   // ✅ 可以复制，引用计数变成2
// Shared1 和 Shared2 都能访问和修改数据
// 问题：不知道谁会修改数据，增加协调成本

// ===== TUniquePtr：独占所有权 =====
TUniquePtr<FPlayerData> Unique1 = MakeUnique<FPlayerData>();
// TUniquePtr<FPlayerData> Unique2 = Unique1;  // ❌ 编译错误！不能复制！
// 只有一个所有者，所有权清晰

// 所有权转移：用MoveTemp/ MoveTemp允许所有权从一个转移到另一个）
TUniquePtr<FPlayerData> Unique2 = MoveTemp(Unique1);
// 现在 Unique2 拥有对象，Unique1 变为 nullptr
// 所有权转移了，不是复制

// 验证
check(Unique1 == nullptr);       // ✅ Unique1已空
check(Unique2.IsValid());        // ✅ Unique2拥有对象
```

### 完整示例

```cpp
// ===== 创建 =====
// MakeUnique：创建对象并返回TUniquePtr
TUniquePtr<FPlayerData> Data = MakeUnique<FPlayerData>();
Data->PlayerName = TEXT("Player1");
Data->MaxHealth = 100.0f;

// ===== 使用（和TSharedPtr一样）=====
if (Data.IsValid())
{
    Data->CurrentHealth = 80.0f;
    UE_LOG(LogTemp, Warning, TEXT("玩家: %s"), *Data->PlayerName);
}

// ===== 获取原始指针 =====
FPlayerData* RawPtr = Data.Get();  // ⚠️ 不要delete！
// 在Data生命周期内RawPtr有效
// Data销毁或转移所有权后RawPtr变成野指针

// ===== 释放所有权 =====
// 方法1：Reset
Data.Reset();  // 立即销毁对象，Data变为空

// 方法2：Release（放弃所有权但不销毁——很少用！）
// FPlayerData* LeakedPtr = Data.Release();  // Data设为空，对象由你负责delete
// 正常不需要Release，用Reset就好

// ===== 独占所有权的价值：明确所有权 =====
class FInventoryManager
{
private:
    // 这个玩家背包数据 "属于" 这个管理器
    // 其他人不能共享，只能获取只读引用
    TUniquePtr<FPlayerData> PlayerInventory;

public:
    FInventoryManager()
    {
        PlayerInventory = MakeUnique<FPlayerData>();
    }

    // 提供只读访问（返回const引用，不能修改）
    const FPlayerData* GetInventoryReadOnly() const
    {
        return PlayerInventory.Get();
    }

    // 唯一可以修改数据的途径
    void AddItem(const FString& ItemName, int32 Count)
    {
        // ... 修改 PlayerInventory
    }
};
```

## TWeakPtr —— 弱引用（打破循环引用）

### 循环引用问题

```cpp
// ===== 循环引用场景（内存泄漏！）=====

// 场景：一个任务系统
struct FTaskNode
{
    FString TaskName;                // 任务名称
    TSharedPtr<FTaskNode> ChildTask; // 子任务
    TSharedPtr<FTaskNode> ParentTask; // ❌ 父任务！

    // 这会导致循环引用！分析如下：
    //
    // Parent → TSharedPtr → Child
    //   ↑                      │
    //   └────── TSharedPtr ────┘
    //
    // Parent的引用计数：1（Child持有Parent）
    // Child的引用计数：1（Parent持有Child）
    // 当外部释放Parent时：
    //   外部释放 → Parent引用计数 1-1 = 0? NO!
    //   Child还持有Parent → Parent引用计数还是1
    //   但外部已经无法访问Parent了！
    //   Parent和Child互相持有，引用计数永远不为0
    //   → 内存泄漏！
};

// ===== 解决方案：将其中一个改用TWeakPtr =====
struct FTaskNodeFixed
{
    FString TaskName;                   // 任务名称
    TSharedPtr<FTaskNodeFixed> ChildTask; // 子任务（强引用：拥有子任务）
    TWeakPtr<FTaskNodeFixed> ParentTask;  // ✅ 父任务用弱引用！

    // 现在引用关系：
    //
    // Parent → TSharedPtr → Child
    //   ↑                      │
    //   └────── TWeakPtr ──────┘（弱引用不增加引用计数）
    //
    // 当外部释放Parent时：
    //   Parent引用计数：1（只有外部持有）
    //   外部释放 → Parent引用计数 = 0 → Parent被销毁
    //   Parent销毁 → Child引用计数 -1 → 如果也是0 → Child被销毁
    //   → 正常释放！没有内存泄漏！
};
```

### TWeakPtr 使用方式

```cpp
// ===== 创建：从TSharedPtr产生 =====
TSharedPtr<FPlayerData> StrongPtr = MakeShared<FPlayerData>();
StrongPtr->PlayerName = TEXT("Player1");

TWeakPtr<FPlayerData> WeakPtr = StrongPtr;  // 从强引用创建弱引用
// 弱引用不增加引用计数，不影响对象生命周期

// ===== 使用：必须先 Pin() 获取临时强引用 =====
// ❌ 错误：不能用 -> 直接访问弱引用
// WeakPtr->PlayerName = TEXT("xxx");  // 编译错误！

// ✅ 正确：Pin() 返回一个临时的 TSharedPtr
TSharedPtr<FPlayerData> Pinned = WeakPtr.Pin();
if (Pinned.IsValid())  // 检查对象是否还活着
{
    Pinned->PlayerName = TEXT("Updated");  // ✅ 安全访问
}
// Pin() 返回的TSharedPtr销毁时，引用计数恢复

// 也可以用这种方式：
if (TSharedPtr<FPlayerData> Temp = WeakPtr.Pin())
{
    Temp->PrintDebugInfo();
}

// ===== 检查有效性 =====
if (WeakPtr.IsValid())  // 检查引用的对象还活着吗
{
    // 对象还活着
}
else
{
    UE_LOG(LogTemp, Warning, TEXT("对象已被销毁"));
}

// ===== 重置 =====
WeakPtr.Reset();  // 清空弱引用
```

### 循环引用完整演示

```cpp
// ===== 演示循环引用问题 =====
void Demonstrate_CircularReference()
{
    UE_LOG(LogTemp, Warning, TEXT("===== 循环引用演示 ====="));

    // 创建两个任务节点
    TSharedPtr<FTaskNode> ParentTask = MakeShared<FTaskNode>();
    ParentTask->TaskName = TEXT("父任务");

    TSharedPtr<FTaskNode> ChildTask = MakeShared<FTaskNode>();
    ChildTask->TaskName = TEXT("子任务");

    // 建立关系（循环引用！）
    ParentTask->ChildTask = ChildTask;   // Parent 持有 Child
    // 引用计数：Parent=1(外部), Child=2(外部+Parent)

    ChildTask->ParentTask = ParentTask;  // Child 持有 Parent（用TSharedPtr时的问题）
    // 引用计数：Parent=2(外部+Child), Child=2(外部+Parent)

    // 释放外部引用
    ParentTask.Reset();  // Parent引用计数: 2 → 1（Child还持有）
    ChildTask.Reset();   // Child引用计数: 2 → 1（Parent还持有）

    // 结果：
    // Parent引用计数 = 1（Child持有Parent）
    // Child引用计数 = 1（Parent持有Child）
    // 两者永不归零 → 内存泄漏！
    // 外部已经无法访问它们了，但内存就是无法释放

    UE_LOG(LogTemp, Error, TEXT("循环引用导致内存泄漏！Parent和Child无法释放"));
}

// ===== 演示TWeakPtr解决方案 =====
void Demonstrate_WeakPtrFix()
{
    UE_LOG(LogTemp, Warning, TEXT("===== TWeakPtr解决方案 ====="));

    // 创建两个修正版任务节点
    TSharedPtr<FTaskNodeFixed> ParentTask = MakeShared<FTaskNodeFixed>();
    ParentTask->TaskName = TEXT("父任务");

    TSharedPtr<FTaskNodeFixed> ChildTask = MakeShared<FTaskNodeFixed>();
    ChildTask->TaskName = TEXT("子任务");

    // 建立关系（无循环引用！）
    ParentTask->ChildTask = ChildTask;   // Parent用TSharedPtr持有Child
    // 引用计数：Parent=1, Child=2

    ChildTask->ParentTask = ParentTask;  // Child用TWeakPtr引用Parent（不增加引用计数）
    // 引用计数：Parent=1, Child=2  ← 没有增加！

    UE_LOG(LogTemp, Warning, TEXT("Parent引用计数: %d"),
           ParentTask.GetSharedReferenceCount());  // 1
    UE_LOG(LogTemp, Warning, TEXT("Child引用计数: %d"),
           ChildTask.GetSharedReferenceCount());   // 2

    // 释放外部引用
    ParentTask.Reset();  // Parent引用计数: 1 → 0 → Parent被销毁
    // Parent销毁 → Child引用计数: 2 → 1

    // 检查：Parent被销毁后，Child的弱引用还能用吗？
    if (TSharedPtr<FTaskNodeFixed> Temp = ChildTask->ParentTask.Pin())
    {
        UE_LOG(LogTemp, Warning, TEXT("Parent还在"));  // 不会执行
    }
    else
    {
        UE_LOG(LogTemp, Warning, TEXT("Parent已被销毁，弱引用返回空"));
    }

    ChildTask.Reset();  // Child引用计数: 1 → 0 → Child被销毁

    UE_LOG(LogTemp, Warning, TEXT("✅ 没有内存泄漏！两个对象都被正确释放"));
}
```

## 智能指针 vs UObject原始指针：什么时候用哪个

```
你的对象是什么类型？
├── UObject 或其子类
│   ├── 你需要"持有"它（不想被GC回收）
│   │   └── 用 UPROPERTY() + 原始指针
│   │       UMyObject* Ptr;  ← UPROPERTY保护
│   │
│   ├── 你只是"观察"它（允许被GC回收）
│   │   └── 用 TWeakObjectPtr<UMyObject>
│   │       对象被GC回收后，IsValid()返回false
│   │
│   └── 临时局部变量
│       └── 直接用原始指针
│           UMyObject* Temp = GetSomething();
│           // 函数返回前GC不会回收
│
└── 纯C++对象（非UObject）
    ├── 多个地方需要共享所有权
    │   ├── TSharedPtr：可为空
    │   └── TSharedRef：保证非空（推荐做参数/返回值）
    │
    ├── 只有一个所有者
    │   └── TUniquePtr：独占所有权
    │
    ├── 打破循环引用
    │   └── TWeakPtr：从TSharedPtr产生
    │
    └── 栈上临时使用
        └── 直接值类型
            FMyStruct Data;  // 不用指针
```

### 决策速查表

| 场景                        | 用什么                  | 示例                                        |
| --------------------------- | ----------------------- | ------------------------------------------- |
| 持有UObject引用（成员变量） | `UPROPERTY() T*`        | `UPROPERTY() AActor* Owner;`                |
| 观察UObject（不阻止GC）     | `TWeakObjectPtr<T>`     | `TWeakObjectPtr<AActor> Target;`            |
| 多个地方共享纯C++数据       | `TSharedPtr<T>`         | `TSharedPtr<FGameConfig> Config;`           |
| 函数参数（保证非空）        | `const TSharedRef<T>&`  | `void Process(const TSharedRef<FData>& D);` |
| 单例/独占所有权             | `TUniquePtr<T>`         | `TUniquePtr<FWorldData> World;`             |
| 打破循环引用                | `TWeakPtr<T>`           | `TWeakPtr<FTaskNode> Parent;`               |
| 临时局部变量                | `T*` 或 `TSharedPtr<T>` | `auto Ptr = Config.Get();`                  |

## 智能指针最佳实践

```cpp
// ===== ✅ 正确做法 =====

// 1. 优先使用 MakeShared / MakeUnique 创建
TSharedPtr<FData> Data = MakeShared<FData>();      // ✅ 一次分配，效率高
TUniquePtr<FData> Unique = MakeUnique<FData>();     // ✅ 同Subobject

// 2. 函数参数用 const TSharedRef<T>& 表示"我需要的不是空数据"
void ProcessData(const TSharedRef<FData>& Data);    // ✅ 语义清晰

// 3. 函数参数用 const TSharedPtr<T>& 表示"可接受空数据"
void MaybeProcessData(const TSharedPtr<FData>& Data); // ✅ 语义清晰

// 4. 返回值用TSharedRef表示"总是返回有效结果"
TSharedRef<FData> CreateDefaultData();               // ✅ 调用者无需判空

// 5. 循环引用时用TWeakPtr打破
struct FNode
{
    TSharedPtr<FNode> Child;
    TWeakPtr<FNode> Parent;  // ✅ 打破循环
};

// 6. 独占所有权用TUniquePtr
class FManager
{
    TUniquePtr<FInternalData> Data;  // ✅ 只有我能用
};

// ===== ❌ 错误做法 =====

// 1. 用new创建后直接传给TSharedPtr构造函数（不推荐）
// TSharedPtr<FData> Data(new FData());  // ❌ 两次内存分配
// 用 MakeShared 代替

// 2. 从TSharedPtr.Get()获取原始指针然后存储（野指针）
void BadCode()
{
    TSharedPtr<FData> Data = MakeShared<FData>();
    FData* Raw = Data.Get();  // 获取原始指针

    Data.Reset();  // 释放Data

    Raw->DoSomething();  // ❌ CRASH! 野指针！
}

// 3. 用TSharedPtr管理UObject
// TSharedPtr<AActor> ActorPtr;  // ❌ UObject应该用GC管理！
// 正确做法：
// UPROPERTY() AActor* ActorRef;

// 4. TSharedRef作为成员变量（不推荐）
struct FBadStruct
{
    // TSharedRef<FData> Data;  // ❌ TSharedRef必须初始化，导致无法默认构造
    // 成员变量用TSharedPtr，参数传递用TSharedRef
};

// 5. 忘记用TWeakPtr打破循环引用 → 内存泄漏（见上面示例）
```
