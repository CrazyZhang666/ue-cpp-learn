## 4.1 完整的伤害计算管道

GAS的伤害计算不是一个简单的"攻击力 - 防御力"，而是一个多阶段的管道。理解每个阶段的作用，才能写出正确的伤害系统。

### 4.1.1 伤害管道的6个阶段

```
阶段1：GA层 — 设置基础伤害
  攻击方GA设置SetByCaller传入：技能倍率、元素类型、额外标记
  ↓
阶段2：MMC层 — 计算基础伤害
  DamageCalculation::CalculateBaseMagnitude
  公式：BaseDamage = AttackPower × SkillMultiplier
  ↓
阶段3：MMC层 — 计算有效伤害
  在同一MMC中：
  防御减免 = DefensePower / (DefensePower + 100)
  EffectiveDamage = BaseDamage × (1 - 防御减免)
  ↓
阶段4：MMC层 — 暴击判定
  随机判定暴击 → EffectiveDamage × CritMultiplier
  ↓
阶段5：GE层 — 属性修改器应用
  GE的Modifier使用MMC结果作为Magnitude
  写入Target的 IncomingDamage 临时属性
  ↓
阶段6：AttributeSet层 — PostGameplayEffectExecute
  从IncomingDamage读取最终伤害值
  应用护甲穿透、元素抗性、格挡判定
  从Health中扣除最终伤害
  触发死亡判定
  ↓
结果：防御方的Health属性被正确减少
```

```cpp
// ====================================================================
// 元素伤害逻辑：在CalculateBaseMagnitude中，防御减免之后调用
// ====================================================================

float UDamageCalculation::CalculateElementalReduction(
    const FGameplayEffectSpec& Spec,
    const FAggregatorEvaluateParameters& TargetAttributes) const
{
    // ================================================================
    // 第一步：检查伤害类型（通过GE的动态标签判断）
    // ================================================================

    // 获取GE Spec上的动态标签
    const FGameplayTagContainer& DynamicTags = Spec.DynamicGrantedTags;

    // 判断伤害对应的元素类型
    FGameplayTag ElementalTag;
    bool bIsElemental = false;

    if (DynamicTags.HasTag(FGameplayTag::RequestGameplayTag("Combat.Damage.Fire")))
    {
        ElementalTag = FGameplayTag::RequestGameplayTag("Combat.Damage.Fire");
        bIsElemental = true;
    }
    else if (DynamicTags.HasTag(FGameplayTag::RequestGameplayTag("Combat.Damage.Ice")))
    {
        ElementalTag = FGameplayTag::RequestGameplayTag("Combat.Damage.Ice");
        bIsElemental = true;
    }
    else if (DynamicTags.HasTag(FGameplayTag::RequestGameplayTag("Combat.Damage.Lightning")))
    {
        ElementalTag = FGameplayTag::RequestGameplayTag("Combat.Damage.Lightning");
        bIsElemental = true;
    }

    if (!bIsElemental)
    {
        return 0.0f;  // 物理伤害，无元素减免
    }

    // ================================================================
    // 第二步：获取对应的元素抗性值
    // ================================================================

    float ResistValue = 0.0f;

    if (DynamicTags.HasTag(FGameplayTag::RequestGameplayTag("Combat.Damage.Fire")))
    {
        if (!FireResistCapture.AttemptCalculateCapturedAttributeMagnitude(
                Spec, TargetAttributes, ResistValue))
        {
            ResistValue = 0.0f;
        }
    }
    else if (DynamicTags.HasTag(FGameplayTag::RequestGameplayTag("Combat.Damage.Ice")))
    {
        if (!IceResistCapture.AttemptCalculateCapturedAttributeMagnitude(
                Spec, TargetAttributes, ResistValue))
        {
            ResistValue = 0.0f;
        }
    }
    else
    {
        if (!LightningResistCapture.AttemptCalculateCapturedAttributeMagnitude(
                Spec, TargetAttributes, ResistValue))
        {
            ResistValue = 0.0f;
        }
    }

    // ================================================================
    // 第三步：计算元素抗性减免
    // ================================================================
    // 抗性值范围：-1.0（弱点，200%伤害）到 1.0（免疫，0%伤害）
    // ResistValue = 0.3 → 减免30%元素伤害
    // ResistValue = -0.3 → 增加30%元素伤害（弱点）

    float ElementalMultiplier = 1.0f - ResistValue;

    // 限制范围：伤害不能低于10%（防止完全免疫使游戏过于简单）
    // 如果你需要完全免疫，可以在GE的动态标签中添加免疫标签
    ElementalMultiplier = FMath::Clamp(ElementalMultiplier, 0.1f, 2.0f);

    return ElementalMultiplier;
}
```

### 4.1.3 PostGameplayEffectExecute 的完整伤害处理

更新 `RPGAttributeSet.cpp` 中的 `PostGameplayEffectExecute`，添加更多处理阶段：

```cpp
// ====================================================================
// 更新后的 PostGameplayEffectExecute — 完整伤害管道阶段6
// ====================================================================

void URPGAttributeSet::PostGameplayEffectExecute(const FGameplayEffectModCallbackData& Data)
{
    Super::PostGameplayEffectExecute(Data);

    const FGameplayEffectContextHandle ContextHandle = Data.EffectSpec.GetContext();
    const FGameplayAttribute& ModifiedAttribute = Data.EvaluatedData.Attribute;

    // ================================================================
    // 处理伤害（IncomingDamage）
    // ================================================================
    if (ModifiedAttribute == GetIncomingDamageAttribute())
    {
        float LocalDamage = GetIncomingDamage();

        if (LocalDamage > 0.0f)
        {
            // --- 伤害管道阶段6A：读取GE标签判断伤害类型 ---
            // 从GE的DynamicGrantedTags中获取伤害类型信息
            const FGameplayTagContainer& SourceTags = *Data.EffectSpec.CapturedSourceTags.GetAggregatedTags();

            // --- 伤害管道阶段6B：伤害类型特殊处理 ---
            // 例如：火焰伤害可能会点燃目标（触发DOT）
            if (SourceTags.HasTag(FGameplayTag::RequestGameplayTag("Combat.Damage.Fire")))
            {
                // 火焰伤害：额外减少护甲？ 触发燃烧效果？
                // 实际游戏中通过GameplayEvent触发对应的DOT技能
                UE_LOG(LogTemp, Verbose, TEXT("RPGAttributeSet: Fire damage detected"));
            }

            // --- 伤害管道阶段6C：格挡判定 ---
            // 如果角色持有盾牌且有"格挡中"标签，进行格挡判定
            UAbilitySystemComponent* TargetASC = Data.Target.GetOwner()
                ? Cast<UAbilitySystemComponent>(Data.Target.GetOwner()->FindComponentByClass(
                    UAbilitySystemComponent::StaticClass()))
                : nullptr;

            if (TargetASC)
            {
                // 检查是否处于格挡状态
                if (TargetASC->HasMatchingGameplayTag(
                    FGameplayTag::RequestGameplayTag("State.Blocking")))
                {
                    // 读取格挡率属性（如果有的话，需要在AttributeSet中定义）
                    // 简化处理：格挡固定减免50%伤害
                    const float BlockReduction = 0.5f;
                    LocalDamage *= (1.0f - BlockReduction);

                    UE_LOG(LogTemp, Log, TEXT("RPGAttributeSet: Damage blocked! Reduced to %.1f"),
                        LocalDamage);

                    // 可选：添加格挡GameplayCue
                }
            }

            // --- 伤害管道阶段6D：护盾吸收 ---
            // 如果角色有护盾，优先扣除护盾值
            // 需要在AttributeSet中添加 Shield 属性
            // float RemainingShield = GetShield();
            // if (RemainingShield > 0.0f)
            // {
            //     float Absorbed = FMath::Min(LocalDamage, RemainingShield);
            //     LocalDamage -= Absorbed;
            //     SetShield(RemainingShield - Absorbed);
            // }

            // --- 伤害管道阶段6E：应用最终伤害 ---
            const float NewHealth = GetHealth() - LocalDamage;
            SetHealth(FMath::Clamp(NewHealth, 0.0f, GetMaxHealth()));

            // --- 伤害管道阶段6F：死亡判定 ---
            if (GetHealth() <= 0.0f)
            {
                // 获取攻击者和防御者
                AActor* TargetActor = Data.Target.GetOwner();
                AActor* SourceActor = ContextHandle.GetInstigator();

                // 这里应该通过GameplayEvent或委托通知死亡
                // 让专门的死亡处理GA来处理复活、掉落等逻辑
                UE_LOG(LogTemp, Warning,
                    TEXT("RPGAttributeSet: %s killed by %s! Damage: %.1f"),
                    *GetNameSafe(TargetActor),
                    *GetNameSafe(SourceActor),
                    LocalDamage);

                // 发送死亡事件
                // FGameplayEventData DeathEvent;
                // DeathEvent.Instigator = SourceActor;
                // UAbilitySystemBlueprintLibrary::SendGameplayEventToActor(
                //     Cast<AActor>(Data.Target.GetOwner()),
                //     FGameplayTag::RequestGameplayTag("Event.Death"),
                //     DeathEvent
                // );
            }

            // --- 伤害管道阶段6G：受击反馈 ---
            // 这里通过GameplayCue播放受击特效
            // GAS会自动处理GameplayCue的触发
        }

        // 重置临时属性
        SetIncomingDamage(0.0f);
    }

    // --- 处理治疗（与之前相同）---
    else if (ModifiedAttribute == GetIncomingHealingAttribute())
    {
        float LocalHealing = GetIncomingHealing();
        if (LocalHealing > 0.0f)
        {
            const float NewHealth = GetHealth() + LocalHealing;
            SetHealth(FMath::Clamp(NewHealth, 0.0f, GetMaxHealth()));
        }
        SetIncomingHealing(0.0f);
    }
}
```

## 4.2 状态效果系统（利用GameplayTag）

### 4.2.1 状态效果的设计哲学

在GAS中，状态效果（如眩晕、沉默）不是通过传统的`Enum`或`bool`标志实现的，而是通过**GameplayTag**实现。这带来了巨大的灵活性：

```cpp
// ❌ 传统方式（不推荐）
bool bIsStunned = false;
bool bIsSilenced = false;
bool bIsRooted = false;
// 问题：每次加新状态要加新bool，代码膨胀

// ✅ GAS方式（推荐）
// 给角色添加 GameplayTag
// State.Stunned → 眩晕中
// State.Silenced → 沉默中
// State.Rooted → 定身中
// 任何系统都可以通过检查Tag了解角色状态
```

### 4.2.2 状态效果GameplayTag定义

在项目设置中定义以下GameplayTag层级：

```yaml
# ====================================================================
# 在 Project Settings → GameplayTags 中添加
# ====================================================================

State:
  State.Dead                      # 死亡（阻止所有操作）
  State.Stunned                   # 眩晕（阻止技能和移动）
  State.Silenced                  # 沉默（阻止技能，允许移动）
  State.Rooted                    # 定身（允许技能，阻止移动）
  State.Invincible                # 无敌（免疫所有伤害）
  State.Blocking                  # 格挡中（减少正面伤害）
  State.Dodging                   # 闪避中（免疫物理伤害）
  State.Staggered                 # 硬直中（短时间的打断状态）
  State.Asleep                    # 睡眠中
  State.Frozen                    # 冰冻中（=眩晕+定身）
  State.Feared                    # 恐惧中（角色随机移动+无法使用技能）

Ability:
  Ability.State:
    Ability.State.Attacking       # 攻击中（技能激活期间）
    Ability.State.Casting         # 施法中（读条技能期间）
    Ability.State.Dashing         # 冲刺中

Combat:
  Combat.Damage:
    Combat.Damage.Physical        # 物理伤害标记
    Combat.Damage.Fire            # 火焰伤害标记
    Combat.Damage.Ice             # 冰霜伤害标记
    Combat.Damage.Lightning       # 闪电伤害标记
    Combat.Damage.True            # 真实伤害（无视防御/抗性）
  
  Combat.Result:
    Combat.Result.Critical        # 暴击标记
    Combat.Result.Blocked         # 被格挡标记
    Combat.Result.Dodged          # 被闪避标记
```

### 4.2.3 状态效果管理器

创建文件 `Source/YourProject/Public/GAS/Status/StatusEffectManager.h`：

```cpp
// ====================================================================
// 文件：Source/YourProject/Public/GAS/Status/StatusEffectManager.h
// 描述：状态效果管理器 — 集中管理角色的所有状态效果
// ====================================================================

#pragma once

#include "CoreMinimal.h"
#include "GameplayTagContainer.h"

class UAbilitySystemComponent;

/**
 * FStatusEffectManager — 状态效果管理（非UObject，作为Utility使用）
 *
 * 使用方式：
 *   FStatusEffectManager StatusMgr(ASC);
 *   StatusMgr.ApplyStun(3.0f);   // 眩晕3秒
 *   StatusMgr.RemoveStun();      // 移除眩晕
 *   bool bStunned = StatusMgr.IsStunned();
 *
 * 注意：实际的状态持续时间由GE管理（Duration GE）
 * 这个Manager类简化了通用状态操作的代码
 */
struct YOURPROJECT_API FStatusEffectManager
{
public:
    /** 构造函数 — 绑定到角色的ASC */
    explicit FStatusEffectManager(UAbilitySystemComponent* InASC);

    // ================================================================
    // 硬控（Hard CC）— 完全/部分剥夺控制权
    // ================================================================

    /** 眩晕：阻止技能 + 阻止移动，持续Duration秒 */
    void ApplyStun(float Duration);

    /** 移除眩晕 */
    void RemoveStun();

    /** 是否眩晕中 */
    bool IsStunned() const;

    /** 定身：阻止移动，允许技能 */
    void ApplyRoot(float Duration);
    void RemoveRoot();
    bool IsRooted() const;

    /** 沉默：阻止技能，允许移动 */
    void ApplySilence(float Duration);
    void RemoveSilence();
    bool IsSilenced() const;

    /** 冰冻：= 眩晕 + 定身（完全无法行动） */
    void ApplyFrozen(float Duration);

    // ================================================================
    // 软控（Soft CC）— 部分限制
    // ================================================================

    /** 减速（需要AttributeSet支持，这里只用MoveSpeedMultiplier实现） */
    void ApplySlow(float SlowPercent, float Duration);

    // ================================================================
    // 防御状态
    // ================================================================

    /** 无敌：免疫所有伤害 */
    void ApplyInvincible(float Duration);
    void RemoveInvincible();
    bool IsInvincible() const;

    /** 格挡：减少正面伤害 */
    void ApplyBlocking(float Duration);

    // ================================================================
    // 持续效果
    // ================================================================

    /** DOT（Damage Over Time）：周期性造成伤害 */
    void ApplyDOT(float DamagePerTick, float TickInterval, float Duration);

    /** HOT（Heal Over Time）：周期性治疗 */
    void ApplyHOT(float HealPerTick, float TickInterval, float Duration);

    // ================================================================
    // 查询接口
    // ================================================================

    /** 是否可以行动（非死亡、非眩晕、非冰冻） */
    bool CanAct() const;

    /** 是否可以使用技能 */
    bool CanUseAbility() const;

    /** 是否可以移动 */
    bool CanMove() const;

    /** 是否可以受到伤害 */
    bool CanTakeDamage() const;

private:
    /** 绑定的ASC */
    UAbilitySystemComponent* ASC;

    /** 添加LooseTag（临时标签，持续到手动移除） */
    void AddTag(const FGameplayTag& Tag);

    /** 移除LooseTag */
    void RemoveTag(const FGameplayTag& Tag);

    /** 是否有指定Tag */
    bool HasTag(const FGameplayTag& Tag) const;
};
```

### 4.2.4 状态效果管理器实现

创建文件 `Source/YourProject/Private/GAS/Status/StatusEffectManager.cpp`：

```cpp
// ====================================================================
// 文件：Source/YourProject/Private/GAS/Status/StatusEffectManager.cpp
// 描述：状态效果管理器实现
// ====================================================================

#include "GAS/Status/StatusEffectManager.h"
#include "AbilitySystemComponent.h"

FStatusEffectManager::FStatusEffectManager(UAbilitySystemComponent* InASC)
    : ASC(InASC)
{
}

// ================================================================
// 眩晕
// ================================================================

void FStatusEffectManager::ApplyStun(float Duration)
{
    // 添加眩晕标签到角色的ASC
    AddTag(FGameplayTag::RequestGameplayTag("State.Stunned"));

    // 设置计时器在Duration秒后自动移除
    if (ASC && ASC->GetOwner())
    {
        ASC->GetOwner()->GetWorldTimerManager().SetTimer(
            // 使用Lambda在延迟后移除眩晕
            // 注意：需要捕获ASC的弱引用防止悬空
            FTimerHandle(),
            [WeakASC = TWeakObjectPtr<UAbilitySystemComponent>(ASC)]()
            {
                if (WeakASC.IsValid())
                {
                    WeakASC->RemoveLooseGameplayTag(
                        FGameplayTag::RequestGameplayTag("State.Stunned"));
                }
            },
            Duration,
            false  // 不循环
        );
    }
}

void FStatusEffectManager::RemoveStun()
{
    RemoveTag(FGameplayTag::RequestGameplayTag("State.Stunned"));
}

bool FStatusEffectManager::IsStunned() const
{
    return HasTag(FGameplayTag::RequestGameplayTag("State.Stunned"));
}

// ================================================================
// 定身
// ================================================================

void FStatusEffectManager::ApplyRoot(float Duration)
{
    AddTag(FGameplayTag::RequestGameplayTag("State.Rooted"));

    if (ASC && ASC->GetOwner())
    {
        ASC->GetOwner()->GetWorldTimerManager().SetTimer(
            FTimerHandle(),
            [WeakASC = TWeakObjectPtr<UAbilitySystemComponent>(ASC)]()
            {
                if (WeakASC.IsValid())
                {
                    WeakASC->RemoveLooseGameplayTag(
                        FGameplayTag::RequestGameplayTag("State.Rooted"));
                }
            },
            Duration,
            false
        );
    }
}

void FStatusEffectManager::RemoveRoot()
{
    RemoveTag(FGameplayTag::RequestGameplayTag("State.Rooted"));
}

bool FStatusEffectManager::IsRooted() const
{
    return HasTag(FGameplayTag::RequestGameplayTag("State.Rooted"));
}

// ================================================================
// 沉默
// ================================================================

void FStatusEffectManager::ApplySilence(float Duration)
```

```cpp
// ====================================================================
// 文件：Source/YourProject/Private/GAS/Ability/GA_Stun.cpp
// 描述：眩晕技能实现
// ====================================================================

#include "GAS/Ability/GA_Stun.h"
#include "Abilities/Tasks/AbilityTask_PlayMontageAndWait.h"
#include "AbilitySystemComponent.h"
#include "AbilitySystemGlobals.h"

UGA_Stun::UGA_Stun()
{
    AbilityTypeTags.AddTag(FGameplayTag::RequestGameplayTag("Ability.Type.CrowdControl"));
    AbilityTypeTags.AddTag(FGameplayTag::RequestGameplayTag("Ability.Type.Stun"));

    ActivationOwnedTags.AddTag(FGameplayTag::RequestGameplayTag("Ability.State.Attacking"));

    ActivationBlockedTags.AddTag(FGameplayTag::RequestGameplayTag("State.Dead"));
    ActivationBlockedTags.AddTag(FGameplayTag::RequestGameplayTag("State.Stunned"));

    StunDuration = 3.0f;

    NetExecutionPolicy = EGameplayAbilityNetExecutionPolicy::ServerOnly;
}

void UGA_Stun::ActivateAbility(
    const FGameplayAbilitySpecHandle Handle,
    const FGameplayAbilityActorInfo* ActorInfo,
    const FGameplayAbilityActivationInfo ActivationInfo,
    const FGameplayEventData* TriggerEventData)
{
    Super::ActivateAbility(Handle, ActorInfo, ActivationInfo, TriggerEventData);

    if (!CommitAbility(Handle, ActorInfo, ActivationInfo))
    {
        EndAbility(Handle, ActorInfo, ActivationInfo, true, true);
        return;
    }

    // ================================================================
    // 获取目标（从TriggerEventData中读取）
    // ================================================================
    // 在实际应用中，目标通过瞄准系统确定，然后通过GameplayEvent传递
    // 这里简化：眩晕自身周围的敌人（近战眩晕）

    if (TriggerEventData && TriggerEventData->TargetData.Num() > 0)
    {
        // 有目标数据：对目标应用眩晕GE
        const FGameplayAbilityTargetDataHandle& TargetData = TriggerEventData->TargetData;

        for (int32 i = 0; i < TargetData.Num(); ++i)
        {
            AActor* TargetActor = const_cast<FGameplayAbilityTargetData*>(
                TargetData.Get(i))->GetActors()[0].Get();

            if (IsValid(TargetActor))
            {
                UAbilitySystemComponent* TargetASC =
                    UAbilitySystemGlobals::GetAbilitySystemComponentFromActor(TargetActor);

                if (IsValid(TargetASC) && IsValid(StunEffectClass))
                {
                    // --- 检查状态免疫 ---
                    if (TargetASC->HasMatchingGameplayTag(
                        FGameplayTag::RequestGameplayTag("State.Immune.Stun")))
                    {
                        // 目标免疫眩晕
                        UE_LOG(LogTemp, Log, TEXT("GA_Stun: Target is immune to stun"));
                        continue;
                    }

                    // --- 应用眩晕GE到目标 ---
                    FGameplayEffectContextHandle Context = GetSourceASC()->MakeEffectContext();

                    FGameplayEffectSpecHandle SpecHandle = GetSourceASC()->MakeOutgoingSpec(
                        StunEffectClass, GetAbilityLevel(), Context);

                    if (SpecHandle.IsValid())
                    {
                        // 设置眩晕持续时间（SetByCaller）
                        SpecHandle.Data->SetSetByCallerMagnitude(
                            FGameplayTag::RequestGameplayTag("Data.StunDuration"),
                            StunDuration
                        );

                        GetSourceASC()->ApplyGameplayEffectSpecToTarget(
                            *SpecHandle.Data, TargetASC);
                    }
                }
            }
        }
    }
    else
    {
        // 无目标数据：对自身前方扇形区域的敌人应用眩晕
        // 实现类似GA_MeleeAttack的扇形检测 + 眩晕GE
    }

    // 播放动画
    PlayStunAnimation();

    // 结束技能
    EndAbility(Handle, ActorInfo, ActivationInfo, true, false);
}

void UGA_Stun::PlayStunAnimation()
{
    if (AbilityMontage)
    {
        UAbilityTask_PlayMontageAndWait* MontageTask =
            UAbilityTask_PlayMontageAndWait::CreatePlayMontageAndWaitProxy(
                this, NAME_None, AbilityMontage, MontagePlayRate);
        MontageTask->ReadyForActivation();
    }
}
```

### 4.3.2 状态免疫和优先级

```cpp
// ====================================================================
// 状态免疫标签定义
// ====================================================================

// 在项目设置中添加以下免疫标签：
State.Immune.Stun         # 免疫眩晕
State.Immune.Silence      # 免疫沉默
State.Immune.Root         # 免疫定身
State.Immune.All          # 免疫所有控制效果

// ====================================================================
// 在眩晕GE的配置中检查免疫
// ====================================================================

// GE_Stun 的 Ongoing Tag Requirements（持续标签要求）:
//   这个GE只在目标没有免疫标签时才能生效
//   Ongoing Tag Requirements:
//     Require Tags: (empty)
//     Ignore Tags: State.Immune.Stun, State.Immune.All
//   → 如果目标拥有State.Immune.Stun标签，眩晕GE不会应用

// ====================================================================
// 状态优先级
// ====================================================================

// 状态的优先级决定什么状态可以覆盖什么状态：
//
// 高优先级 → 低优先级（同一类型的覆盖关系）：
//   Frozen > Stun > Silence > Root
//
// 冰冻会覆盖眩晕（因为冰冻更强，需要同时移除眩晕标签）
// 眩晕会覆盖沉默
// 沉默覆盖定身
//
// 不同优先级的状态可以共存：
//   Stun + Root = Frozen（如果设计如此）
//   Silence + Root = 沉默+定身（可以共存，各自影响不同行为）
//
// 实现：在应用新状态前，先移除被覆盖的状态标签

void FStatusEffectManager::ApplyStun(float Duration)
{
    // 眩晕会覆盖定身（眩晕期间无法移动，定身没有意义）
    RemoveRoot();

    // 但保留沉默？取决于设计
    // 有些游戏：眩晕 = 无法移动 + 无法技能（所以也移除沉默）
    // 有些游戏：眩晕 = 只影响移动，保留其他状态

    AddTag(FGameplayTag::RequestGameplayTag("State.Stunned"));

    // ... 设置计时器
}

void FStatusEffectManager::ApplyFrozen(float Duration)
{
    // 冰冻 = 眩晕 + 定身
    // 移除所有被覆盖的状态
    RemoveStun();
    RemoveRoot();

    AddTag(FGameplayTag::RequestGameplayTag("State.Stunned"));
    AddTag(FGameplayTag::RequestGameplayTag("State.Rooted"));
    AddTag(FGameplayTag::RequestGameplayTag("State.Frozen"));

    // ... 设置计时器
}
```

### 4.4.1 WaitGameplayTag 的使用场景

`UAbilityTask_WaitGameplayTagAdded` 和 `UAbilityTask_WaitGameplayTagRemoved` 可以在GA执行期间等待特定状态：

```cpp
// ====================================================================
// 示例：在GA中等待眩晕结束，然后自动恢复
// ====================================================================

void UGA_SomeChannelingAbility::ActivateAbility(...)
{
    // --- 正常激活逻辑 ---
    Super::ActivateAbility(Handle, ActorInfo, ActivationInfo, TriggerEventData);

    if (!CommitAbility(Handle, ActorInfo, ActivationInfo))
    {
        EndAbility(Handle, ActorInfo, ActivationInfo, true, true);
        return;
    }

    // --- 创建等待眩晕的Task ---
    // 如果在技能执行期间被眩晕，自动取消技能
    UAbilityTask_WaitGameplayTagAdded* WaitStunTask =
        UAbilityTask_WaitGameplayTagAdded::WaitGameplayTagAdd(
            this,                                               // Owning GA
            FGameplayTag::RequestGameplayTag("State.Stunned"),  // 等待这个Tag
            nullptr,                                            // 可选目标
            false                                               // 只触发一次
        );

    WaitStunTask->Added.AddDynamic(this, &UGA_SomeChannelingAbility::OnStunnedDuringAbility);
    WaitStunTask->ReadyForActivation();

    // --- 技能正常执行 ---
    // ...
}

void UGA_SomeChannelingAbility::OnStunnedDuringAbility()
{
    // 在技能执行期间被眩晕 → 打断技能
    UE_LOG(LogTemp, Log, TEXT("Ability interrupted by Stun!"));

    // 取消技能（会触发EndAbility的清理逻辑）
    CancelAbility(CurrentSpecHandle, CurrentActorInfo, CurrentActivationInfo, true);
}
```

```cpp
// ====================================================================

        ASC->RegisterGameplayTagEvent(
            FGameplayTag::RequestGameplayTag("State.Invincible"),
            EGameplayTagEventType::NewOrRemoved
        ).AddUObject(this, &URPGStatusWidget::OnInvincibleChanged);
    }
```
