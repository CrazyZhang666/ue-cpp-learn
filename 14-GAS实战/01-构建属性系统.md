## 1.2 AttributeSet（属性集）——属性的数据载体

### 1.2.1 核心概念

`UAttributeSet` 是GAS中存储角色属性的数据结构。它本质上是一个 `UObject` 子类，但被GAS框架特殊处理：

- **网络同步**：属性变化自动通过 `OnRep_*` 复制到客户端
- **变化预测**：支持客户端预测（用于减少网络延迟体感）
- **聚合计算**：通过MMC（ModMagnitudeCalculation）实现跨属性计算
- **变化回调**：属性被修改前后可以执行逻辑（Pre/PostGameplayEffectExecute）

### 1.2.2 创建RPGA属性集头文件

创建文件 `Source/YourProject/Public/GAS/AttributeSet/RPGAttributeSet.h`：

> ⚠️ **重要提醒**：在创建文件前，先在UE编辑器中创建对应的目录结构（Content Browser → Add New → New C++ Class），这样UE会自动生成基础代码并注册到模块中。手动创建文件需要额外注册模块，容易出错。

```cpp
// ====================================================================
// 文件：Source/YourProject/Public/GAS/AttributeSet/RPGAttributeSet.h
// 描述：RPG属性集 — 包含角色的所有可成长属性
// ====================================================================

#pragma once

#include "CoreMinimal.h"
#include "AttributeSet.h"                           // GAS属性集基类
#include "AbilitySystemComponent.h"                 // 提供GetAbilitySystemComponent等工具函数
#include "RPGAttributeSet.generated.h"              // 必须最后引入（UE反射系统要求）

// ====================================================================
// 宏定义：辅助声明属性访问器
// ====================================================================
// 这是GAS官方推荐的属性声明宏，用于减少重复代码
// 每个属性需要：FGameplayAttributeData成员 + Getter + Setter + Init + OnRep函数声明

// ATTRIBUTE_ACCESSORS 宏会自动生成 Get{Name}Attribute、Set{Name}BaseValue、
// Set{Name}CurrentValue、Init{Name} 等函数
// 这是UE GAS官方提供的宏，定义在AttributeSet.h中

/**
 * URPGAttributeSet — RPG游戏的核心属性集合
 *
 * 设计原则：
 * 1. 主要属性（Vital）使用当前值+最大值的配对模式
 * 2. 战斗属性使用单一数值
 * 3. 元素抗性使用百分比形式（0.0~1.0，1.0=完全免疫）
 * 4. 所有属性通过FGameplayAttributeData存储（支持网络同步和预测）
 */
UCLASS()
class YOURPROJECT_API URPGAttributeSet : public UAttributeSet
{
    GENERATED_BODY()

public:
    // 构造函数
    URPGAttributeSet();

    // ===================================================================
    // 第一部分：主要属性（Vital Attributes） — 生命/法力/体力
    // ===================================================================
    // 命名约定：
    //   Health = 当前值
    //   MaxHealth = 最大值
    //   Health的最大值由MaxHealth属性限制（通过GE的Modifier中的Clamp实现）
    // ===================================================================

    /** 当前生命值 — 降到0时角色死亡 */
    UPROPERTY(BlueprintReadOnly, Category = "Vital|Health", ReplicatedUsing = OnRep_Health)
    FGameplayAttributeData Health;
    ATTRIBUTE_ACCESSORS(URPGAttributeSet, Health)  // 自动生成Getter/Setter/Init函数

    /** 最大生命值 — 由装备、等级、Buff等影响 */
    UPROPERTY(BlueprintReadOnly, Category = "Vital|Health", ReplicatedUsing = OnRep_MaxHealth)
    FGameplayAttributeData MaxHealth;
    ATTRIBUTE_ACCESSORS(URPGAttributeSet, MaxHealth)

    /** 当前法力值 — 施放技能消耗法力 */
    UPROPERTY(BlueprintReadOnly, Category = "Vital|Mana", ReplicatedUsing = OnRep_Mana)
    FGameplayAttributeData Mana;
    ATTRIBUTE_ACCESSORS(URPGAttributeSet, Mana)

    /** 最大法力值 */
    UPROPERTY(BlueprintReadOnly, Category = "Vital|Mana", ReplicatedUsing = OnRep_MaxMana)
    FGameplayAttributeData MaxMana;
    ATTRIBUTE_ACCESSORS(URPGAttributeSet, MaxMana)

    /** 当前体力值 — 闪避、冲刺消耗体力 */
    UPROPERTY(BlueprintReadOnly, Category = "Vital|Stamina", ReplicatedUsing = OnRep_Stamina)
    FGameplayAttributeData Stamina;
    ATTRIBUTE_ACCESSORS(URPGAttributeSet, Stamina)

    /** 最大体力值 */
    UPROPERTY(BlueprintReadOnly, Category = "Vital|Stamina", ReplicatedUsing = OnRep_MaxStamina)
    FGameplayAttributeData MaxStamina;
    ATTRIBUTE_ACCESSORS(URPGAttributeSet, MaxStamina)

    // ===================================================================
    // 第二部分：战斗属性（Combat Attributes）
    // ===================================================================
    // 这些属性直接影响战斗公式的计算结果
    // ===================================================================

    /** 攻击力 — 影响所有伤害类技能的最终伤害值 */
    UPROPERTY(BlueprintReadOnly, Category = "Combat", ReplicatedUsing = OnRep_AttackPower)
    FGameplayAttributeData AttackPower;
    ATTRIBUTE_ACCESSORS(URPGAttributeSet, AttackPower)

    /** 防御力 — 减少受到的物理伤害 */
    UPROPERTY(BlueprintReadOnly, Category = "Combat", ReplicatedUsing = OnRep_DefensePower)
    FGameplayAttributeData DefensePower;
    ATTRIBUTE_ACCESSORS(URPGAttributeSet, DefensePower)

    /** 暴击率 — 0.0~1.0，0.05=5%暴击率 */
    UPROPERTY(BlueprintReadOnly, Category = "Combat", ReplicatedUsing = OnRep_CriticalRate)
    FGameplayAttributeData CriticalRate;
    ATTRIBUTE_ACCESSORS(URPGAttributeSet, CriticalRate)

    /** 暴击伤害倍率 — 1.5=暴击时造成150%伤害 */
    UPROPERTY(BlueprintReadOnly, Category = "Combat", ReplicatedUsing = OnRep_CriticalDamage)
    FGameplayAttributeData CriticalDamage;
    ATTRIBUTE_ACCESSORS(URPGAttributeSet, CriticalDamage)

    // ===================================================================
    // 第三部分：元素抗性（Elemental Resistances）
    // ===================================================================
    // 0.0 = 无抗性（受到全额元素伤害）
    // 1.0 = 完全免疫该元素伤害
    // 负数 = 弱点（受到额外伤害）
    // ===================================================================

    /** 火焰抗性 */
    UPROPERTY(BlueprintReadOnly, Category = "Resistance", ReplicatedUsing = OnRep_FireResist)
    FGameplayAttributeData FireResist;
    ATTRIBUTE_ACCESSORS(URPGAttributeSet, FireResist)

    /** 冰霜抗性 */
    UPROPERTY(BlueprintReadOnly, Category = "Resistance", ReplicatedUsing = OnRep_IceResist)
    FGameplayAttributeData IceResist;
    ATTRIBUTE_ACCESSORS(URPGAttributeSet, IceResist)

    /** 闪电抗性 */
    UPROPERTY(BlueprintReadOnly, Category = "Resistance", ReplicatedUsing = OnRep_LightningResist)
    FGameplayAttributeData LightningResist;
    ATTRIBUTE_ACCESSORS(URPGAttributeSet, LightningResist)

    // ===================================================================
    // 第四部分：临时/派生属性（Meta Attributes）
    // ===================================================================
    // 这些属性不在角色身上永久存储，仅用于GE执行期间的中间计算
    // 通常在每个GE被执行时重置为0，计算结束后读取结果
    // ===================================================================

    /**
     * 受到的最终伤害值（临时属性）
     * 在PreGameplayEffectExecute中重置为0
     * 在PostGameplayEffectExecute中读取并应用到Health
     * 使用临时属性可以避免循环依赖问题
     */
    UPROPERTY(BlueprintReadOnly, Category = "Meta")
    FGameplayAttributeData IncomingDamage;
    ATTRIBUTE_ACCESSORS(URPGAttributeSet, IncomingDamage)

    /** 受到的最终治疗值（临时属性） */
    UPROPERTY(BlueprintReadOnly, Category = "Meta")
    FGameplayAttributeData IncomingHealing;
    ATTRIBUTE_ACCESSORS(URPGAttributeSet, IncomingHealing)

    // ===================================================================
    // 网络复制回调（OnRep函数）
    // ===================================================================
    // 当属性在服务器上变化并复制到客户端时自动调用
    // 在这里可以绑定UI更新逻辑
    // ===================================================================

    UFUNCTION()
    virtual void OnRep_Health(const FGameplayAttributeData& OldValue);

    UFUNCTION()
    virtual void OnRep_MaxHealth(const FGameplayAttributeData& OldValue);

    UFUNCTION()
    virtual void OnRep_Mana(const FGameplayAttributeData& OldValue);

    UFUNCTION()
    virtual void OnRep_MaxMana(const FGameplayAttributeData& OldValue);

    UFUNCTION()
    virtual void OnRep_Stamina(const FGameplayAttributeData& OldValue);

    UFUNCTION()
    virtual void OnRep_MaxStamina(const FGameplayAttributeData& OldValue);

    UFUNCTION()
    virtual void OnRep_AttackPower(const FGameplayAttributeData& OldValue);

    UFUNCTION()
    virtual void OnRep_DefensePower(const FGameplayAttributeData& OldValue);

    UFUNCTION()
    virtual void OnRep_CriticalRate(const FGameplayAttributeData& OldValue);

    UFUNCTION()
    virtual void OnRep_CriticalDamage(const FGameplayAttributeData& OldValue);

    UFUNCTION()
    virtual void OnRep_FireResist(const FGameplayAttributeData& OldValue);

    UFUNCTION()
    virtual void OnRep_IceResist(const FGameplayAttributeData& OldValue);

    UFUNCTION()
    virtual void OnRep_LightningResist(const FGameplayAttributeData& OldValue);

    // ===================================================================
    // 重写：属性变化前/后处理
    // ===================================================================
    // 这是GAS中最核心的回调之一，在此实现：
    // 1. Clamp（钳制）属性值到合法范围
    // 2. 伤害/治疗的最终计算
    // 3. 特殊效果触发（如血量低于阈值）
    // ===================================================================

    /**
     * PreAttributeChange — 属性即将被修改时调用
     * 用途：钳制属性值（例如Health不能超过MaxHealth）
     * ⚠️ 注意：此函数在属性真正改变之前调用，但此时NewValue还未应用到属性
     * ⚠️ 不要在此函数中触发游戏逻辑（如死亡判定）— 应放在PostGameplayEffectExecute中
     */
    virtual void PreAttributeChange(const FGameplayAttribute& Attribute, float& NewValue) override;

    /**
     * PostGameplayEffectExecute — GE执行完毕后调用
     * 用途：处理GE执行后的逻辑（伤害应用、死亡判定、Buff触发等）
     * ⚠️ 这是处理伤害/治疗逻辑的唯一正确位置
     * ⚠️ PreAttributeChange只做钳制，PostGameplayEffectExecute做业务逻辑
     */
    virtual void PostGameplayEffectExecute(const FGameplayEffectModCallbackData& Data) override;

    /**
     * PreAttributeBaseChange — 属性的BaseValue即将被修改时调用
     * 用途：钳制BaseValue（BaseValue通常用于永久性属性变化）
     */
    virtual void PreAttributeBaseChange(const FGameplayAttribute& Attribute, float& NewValue) const override;

private:
    // ===================================================================
    // 内部辅助函数
    // ===================================================================

    /** 将所有临时属性（IncomingDamage/IncomingHealing）重置为0 */
    void ResetMetaAttributes();

    /** 钳制主要属性：确保当前值不超过最大值，且不低于0 */
    static void ClampVitalAttribute(const FGameplayAttribute& Attribute, float& NewValue,
        const FGameplayAttributeData& MaxValueAttribute);
};
```

```cpp
// 构造函数
// ====================================================================
URPGAttributeSet::URPGAttributeSet()
{
    // --- 初始化所有主要属性 ---
    // 使用FGameplayAttributeData的默认值语法
    // BaseValue = 基础值（永久性变化用）
    // CurrentValue = 当前值（临时Buff/Debuff用）

    // 生命相关
    InitHealth(100.0f);          // 初始生命值100
    InitMaxHealth(100.0f);       // 初始最大生命值100

    // 法力相关

    InitMaxMana(50.0f);          // 初始最大法力值50

    // 体力相关

    InitMaxStamina(80.0f);       // 初始最大体力值80

    InitCriticalRate(0.05f);     // 初始暴击率5%（用小数表示）

    // --- 初始化元素抗性 ---
    InitFireResist(0.0f);        // 初始火焰抗性0%
    InitIceResist(0.0f);         // 初始冰霜抗性0%

    InitIncomingHealing(0.0f);   // 受到的治疗初始为0
}

// ====================================================================
// PreAttributeChange — 属性被修改前的最终检查点
// ====================================================================
void URPGAttributeSet::PreAttributeChange(const FGameplayAttribute& Attribute, float& NewValue)
{
    Super::PreAttributeChange(Attribute, NewValue);

    // 钳制主要属性：保证当前值不超过最大值且不低于0
    // 例如：如果MaxHealth=100，NewValue=150，则自动修正为100
    if (Attribute == GetHealthAttribute())
    {
        ClampVitalAttribute(Attribute, NewValue, MaxHealth);
    }
    else if (Attribute == GetManaAttribute())
    {
        ClampVitalAttribute(Attribute, NewValue, MaxMana);
    }
    else if (Attribute == GetStaminaAttribute())
    {
        ClampVitalAttribute(Attribute, NewValue, MaxStamina);
    }
    // 注意：这里只做钳制，不做业务逻辑
    // ❌ 错误：在这里处理死亡逻辑
    // ✅ 正确：在PostGameplayEffectExecute中处理死亡逻辑
}

// ====================================================================
// PreAttributeBaseChange — BaseValue被修改前调用
// ====================================================================
void URPGAttributeSet::PreAttributeBaseChange(const FGameplayAttribute& Attribute, float& NewValue) const
{
    Super::PreAttributeBaseChange(Attribute, NewValue);

    // BaseValue通常表示永久性变化（如升级增加的属性）
    // 同样需要钳制到合理范围
    if (Attribute == GetHealthAttribute())
    {
        ClampVitalAttribute(Attribute, NewValue, MaxHealth);
    }
    else if (Attribute == GetManaAttribute())
    {
        ClampVitalAttribute(Attribute, NewValue, MaxMana);
    }
    else if (Attribute == GetStaminaAttribute())
    {
        ClampVitalAttribute(Attribute, NewValue, MaxStamina);
    }
}

// ====================================================================
// PostGameplayEffectExecute — GE执行完毕后的处理（最核心的函数！）
// ====================================================================
void URPGAttributeSet::PostGameplayEffectExecute(const FGameplayEffectModCallbackData& Data)
{
    Super::PostGameplayEffectExecute(Data);

    // --- 第一步：获取上下文信息 ---
    // Data.EvaluatedData.Attribute 告诉你哪个属性被修改了
    // Data.EvaluatedData.Magnitude 告诉你修改量是多少
    // Data.EffectSpec 包含GE的完整信息（等级、标签、源对象等）

    const FGameplayEffectContextHandle ContextHandle = Data.EffectSpec.GetContext();
    const FGameplayAttribute& ModifiedAttribute = Data.EvaluatedData.Attribute;

    // --- 第二步：处理临时属性（IncomingDamage）---
    if (ModifiedAttribute == GetIncomingDamageAttribute())
    {
        // 读取GE计算后写入到IncomingDamage的值
        const float LocalIncomingDamage = GetIncomingDamage();

        // 防御检查：伤害必须大于0才有意义
        if (LocalIncomingDamage > 0.0f)
        {
            // 计算实际生命值减少
            // 注意：IncomingDamage已经在MMC中完成了防御力减免等计算
            // 这里直接应用即可
            const float NewHealth = GetHealth() - LocalIncomingDamage;

            // 设置新生命值（PreAttributeChange会自动钳制到0~MaxHealth范围）
            SetHealth(FMath::Clamp(NewHealth, 0.0f, GetMaxHealth()));

            // === 在这里实现死亡判定 ===
            // ✅ 正确位置：PostGameplayEffectExecute
            if (GetHealth() <= 0.0f && LocalIncomingDamage > 0.0f)
            {
                // 获取受伤的角色
                AActor* TargetActor = Data.Target.GetOwner();
                if (ACharacter* TargetCharacter = Cast<ACharacter>(TargetActor))
                {
                    // 通过ASC广播死亡事件
                    // 实战中可以通过GameplayEvent或委托通知
                    UE_LOG(LogTemp, Warning, TEXT("RPGAttributeSet: Character Died! Health reached 0"));

                    // 可选：发送GameplayEvent来触发死亡技能
                    // FGameplayEventData EventData;
                    // EventData.Instigator = Data.EffectSpec.GetContext().GetInstigator();
                    // UAbilitySystemBlueprintLibrary::SendGameplayEventToActor(
                    //     TargetCharacter,
                    //     FGameplayTag::RequestGameplayTag("Event.Death"),
                    //     EventData
                    // );
                }
            }
        }

        // 重置临时属性，为下一个GE做准备
        SetIncomingDamage(0.0f);
    }

    // --- 第三步：处理临时属性（IncomingHealing）---
    else if (ModifiedAttribute == GetIncomingHealingAttribute())
    {
        const float LocalIncomingHealing = GetIncomingHealing();

        if (LocalIncomingHealing > 0.0f)
        {
            // 治疗不能超过最大生命值
            const float NewHealth = GetHealth() + LocalIncomingHealing;
            SetHealth(FMath::Clamp(NewHealth, 0.0f, GetMaxHealth()));
        }

        // 重置临时属性
        SetIncomingHealing(0.0f);
    }
}
```

## 1.3 MMC（ModMagnitudeCalculation）——属性计算引擎

### 1.3.1 什么是MMC？

MMC（全称 `UGameplayModMagnitudeCalculation`）是GAS中用于**动态计算属性修改量**的类。与简单的固定数值GE不同，MMC可以从多个来源获取属性值并进行数学运算。

**典型使用场景：**
- 伤害公式：`最终伤害 = 攻击力 * 技能倍率 - 防御力`
- 治疗公式：`治疗量 = 最大生命值 * 0.1 + 治疗加成`
- 护盾公式：`护盾量 = 最大法力值 * 0.3`

### 1.3.2 创建伤害计算MMC

创建文件 `Source/YourProject/Public/GAS/Calculation/DamageCalculation.h`：

```cpp
// ====================================================================
// 文件：Source/YourProject/Public/GAS/Calculation/DamageCalculation.h
// 描述：伤害计算MMC — 实现完整的伤害计算公式
// ====================================================================

#pragma once

#include "CoreMinimal.h"
#include "GameplayModMagnitudeCalculation.h"        // MMC基类
#include "DamageCalculation.generated.h"

/**
 * UDamageCalculation — 通用伤害计算类
 *
 * 伤害公式：
 *   BaseDamage = 攻击力 * 技能倍率
 *   EffectiveDamage = BaseDamage * (1 - 防御减免率)
 *   CriticalBonus = EffectiveDamage * (暴击倍率 - 1.0) [如果暴击]
 *   FinalDamage = EffectiveDamage + CriticalBonus
 *
 * 设计思路：
 * 1. 技能倍率通过GE的SetByCaller传入（使用GameplayTag）
 * 2. 攻击力从Source（攻击者）的属性集中读取
 * 3. 防御力从Target（防御者）的属性集中读取
 * 4. 是否暴击由攻击者的暴击率随机判定
 */
UCLASS()
class YOURPROJECT_API UDamageCalculation : public UGameplayModMagnitudeCalculation
{
    GENERATED_BODY()

public:
    UDamageCalculation();

    // ================================================================
    // 核心覆写函数 — 所有MMC必须实现此函数
    // ================================================================
    // @param Spec          — 当前正在执行的GE的规格
    // @param SourceTags    — Source（攻击方）的GameplayTag聚合
    // @param TargetTags    — Target（防御方）的GameplayTag聚合
    // @param SourceAttributes — Source的AttributeSet捕获（可从中读取属性值）
    // @param TargetAttributes — Target的AttributeSet捕获（可从中读取属性值）
    // @return              — 计算出的属性修改量（会应用到GE目标属性上）
    // ================================================================
    virtual float CalculateBaseMagnitude_Implementation(
        const FGameplayEffectSpec& Spec,
        const FGameplayTagContainer* SourceTags,
        const FGameplayTagContainer* TargetTags,
        const FAggregatorEvaluateParameters& SourceAttributes,
        const FAggregatorEvaluateParameters& TargetAttributes
    ) const override;

protected:
    // ================================================================
    // 属性捕获定义 — 告诉GAS我们需要读取哪些属性
    // ================================================================
    // 这些成员变量在编辑器中配置（或在构造函数中硬编码）
    // FGameplayEffectAttributeCaptureDefinition 包含：
    //   - AttributeToCapture：要捕获的属性
    //   - EGameplayEffectAttributeCaptureSource：从谁身上捕获（Source还是Target）
    //   - bSnapshot：是否快照（true=使用GE应用时的值，false=使用计算时的值）
    // ================================================================

    /** 捕获：攻击者的攻击力属性 */
    UPROPERTY(EditDefaultsOnly, Category = "Calculation|Capture")
    FGameplayEffectAttributeCaptureDefinition AttackPowerCapture;

    /** 捕获：防御者的防御力属性 */
    UPROPERTY(EditDefaultsOnly, Category = "Calculation|Capture")
    FGameplayEffectAttributeCaptureDefinition DefensePowerCapture;

    /** 捕获：攻击者的暴击率属性 */
    UPROPERTY(EditDefaultsOnly, Category = "Calculation|Capture")
    FGameplayEffectAttributeCaptureDefinition CriticalRateCapture;

    /** 捕获：攻击者的暴击伤害倍率属性 */
    UPROPERTY(EditDefaultsOnly, Category = "Calculation|Capture")
    FGameplayEffectAttributeCaptureDefinition CriticalDamageCapture;

    // ================================================================
    // 可配置参数
    // ================================================================

    /**
     * 防御减免系数
     * 公式：减免率 = Defense / (Defense + DefenseCoefficient)
     * 默认100：当防御力=100时，减免率为50%
     * 降低此值会使防御力的效果更明显
     */
    UPROPERTY(EditDefaultsOnly, Category = "Calculation|Formula")
    float DefenseCoefficient = 100.0f;
};
```

### 1.3.3 创建伤害计算MMC实现

创建文件 `Source/YourProject/Private/GAS/Calculation/DamageCalculation.cpp`：

```cpp
// ====================================================================
// 文件：Source/YourProject/Private/GAS/Calculation/DamageCalculation.cpp
// 描述：伤害计算MMC的实现
// ====================================================================

#include "GAS/Calculation/DamageCalculation.h"
#include "GAS/AttributeSet/RPGAttributeSet.h"        // 我们的属性集

UDamageCalculation::UDamageCalculation()
{
    // ================================================================
    // 构造函数 — 配置属性捕获
    // ================================================================
    // 这里定义需要从Source和Target读取哪些属性
    // 每个捕获定义3个要素：哪个属性、从谁身上捕获、是否快照

    // --- 捕获：攻击者的攻击力 ---
    AttackPowerCapture.AttributeToCapture = URPGAttributeSet::GetAttackPowerAttribute();
    AttackPowerCapture.AttributeSource = EGameplayEffectAttributeCaptureSource::Source;
    // bSnapshot = false：使用计算时刻的实时值（而非GE应用时刻的快照值）
    // 为什么不用快照：如果攻击者在GE执行过程中被Buff影响，我们希望使用最新值
    AttackPowerCapture.bSnapshot = false;

    // --- 捕获：防御者的防御力 ---
    DefensePowerCapture.AttributeToCapture = URPGAttributeSet::GetDefensePowerAttribute();
    DefensePowerCapture.AttributeSource = EGameplayEffectAttributeCaptureSource::Target;
    DefensePowerCapture.bSnapshot = false;

    // --- 捕获：攻击者的暴击率 ---
    CriticalRateCapture.AttributeToCapture = URPGAttributeSet::GetCriticalRateAttribute();
    CriticalRateCapture.AttributeSource = EGameplayEffectAttributeCaptureSource::Source;
    CriticalRateCapture.bSnapshot = false;

    // --- 捕获：攻击者的暴击伤害倍率 ---
    CriticalDamageCapture.AttributeToCapture = URPGAttributeSet::GetCriticalDamageAttribute();
    CriticalDamageCapture.AttributeSource = EGameplayEffectAttributeCaptureSource::Source;
    CriticalDamageCapture.bSnapshot = false;

    // ================================================================
    // ⚠️ 关键步骤：必须将捕获定义注册到RelevantAttributesToCapture
    // ================================================================
    // 如果不注册，GAS将不会捕获这些属性值，计算结果永远为0
    RelevantAttributesToCapture.Add(AttackPowerCapture);
    RelevantAttributesToCapture.Add(DefensePowerCapture);
    RelevantAttributesToCapture.Add(CriticalRateCapture);
    RelevantAttributesToCapture.Add(CriticalDamageCapture);

    // ================================================================
    // 设置防御减免系数
    // ================================================================
    // 公式：减免率 = Defense / (Defense + 100)
    // 防御力100 → 减免率 = 100/200 = 50%
    // 防御力200 → 减免率 = 200/300 = 66.7%
    // 防御力无限大 → 减免率趋近100%
    // 这种公式叫"边际递减公式"，保证堆防御力永远有收益但不至于无敌
    DefenseCoefficient = 100.0f;
}

float UDamageCalculation::CalculateBaseMagnitude_Implementation(
    const FGameplayEffectSpec& Spec,
    const FGameplayTagContainer* SourceTags,
    const FGameplayTagContainer* TargetTags,
    const FAggregatorEvaluateParameters& SourceAttributes,
    const FAggregatorEvaluateParameters& TargetAttributes) const
{
    // ================================================================
    // 第一步：从Source获取攻击者的属性值
    // ================================================================

    float AttackPower = 0.0f;
    // AttemptCalculateCapturedAttributeMagnitude 尝试从捕获中读取属性值
    // 参数1：捕获定义（哪个属性、从哪里捕获）
    // 参数2：Source的评估参数（包含Source的Aggregator数据）
    // 参数3：输出参数，接收捕获到的属性值
    // 返回值：true=成功捕获，false=捕获失败（例如Source没有这个AttributeSet）
    if (!AttackPowerCapture.AttemptCalculateCapturedAttributeMagnitude(
            Spec, SourceAttributes, AttackPower))
    {
        // 捕获失败时的默认值（攻击力为0）
        AttackPower = 0.0f;
        UE_LOG(LogTemp, Warning, TEXT("DamageCalculation: Failed to capture AttackPower from Source"));
    }

    // ================================================================
    // 第二步：从Target获取防御者的属性值
    // ================================================================

    float DefensePower = 0.0f;
    if (!DefensePowerCapture.AttemptCalculateCapturedAttributeMagnitude(
            Spec, TargetAttributes, DefensePower))
    {
        DefensePower = 0.0f;
    }

    // ================================================================
    // 第三步：读取SetByCaller传递的技能倍率
    // ================================================================
    // SetByCaller是GE中传递动态参数的机制
    // 技能在激活时通过FGameplayAbilitySpecHandle → SetByCallerTagMagnitude传入倍率
    // 例如：火球术倍率=1.5，普通攻击倍率=1.0

    const FGameplayTag SkillMultiplierTag = FGameplayTag::RequestGameplayTag(
        FName("Data.SkillMultiplier"),  // 标签名称（必须与GA中设置的标签一致）
        false                           // false = 如果标签不存在不报错，返回空标签
    );

    float SkillMultiplier = 1.0f;  // 默认技能倍率1.0（100%攻击力伤害）
    if (SkillMultiplierTag.IsValid())
    {
        // 尝试从GE Spec中读取SetByCaller的值
        // GetSetByCallerMagnitude：读取指定标签关联的数值
        // 如果该标签没有被设置，返回false，SafeMultiplyValue保持为1.0f
        SkillMultiplier = Spec.GetSetByCallerMagnitude(SkillMultiplierTag, true, 1.0f);
    }

    // ================================================================
    // 第四步：计算基础伤害（攻击力 × 技能倍率）
    // ================================================================

    const float BaseDamage = AttackPower * SkillMultiplier;

    UE_LOG(LogTemp, Verbose, TEXT("DamageCalculation: AttackPower=%.1f, Multiplier=%.2f, BaseDamage=%.1f"),
        AttackPower, SkillMultiplier, BaseDamage);

    // ================================================================
    // 第五步：计算防御减免（使用边际递减公式）
    // ================================================================

    // 减免率 = Defense / (Defense + DefenseCoefficient)
    // 注意：如果DefenseCoefficient为0会导致除零错误
    const float DefenseReductionRate = (DefenseCoefficient > 0.0f)
        ? DefensePower / (DefensePower + DefenseCoefficient)
        : 0.0f;

    // 有效伤害 = 基础伤害 × (1 - 减免率)
    const float EffectiveDamage = BaseDamage * (1.0f - DefenseReductionRate);

    // ================================================================
    // 第六步：暴击判定
    // ================================================================

    // 获取暴击率
    float CriticalRate = 0.0f;
    if (!CriticalRateCapture.AttemptCalculateCapturedAttributeMagnitude(
            Spec, SourceAttributes, CriticalRate))
    {
        CriticalRate = 0.0f;
    }

    // 获取暴击伤害倍率
    float CriticalDamageMultiplier = 1.5f;  // 默认150%
    if (!CriticalDamageCapture.AttemptCalculateCapturedAttributeMagnitude(
            Spec, SourceAttributes, CriticalDamageMultiplier))
    {
        CriticalDamageMultiplier = 1.5f;
    }

    // 生成0~1之间的随机数，判断是否暴击
    const float RandomRoll = FMath::FRand();
    const bool bIsCritical = (RandomRoll < CriticalRate);

    // 计算最终伤害
    float FinalDamage = EffectiveDamage;

    if (bIsCritical)
    {
        // 暴击伤害 = 有效伤害 × 暴击倍率
        FinalDamage = EffectiveDamage * CriticalDamageMultiplier;

        UE_LOG(LogTemp, Log, TEXT("DamageCalculation: CRIT! Roll=%.3f, Rate=%.3f, Final=%.1f (x%.1f)"),
            RandomRoll, CriticalRate, FinalDamage, CriticalDamageMultiplier);

        // ================================================================
        // 注意：这里只是计算属性修改量的数值
        // 为了让UI显示"暴击"文字/效果，应该通过GameplayCue或GameplayTag传递暴击信息
        // 这里通过动态标签标记此次伤害为暴击
        // ================================================================

        // 在实际项目中，可以通过以下方式传递暴击信息：
        // Spec.DynamicGrantedTags.AddTag(FGameplayTag::RequestGameplayTag("Combat.Critical"));
        // 但由于Spec是const引用，需要在GA创建GE时设置
    }
    else
    {
        UE_LOG(LogTemp, Verbose, TEXT("DamageCalculation: Normal hit, Final=%.1f"), FinalDamage);
    }

    // ================================================================
    // 第七步：确保伤害不为负数（防御力再高也不会反弹伤害）
    // ================================================================

    FinalDamage = FMath::Max(FinalDamage, 0.0f);

    // 返回计算结果 → GAS会将此值应用到GE的目标属性上（通常是IncomingDamage）
    return FinalDamage;
}
```

```cpp
void ARPGCharacter::InitializeGAS()
{
    // ================================================================
    // 第一步：创建并注册AttributeSet
    // ================================================================

    if (IsValid(AbilitySystemComponent))
    {
        // 创建属性集实例（UAttributeSet需要手动创建并注册到ASC）
        // ASC内部会管理AttributeSet的生命周期
        AttributeSet = NewObject<URPGAttributeSet>(this, TEXT("RPGAttributeSet"));

        if (IsValid(AttributeSet))
        {
            // 将AttributeSet注册到ASC
            // 此后ASC才能通过GetAttributeSet找到它，GE才能修改它的属性
            AbilitySystemComponent->AddAttributeSetSubobject(AttributeSet);

            UE_LOG(LogTemp, Log, TEXT("RPGCharacter: AttributeSet created and registered"));
        }

        // ================================================================
        // 第二步：初始化ASC自身
        // ================================================================
        // InitAbilityActorInfo将ASC的Owner和Avatar关联起来
        // Owner = 拥有此ASC的Actor（通常是角色本身，用于权限检查）
        // Avatar = 代表此ASC在世界中的物理Actor（通常也是角色本身）
        // ⚠️ 必须在应用任何GE或激活任何GA之前调用
        AbilitySystemComponent->InitAbilityActorInfo(this, this);

        // ================================================================
        // 第三步：应用初始属性GE
        // ================================================================
        // 仅在服务器端执行（或在单机模式下的服务端逻辑）
        // 如果是多人游戏，GetLocalRole() == ROLE_Authority 确保只在服务器执行

        if (HasAuthority())
        {
            // --- 应用默认属性 ---
            if (DefaultAttributeGE)
            {
                // MakeEffectContext 创建GE上下文（包含Source、Instigator等信息）
                FGameplayEffectContextHandle ContextHandle =
                    AbilitySystemComponent->MakeEffectContext();

                // MakeOutgoingSpec 基于GE类创建GE规格
                // 参数1：GE类（TSubclassOf<UGameplayEffect>）
                // 参数2：GE等级（1级为基础等级，等级影响GE中的Scalable Float值）
                // 参数3：Context（包含谁应用这个GE的信息）
                FGameplayEffectSpecHandle SpecHandle =
                    AbilitySystemComponent->MakeOutgoingSpec(DefaultAttributeGE, 1.0f, ContextHandle);

                if (SpecHandle.IsValid())
                {
                    // ApplyGameplayEffectSpecToSelf 将GE应用到ASC自身
                    // 返回值是一个ActiveGameplayEffectHandle，可用于后续移除或查询
                    AbilitySystemComponent->ApplyGameplayEffectSpecToSelf(*SpecHandle.Data.Get());

                    UE_LOG(LogTemp, Log, TEXT("RPGCharacter: Default attributes applied"));
                }
            }

            // --- 应用等级属性（如果配置了） ---
            if (Level1AttributeGE)
            {
                FGameplayEffectContextHandle ContextHandle =
                    AbilitySystemComponent->MakeEffectContext();

                FGameplayEffectSpecHandle SpecHandle =
                    AbilitySystemComponent->MakeOutgoingSpec(Level1AttributeGE, 1.0f, ContextHandle);

                if (SpecHandle.IsValid())
                {
                    AbilitySystemComponent->ApplyGameplayEffectSpecToSelf(*SpecHandle.Data.Get());
                }
            }
        }

        // ================================================================
        // 第四步：绑定属性变化委托到UI回调
        // ================================================================
        BindAttributeChangeDelegates();
    }

}
```

### 1.5.3 GE操作类型详解

| 操作类型 | 说明 | 使用场景 | 示例 |
|---------|------|---------|------|
| `Add` | 在当前值上加上指定量 | 升级增加属性、Buff | 攻击力+10 |
| `Multiply` | 在当前值上乘以指定系数 | 百分比Buff | 攻击力×1.2（增加20%） |
| `Override` | 直接覆盖当前值 | 初始化属性、固定值设置 | 生命值=100 |
| `Divide` | 在当前值上除以指定系数 | 百分比Debuff | 攻击力÷2（减半） |

### 1.5.4 在编辑器中创建GE的步骤

1. **打开Content Browser** → 右键 → Miscellaneous → Data Asset

   - `Gameplay Effect` → `Duration Policy`：选择 `Instant`
   - `Modifiers` → 添加每个属性修改器
