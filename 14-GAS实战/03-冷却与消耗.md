# 03 - 冷却与消耗

> **本章目标**：彻底理解GAS冷却与消耗系统的工作原理，创建可复用的冷却/消耗GE模板，并实现冷却UI显示和消耗不足反馈。

---

## 3.1 冷却与消耗在GAS中的角色

在GAS中，冷却和消耗不是独立系统，而是通过GameplayEffect实现的。这意味着：

- **冷却** = 一个 `Duration` 类型的GE，给角色添加一个 `CooldownTag`
- **消耗** = 一个 `Instant` 类型的GE，从角色的资源属性中减去数值

这种设计的优势：
- ✅ 冷却和消耗可以被其他系统"看到"（通过GameplayTag）
- ✅ UI可以通过监听GameplayTag变化来更新冷却显示
- ✅ 冷却时间可以通过GE等级动态调整
- ✅ 可以创建"重置所有冷却"的GE（移除冷却Tag）

---

## 3.2 冷却系统详解

### 3.2.1 冷却的核心机制

```
技能激活时的冷却流程：

1. GA::ActivateAbility 被调用
2. GA::CommitAbility 被调用
3. CommitAbility 内部调用 ApplyCooldown
4. ApplyCooldown 创建并应用冷却GE到自身
5. 冷却GE包含：
   - Duration Policy = HasDuration（持续一段时间）
   - GrantedTags = 冷却Tag（如 Ability.Cooldown.Fireball）
   - Duration = 冷却时间（如 8秒）
6. 在冷却期间：
   - 角色拥有冷却Tag
   - ASC检查CanActivateAbility时，发现冷却Tag → 阻止激活
7. 冷却GE过期后：
   - 冷却Tag自动移除
   - 技能可以再次激活
```

```cpp
// 2. 8秒后GE过期，标签自动移除
// 3. 在拥有此标签期间，火球术技能的 CanActivateAbility 返回false
```

### 3.2.4 共享冷却的实现

如果需要实现"火球术和陨石术共享冷却"：

**方法一：使用相同的冷却Tag（推荐）**

```
  Duration: 8.0秒
  GrantedTags: Ability.Cooldown.FireSpells

火球术的GA配置：
  CooldownTags: { Ability.Cooldown.FireSpells }
  CooldownEffectClass: GE_Cooldown_FireSpells

陨石术的GA配置：
  CooldownTags: { Ability.Cooldown.FireSpells }    # 使用相同标签！
  CooldownEffectClass: GE_Cooldown_FireSpells       # 使用相同GE！

```

```
# - 自身冷却8秒

  Duration: 8.0秒
```

### 3.2.5 查询冷却剩余时间（C++实现）

在 `RPGGameplayAbility.cpp` 中添加冷却查询函数：

```cpp
// ====================================================================
// 添加到 URPGGameplayAbility 的公共函数中
// ====================================================================

/**
 * 获取技能的剩余冷却时间
 * @param Handle       — 技能句柄
 * @param ActorInfo    — 角色信息
 * @return             — 剩余冷却时间（秒），0表示不在冷却中
 *
 * 使用示例（在蓝图或C++的UI代码中）:
 *   float Remaining = Ability->GetCooldownTimeRemaining(Handle, ActorInfo);
 *   if (Remaining > 0) { // 显示冷却倒计时 }
 */
UFUNCTION(BlueprintCallable, Category = "Ability|Cooldown")
float GetCooldownTimeRemaining(
    const FGameplayAbilitySpecHandle& Handle,
    const FGameplayAbilityActorInfo* ActorInfo) const
{
    // ================================================================
    // 方法一：使用CooldownTags查询（推荐）
    // ================================================================

    if (!ActorInfo || !ActorInfo->AbilitySystemComponent.IsValid())
    {
        return 0.0f;
    }

    if (CooldownTags.IsEmpty())
    {
        // 技能没有配置冷却标签，无需检查
        return 0.0f;
    }

    UAbilitySystemComponent* ASC = ActorInfo->AbilitySystemComponent.Get();

    // ================================================================
    // 查询冷却剩余时间
    // ================================================================
    // 方法：遍历所有冷却标签，找到拥有该标签的Active GE
    // 然后查询该GE的剩余持续时间

    float LongestCooldown = 0.0f;

    // 获取所有活跃的GE
    TArray<FActiveGameplayEffectHandle> ActiveEffects;
    ASC->GetActiveEffects(FGameplayEffectQuery::MakeQuery_MatchAnyOwningTags(CooldownTags),
        ActiveEffects);

    for (const FActiveGameplayEffectHandle& EffectHandle : ActiveEffects)
    {
        // 获取该GE的剩余时间和总持续时间
        float TimeRemaining = 0.0f;
        float Duration = 0.0f;

        if (ASC->GetGameplayEffectRemainingDuration(EffectHandle, TimeRemaining, Duration))
        {
            // 取最长的冷却时间（如果有多个冷却标签生效）
            LongestCooldown = FMath::Max(LongestCooldown, TimeRemaining);
        }
    }

    return LongestCooldown;
}

/**
 * 获取冷却进度（0.0~1.0），用于UI进度条
 * @return 0.0 = 冷却完成，1.0 = 冷却刚开始
 */
UFUNCTION(BlueprintCallable, Category = "Ability|Cooldown")
float GetCooldownProgress(
    const FGameplayAbilitySpecHandle& Handle,
    const FGameplayAbilityActorInfo* ActorInfo) const
{
    // 查询冷却总时长和剩余时间
    float TotalDuration = GetCooldownTotalDuration();
    float TimeRemaining = GetCooldownTimeRemaining(Handle, ActorInfo);

    if (TotalDuration <= 0.0f)
    {
        return 0.0f;  // 没有冷却
    }

    return TimeRemaining / TotalDuration;
}

/**
 * 获取冷却总时长（从CooldownGE中读取）
 */
UFUNCTION(BlueprintCallable, Category = "Ability|Cooldown")
float GetCooldownTotalDuration() const
{
    // 从CooldownEffectClass中读取默认持续时间
    if (CooldownEffectClass)
    {
        // 获取GE的CDO
        const UGameplayEffect* DefaultGE = CooldownEffectClass->GetDefaultObject<UGameplayEffect>();
        if (DefaultGE)
        {
            // 从GE定义中获取持续时间
            return DefaultGE->DurationMagnitude.GetValue();
        }
    }

    // 如果没有配置冷却GE，使用CooldownDuration参数
    return CooldownDuration.GetValue();
}
```

## 3.3 消耗系统详解

### 3.3.1 消耗的核心机制

```
技能激活时的消耗检查流程：

1. GA::ActivateAbility 被调用
2. GA::CommitAbility 被调用
3. CommitAbility 内部检查消耗：
   a. 读取 CostGameplayEffectClass
   b. 模拟应用消耗GE（不真正应用）
   c. 检查GE修改的属性是否会降到0以下
   d. 如果会 → CommitAbility 返回 false
   e. 如果不会 → 真正应用消耗GE
4. 消耗GE修改对应的资源属性（Mana、Stamina等）
5. 属性变化触发UI更新
```

### 3.3.2 消耗GameplayEffect的示例配置

在编辑器中创建 `GE_Cost_Fireball`：

```yaml
GameplayEffect: GE_Cost_Fireball
  类型: Instant（立即生效）
  等级: 1

  Modifiers:
    - 属性: RPGAttributeSet.Mana
      操作: Add（加上负值 = 减少）
      量级: -25.0
      # 注意：在编辑器中配置Magnitude时，
      # 选择 "Scalable Float" → 值填入负数
      # 或者使用MMC动态计算消耗

    - 属性: RPGAttributeSet.Stamina
      操作: Add
      量级: -5.0
      # 同时消耗5点体力
```

### 3.3.3 CommitAbility的消耗检查源码解析

了解 `CommitAbility` 内部做了什么，有助于理解为什么它这么重要：

```cpp
// ====================================================================
// 以下来自 UGameplayAbility::CommitAbility 的简化逻辑（非源码，是概念解释）
// ====================================================================

bool UGameplayAbility::CommitAbility(
    const FGameplayAbilitySpecHandle Handle,
    const FGameplayAbilityActorInfo* ActorInfo,
    const FGameplayAbilityActivationInfo ActivationInfo)
{
    // 第一步：检查冷却
    if (!CheckCooldown(Handle, ActorInfo))
    {
        UE_LOG(LogTemp, Warning, TEXT("CommitAbility: Cooldown check failed"));
        return false;  // ❌ 冷却中
    }

    // 第二步：检查消耗
    if (!CheckCost(Handle, ActorInfo))
    {
        UE_LOG(LogTemp, Warning, TEXT("CommitAbility: Cost check failed"));
        return false;  // ❌ 消耗不足
    }

    // 第三步：应用冷却
    ApplyCooldown(Handle, ActorInfo, ActivationInfo);

    // 第四步：应用消耗
    ApplyCost(Handle, ActorInfo, ActivationInfo);

    // 第五步：标记技能已提交
    // ...

    return true;  // ✅ 提交成功
}

bool UGameplayAbility::CheckCost(const FGameplayAbilitySpecHandle Handle,
    const FGameplayAbilityActorInfo* ActorInfo)
{
    if (!CostGameplayEffectClass)
    {
        return true;  // 没有消耗GE，总是通过
    }

    UAbilitySystemComponent* ASC = ActorInfo->AbilitySystemComponent.Get();
    if (!ASC)
    {
        return false;
    }

    // ================================================================
    // 关键：CanApplyAttributeModifiers 模拟应用GE
    // ================================================================
    // 这个函数会"假装"应用消耗GE，检查应用后属性值是否会变成负数
    // 如果会，返回false
    // 注意：这个检查只适用于数值属性（如Mana、Stamina）
    // 对于标签类型的消耗（如"需要持有某物品"），需要在GA中手动检查

    FGameplayEffectSpecHandle SpecHandle = MakeOutgoingGameplayEffectSpec(
        CostGameplayEffectClass, GetAbilityLevel());

    if (!SpecHandle.IsValid())
    {
        return false;
    }

    // 模拟应用：检查但不真实修改属性
    return ASC->CanApplyAttributeModifiers(
        *SpecHandle.Data.Get(),
        ASC
    );
}
```

```cpp
 */

    if (!AbilitySystemComponent)
    {
        return false;
    }

    // --- 找到技能规格 ---
    FGameplayAbilitySpec* Spec = AbilitySystemComponent->FindAbilitySpecFromClass(AbilityClass);
    if (!Spec)
    {
        return false;
    }

    // --- 尝试激活 ---
    if (AbilitySystemComponent->TryActivateAbility(Spec->Handle))
    {
        // ✅ 成功激活
        return true;
    }

    // ❌ 激活失败 → 提供反馈

    // --- 检查失败原因 ---
    UGameplayAbility* AbilityCDO = AbilityClass->GetDefaultObject<UGameplayAbility>();
    URPGGameplayAbility* RPGAbility = Cast<URPGGameplayAbility>(AbilityCDO);

    if (RPGAbility)
    {
        const FGameplayTagContainer& BlockedTags = RPGAbility->ActivationBlockedTags;

        // 原因1：被阻止（眩晕、沉默等）
        if (AbilitySystemComponent->HasAnyMatchingGameplayTags(BlockedTags))
        {
            // UI反馈：显示"被控制，无法使用技能"
            OnSkillBlockedByStatus(RPGAbility);
            return false;
        }

        // 原因2：冷却中
        const FGameplayTagContainer& CooldownTags = RPGAbility->CooldownTags;
        if (AbilitySystemComponent->HasAnyMatchingGameplayTags(CooldownTags))
        {
            // UI反馈：技能图标闪烁提示冷却中
            OnSkillOnCooldown(RPGAbility);
            return false;
        }

        // 原因3：消耗不足
        // 检查法力是否足够
        URPGAttributeSet* AttrSet = GetRPGAttributeSet();
        if (AttrSet && RPGAbility->ManaCost.GetValue() > AttrSet->GetMana())
        {
            // UI反馈：法力条闪烁红色 + "法力不足"提示
            OnInsufficientResource(RPGAbility, TEXT("法力不足"));
            return false;
        }

        // 检查体力是否足够
        if (AttrSet && RPGAbility->StaminaCost.GetValue() > AttrSet->GetStamina())
        {
            // UI反馈：体力条闪烁红色 + "体力不足"提示
            OnInsufficientResource(RPGAbility, TEXT("体力不足"));
            return false;
        }
    }

    return false;
}
```

### 3.4.1 标准冷却GE模板

```yaml
# GE_Cooldown_模板

GameplayEffect: GE_Cooldown_[SkillName]

  # --- 基本配置 ---

    Calculation Type: Scalable Float # 可缩放的浮点数
    Value: 8.0                       # 8秒冷却（可在GA中按等级缩放）
  
  # --- 标签配置 ---

        - Ability.Cooldown.[SkillName]

      (Empty)
    
    Remove Gameplay Effects with Tags: # 移除旧冷却（如果有的话）
      (通常不需要，每个冷却独立运行)
  
  # --- 修改器 ---
  Modifiers:
    # 冷却GE通常不需要修改属性
    (Empty)
  
  # --- 堆叠设置 ---
  Stacking:
    Stacking Type: None              # 冷却不堆叠（每个技能独立冷却）

    Display Name: "冷却中"
```

## 3.5 技能资源管理最佳实践

### 3.5.1 设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **统一接口** | 所有技能使用相同的消耗机制（CostGE） | 不会有些技能直接减Mana，有些用GE |
| **属性驱动** | 消耗量通过AttributeSet读取，不硬编码 | 消耗= 基础消耗 × (1 - 冷却缩减%) |
| **标签化** | 每个冷却有独立的GameplayTag | 便于UI监听和查询 |
| **可扩展** | 冷却和消耗支持等级缩放 | Lv1:8秒, Lv2:7秒, Lv3:6秒 |
| **反馈友好** | 激活失败时提供明确的失败原因 | "法力不足" vs 默默失败 |

### 3.5.2 常见反模式

```cpp
// ❌ 反模式1：在GA中手动减Mana
void UGA_BadExample::ActivateAbility(...)
{
    // ❌ 错误做法：直接修改属性
    AttributeSet->SetMana(AttributeSet->GetMana() - 20.0f);

    // 问题：
    // 1. GAS不知道Mana被消耗了（没有通过GE）
    // 2. 网络复制不会正确触发
    // 3. UI不会自动更新
    // 4. 无法被其他系统监听
}

// ✅ 正确做法：使用CostGE
void UGA_GoodExample::ActivateAbility(...)
{
    // ✅ 使用CommitAbility，内部通过GE消耗资源
    if (!CommitAbility(Handle, ActorInfo, ActivationInfo))
    {
        EndAbility(Handle, ActorInfo, ActivationInfo, true, true);
        return;
    }
    // 消耗已通过GE正确应用
}

// ❌ 反模式2：硬编码冷却时间
void UGA_BadExample2::ActivateAbility(...)
{
    // ❌ 错误：没有使用冷却GE，手动管理冷却
    GetWorld()->GetTimerManager().SetTimer(CooldownTimer, 8.0f, false);
    bIsOnCooldown = true;

    // 问题：
    // 1. 没有GameplayTag标记冷却状态
    // 2. 其他系统无法感知冷却
    // 3. 无法通过GE修改冷却时间（如冷却缩减Buff）
}

// ✅ 正确做法：使用CooldownGE
void UGA_GoodExample2::ActivateAbility(...)
{
    // ✅ CommitAbility会处理冷却GE的创建和应用
    CommitAbility(Handle, ActorInfo, ActivationInfo);
    // 冷却标签已添加，其他系统可查询
}

// ❌ 反模式3：在UI中硬编码冷却查询
void UCooldownWidget::NativeTick(...)
{
    // ❌ 每帧查询属性表
    float CD = GetAbilityCD(AbilityName);
    UpdateUI(CD);

    // 问题：
    // 1. 轮询浪费性能
    // 2. 硬编码技能名，不易扩展
}

// ✅ 正确做法：监听GameplayTag变化
void UCooldownWidget::NativeConstruct()
{
    // ✅ 绑定到ASC的GameplayTag变化委托
    ASC->RegisterGameplayTagEvent(
        CooldownTag,
        EGameplayTagEventType::NewOrRemoved
    ).AddUObject(this, &UCooldownWidget::OnCooldownTagChanged);

    // 只有当冷却标签真正变化时才更新UI
}
```

### 3.5.3 冷却缩减（CDR）系统的实现思路

```cpp
// ====================================================================
// 概念：冷却缩减属性
// ====================================================================

// 1. 在RPGAttributeSet中添加属性：
//    FGameplayAttributeData CooldownReduction;  // 0.0~1.0，0.4=40%冷却缩减

// 2. 创建CDR的MMC（CoolDownReductionCalculation）:
//    在CalculateBaseMagnitude中读取CooldownReduction属性
//    应用于冷却GE的持续时间修改

// 3. 冷却GE的Modifier使用MMC计算实际冷却时间：
//    Modifier:
//      - Attribute: (特殊：修改GE自身Duration)
//      - Modifier Op: Override
//      - Magnitude:
//          Calculation Type: Custom Calculation Class
//          Class: CoolDownReductionCalculation

// 4. MMC内部计算：
//    ActualCooldown = BaseCooldown * (1.0 - CooldownReduction)
//    例如：8秒冷却，40%CDR → 4.8秒
```
