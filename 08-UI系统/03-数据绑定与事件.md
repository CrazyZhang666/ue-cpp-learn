## 1. BindWidget深入：绑定UMG控件到C++变量

### 1.1 回顾BindWidget

在上一章我们学习了 `BindWidget` 的基本用法。这里做一个快速回顾并进行更深入的讲解。

```cpp
// BindWidget的核心规则（再强调一遍！）：
// 1. C++变量名 == 蓝图控件名（大小写敏感）
// 2. 变量类型 == 控件类型
// 3. 必须用 meta = (BindWidget) 标记
// 4. 必须是 TObjectPtr<T> 类型

// 正确的示例：
UPROPERTY(meta = (BindWidget))
TObjectPtr<UButton> MyButton;  // ← 名字是 "MyButton"，类型是 UButton
                                // 蓝图中必须有名为 "MyButton" 的Button控件
```

### 1.3 完整的BindWidget示例

下面是一个完整的头文件，展示了如何在一个Widget中绑定多种控件：

```cpp
// 文件：MyCompleteWidget.h
#pragma once

#include "CoreMinimal.h"
#include "Blueprint/UserWidget.h"

// ==================== 引入所需控件的头文件 ====================
#include "Components/Button.h"              // Button控件
#include "Components/TextBlock.h"           // TextBlock控件
#include "Components/ProgressBar.h"         // ProgressBar控件
#include "Components/Slider.h"              // Slider控件
#include "Components/Image.h"               // Image控件
#include "Components/ScrollBox.h"           // ScrollBox控件
#include "Components/EditableTextBox.h"     // EditableTextBox控件
#include "Components/CheckBox.h"            // CheckBox控件
#include "Components/CanvasPanel.h"         // CanvasPanel容器
#include "Components/HorizontalBox.h"       // HorizontalBox容器
#include "Components/VerticalBox.h"         // VerticalBox容器
#include "Components/Overlay.h"             // Overlay容器

#include "MyCompleteWidget.generated.h"

UCLASS()
class YOURPROJECT_API UMyCompleteWidget : public UUserWidget
{
    GENERATED_BODY()

public:
    // ==================== 必须绑定的控件 ====================

    // 一个开始按钮 — 在蓝图中必须存在同名控件
    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UButton> StartButton;

    // 一个标题文字 — 在蓝图中必须存在同名控件
    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UTextBlock> TitleText;

    // 一个血条 — 在蓝图中必须存在同名控件
    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UProgressBar> HealthProgressBar;

    // 一个音量滑块 — 在蓝图中必须存在同名控件
    UPROPERTY(meta = (BindWidget))
    TObjectPtr<USlider> VolumeSlider;

    // 一个角色头像图片 — 在蓝图中必须存在同名控件
    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UImage> AvatarImage;

    // ==================== 可选绑定的控件（有就绑定，没有也行）====================

    // 一个可选的调试文字 — 蓝图中可以不存在
    UPROPERTY(meta = (BindWidgetOptional))
    TObjectPtr<UTextBlock> DebugText;

    // 一个可选的确认按钮
    UPROPERTY(meta = (BindWidgetOptional))
    TObjectPtr<UButton> ConfirmButton;

    // ==================== 容器控件绑定 ====================
    // 容器也可以BindWidget！这样在C++中可以动态添加子控件

    // 物品列表容器
    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UVerticalBox> ItemListBox;

    // 技能栏容器
    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UHorizontalBox> SkillBarBox;

protected:
    virtual void NativeConstruct() override;
};
```

### 1.4 常见绑定失败原因

这里汇总了所有可能导致 `BindWidget` 失败的原因和解决方案：

| 失败原因                  | 现象                          | 解决方案                                        |
| ------------------------- | ----------------------------- | ----------------------------------------------- |
| 变量名和控件名不一致      | 运行时变量为nullptr           | 检查大小写，确保完全一致                        |
| 类型不匹配                | 运行时变量为nullptr           | 检查类型（UTextBlock不能绑TextBlock之外的控件） |
| 蓝图中没有创建控件        | 运行时变量为nullptr           | 检查蓝图中是否有对应控件                        |
| 使用StaticClass而非蓝图类 | 所有BindWidget变量都为nullptr | 必须使用蓝图类（TSubclassOf方式）               |
| 在构造函数中访问控件      | 控件还未创建                  | 改在NativeConstruct中访问                       |
| 控件放在错误的父容器下    | BindWidget找不到              | 控件必须在Widget的根层级或子层级中              |

### 1.5 调试技巧：检查绑定是否成功

```cpp
void UMyCompleteWidget::NativeConstruct()
{
    Super::NativeConstruct();

    // ==================== 调试方法1：逐个检查控件是否绑定成功 ====================
    // 使用checkf在开发阶段强制检查（Debug模式下会触发断言）
    // 只在非发布版本中检查！
    #if WITH_EDITOR
    {
        checkf(StartButton, TEXT("BindWidget失败：StartButton为nullptr！"));
        checkf(TitleText, TEXT("BindWidget失败：TitleText为nullptr！"));
        checkf(HealthProgressBar, TEXT("BindWidget失败：HealthProgressBar为nullptr！"));
        checkf(VolumeSlider, TEXT("BindWidget失败：VolumeSlider为nullptr！"));
    }
    #endif

    // ==================== 调试方法2：使用IsValid检查 ====================
    // IsValid同时检查指针是否为nullptr和对象是否被标记为待销毁
    if (!IsValid(StartButton))
    {
        // UE_LOG会输出Error级别的日志，在编辑器的Output Log中能看到红色错误信息
        UE_LOG(LogTemp, Error, TEXT("[UMyCompleteWidget] StartButton绑定失败！"));
        return; // 如果核心控件绑定失败，提前返回避免后续崩溃
    }

    // ==================== 调试方法3：使用ensure和ensureMsgf ====================
    // ensure：条件为false时触发断言，但不会崩溃（只在编辑器中警告）
    // 比checkf更安全，适合可能为nullptr的可选控件
    ensureMsgf(TitleText, TEXT("TitleText绑定失败，请检查蓝图中的控件命名"));
    ensureMsgf(HealthProgressBar, TEXT("HealthProgressBar绑定失败"));

    // ==================== 调试方法4：可选控件的安全检查 ====================
    // 对于BindWidgetOptional绑定的控件，执行操作前必须检查
    if (DebugText)
    {
        // 可选控件存在，安全使用
        DebugText->SetText(FText::FromString(TEXT("调试模式开启")));
    }
    else
    {
        // 可选控件不存在，跳过相关逻辑（不报错）
        UE_LOG(LogTemp, Verbose, TEXT("DebugText未绑定，跳过调试文字显示"));
    }

    UE_LOG(LogTemp, Log, TEXT("所有控件绑定检查完成"));
}
```

**四种检查方式的区别：**

| 方法                        | 失败时行为                      | 何时使用                           |
| --------------------------- | ------------------------------- | ---------------------------------- |
| `check()` / `checkf()`      | 立即崩溃（仅Debug/Development） | 开发阶段强制检查（必须存在的东西） |
| `ensure()` / `ensureMsgf()` | 警告 + 调用栈（不会崩溃）       | 可能为nullptr但应该存在的控件      |
| `verify()` / `verifyf()`    | 和check一样但不跳过Shipping版本 | 核心功能，任何版本都要检查         |
| `if (ptr)` 安全检查         | 静默跳过                        | BindWidgetOptional绑定的可选控件   |

## 2. UButton的OnClicked事件处理

### 2.1 事件绑定三要素

在C++中，为按钮绑定点击事件需要确定三样东西：

```
要素1：绑定的目标对象（this — 当前Widget实例）
要素2：绑定的目标函数（&UMyWidget::OnMyButtonClicked — 点击后调用的函数）
要素3：绑定操作（AddDynamic — 执行绑定动作）
```

### 2.2 OnClicked的基本绑定方式

```cpp
// 文件：MyMenuWidget.h
#pragma once

#include "CoreMinimal.h"
#include "Blueprint/UserWidget.h"
#include "Components/Button.h"   // ← Button头文件
#include "Components/TextBlock.h"
#include "MyMenuWidget.generated.h"

UCLASS()
class YOURPROJECT_API UMyMenuWidget : public UUserWidget
{
    GENERATED_BODY()

protected:
    virtual void NativeConstruct() override;

private:
    // ==================== 绑定的控件 ====================
    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UButton> StartGameButton;

    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UButton> SettingsButton;

    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UButton> QuitButton;

    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UTextBlock> WelcomeText;

    // ==================== 事件处理函数 ====================
    // 这些函数会在对应的按钮被点击时自动调用
    // 函数签名必须是：void FunctionName()（无参数，无返回值）

    // UFUNCTION() 宏让这个函数可以被UE的委托系统调用
    UFUNCTION()
    void OnStartGameClicked();   // 开始游戏按钮的处理函数

    UFUNCTION()
    void OnSettingsClicked();    // 设置按钮的处理函数

    UFUNCTION()
    void OnQuitClicked();        // 退出按钮的处理函数
};
```

```cpp
// 文件：MyMenuWidget.cpp
#include "MyMenuWidget.h"
#include "Kismet/GameplayStatics.h"

void UMyMenuWidget::NativeConstruct()
{
    Super::NativeConstruct();

    // ==================== 绑定按钮点击事件 ====================

    // 方式A：逐个绑定（清晰、推荐新手使用）
    if (StartGameButton)
    {
        // AddDynamic 是 UE 委托系统的绑定函数
        // 参数1：绑定的对象（this — 当前Widget）
        // 参数2：要调用的成员函数（取函数指针）
        StartGameButton->OnClicked.AddDynamic(this, &UMyMenuWidget::OnStartGameClicked);
    }

    if (SettingsButton)
    {
        SettingsButton->OnClicked.AddDynamic(this, &UMyMenuWidget::OnSettingsClicked);
    }

    if (QuitButton)
    {
        QuitButton->OnClicked.AddDynamic(this, &UMyMenuWidget::OnQuitClicked);
    }

    UE_LOG(LogTemp, Log, TEXT("所有按钮事件绑定完成"));
}

// ==================== 事件处理函数的实现 ====================

void UMyMenuWidget::OnStartGameClicked()
{
    UE_LOG(LogTemp, Log, TEXT("开始游戏按钮被点击！"));

    // 这里写开始游戏的逻辑：
    // 例如：打开关卡、关闭菜单、开始游戏等
    // UGameplayStatics::OpenLevel(GetWorld(), TEXT("GameLevel"));
}

void UMyMenuWidget::OnSettingsClicked()
{
    UE_LOG(LogTemp, Log, TEXT("设置按钮被点击！"));

    // 这里写打开设置界面的逻辑：
    // 例如：显示设置面板、隐藏主菜单等
}

void UMyMenuWidget::OnQuitClicked()
{
    UE_LOG(LogTemp, Log, TEXT("退出按钮被点击！"));

    // 退出游戏
    if (APlayerController* PC = GetOwningPlayer())
    {
        // ConsoleCommand 可以执行控制台命令
        // "quit" 命令会退出游戏
        PC->ConsoleCommand(TEXT("quit"));
    }
}
```

### 2.3 OnClicked事件的其他绑定方式

```cpp
// ==================== 方式B：使用AddUniqueDynamic（防止重复绑定）====================
// AddUniqueDynamic 会检查是否已经绑定了相同的函数，避免重复
StartGameButton->OnClicked.AddUniqueDynamic(this, &UMyMenuWidget::OnStartGameClicked);
// 如果多次调用AddUniqueDynamic绑定同一个函数，只会生效一次

// ==================== 方式C：使用Lambda表达式（C++11）====================
// 适合简单的、不需要复用的逻辑
SettingsButton->OnClicked.AddDynamic(this, [this]()
{
    // 直接在Lambda中写处理逻辑
    UE_LOG(LogTemp, Log, TEXT("设置按钮被点击（Lambda）！"));
    // 可以捕获this，访问Widget的成员变量和方法
    WelcomeText->SetText(FText::FromString(TEXT("进入设置...")));
});
// ⚠️ 注意：Lambda中的this要小心生命周期！确保Widget在Lambda执行时还存在

// ==================== 方式D：使用AddWeakLambda（更安全的Lambda）====================
// AddWeakLambda 会自动检查对象是否还有效
SettingsButton->OnClicked.AddWeakLambda(this, [this]()
{
    // 如果this（Widget）已经被销毁，这个Lambda不会被调用
    UE_LOG(LogTemp, Log, TEXT("安全的Lambda调用"));
});

// ==================== 方式E：绑定到UFUNCTION标记的函数（蓝图可调用）====================
// 如果你想让蓝图也能绑定这个事件，使用UFUNCTION
UFUNCTION()
void HandleButtonClick(); // 在头文件中声明

MyButton->OnClicked.AddDynamic(this, &UMyWidget::HandleButtonClick);
```

### 2.4 解绑事件（RemoveDynamic / RemoveAll）

```cpp
void UMyMenuWidget::NativeDestruct()
{
    // ==================== 解绑事件 ====================
    // 在Widget销毁前，解除按钮事件的绑定（防止悬空指针）

    if (StartGameButton)
    {
        // 方式1：解除特定函数的绑定
        StartGameButton->OnClicked.RemoveDynamic(this, &UMyMenuWidget::OnStartGameClicked);
    }

    if (SettingsButton)
    {
        // 方式2：解除某个对象的所有绑定
        SettingsButton->OnClicked.RemoveAll(this);
    }

    // 方式3：清除该委托的所有绑定（不推荐，可能影响其他对象）
    // QuitButton->OnClicked.Clear();

    Super::NativeDestruct();
}
```

```
✅ 在NativeDestruct中解绑事件是一个好习惯
✅ 使用AddUniqueDynamic防止重复绑定
✅ Lambda中使用AddWeakLambda更安全
❌ 忘记在NativeDestruct中解绑 → 可能导致悬空指针崩溃
❌ 使用.Clear()会清除所有对象的绑定 → 可能误伤其他代码
```

### 2.5 OnClicked事件的完整参数

实际上 `OnClicked` 是一个多播委托，它的更多操作：

```cpp
// 检查是否已绑定
if (StartGameButton->OnClicked.IsBound())
{
    UE_LOG(LogTemp, Log, TEXT("OnClicked已绑定"));
}

// 手动执行（就像模拟点击）
StartGameButton->OnClicked.Broadcast();  // 触发所有绑定的函数

// 检查是否已绑定到特定函数
bool bIsBound = StartGameButton->OnClicked.IsAlreadyBound(
    this, &UMyMenuWidget::OnStartGameClicked
);
```

## 3. 在C++中更新TextBlock和ProgressBar

### 3.1 更新TextBlock的各种方式

TextBlock是最常用的显示控件之一，掌握它的多种更新方式非常重要：

```cpp
// 文件：演示代码（放在任何Widget的NativeConstruct或自定义方法中）

// ==================== 方式1：使用SetText + FText::FromString ====================
// 最简单直接的方式 — 从FString创建FText
MyTextBlock->SetText(FText::FromString(TEXT("欢迎来到游戏！")));
// TEXT() 宏：确保字符串在Unicode环境下正确处理

// ==================== 方式2：使用FText::Format（格式化字符串）====================
// 适合需要拼接变量数值的场景
int32 PlayerLevel = 15;
int32 PlayerExp = 340;
int32 ExpToNextLevel = 500;

MyTextBlock->SetText(FText::Format(
    FText::FromString(TEXT("等级: {0}  经验: {1}/{2}")),
    // 用 {0} {1} {2} 作为占位符
    FText::AsNumber(PlayerLevel),   // {0} = 15
    FText::AsNumber(PlayerExp),     // {1} = 340
    FText::AsNumber(ExpToNextLevel) // {2} = 500
));
// 结果显示："等级: 15  经验: 340/500"

// ==================== 方式3：使用NSLOCTEXT（本地化文本）====================
// NSLOCTEXT用于需要多语言支持的文本
MyTextBlock->SetText(NSLOCTEXT("MyUI", "Welcome", "欢迎"));
// 参数1：命名空间（用于组织和查找文本）
// 参数2：键（唯一标识这段文本）
// 参数3：默认文本（本语言的文字）

// ==================== 方式4：使用LOCTEXT（简写版）====================
// 在函数内部使用LOCTEXT更方便
MyTextBlock->SetText(LOCTEXT("SaveSuccess", "保存成功！"));

// ==================== 方式5：FText::AsNumber（数字转文本）====================
MyTextBlock->SetText(FText::AsNumber(999));       // 显示 "999"
MyTextBlock->SetText(FText::AsPercent(0.75f));    // 显示 "75%"
MyTextBlock->SetText(FText::AsCurrencyBase(100, TEXT("USD"))); // 显示对应货币格式

// ==================== 方式6：FText::AsTime（时间格式）====================
float TimeInSeconds = 125.5f;
MyTextBlock->SetText(FText::AsTimespan(FTimespan::FromSeconds(TimeInSeconds)));
// 显示 "02:05"
```

### 3.2 FText vs FString vs FName

**非常重要！** 这三个字符串类型的区别必须搞清楚：

| 类型      | 用途             | 是否可本地化 | 性能     | 使用场景                 |
| --------- | ---------------- | ------------ | -------- | ------------------------ |
| `FText`   | 显示给玩家的文字 | **是**       | 较慢     | UI文字、对话、道具说明   |
| `FString` | 通用字符串处理   | 否           | 一般     | 文件路径、数据处理、日志 |
| `FName`   | 标识符/键        | 否           | **最快** | 资源名、标签、查找键     |

```cpp
// 转换速查表：
FString MyString = TEXT("Hello");
FName MyName = FName(TEXT("Hello"));
FText MyText = FText::FromString(TEXT("Hello"));

// FString → FText
FText Text1 = FText::FromString(MyString);

// FText → FString
FString String1 = MyText.ToString();

// FString → FName
FName Name1 = FName(*MyString);

// FName → FString
FString String2 = MyName.ToString();

// FName → FText（间接）
FText Text2 = FText::FromName(MyName);
// 或
FText Text3 = FText::FromString(MyName.ToString());

// FText → FName（不支持直接转换，需要先转FString）
// FName不支持直接从FText转换（因为FText可能包含本地化信息）
```

```
✅ UI相关的所有文字用FText
✅ 文件路径、日志、调试用FString
✅ 资源名、快速查找用FName
❌ 在UI中直接使用FString（SetText不接受FString参数，需要转换）
❌ 频繁创建FText（用NSLOCTEXT/LOCTEXT静态创建）
```

## 4. Slate属性绑定（C++版UMG Bind）

### 4.1 什么是属性绑定

**一句话理解：** 属性绑定可以让TextBlock、ProgressBar等控件的显示内容自动从某个函数获取，而不需要手动每帧调用SetText/SetPercent。

```
手动更新 vs 属性绑定：

手动更新（每帧调用）：
    NativeTick() {
        HealthText->SetText(GetHealthString());  ← 每帧手动设置
    }

属性绑定（自动同步）：
    HealthText->TextDelegate.BindDynamic(this, &GetHealthFunc);
    // ↑ 只需要绑定一次，之后引擎自动每帧调用GetHealthFunc更新文字
```

### 4.2 TextBlock的属性绑定

```cpp
// 文件：MyHUD.h
#pragma once

#include "CoreMinimal.h"
#include "Blueprint/UserWidget.h"
#include "Components/TextBlock.h"
#include "MyHUD.generated.h"

UCLASS()
class YOURPROJECT_API UMyHUD : public UUserWidget

protected:

private:
    // ==================== 绑定的控件 ====================

    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UTextBlock> ScoreText;

    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UTextBlock> TimeText;

    // ==================== 属性绑定函数 ====================
    // 这些函数的返回值会自动显示在对应的TextBlock上
    // 重要：函数签名必须是 FText FunctionName()（返回FText，无参数）

    // 获取血量文字的函数
    UFUNCTION()
    FText GetHealthText() const;

    // 获取分数文字的函数
    UFUNCTION()
    FText GetScoreText() const;

    // 获取时间文字的函数
    UFUNCTION()
    FText GetTimeText() const;

    // ==================== 数据源 ====================

    float MaxHealth = 100.0f;
```

```cpp
// 文件：MyHUD.cpp
#include "MyHUD.h"

void UMyHUD::NativeConstruct()
{
    Super::NativeConstruct();

    // ==================== 属性绑定 ====================
    // TextDelegate是UTextBlock内部的一个委托
    // BindDynamic将委托绑定到指定的成员函数
    // 绑定后，引擎每帧自动调用绑定函数获取最新文字

    if (HealthText)
    {
        // BindDynamic：绑定动态委托
        // 参数1：this（绑定的对象）
        // 参数2：&UMyHUD::GetHealthText（绑定的成员函数）
        HealthText->TextDelegate.BindDynamic(this, &UMyHUD::GetHealthText);
    }

    if (ScoreText)
    {
        ScoreText->TextDelegate.BindDynamic(this, &UMyHUD::GetScoreText);
    }

    if (TimeText)
    {
        TimeText->TextDelegate.BindDynamic(this, &UMyHUD::GetTimeText);
    }

    UE_LOG(LogTemp, Log, TEXT("TextBlock属性绑定完成"));
}

// ==================== 绑定函数的实现 ====================
// 这些函数不需要手动调用，引擎会自动调用它们！

FText UMyHUD::GetHealthText() const
{
    // 返回当前血量的格式化文字
    return FText::Format(
        FText::FromString(TEXT("血量: {0}/{1}")),
        FText::AsNumber(FMath::RoundToInt(CurrentHealth)),
        FText::AsNumber(FMath::RoundToInt(MaxHealth))
    );
}

FText UMyHUD::GetScoreText() const
{
    // 返回当前分数的格式化文字
    return FText::Format(
        FText::FromString(TEXT("分数: {0}")),
        FText::AsNumber(CurrentScore)
    );
}

FText UMyHUD::GetTimeText() const
{
    // 将秒数转换为 分:秒 格式
    int32 Minutes = FMath::FloorToInt(GameTime / 60.0f);
    int32 Seconds = FMath::FloorToInt(FMath::Fmod(GameTime, 60.0f));

    return FText::Format(
        FText::FromString(TEXT("{0}:{1}")),
        FText::AsNumber(Minutes),
        FText::AsNumber(Seconds)
    );
}
```

### 4.4 属性绑定的完整示例

```cpp
// 一个完整的属性绑定示例：健康指示器

// 头文件中的绑定函数声明：
UFUNCTION()
float GetHealthPercent() const;     // 返回血量百分比（给ProgressBar用）

UFUNCTION()
FText GetHealthText() const;        // 返回血量文字（给TextBlock用）

UFUNCTION()
FLinearColor GetHealthColor() const; // 返回血量颜色（给Image用）

// 实现：
float UMyWidget::GetHealthPercent() const
{
    float Ratio = (MaxHealth > 0.0f) ? (CurrentHealth / MaxHealth) : 0.0f;
    return FMath::Clamp(Ratio, 0.0f, 1.0f);  // 限制在 0.0~1.0
}

FText UMyWidget::GetHealthText() const
{
    return FText::Format(
        NSLOCTEXT("HUD", "HealthFormat", "{0}/{1}"),
        FText::AsNumber(FMath::RoundToInt(CurrentHealth)),
        FText::AsNumber(FMath::RoundToInt(MaxHealth))
    );
}

FLinearColor UMyWidget::GetHealthColor() const
{
    float Ratio = GetHealthPercent();

    if (Ratio > 0.65f)
        return FLinearColor::Green;
    else if (Ratio > 0.35f)
        return FLinearColor::Yellow;
    else
        return FLinearColor::Red;
}

// 在NativeConstruct中绑定：
void UMyWidget::NativeConstruct()
{
    Super::NativeConstruct();

    // ProgressBar绑定到GetHealthPercent函数 → 自动更新进度
    if (HealthBar)
    {
        HealthBar->PercentDelegate.BindDynamic(this, &UMyWidget::GetHealthPercent);
    }

    // TextBlock绑定到GetHealthText函数 → 自动更新文字
    if (HealthText)
    {
        HealthText->TextDelegate.BindDynamic(this, &UMyWidget::GetHealthText);
    }

    // Image绑定到GetHealthColor函数 → 自动更新颜色
    if (HealthColorImage)
    {
        HealthColorImage->ColorAndOpacityDelegate.BindDynamic(this, &UMyWidget::GetHealthColor);
    }
}
```

### 4.5 属性绑定 vs 手动更新

| 对比维度 | 属性绑定                     | 手动更新（每帧SetText）          |
| -------- | ---------------------------- | -------------------------------- |
| 代码量   | 少（绑定一次，自动更新）     | 多（每帧手动调用）               |
| 性能     | 好（引擎优化过的路径）       | 一般（需要额外的函数调用）       |
| 灵活性   | 较低（只能通过返回值控制）   | 高（可以在任何时机精确控制）     |
| 适用场景 | 持续变化的数据（血量、时间） | 事件驱动的更新（按钮文字、提示） |
| 复杂度   | 简单                         | 中等                             |

```
✅ 持续变化的数据（血条、时间、分数）使用属性绑定
✅ 事件驱动的更新（收到伤害、拾取物品）使用直接调用
✅ 可以混合使用：大部分用绑定，特殊情况直接调用
❌ 不要在绑定函数中做复杂计算（绑定函数每帧调用，太重会影响性能）
```

## 5. 动画播放（PlayAnimation）

### 5.1 UMG中的动画

在UMG编辑器中，你可以在Timeline面板创建动画（如淡入淡出、移动、缩放等）。在C++中，你可以通过 `PlayAnimation` 来控制这些动画的播放。

### 5.2 创建和绑定动画

**步骤1：在UMG编辑器中创建动画**

1. 打开Widget Blueprint
2. 在底部找到 `Animations` 面板（如果没有，点击 `Window → Animations`）
3. 点击 `+ Animation` 按钮创建新的动画
4. 给动画取名（如 `FadeIn`、`SlideUp`、`Pulse`）
5. 在Timeline中添加关键帧，调整控件属性（如位置、透明度等）

**步骤2：在C++中绑定动画**

```cpp
// 文件：MyAnimatedWidget.h
#pragma once

#include "CoreMinimal.h"
#include "Blueprint/UserWidget.h"
#include "Animation/WidgetAnimation.h"  // ← 播放动画需要的头文件
#include "MyAnimatedWidget.generated.h"

UCLASS()
class YOURPROJECT_API UMyAnimatedWidget : public UUserWidget
{
    GENERATED_BODY()

public:
    // ==================== 动画控制函数 ====================

    // 播放淡入动画
    UFUNCTION(BlueprintCallable, Category = "Animation")

    // 绑定名为 "FadeOutAnim" 的动画（必须存在于蓝图中）
    UPROPERTY(meta = (BindWidgetAnim))

    UPROPERTY(meta = (BindWidgetAnim))
```

```cpp
UUMGSequencePlayer* PlayAnimation(
```

### 5.4 动画播放的完整示例

```cpp
// 文件：MyAnimatedWidget.cpp
#include "MyAnimatedWidget.h"

void UMyAnimatedWidget::NativeConstruct()
{
    Super::NativeConstruct();

    // 检查动画绑定是否成功
    if (!FadeInAnim)
    {
        UE_LOG(LogTemp, Warning, TEXT("FadeInAnim 绑定失败！"));
    }
}

// ==================== 播放淡入动画 ====================
void UMyAnimatedWidget::PlayFadeInAnimation()
{
    // 安全检查
    if (!FadeInAnim)
    {
        UE_LOG(LogTemp, Warning, TEXT("FadeInAnim 不存在，无法播放"));
        return;
    }

    // 检查动画是否已经在播放（避免重复播放）
    // IsAnimationPlaying 是 UUserWidget 提供的检查函数
    if (IsAnimationPlaying(FadeInAnim))
    {
        UE_LOG(LogTemp, Warning, TEXT("淡入动画已经在播放中"));
        return;
    }

    // 播放动画
    // 参数说明：
    //   FadeInAnim    — 要播放的动画
    //   0.0f          — 从第0秒开始播放（即从头开始）
    //   1             — 播放1次（不循环）
    //   Forward       — 正向播放
    //   1.0f          — 正常速度
    //   false         — 播放结束后保持最终状态（不还原）
    PlayAnimation(FadeInAnim, 0.0f, 1,
        EUMGSequencePlayMode::Forward, 1.0f, false);

    UE_LOG(LogTemp, Log, TEXT("播放淡入动画"));
}

// ==================== 播放淡出动画并关闭Widget ====================
void UMyAnimatedWidget::PlayFadeOutAnimation()
{
    if (!FadeOutAnim)
    {
        UE_LOG(LogTemp, Warning, TEXT("FadeOutAnim 不存在"));
        return;
    }

    // 播放淡出动画
    // 注意：bRestoreState = false，播放结束后保持动画最后一帧的状态
    UUMGSequencePlayer* Player = PlayAnimation(
        FadeOutAnim,                           // 淡出动画
        0.0f,                                  // 从头开始
        1,                                     // 播放1次
        EUMGSequencePlayMode::Forward,         // 正向播放
        1.0f,                                  // 正常速度
        false                                  // 保持最终状态
    );

    // 如果播放器创建成功，绑定播放完成后的回调
    if (Player)
    {
        // OnSequenceFinishedPlaying 是播放器提供的委托
        // 动画播放完成后自动调用
        Player->OnSequenceFinishedPlaying().AddWeakLambda(this, [this](UUMGSequencePlayer& InPlayer)
        {
            // 动画播放完毕 → 关闭Widget
            RemoveFromParent();
            UE_LOG(LogTemp, Log, TEXT("淡出动画完成，Widget已移除"));
        });
    }
}

// ==================== 播放脉冲动画（无限循环）====================
void UMyAnimatedWidget::PlayPulseAnimation()
{
    if (!PulseAnim) return;

    if (IsAnimationPlaying(PulseAnim)) return;

    // 播放脉冲动画，设置为无限循环
    PlayAnimation(
        PulseAnim,                            // 脉冲动画
        0.0f,                                 // 从头开始
        0,                                    // ★ NumLoopsToPlay = 0 表示无限循环！
        EUMGSequencePlayMode::Forward,        // 正向播放
        1.0f,                                 // 正常速度
        false                                 // 保持最终状态
    );

    UE_LOG(LogTemp, Log, TEXT("脉冲动画开始循环播放"));
}

// ==================== 停止所有动画 ====================
void UMyAnimatedWidget::StopAllAnimations()
{
    // 逐个停止每个动画
    if (FadeInAnim && IsAnimationPlaying(FadeInAnim))
    {
        StopAnimation(FadeInAnim);  // ← UUserWidget提供的停止动画函数
    }

    if (FadeOutAnim && IsAnimationPlaying(FadeOutAnim))
    {
        StopAnimation(FadeOutAnim);
    }

    if (PulseAnim && IsAnimationPlaying(PulseAnim))
    {
        StopAnimation(PulseAnim);
    }

    UE_LOG(LogTemp, Log, TEXT("所有动画已停止"));
}
```

```cpp
void AdvancedAnimationControl(UMyAnimatedWidget* Widget)
{
    if (!Widget->FadeInAnim) return;

    // 播放动画并获取播放器指针
```

## 6. 输入模式切换

### 6.1 为什么需要切换输入模式

游戏中的UI和游戏玩法对输入的需求是不同的：

```
游戏模式（Game Only）：
- 鼠标控制视角（隐藏鼠标光标）
- 键盘控制角色移动
- 没有鼠标光标

UI模式（UI Only）：
- 鼠标光标可见，用于点击按钮
- 键盘输入被UI拦截，不影响角色
- 典型的菜单、背包、对话框场景

混合模式（Game and UI）：
- 鼠标光标可见（可以点击UI）
- 同时角色也能响应键盘输入
- 典型的使用场景：HUD交互
```

### 6.2 三种输入模式详解

```cpp
// ==================== 模式1：SetInputMode_UIOnly ====================
// 纯UI模式 — 只响应UI输入，不响应游戏输入
void SwitchToUIMode(APlayerController* PC)
{
    if (!PC) return;

    // 创建一个FInputModeUIOnly结构体
    FInputModeUIOnly InputMode;

    // 设置哪个Widget接收输入（通常是顶层菜单Widget）
    InputMode.SetWidgetToFocus(SomeWidget->TakeWidget());
    // TakeWidget()：获取Widget底层的Slate对象

    // 设置鼠标锁定模式
    InputMode.SetLockMouseToViewportBehavior(EMouseLockMode::DoNotLock);
    // EMouseLockMode选项：
    //   DoNotLock    — 鼠标不锁定，可以自由移出窗口
    //   LockOnCapture — 鼠标被锁定在窗口内（需要点击窗口内才能移出）
    //   LockAlways    — 鼠标始终锁定在窗口内
    //   LockInFullscreen — 全屏时锁定

    // 应用输入模式
    PC->SetInputMode(InputMode);

    // 显示鼠标光标（SetInputMode_UIOnly会自动显示，但显式设置更安全）
    PC->bShowMouseCursor = true;
}

// ==================== 模式2：SetInputMode_GameOnly ====================
// 纯游戏模式 — 只响应游戏输入，不响应UI输入
void SwitchToGameMode(APlayerController* PC)
{
    if (!PC) return;

    // 创建FInputModeGameOnly结构体
    FInputModeGameOnly InputMode;

    // 不消耗鼠标按下事件（让鼠标点击也能触发游戏中的动作）
    InputMode.SetConsumeCaptureMouseDown(false);

    // 应用输入模式
    PC->SetInputMode(InputMode);

    // 隐藏鼠标光标（游戏模式下通常不需要）
    PC->bShowMouseCursor = false;
}

// ==================== 模式3：SetInputMode_GameAndUI ====================
// 混合模式 — 同时响应UI和游戏输入
void SwitchToGameAndUIMode(APlayerController* PC, UUserWidget* HUDWidget)
{
    if (!PC) return;

    // 创建FInputModeGameAndUI结构体
    FInputModeGameAndUI InputMode;

    // 设置要接收UI输入的Widget（可以是HUD）
    if (HUDWidget)
    {
        InputMode.SetWidgetToFocus(HUDWidget->TakeWidget());
    }

    // 鼠标不锁定在窗口内
    InputMode.SetLockMouseToViewportBehavior(EMouseLockMode::DoNotLock);

    // 点击UI区域时不触发游戏中的点击事件（例如不开枪）
    InputMode.SetHideCursorDuringCapture(false);

    // 应用输入模式
    PC->SetInputMode(InputMode);

    // 显示鼠标光标（方便点击UI）
    PC->bShowMouseCursor = true;
}
```

```cpp
{
    // 步骤1：如果Widget还没创建，先创建
    if (!InventoryWidget)

    FInputModeGameOnly InputMode;
    SetInputMode(InputMode);
    bShowMouseCursor = false;

    // 步骤3：恢复游戏（如果之前暂停了）
    // SetPause(false);

    bIsInventoryOpen = false;
    UE_LOG(LogTemp, Log, TEXT("背包已关闭 → 输入模式：GameOnly"));
}
```

### 6.4 输入模式切换的注意事项

```
✅ 打开任何需要鼠标点击的UI时，切换到GameAndUI或UIOnly模式
✅ 关闭UI后，记得切换回GameOnly模式
✅ 使用SetWidgetToFocus确保焦点正确的Widget
✅ 记录当前状态，避免重复切换

❌ 忘记切回GameOnly模式 → 玩家无法控制角色
❌ 打开多层UI时反复切换输入模式 → 让最上层的UI处理输入即可
❌ 在没有UI的情况下使用UIOnly → 鼠标可见但什么也不能点
❌ 切换输入模式时忘记同步bShowMouseCursor
```

```cpp
    // 因为已经绑定了属性委托，引擎会自动更新显示！

// ==================== 属性绑定函数 ====================

float UPlayerStatusWidget::GetHealthPercent() const
{
    // 计算血量百分比，限制在0.0~1.0之间
    if (MaxHealth <= 0.0f) return 0.0f;
    return FMath::Clamp(CurrentHealth / MaxHealth, 0.0f, 1.0f);
}

FText UPlayerStatusWidget::GetHealthText() const
{
    // 返回格式化的血量文字："85 / 100"
    return FText::Format(
        FText::FromString(TEXT("{0} / {1}")),
        FText::AsNumber(FMath::RoundToInt(CurrentHealth)),
        FText::AsNumber(FMath::RoundToInt(MaxHealth))
    );
}

float UPlayerStatusWidget::GetExpPercent() const
{
    // 计算经验百分比
    if (MaxExp <= 0.0f) return 0.0f;
    return FMath::Clamp(CurrentExp / MaxExp, 0.0f, 1.0f);
}

FText UPlayerStatusWidget::GetLevelText() const
{
    // 返回等级文字："LV.1"
    return FText::Format(
        FText::FromString(TEXT("LV.{0}")),
        FText::AsNumber(CurrentLevel)
    );
}

FLinearColor UPlayerStatusWidget::GetHealthBarColor() const
{
    // 根据血量百分比返回不同的颜色
    float Percent = GetHealthPercent();

    if (Percent > 0.65f)
    {
        // 血量充足 → 绿色
        return FLinearColor(0.0f, 0.8f, 0.0f);  // RGB
    }
    else if (Percent > 0.35f)
    {
        // 血量中等 → 黄色
        return FLinearColor(0.8f, 0.8f, 0.0f);
    }
    else if (Percent > 0.15f)
    {
        // 血量较低 → 橙色
        return FLinearColor(1.0f, 0.5f, 0.0f);
    }
    else
    {
        // 血量危险 → 红色
        return FLinearColor(1.0f, 0.0f, 0.0f);
    }
}

// ==================== 事件处理 ====================

void UPlayerStatusWidget::OnLevelUpButtonClicked()
{
    UE_LOG(LogTemp, Log, TEXT("===== 升级按钮被点击 ====="));

    // 步骤1：增加经验值
    CurrentExp += ExpPerLevelUp;

    // 步骤2：检查是否可以升级
    // while循环处理连续升级的情况（一次加的经验可能够升多级）
    while (CurrentExp >= MaxExp && MaxExp > 0.0f)
    {
        // 经验值减去升级所需经验
        CurrentExp -= MaxExp;

        // 等级+1
        CurrentLevel++;

        // 每升一级，升级所需经验增加一点（可选，让游戏更有挑战）
        MaxExp = MaxExp * 1.2f;  // 每次升级所需经验增加20%

        // 升级后回满血量（作为升级奖励）
        CurrentHealth = MaxHealth;

        UE_LOG(LogTemp, Log, TEXT("升级！当前等级: %d，下级需要经验: %.0f"),
            CurrentLevel, MaxExp);
    }

    // 步骤3：确保经验值不会超过上限
    CurrentExp = FMath::Clamp(CurrentExp, 0.0f, MaxExp - FLT_EPSILON);

    // 注意：不需要手动更新进度条和文字！
    // 因为属性绑定会自动同步最新的数据到UI上

    UE_LOG(LogTemp, Log, TEXT("当前状态 — 等级:%d, 经验:%.0f/%.0f, 血量:%.0f/%.0f"),
        CurrentLevel, CurrentExp, MaxExp, CurrentHealth, MaxHealth);
}

// ==================== 动画控制 ====================

void UPlayerStatusWidget::ShowWithAnimation()
{
    // 先设置为可见
    SetVisibility(ESlateVisibility::Visible);

    // 播放入场动画
    if (ShowAnimation)
    {
        PlayAnimation(ShowAnimation, 0.0f, 1,
            EUMGSequencePlayMode::Forward, 1.0f, false);
    }
}

void UPlayerStatusWidget::HideWithAnimation()
{
    if (!HideAnimation)
    {
        // 如果没有退场动画，直接隐藏
        SetVisibility(ESlateVisibility::Collapsed);
        return;
    }

    // 播放退场动画
    UUMGSequencePlayer* Player = PlayAnimation(
        HideAnimation, 0.0f, 1,
        EUMGSequencePlayMode::Forward, 1.0f, false
    );

    // 动画完成后隐藏
    if (Player)
    {
        Player->OnSequenceFinishedPlaying().AddWeakLambda(this, [this](UUMGSequencePlayer&)
        {
            SetVisibility(ESlateVisibility::Collapsed);
            UE_LOG(LogTemp, Log, TEXT("退场动画播放完成，Widget已隐藏"));
        });
    }
}
```
