### 1.4 交互流程

```
玩家按下 Tab 键：
    → PlayerController::OnInventoryKeyPressed()
        → UIManager::ToggleInventory()
            → 如果背包关闭 → 显示背包 + 切换到 GameAndUI 模式
            → 如果背包打开 → 隐藏背包 + 切换到 GameOnly 模式

玩家按下 ESC 键：
    → PlayerController::OnPauseKeyPressed()
        → UIManager::ToggleMainMenu()
            → 如果菜单关闭 → 显示菜单 + 暂停游戏 + 切换到 UIOnly 模式
            → 如果菜单打开 → 隐藏菜单 + 恢复游戏 + 切换到 GameOnly 模式

外部游戏逻辑更新数据：
    → UIManager 的公开方法（如 UpdateHealth, UpdateAmmo）
        → 转发到对应 Widget 的更新方法
            → Widget 内部通过属性绑定或直接更新改变显示
```

## 2. UMainHUD类：主游戏界面

### 2.1 功能描述

`UMainHUD` 是游戏进行中始终显示的HUD界面，包含：

- **血条** — ProgressBar，绑定到玩家血量百分比，根据血量改变颜色
- **弹药显示** — TextBlock，显示当前弹药/最大弹药
- **小地图指示器** — Image，显示小地图（示意）
- **伤害数字弹出** — 受伤时在屏幕中央短暂显示伤害数字（浮动文本）

### 2.2 头文件

```cpp
// 文件：MainHUD.h
#pragma once

#include "CoreMinimal.h"
#include "Blueprint/UserWidget.h"

// ==================== 必要的头文件 ====================
#include "Components/ProgressBar.h"    // ProgressBar控件 — 用于血条
#include "Components/TextBlock.h"      // TextBlock控件 — 用于弹药/伤害文字
#include "Components/Image.h"          // Image控件 — 用于小地图
#include "Components/CanvasPanel.h"    // CanvasPanel — 用于容器
#include "Components/Overlay.h"        // Overlay — 用于层叠布局
#include "Animation/WidgetAnimation.h" // 动画支持 — 伤害数字的弹出动画

#include "MainHUD.generated.h"

/**
 * 主游戏HUD
 * 显示血量、弹药、小地图和伤害数字
 */
UCLASS()
class YOURPROJECT_API UMainHUD : public UUserWidget
{
    GENERATED_BODY()

public:
    // ==================== 构造函数 ====================
    UMainHUD(const FObjectInitializer& ObjectInitializer);

    // ==================== 公开更新方法 ====================
    // 这些方法由UIManager或外部系统调用

    // 更新血量显示
    // @param CurrentHealth — 当前血量
    // @param MaxHealth — 最大血量
    UFUNCTION(BlueprintCallable, Category = "HUD")
    void UpdateHealth(float CurrentHealth, float MaxHealth);

    // 更新弹药显示
    // @param CurrentAmmo — 当前弹药
    // @param MaxAmmo — 最大弹药
    UFUNCTION(BlueprintCallable, Category = "HUD")
    void UpdateAmmo(int32 CurrentAmmo, int32 MaxAmmo);

    // 显示伤害数字
    // @param Damage — 伤害数值
    // @param WorldPosition — 伤害发生的世界坐标（用于计算屏幕位置）
    UFUNCTION(BlueprintCallable, Category = "HUD")
    void ShowDamageNumber(float Damage, FVector WorldPosition);

    // 更新小地图（简化版：旋转一个指示器表示玩家朝向）
    // @param PlayerRotationYaw — 玩家的Yaw旋转角度
    UFUNCTION(BlueprintCallable, Category = "HUD")
    void UpdateMiniMap(float PlayerRotationYaw);

protected:
    // ==================== 生命周期 ====================
    virtual void NativeConstruct() override;

private:
    // ==================== 绑定的控件（BindWidget）====================
    // 命名规则：变量名必须和蓝图中控件的名字完全一致

    // 血条进度条 → 蓝图控件名: "HealthBar"
    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UProgressBar> HealthBar;

    // 血量文字 → 蓝图控件名: "HealthText"
    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UTextBlock> HealthText;

    // 弹药文字 → 蓝图控件名: "AmmoText"
    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UTextBlock> AmmoText;

    // 武器名称文字 → 蓝图控件名: "WeaponNameText"
    UPROPERTY(meta = (BindWidgetOptional))  // 可选绑定
    TObjectPtr<UTextBlock> WeaponNameText;

    // 小地图图片 → 蓝图控件名: "MiniMapImage"
    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UImage> MiniMapImage;

    // 小地图上玩家方向指示器 → 蓝图控件名: "PlayerDirectionIndicator"
    UPROPERTY(meta = (BindWidgetOptional))
    TObjectPtr<UImage> PlayerDirectionIndicator;

    // 伤害数字的容器（CanvasPanel，伤害数字在其中自由定位）→ 蓝图控件名: "DamageNumberCanvas"
    UPROPERTY(meta = (BindWidget))
    TObjectPtr<UCanvasPanel> DamageNumberCanvas;

    // ==================== 绑定的动画 ====================
    // 伤害数字弹出动画 → 蓝图中的动画名: "DamagePopAnim"
    // 这个动画在UMG编辑器中创建，包含文字从下方往上飘并淡出的效果
    UPROPERTY(meta = (BindWidgetAnimOptional))
    TObjectPtr<UWidgetAnimation> DamagePopAnim;

    // ==================== 属性绑定函数 ====================
    // 这些函数被BindDynamic绑定后，引擎自动调用它们来更新UI

    // 获取血量百分比 → 绑定到ProgressBar
    UFUNCTION()
    float GetHealthPercent() const;

    // 获取血量文字 → 绑定到TextBlock
    UFUNCTION()
    FText GetHealthText() const;

    // 获取弹药文字 → 绑定到TextBlock
    UFUNCTION()
    FText GetAmmoText() const;

    // 获取血条颜色 → 绑定到ProgressBar的颜色
    UFUNCTION()
    FLinearColor GetHealthBarColor() const;

    // ==================== 内部数据 ====================
    // 使用属性绑定后，这些数据变化会自动反映到UI上

    float CurrentHealth = 100.0f;   // 当前血量
    float MaxHealth = 100.0f;       // 最大血量
    int32 CurrentAmmo = 30;         // 当前弹药
    int32 MaxAmmo = 30;             // 最大弹药
    float PlayerYawRotation = 0.0f; // 玩家朝向角度
};
```

### 2.3 实现文件

```cpp
// 文件：MainHUD.cpp
#include "MainHUD.h"
#include "Components/CanvasPanelSlot.h"  // CanvasPanelSlot — 用于在Canvas中控制子控件位置
#include "GameFramework/PlayerController.h"

// ==================== 构造函数 ====================
UMainHUD::UMainHUD(const FObjectInitializer& ObjectInitializer)
    : Super(ObjectInitializer)
{
    // HUD需要通过属性绑定自动更新，不需要启用Tick
    // bHasScriptImplementedTick 保持默认的 false 即可
}

// ==================== NativeConstruct ====================
void UMainHUD::NativeConstruct()
{
    Super::NativeConstruct();

    // ---------- 1. 验证核心控件绑定 ----------
    checkf(HealthBar, TEXT("UMainHUD: HealthBar 绑定失败！请检查蓝图中是否存在名为 HealthBar 的控件"));
    checkf(HealthText, TEXT("UMainHUD: HealthText 绑定失败！"));
    checkf(AmmoText, TEXT("UMainHUD: AmmoText 绑定失败！"));
    checkf(MiniMapImage, TEXT("UMainHUD: MiniMapImage 绑定失败！"));
    checkf(DamageNumberCanvas, TEXT("UMainHUD: DamageNumberCanvas 绑定失败！"));

    // ---------- 2. 设置属性绑定 ----------
    // 这里的绑定让UI自动从数据中获取最新的显示内容
    // 不需要在Tick中手动更新！

    // 血条进度 → 自动调用GetHealthPercent
    HealthBar->PercentDelegate.BindDynamic(this, &UMainHUD::GetHealthPercent);

    // 血量文字 → 自动调用GetHealthText
    HealthText->TextDelegate.BindDynamic(this, &UMainHUD::GetHealthText);

    // 弹药文字 → 自动调用GetAmmoText
    AmmoText->TextDelegate.BindDynamic(this, &UMainHUD::GetAmmoText);

    // 血条颜色 → 自动调用GetHealthBarColor
    HealthBar->ColorAndOpacityDelegate.BindDynamic(this, &UMainHUD::GetHealthBarColor);

    // ---------- 3. 初始化显示 ----------
    // 属性绑定虽然会自动更新，但初始值需要手动触发一次（或者等第一帧）
    // 这里显式调用更新方法确保初始显示正确
    UpdateHealth(CurrentHealth, MaxHealth);
    UpdateAmmo(CurrentAmmo, MaxAmmo);

    UE_LOG(LogTemp, Log, TEXT("UMainHUD 初始化完成（属性绑定已设置）"));
}

// ==================== UpdateHealth ====================
void UMainHUD::UpdateHealth(float InCurrentHealth, float InMaxHealth)
{
    // 更新内存中的数据
    CurrentHealth = FMath::Max(0.0f, InCurrentHealth);
    MaxHealth = FMath::Max(1.0f, InMaxHealth);  // 避免除以0

    // 注意：不需要手动更新UI！
    // 因为属性绑定会自动从GetHealthPercent/GetHealthText/GetHealthBarColor
    // 获取最新的数据。这里只需要更新内部变量即可。
}

// ==================== UpdateAmmo ====================
void UMainHUD::UpdateAmmo(int32 InCurrentAmmo, int32 InMaxAmmo)
{
    CurrentAmmo = FMath::Max(0, InCurrentAmmo);
    MaxAmmo = FMath::Max(1, InMaxAmmo);

    // 同上，属性绑定会自动更新UI
}

// ==================== ShowDamageNumber ====================
void UMainHUD::ShowDamageNumber(float Damage, FVector WorldPosition)
{
    // ==================== 步骤1：将世界坐标转换为屏幕坐标 ====================
    APlayerController* PC = GetOwningPlayer();
    if (!PC)
    {
        UE_LOG(LogTemp, Warning, TEXT("没有PlayerController，无法显示伤害数字"));
        return;
    }

    // ProjectWorldLocationToScreen 将3D世界坐标投影到2D屏幕坐标
    FVector2D ScreenPosition;  // 输出：屏幕上的像素坐标
    bool bIsOnScreen = PC->ProjectWorldLocationToScreen(
        WorldPosition,    // 世界坐标（伤害发生的位置）
        ScreenPosition,   // 输出的屏幕位置
        true              // true = 考虑屏幕边缘（超出屏幕会修正）
    );

    // 如果不在屏幕上（例如在玩家背后），就不显示
    if (!bIsOnScreen)
    {
        return;
    }

    // ==================== 步骤2：创建伤害数字的TextBlock ====================
    // 动态创建一个TextBlock作为伤害数字
    UTextBlock* DamageText = NewObject<UTextBlock>(this, UTextBlock::StaticClass());
    if (!DamageText)
    {
        return;
    }

    // 设置显示文字（伤害数值，取整显示）
    DamageText->SetText(FText::AsNumber(FMath::RoundToInt(Damage)));

    // 设置文字样式
    // 从字体资产加载（需要在项目中有一个Font资源）
    // 如果没有专门的Font，可以留空使用默认字体
    FSlateFontInfo FontInfo;
    FontInfo.Size = 28;  // 字体大小
    DamageText->SetFont(FontInfo);

    // 根据伤害大小改变颜色：伤害越大颜色越红
    FLinearColor DamageColor;
    if (Damage >= 50.0f)
    {
        DamageColor = FLinearColor(1.0f, 0.2f, 0.0f);  // 大伤害 → 深橙红色
    }
    else if (Damage >= 20.0f)
    {
        DamageColor = FLinearColor(1.0f, 0.6f, 0.0f);  // 中伤害 → 橙色
    }
    else
    {
        DamageColor = FLinearColor(1.0f, 1.0f, 0.0f);  // 小伤害 → 黄色
    }
    DamageText->SetColorAndOpacity(FSlateColor(DamageColor));

    // 添加阴影让文字更清晰可读
    DamageText->SetShadowOffset(FVector2D(1.0f, 1.0f));    // 阴影偏移
    DamageText->SetShadowColorAndOpacity(FLinearColor::Black); // 阴影颜色

    // ==================== 步骤3：将伤害数字添加到CanvasPanel ====================
    // AddChild 将DamageText添加为CanvasPanel的子控件
    // 返回值是一个UPanelSlot（这里是UCanvasPanelSlot）
    UCanvasPanelSlot* CanvasSlot = Cast<UCanvasPanelSlot>(
        DamageNumberCanvas->AddChild(DamageText)
    );

    if (CanvasSlot)
    {
        // 设置伤害数字在Canvas中的位置（屏幕坐标）
        CanvasSlot->SetPosition(ScreenPosition);

        // 设置控件大小
        CanvasSlot->SetSize(FVector2D(100.0f, 40.0f));

        // 设置对齐方式（居中显示在指定位置上）
        CanvasSlot->SetAlignment(FVector2D(0.5f, 0.5f));

        // 设置ZOrder（确保显示在最前面）
        CanvasSlot->SetZOrder(999);
    }

    // ==================== 步骤4：设置定时器，让伤害数字在一段时间后自动消失 ====================
    // 获取世界（用于获取定时器管理器）
    if (UWorld* World = GetWorld())
    {
        // FTimerHandle 是定时器的句柄，用于管理和取消定时器
        FTimerHandle TimerHandle;

        // SetTimer 设置一个延迟执行的函数
        World->GetTimerManager().SetTimer(
            TimerHandle,                                              // 定时器句柄
            [DamageText]()                                            // Lambda：定时器到期后执行
            {
                // 从父容器中移除伤害数字
                if (DamageText)
                {
                    DamageText->RemoveFromParent();
                }
            },
            1.5f,                                                     // 延迟1.5秒后执行
            false                                                     // false = 不循环
        );
    }

    // ==================== （可选）步骤5：播放弹出动画 ====================
    // 如果蓝图中定义了DamagePopAnim动画，可以在这里播放
    if (DamagePopAnim)
    {
        // 在DamageText上播放弹出动画
        // 注意：PlayAnimation默认在Widget自身播放。如果想在子控件上播放，
        // 需要在蓝图Timeline中正确设置动画目标
        PlayAnimation(DamagePopAnim, 0.0f, 1,
            EUMGSequencePlayMode::Forward, 1.0f, false);
    }
}

// ==================== UpdateMiniMap ====================
void UMainHUD::UpdateMiniMap(float InPlayerRotationYaw)
{
    // 保存玩家朝向
    PlayerYawRotation = InPlayerRotationYaw;

    // 如果小地图指示器存在，旋转它来表示玩家的朝向
    if (PlayerDirectionIndicator)
    {
        // SetRenderTransformAngle 设置控件的渲染旋转角度
        // 注意：这是渲染层面的旋转（纯视觉效果），不影响布局
        PlayerDirectionIndicator->SetRenderTransformAngle(-PlayerYawRotation);
        // 取负号是因为UMG的坐标系和游戏世界坐标系的Y轴方向不同
    }
}

// ==================== 属性绑定函数实现 ====================

float UMainHUD::GetHealthPercent() const
{
    // 计算血量百分比，限制在0.0~1.0之间
    if (MaxHealth <= 0.0f)
    {
        return 0.0f;
    }
    return FMath::Clamp(CurrentHealth / MaxHealth, 0.0f, 1.0f);
}

FText UMainHUD::GetHealthText() const
{
    // 返回格式化的血量文字
    // "{0} / {1}" → "75 / 100"
    return FText::Format(
        FText::FromString(TEXT("{0} / {1}")),
        FText::AsNumber(FMath::RoundToInt(CurrentHealth)),
        FText::AsNumber(FMath::RoundToInt(MaxHealth))
    );
}

FText UMainHUD::GetAmmoText() const
{
    // 返回格式化的弹药文字
    // "{0} / {1}" → "25 / 30"
    return FText::Format(
        FText::FromString(TEXT("{0} / {1}")),
        FText::AsNumber(CurrentAmmo),
        FText::AsNumber(MaxAmmo)
    );
}

FLinearColor UMainHUD::GetHealthBarColor() const
{
    // 根据血量百分比返回颜色
    float Percent = GetHealthPercent();

    if (Percent > 0.65f)
    {
        return FLinearColor(0.0f, 0.8f, 0.0f);   // 绿色 — 血量充足
    }
    else if (Percent > 0.35f)
    {
        return FLinearColor(0.8f, 0.8f, 0.0f);   // 黄色 — 血量警告
    }
    else if (Percent > 0.15f)
    {
        return FLinearColor(1.0f, 0.5f, 0.0f);   // 橙色 — 血量危险
    }
    else
    {
        return FLinearColor(1.0f, 0.0f, 0.0f);   // 红色 — 血量危急
    }
}
```

### 2.4 在蓝图中配合的布局设计

为了让C++代码正常工作，你需要在编辑器中创建对应的Widget Blueprint（如`WBP_MainHUD`）：

```
WBP_MainHUD 的Hierarchy结构建议：

CanvasPanel (根)
├── HorizontalBox (底部技能栏容器)
│   └── ...（技能图标）
│
├── Overlay (左上角血量区域，锚点=左上)
│   ├── Image (血条背景)
│   ├── ProgressBar (HealthBar — 血条，锚点=左拉伸)
│   └── TextBlock (HealthText — 血量数字)
│
├── TextBlock (AmmoText — 弹药数字，锚点=右下)
│
├── Image (MiniMapImage — 小地图，锚点=右上)
```

```cpp
// ==================== RefreshInventory ====================
void UInventoryWidget::RefreshInventory(const TArray<FInventoryItem>& Items)
{
    // 步骤1：清空当前网格中的所有物品格子
    ItemGridPanel->ClearChildren();

    // 步骤2：遍历物品数组，逐个创建格子
    int32 ItemCount = Items.Num();
    for (int32 i = 0; i < ItemCount; ++i)
    {
        // 计算行列位置
        int32 Row = i / GridColumns;     // 行号：0, 0, 0, 0, 0, 1, 1, ...
        int32 Column = i % GridColumns;  // 列号：0, 1, 2, 3, 4, 0, 1, ...

        // 拒绝超出网格容量的物品（做防御性检查）
        if (Row >= GridRows)
        {
            UE_LOG(LogTemp, Warning, TEXT("物品数量超出网格容量！物品%d将被忽略"), i);
            continue;
        }

        // 创建单个物品格子
        CreateItemSlot(Items[i], Row, Column);
    }

    // 步骤3：更新汇总信息
    UpdateInfoText(ItemCount, GridColumns * GridRows);
}

// ==================== ClearInventory ====================
void UInventoryWidget::ClearInventory()
{
    // 清除网格中的所有子控件
    ItemGridPanel->ClearChildren();

    // 更新显示
    UpdateInfoText(0, GridColumns * GridRows);
}

// ==================== CreateItemSlot ====================
void UInventoryWidget::CreateItemSlot(const FInventoryItem& Item, int32 Row, int32 Column)
{
    // ==================== 步骤1：创建物品格子的最外层容器 ====================
    // 使用Border作为每个格子的背景框
    UBorder* SlotBorder = NewObject<UBorder>(this, UBorder::StaticClass());
    if (!SlotBorder) return;

    // 设置Border的背景颜色（根据物品稀有度）
    FLinearColor RarityColor;
    switch (Item.Rarity)
    {
        case 0: RarityColor = FLinearColor(0.4f, 0.4f, 0.4f); break;  // 灰色 — 普通
        case 1: RarityColor = FLinearColor(0.2f, 0.4f, 1.0f); break;  // 蓝色 — 稀有
        case 2: RarityColor = FLinearColor(0.6f, 0.2f, 1.0f); break;  // 紫色 — 史诗
        case 3: RarityColor = FLinearColor(1.0f, 0.5f, 0.0f); break;  // 橙色 — 传说
        default: RarityColor = FLinearColor(0.4f, 0.4f, 0.4f); break;
    }
    SlotBorder->SetBrushColor(RarityColor);

    // 设置内边距（让内容不贴边）
    SlotBorder->SetPadding(FMargin(2.0f));

    // 设置悬停事件（鼠标放上去时显示详情）
    // ⚠️ 简化处理：实际项目中需要更复杂的悬停事件绑定
    // 这里使用Button来处理点击和悬停

    // ==================== 步骤2：创建内部的Overlay（图标+数量层叠）====================
    UOverlay* ItemOverlay = NewObject<UOverlay>(this, UOverlay::StaticClass());
    if (!ItemOverlay) return;

    // ==================== 步骤3：创建物品图标（Image）====================
    UImage* ItemIconImage = NewObject<UImage>(this, UImage::StaticClass());
    if (ItemIconImage)
    {
        // 设置图标纹理
        if (Item.ItemIcon)
        {
            // 有图标就用设置好的图标
            ItemIconImage->SetBrushFromTexture(Item.ItemIcon);
        }
        else
        {
            // 没有图标就用默认的灰色方块
            // 创建一个纯色纹理作为占位符
            ItemIconImage->SetColorAndOpacity(FLinearColor(0.2f, 0.2f, 0.2f));
        }

        // 将图标添加到Overlay中，设置为填充整个空间
        UOverlaySlot* IconSlot = ItemOverlay->AddChildToOverlay(ItemIconImage);
        if (IconSlot)
        {
            // 水平方向填充整个Overlay
            IconSlot->SetHorizontalAlignment(HAlign_Fill);
            // 垂直方向填充整个Overlay
            IconSlot->SetVerticalAlignment(VAlign_Fill);
        }
    }

    // ==================== 步骤4：创建数量文字（TextBlock）====================
    if (Item.Quantity > 1)  // 数量大于1才显示数字
    {
        UTextBlock* QuantityText = NewObject<UTextBlock>(this, UTextBlock::StaticClass());
        if (QuantityText)
        {
            // 设置数量文字
            QuantityText->SetText(FText::AsNumber(Item.Quantity));

            // 设置文字样式
            FSlateFontInfo QuantityFont;
            QuantityFont.Size = 12;  // 比图标小的字号
            QuantityText->SetFont(QuantityFont);

            // 白色文字
            QuantityText->SetColorAndOpacity(FSlateColor(FLinearColor::White));

            // 文字阴影提高可读性
            QuantityText->SetShadowOffset(FVector2D(1.0f, 1.0f));
            QuantityText->SetShadowColorAndOpacity(FLinearColor::Black);

            // 将数量文字添加到Overlay（右下角对齐）
            UOverlaySlot* QtySlot = ItemOverlay->AddChildToOverlay(QuantityText);
            if (QtySlot)
            {
                // 右下角对齐
                QtySlot->SetHorizontalAlignment(HAlign_Right);
                QtySlot->SetVerticalAlignment(VAlign_Bottom);
                // 距离边缘一点距离
                QtySlot->SetPadding(FMargin(0.0f, 0.0f, 4.0f, 2.0f));
            }
        }
    }

    // ==================== 步骤5：将Overlay放入Border ====================
    // Border的AddChild返回UPanelSlot，这里我们不需要特别配置
    SlotBorder->AddChild(ItemOverlay);

    // ==================== 步骤6：将整个格子放入UniformGridPanel ====================
    UUniformGridSlot* GridSlot = ItemGridPanel->AddChildToUniformGrid(
        SlotBorder,  // 要添加的控件
        Row,         // 行号
        Column       // 列号
    );

    // 注意：UniformGridPanel会自动分配相同的大小给每个格子
    // 如果需要精确控制每个格子的大小，可以在蓝图中设置Slot的Padding
}
```

```cpp
    InitializeResolutionOptions();

    // ---------- 7. 加载当前设置 ----------
    // 从游戏用户设置中读取当前的分辨率、全屏、音量等
    if (UGameUserSettings* Settings = GEngine->GetGameUserSettings())

    SettingsPanel->SetVisibility(ESlateVisibility::Collapsed);
    bSettingsPanelVisible = false;

    // 设置按钮文字
    if (StartGameButtonText)
```

## 5. UIManager：UI管理器

### 5.1 为什么需要UIManager

将所有UI的创建、显示、隐藏集中在UIManager中管理，有以下好处：

- **集中控制** — 所有UI操作在一个地方，容易维护
- **状态管理** — 避免多个UI之间互相干扰
- **输入模式** — 统一管理输入模式的切换
- **ZOrder** — 统一管理和分配ZOrder层级

```cpp
// 文件：UIManager.cpp
#include "UIManager.h"
#include "UI/MainHUD.h"
#include "UI/InventoryWidget.h"
#include "UI/MainMenuWidget.h"
#include "GameFramework/PlayerController.h"
#include "Blueprint/UserWidget.h"

// ==================== Initialize ====================
void UUIManager::Initialize(
    APlayerController* InOwner,
    TSubclassOf<UMainHUD> InHUDClass,
    TSubclassOf<UInventoryWidget> InInventoryClass,
    TSubclassOf<UMainMenuWidget> InMenuClass)
{
    // 步骤1：保存参数
    OwnerPC = InOwner;
    HUDClass = InHUDClass;
    InventoryClass = InInventoryClass;
    MenuClass = InMenuClass;

    if (!OwnerPC)
    {
        UE_LOG(LogTemp, Error, TEXT("UIManager: OwnerPC 为空！"));
        return;
    }

    if (!HUDClass)
    {
        UE_LOG(LogTemp, Error, TEXT("UIManager: HUDClass 未设置！"));
        return;
    }

    // 步骤2：创建HUD（游戏运行时就一直显示）
    HUDWidget = CreateWidget<UMainHUD>(OwnerPC, HUDClass);
    if (HUDWidget)
    {
        HUDWidget->AddToViewport(0);  // ZOrder=0，最底层
        UE_LOG(LogTemp, Log, TEXT("HUD 创建并显示成功"));
    }
    else
    {
        UE_LOG(LogTemp, Error, TEXT("创建 HUD 失败"));
    }

    // 步骤3：预创建背包（但不显示，等玩家按Tab时再显示）
    if (InventoryClass)
    {
        InventoryWidget = CreateWidget<UInventoryWidget>(OwnerPC, InventoryClass);
        if (InventoryWidget)
        {
            // 创建后设为Collapsed（不可见且不占空间）
            InventoryWidget->SetVisibility(ESlateVisibility::Collapsed);
        }
    }

    // 步骤4：预创建主菜单（但不显示，等玩家按ESC时再显示）
    if (MenuClass)
    {
        MenuWidget = CreateWidget<UMainMenuWidget>(OwnerPC, MenuClass);
        if (MenuWidget)
        {
            MenuWidget->SetVisibility(ESlateVisibility::Collapsed);
        }
    }

    // 步骤5：确保初始输入模式为游戏模式
    SetGameInputMode();

    UE_LOG(LogTemp, Log, TEXT("UIManager 初始化完成"));
}

// ==================== 背包控制 ====================

void UUIManager::ToggleInventory()
{
    // 如果主菜单打开，不允许打开背包（避免UI重叠混乱）
    if (bIsMainMenuOpen)
    {
        UE_LOG(LogTemp, Warning, TEXT("主菜单打开时不允许操作背包"));
        return;
    }

    // 如果背包还没创建（预创建失败的情况），尝试创建
    if (!InventoryWidget && InventoryClass && OwnerPC)
    {
        InventoryWidget = CreateWidget<UInventoryWidget>(OwnerPC, InventoryClass);
        if (InventoryWidget)
        {
            InventoryWidget->SetVisibility(ESlateVisibility::Collapsed);
        }
    }

    if (!InventoryWidget)
    {
        UE_LOG(LogTemp, Error, TEXT("背包Widget未创建"));
        return;
    }

    // 切换背包状态
    bIsInventoryOpen = !bIsInventoryOpen;

    if (bIsInventoryOpen)
    {
        // --- 打开背包 ---
        // 确保在视口中（如果之前被RemoveFromParent了）
        if (!InventoryWidget->IsInViewport())
        {
            InventoryWidget->AddToViewport(20);  // ZOrder=20，在HUD之上
        }

        InventoryWidget->SetVisibility(ESlateVisibility::Visible);
        UE_LOG(LogTemp, Log, TEXT("背包已打开"));
    }
    else
    {
        // --- 关闭背包 ---
        InventoryWidget->SetVisibility(ESlateVisibility::Collapsed);
        UE_LOG(LogTemp, Log, TEXT("背包已关闭"));
    }

    // 根据当前UI状态更新输入模式
    UpdateInputMode();
}

// ==================== 主菜单控制 ====================

void UUIManager::ToggleMainMenu()
{
    // 如果背包打开，先关闭背包再打开菜单（避免UI重叠）
    if (bIsInventoryOpen)
    {
        bIsInventoryOpen = false;
        if (InventoryWidget)
        {
            InventoryWidget->SetVisibility(ESlateVisibility::Collapsed);
        }
    }

    if (!MenuWidget && MenuClass && OwnerPC)
    {
        MenuWidget = CreateWidget<UMainMenuWidget>(OwnerPC, MenuClass);
        if (MenuWidget)
        {
            MenuWidget->SetVisibility(ESlateVisibility::Collapsed);
        }
    }

    if (!MenuWidget)
    {
        UE_LOG(LogTemp, Error, TEXT("主菜单Widget未创建"));
        return;
    }

    // 切换菜单状态
    bIsMainMenuOpen = !bIsMainMenuOpen;

    if (bIsMainMenuOpen)
    {
        // --- 打开主菜单 ---
        if (!MenuWidget->IsInViewport())
        {
            MenuWidget->AddToViewport(80);  // ZOrder=80，最上层
        }

        MenuWidget->SetVisibility(ESlateVisibility::Visible);

        // 暂停游戏
        if (OwnerPC)
        {
            OwnerPC->SetPause(true);
        }

        UE_LOG(LogTemp, Log, TEXT("主菜单已打开 → 游戏已暂停"));
    }
    else
    {
        // --- 关闭主菜单 ---
        MenuWidget->SetVisibility(ESlateVisibility::Collapsed);

        // 恢复游戏
        if (OwnerPC)
        {
            OwnerPC->SetPause(false);
        }

        UE_LOG(LogTemp, Log, TEXT("主菜单已关闭 → 游戏已恢复"));
    }

    // 根据当前UI状态更新输入模式
    UpdateInputMode();
}

// ==================== 输入模式管理 ====================

void UUIManager::UpdateInputMode()
{
    if (!OwnerPC) return;

    if (bIsMainMenuOpen)
    {
        // 主菜单打开 → 纯UI模式，鼠标可见
        SetUIOnlyInputMode(MenuWidget);
    }
    else if (bIsInventoryOpen)
    {
        // 背包打开 → 游戏+UI混合模式，鼠标可见
        SetGameAndUIInputMode(InventoryWidget);
    }
    else
    {
        // 没有任何UI打开 → 纯游戏模式，隐藏鼠标
        SetGameInputMode();
    }
}

void UUIManager::SetGameInputMode()
{
    if (!OwnerPC) return;

    // 纯游戏模式
    FInputModeGameOnly InputMode;
    InputMode.SetConsumeCaptureMouseDown(false);

    OwnerPC->SetInputMode(InputMode);
    OwnerPC->bShowMouseCursor = false;

    UE_LOG(LogTemp, Verbose, TEXT("输入模式: GameOnly"));
}

void UUIManager::SetGameAndUIInputMode(UUserWidget* FocusWidget)
{
    if (!OwnerPC) return;

    // 游戏+UI混合模式
    FInputModeGameAndUI InputMode;

    if (FocusWidget)
    {
        // 设置哪个Widget优先接收输入
        InputMode.SetWidgetToFocus(FocusWidget->TakeWidget());
    }

    // 鼠标不锁定在窗口内
    InputMode.SetLockMouseToViewportBehavior(EMouseLockMode::DoNotLock);

    // 点击UI时不穿透到游戏
    InputMode.SetHideCursorDuringCapture(false);

    OwnerPC->SetInputMode(InputMode);
    OwnerPC->bShowMouseCursor = true;

    UE_LOG(LogTemp, Verbose, TEXT("输入模式: GameAndUI"));
}

void UUIManager::SetUIOnlyInputMode(UUserWidget* FocusWidget)
{
    if (!OwnerPC) return;

    // 纯UI模式
    FInputModeUIOnly InputMode;

    if (FocusWidget)
    {
        InputMode.SetWidgetToFocus(FocusWidget->TakeWidget());
    }

    // 鼠标不锁定
    InputMode.SetLockMouseToViewportBehavior(EMouseLockMode::DoNotLock);

    OwnerPC->SetInputMode(InputMode);
    OwnerPC->bShowMouseCursor = true;

    UE_LOG(LogTemp, Verbose, TEXT("输入模式: UIOnly"));
}
```

## 7. 输入模式切换的完整实现

### 7.1 输入模式切换总览

```
场景                        → 输入模式          → 鼠标        → 游戏状态
─────────────────────────────────────────────────────────────────────────
正常游戏                    → GameOnly          → 隐藏        → 正常运行
按Tab打开背包               → GameAndUI         → 显示        → 正常运行
    （关闭背包后恢复GameOnly）
按ESC打开主菜单             → UIOnly            → 显示        → 暂停
    （关闭菜单后恢复GameOnly）
打开背包时按ESC            → 先关背包，再开菜单  → 显示        → 暂停
打开菜单时按Tab            → 忽略（菜单优先）    → 保持不变    → 保持不变
```

### 7.2 输入模式切换的注意事项（实战要点）

```cpp
/*
 * ==================== 输入模式切换的常见错误 ====================
 *
 * 错误1：只切换输入模式，忘记切换鼠标光标
 * ❌ 错误做法：
 *    SetInputMode(FInputModeGameOnly());
 *    // 忘记设置 bShowMouseCursor = false
 *    // 结果：鼠标仍然可见，但无法点击任何东西
 *
 * ✅ 正确做法：
 *    FInputModeGameOnly InputMode;
 *    SetInputMode(InputMode);
 *    bShowMouseCursor = false;
 *
 * 错误2：打开多层UI时反复切换输入模式
 * ❌ 错误做法：
 *    打开背包 → UIOnly
 *    打开对话框(在背包之上) → UIOnly
 *    关闭对话框 → GameOnly (背包还在！输入模式错误)
 *
 * ✅ 正确做法：
 *    使用UIManager统一管理，根据当前打开的UI集合决定输入模式
 *    IsAnyUIOpen() → 有UI打开就用GameAndUI/UIOnly
 *    !IsAnyUIOpen() → 没有UI打开才用GameOnly
 *
 * 错误3：忘记在关卡切换时清理UI
 * ❌ 错误做法：
 *    关卡切换后UI还在，但World引用失效 → 崩溃
 *
 * ✅ 正确做法：
 *    在EndPlay中调用UIManager->Shutdown()
 *    在BeginPlay中重新创建UIManager
 *
 * 错误4：在输入模式切换中执行耗时操作
 * ❌ 错误做法：
 *    打开背包时加载大量资源再切换输入模式 → 卡顿
 *
 * ✅ 正确做法：
 *    异步加载资源，输入模式切换是瞬间完成的
 */
```

### 7.3 完整的输入模式切换流程图

```
玩家输入
    │
    ▼
PlayerController::OnInventoryKeyPressed() / OnPauseKeyPressed()
    │
    ▼
UIManager::ToggleInventory() / ToggleMainMenu()
    │
    ├─→ 更新内部状态 (bIsInventoryOpen / bIsMainMenuOpen)
    │
    ├─→ 显示/隐藏对应的Widget
    │       │
    │       ├─ 打开：Widget->SetVisibility(Visible)
    │       │         Widget->AddToViewport(ZOrder)
    │       │
    │       └─ 关闭：Widget->SetVisibility(Collapsed)
    │
    ├─→ 调用 UpdateInputMode()
    │       │
    │       ├─ bIsMainMenuOpen == true
    │       │       └─→ SetUIOnlyInputMode()
    │       │            └─→ FInputModeUIOnly + bShowMouseCursor = true
    │       │
    │       ├─ bIsInventoryOpen == true
    │       │       └─→ SetGameAndUIInputMode()
    │       │            └─→ FInputModeGameAndUI + bShowMouseCursor = true
    │       │
    │       └─ 都没有
    │               └─→ SetGameInputMode()
    │                    └─→ FInputModeGameOnly + bShowMouseCursor = false
    │
    └─→ 控制游戏暂停状态
            │
            ├─ 主菜单打开 → SetPause(true)
            └─ 主菜单关闭 → SetPause(false)
```
