## 1. UMG是什么

### 1.1 基本概念

**UMG（Unreal Motion Graphics）** 是虚幻引擎提供的UI制作系统。你可以把它理解为一个"UI编辑器"，让你能够像搭积木一样创建游戏中的各种界面。

| 概念             | 类比生活中的东西                      |
| ---------------- | ------------------------------------- |
| UMG              | 整个UI制作工具，就像Photoshop用来做图 |
| Widget Blueprint | 一个具体的UI界面，就像一张画布        |
| Widget控件       | 画布上的元素，就像画笔和颜料          |
| CanvasPanel      | 画布本身，用来放置其他元素            |

### 1.2 UMG能做什么

- 游戏内的HUD（血条、弹药、小地图等）
- 主菜单界面（开始游戏、设置、退出）
- 背包/物品栏界面
- 对话系统界面
- 设置界面（音量、画质等）
- 加载画面
- 任何你在游戏中看到的2D界面

### 1.3 UMG与Slate的关系

`UMG` 是建立在 `Slate` 之上的上层系统。Slate是UE底层的UI框架（纯C++），而UMG在Slate之上提供了可视化编辑器和蓝图支持，让不熟悉C++的人也能轻松制作UI。

```
┌─────────────────────┐
│      UMG（可视化）    │  ← 新手学习这个
├─────────────────────┤
│   Slate（C++底层）    │  ← 进阶学习这个
├─────────────────────┤
│   渲染系统 / 引擎     │
└─────────────────────┘
```

### 1.4 关键术语对照表

| 英文术语  | 中文释义     | 说明                 |
| --------- | ------------ | -------------------- |
| Widget    | 控件/部件    | UI的基本组成单元     |
| Panel     | 面板/容器    | 用来装其他控件的控件 |
| Blueprint | 蓝图         | UE的可视化脚本系统   |
| UMG       | 虚幻运动图形 | UE的UI制作系统       |
| HUD       | 抬头显示     | 游戏进行中显示的界面 |
| Canvas    | 画布         | 自由布局的容器       |

## 2. UserWidget的概念

### 2.1 UserWidget是什么

`UserWidget` 是UE中所有UI界面的"基类"——它就像是UI的"蓝图"。每一个游戏界面（主菜单、背包、HUD等）都是一个继承自 `UUserWidget` 的类。

```
你创建一个游戏界面
        │
        ▼
   UUserWidget（基类）
        │
        │  继承
        ▼
你的具体界面：UMyHUD / UMyMenu / UMyInventory
```

### 2.2 UserWidget的生命周期

理解Widget的生命周期非常重要，因为很多新手在此犯错：

```
1. Construct()          ← Widget被创建时调用（蓝图层面）
2. NativeConstruct()    ← Widget被创建时调用（C++层面）
      │
3. 显示在屏幕上
      │
4. NativeTick()         ← 每帧调用（如果启用了Tick）
      │
5. NativeDestruct()     ← Widget被销毁时调用（C++层面）
6. Destruct()           ← Widget被销毁时调用（蓝图层面）
```

### 2.3 一个Widget可以做什么

- **显示信息：** 血量、分数、时间、文字提示等
- **接收输入：** 按钮点击、文本框输入、滑块拖动等
- **播放动画：** 界面出现/消失的过渡动画
- **响应用户操作：** 打开背包、关闭菜单等

### 2.4 直观理解

```
想象一下手机App：
┌──────────────────────────────────┐
│  UserWidget = 整个App的界面      │
│  ┌────────────────────────────┐  │
│  │  Panel = 容器（放东西的盒子）│  │
│  │  ┌──────┐  ┌──────────┐   │  │
│  │  │Button│  │ TextBlock│   │  │
│  │  │ 按钮 │  │   文字    │   │  │
│  │  └──────┘  └──────────┘   │  │
│  └────────────────────────────┘  │
└──────────────────────────────────┘
```

## 3. 编辑器中创建Widget Blueprint

### 3.1 创建步骤（一步一步跟我做）

**步骤1：打开内容浏览器**

在UE编辑器底部找到"内容浏览器"（Content Browser）面板。如果没有看到，点击菜单栏 `Window → Content Browser` 打开。

**步骤2：选择创建位置**

在内容浏览器左侧的文件夹树中，选择一个文件夹（建议创建一个 `UI` 或 `Widgets` 文件夹来存放UI资源）。

右键点击文件夹空白处 → 选择 `New Folder` → 命名为 `UI`

**步骤3：创建Widget Blueprint**

右键点击 `UI` 文件夹的空白处 → 选择 `User Interface → Widget Blueprint`

```
右键空白处
    → User Interface
        → Widget Blueprint ✓   ← 选这个
```

**步骤4：命名**

给Widget Blueprint取一个有意义的名字，建议命名规范：

```
✅ WBP_MainMenu        ← WBP = Widget Blueprint 的缩写，推荐
✅ WBP_HUD
✅ WBP_Inventory
❌ NewWidgetBlueprint1  ← 不要用默认名字
❌ UI_1                 ← 名字不够清晰
```

**步骤5：双击打开**

双击刚创建的Widget Blueprint，会打开UMG编辑器。

### 3.2 UMG编辑器界面介绍

打开Widget Blueprint后，你会看到：

```
┌──────────────────────────────────────────────────────────┐
│  菜单栏                                                    │
├────────────┬──────────────────────────┬──────────────────┤
│  Palette   │                          │    Details       │
│  控件面板   │     设计器视图            │    细节面板       │
│  (左侧)    │    (中间)                │    (右侧)        │
│            │  ┌──────────────────┐   │                  │
│  常用控件   │  │                  │   │   选中控件的      │
│  面板容器   │  │   你的UI界面      │   │   所有属性       │
│  按钮文本   │  │   拖拽布局        │   │                  │
│  输入框等   │  │                  │   │                  │
│            │  └──────────────────┘   │                  │
├────────────┴──────────────────────────┴──────────────────┤
│  Hierarchy（层级面板）— 显示所有控件的父子关系              │
└──────────────────────────────────────────────────────────┘
```

**四个主要面板：**

| 面板名称              | 位置 | 作用                             |
| --------------------- | ---- | -------------------------------- |
| Palette（控件面板）   | 左侧 | 列出所有可拖拽的控件，就像工具箱 |
| 设计器视图            | 中间 | 可视化的界面设计区域，所见即所得 |
| Details（细节面板）   | 右侧 | 调整选中控件的具体属性           |
| Hierarchy（层级面板） | 下方 | 显示控件的树形层级关系           |

### 3.3 往界面上添加控件

**方法1：拖拽法（最常用）**

1. 在Palette面板中找到想要的控件（比如 `TextBlock`）
2. 按住鼠标左键拖拽到设计器视图中
3. 松开鼠标，控件就出现在界面上了

**方法2：右键添加**

1. 在设计器视图中右键
2. 在弹出菜单中找到想要的控件类型
3. 点击添加

### 3.4 选择和移动控件

- **选择控件：** 在设计器视图中点击控件，或者在Hierarchy面板中点击控件名
- **移动控件：** 选中后直接拖拽
- **调整大小：** 拖拽控件四周的白色方块（调整手柄）
- **精确调整：** 在Details面板中修改Position和Size数值

## 4. 容器面板详解

> 容器（Panel）是用来"装"其他控件的控件。它们是UI布局的核心。

### 4.1 CanvasPanel（画布面板）

**一句话理解：** 最自由的容器，控件可以放在任何位置，就像在一张白纸上画画。

```
┌──────────────────────────┐
│  CanvasPanel 画布         │
│                           │
│    ┌──┐         ← 可以随便放         │
│    └──┘                   │
│               ┌──┐        │
│               └──┘        │
│       ┌──┐                │
│       └──┘                │
│                           │
└──────────────────────────┘
```

**特点：**

- 子控件通过绝对位置定位（X, Y坐标）
- 也可以使用锚点进行相对定位
- 最灵活，但也最容易在分辨率变化时出问题

**使用场景：**

- 需要精确控制每个元素位置时
- 游戏HUD（小地图在右上角，血量在左下角等）
- Widget Blueprint的默认根容器就是CanvasPanel

**在C++中使用：**

```cpp
// 在C++中创建CanvasPanel
UCanvasPanel* MyCanvas = NewObject<UCanvasPanel>(this);

// 在C++中创建CanvasPanelSlot（用于设置子控件的位置）
UCanvasPanelSlot* Slot = Cast<UCanvasPanelSlot>(MyCanvas->AddChild(MyButton));
if (Slot)
{
    Slot->SetPosition(FVector2D(100.0f, 200.0f));  // 设置位置 X=100, Y=200
    Slot->SetSize(FVector2D(300.0f, 50.0f));       // 设置大小 宽300, 高50
}
```

### 4.2 HorizontalBox（水平盒子）

**一句话理解：** 把所有子控件从左到右排成一行。

```
┌──────────────────────────────────┐
│  HorizontalBox 水平盒子          │
│  ┌──┐  ┌──┐  ┌──┐  ┌──┐        │
│  │A │→ │B │→ │C │→ │D │  水平排列│
│  └──┘  └──┘  └──┘  └──┘        │
└──────────────────────────────────┘
```

**特点：**

- 子控件自动从左到右排列
- 可以设置每个子控件的大小比例
- 自动处理间距和对齐

**使用场景：**

- 技能栏（一排技能图标）
- 顶部菜单栏
- 工具栏按钮组

**在C++中使用：**

```cpp
// 在C++中创建HorizontalBox
UHorizontalBox* HBox = NewObject<UHorizontalBox>(this);

// 在C++中创建HorizontalBoxSlot（用于设置填充和间距）
UHorizontalBoxSlot* Slot = Cast<UHorizontalBoxSlot>(HBox->AddChild(MyButton));
if (Slot)
{
    Slot->SetPadding(FMargin(5.0f));           // 设置内边距为5像素
    Slot->SetHorizontalAlignment(HAlign_Center); // 水平居中对齐
    Slot->SetVerticalAlignment(VAlign_Center);   // 垂直居中对齐
}
```

### 4.3 VerticalBox（垂直盒子）

**一句话理解：** 把所有子控件从上到下排成一列。

```
┌──────┐
│  A   │  ┐
├──────┤  │
│  B   │  │ 垂直排列
├──────┤  │
│  C   │  │
├──────┤  │
│  D   │  ┘
└──────┘
```

**特点：**

- 子控件自动从上到下排列
- 可以设置每个子控件的大小比例
- 自动处理间距和对齐

**使用场景：**

- 任务列表
- 聊天消息列表
- 设置菜单项列表
- 背包物品列表

**在C++中使用：**

```cpp
// 在C++中创建VerticalBox
UVerticalBox* VBox = NewObject<UVerticalBox>(this);

// 在C++中创建VerticalBoxSlot
UVerticalBoxSlot* Slot = Cast<UVerticalBoxSlot>(VBox->AddChild(MyButton));
if (Slot)
{
    Slot->SetPadding(FMargin(10.0f));          // 设置内边距为10像素
    Slot->SetHorizontalAlignment(HAlign_Fill);  // 水平填充
    Slot->SetVerticalAlignment(VAlign_Center);  // 垂直居中
}
```

### 4.4 Overlay（覆盖层）

**一句话理解：** 所有子控件叠在一起，就像一叠透明的纸。

```
┌─────────────────────┐
│         C           │ ← 最上层
│  ┌───────────┐      │
│  │     B     │      │ ← 中间层
│  │  ┌─────┐  │      │
│  │  │  A  │  │      │ ← 最下层
│  │  └─────┘  │      │
│  └───────────┘      │
└─────────────────────┘
```

**特点：**

- 所有子控件占据相同的空间
- 后面的控件覆盖在前面的控件之上
- 适合做层叠效果

**使用场景：**

- 在图片上叠加文字
- 制作进度条（背景 + 填充 + 文字）
- 制作按钮（背景图 + 文字）
- 在角色头顶显示血条和名字

**在C++中使用：**

```cpp
// 在C++中创建Overlay
UOverlay* MyOverlay = NewObject<UOverlay>(this);

// 在C++中创建OverlaySlot（设置对齐方式）
UOverlaySlot* Slot = Cast<UOverlaySlot>(MyOverlay->AddChild(MyWidget));
if (Slot)
{
    Slot->SetHorizontalAlignment(HAlign_Center);  // 水平居中
    Slot->SetVerticalAlignment(VAlign_Center);    // 垂直居中
}
```

### 4.5 WrapBox（自动换行盒子）

**一句话理解：** 子控件从左到右排列，超出宽度后自动换行，就像文字换行一样。

```
┌───────────────────────┐
│ ┌──┐ ┌──┐ ┌──┐ ┌──┐  │
│ │A │ │B │ │C │ │D │  │
│ └──┘ └──┘ └──┘ └──┘  │
│ ┌──┐ ┌──┐             │ ← 放不下了，自动换行
│ │E │ │F │             │
│ └──┘ └──┘             │
└───────────────────────┘
```

**使用场景：**

- 背包网格（自动排列物品图标）
- 标签列表
- 技能图标排列

### 4.6 ScrollBox（滚动盒子）

**一句话理解：** 内容超出显示区域时，出现滚动条可以上下滚动查看。

```
┌───────────┐
│ 内容1     │  ┐
│ 内容2     │  │
│ 内容3     │  │
│ ██████░░░ │  │ ← 滚动条
│ 内容4     │  │
│ 内容5     │  ┘
└───────────┘
```

**使用场景：**

- 背包物品列表（物品太多时）
- 设置界面（选项太多时）
- 聊天记录
- 任务日志

### 4.7 GridPanel / UniformGridPanel（网格面板）

**一句话理解：** 把子控件放在一个表格的指定行列位置。

```
┌────┬────┬────┬────┐
│0,0 │0,1 │0,2 │0,3 │  ← 第0行
├────┼────┼────┼────┤
│1,0 │1,1 │1,2 │1,3 │  ← 第1行
├────┼────┼────┼────┤
│2,0 │2,1 │2,2 │2,3 │  ← 第2行
└────┴────┴────┴────┘
  ↑    ↑    ↑    ↑
  0列  1列  2列  3列
```

- **GridPanel：** 每行每列的大小可以不同
- **UniformGridPanel：** 所有格子大小一致

**使用场景：**

- 背包界面（规律排列的物品图标）
- 技能树界面
- 棋盘游戏界面

### 4.8 容器选择建议

| 需求                     | 推荐容器                     |
| ------------------------ | ---------------------------- |
| 自由放置元素（精确位置） | CanvasPanel                  |
| 水平排列元素             | HorizontalBox                |
| 垂直排列元素             | VerticalBox                  |
| 元素要叠在一起           | Overlay                      |
| 自动换行排列             | WrapBox                      |
| 内容太多需要滚动         | ScrollBox                    |
| 网格/表格排列            | GridPanel / UniformGridPanel |

## 5. 基础控件详解

### 5.1 Button（按钮）

**一句话理解：** 用户点击后触发某种动作的控件。

```
┌─────────────────┐
│    点击我开始      │
└─────────────────┘
    ↑ 按下去会触发事件
```

**核心属性：**

- `Style` — 按钮的视觉样式（普通、悬停、按下、禁用四种状态）
- `IsEnabled` — 是否可点击
- `OnClicked` — 点击事件（最重要！在这里写点击后的逻辑）

**常用操作：**

```
1. 设置按钮文字：在Button下面放一个TextBlock子控件
2. 设置按钮样式：在Details面板 → Appearance → Style
3. 绑定点击事件：在Details面板 → Events → OnClicked → 点击+
```

**正确做法：**

```
✅ Button的子控件通常是一个Overlay包裹的TextBlock或Image
✅ 一定要处理OnClicked事件，空按钮没有意义
✅ 设置不同状态的样式（Normal/Hovered/Pressed）提升体验
```

**错误做法：**

```
❌ Button放在ScrollBox中但不设置IsFocusable，可能导致滚动误触
❌ 按钮太小（<32x32像素），手机/触屏用户点不到
❌ OnClicked事件绑定到重量级操作，导致界面卡顿
```

### 5.2 TextBlock（文本块）

**一句话理解：** 显示一段文字的控件，UI中最基础的显示元素。

```
┌──────────────────────┐
│ 这里是显示的文字内容    │
└──────────────────────┘
```

**核心属性：**

| 属性                  | 作用                         | 示例             |
| --------------------- | ---------------------------- | ---------------- |
| `Text`                | 显示的文字内容               | "玩家名称：小明" |
| `Font`                | 字体设置（字体、大小、样式） | Size: 24, 粗体   |
| `Color and Opacity`   | 文字颜色和透明度             | 红色、半透明     |
| `Justification`       | 文字对齐方式                 | 居中、左对齐     |
| `Auto Wrap Text`      | 是否自动换行                 | true/false       |
| `Shadow Offset/Color` | 文字阴影                     | 偏移(1,1) 黑色   |

**代码中更新文字：**

```cpp
// C++中动态更新TextBlock的文字
MyTextBlock->SetText(FText::FromString(TEXT("消灭敌人: 5/10")));

// 使用格式化文本（支持占位符）
MyTextBlock->SetText(FText::Format(
    NSLOCTEXT("MyUI", "Score", "分数: {0}"),
    CurrentScore
));
```

### 5.3 Image（图片）

**一句话理解：** 显示一张图片（纹理）的控件。

```
┌──────────────┐
│              │
│   🖼️ 图片    │
│              │
└──────────────┘
```

**核心属性：**

| 属性                | 作用                    |
| ------------------- | ----------------------- |
| `Brush`             | 设置图片资源（Texture） |
| `Color and Opacity` | 图片颜色叠加和透明度    |
| `Brush Size`        | 图片的绘制大小          |

**代码中设置图片：**

```cpp
// C++中动态更换图片
// 1. 加载纹理资源
UTexture2D* MyTexture = LoadObject<UTexture2D>(
    nullptr,
    TEXT("/Game/UI/Textures/Icon_Sword")
);

// 2. 设置到Image控件上
if (MyTexture && MyImage)
{
    MyImage->SetBrushFromTexture(MyTexture);
}
```

**常见错误：**

```
✅ Image的Texture参数需要是导入的纹理资源
❌ 直接给Image一个Material会显示异常（Image期望Texture，不是Material）
✅ 用于UI的图片导入时建议设置Compression为"UserInterface2D"
❌ 用4096x4096的图片做一个小图标（浪费内存！用合适大小的图片）
```

### 5.4 ProgressBar（进度条）

**一句话理解：** 显示一个0%~100%的进度，常用于血条、经验条、加载进度。

```
┌──────────────────────────┐
│████████████░░░░░░░░░░░░░│ ← 60% 进度
└──────────────────────────┘
```

**核心属性：**

| 属性                     | 作用                                       |
| ------------------------ | ------------------------------------------ |
| `Percent`                | 当前进度值（0.0 ~ 1.0）                    |
| `Bar Fill Type`          | 填充方向（左到右、右到左、上到下、下到上） |
| `Fill Color and Opacity` | 填充部分的颜色                             |
| `Bar Fill Style`         | 填充区域的样式图片                         |

**代码中更新进度：**

```cpp
// C++中更新血条进度（血量百分比 0.0 ~ 1.0）
float HealthPercent = CurrentHealth / MaxHealth; // 计算血量百分比
HealthBar->SetPercent(HealthPercent);            // 设置进度条

// 数值必须保证在 0.0 ~ 1.0 之间
HealthBar->SetPercent(FMath::Clamp(HealthPercent, 0.0f, 1.0f));
```

### 5.5 Slider（滑块）

## 6. 锚点（Anchors）系统

### 6.1 什么是锚点

**一句话理解：** 锚点决定了一个控件"黏在"父容器的哪个位置，以及在窗口大小变化时控件如何伸缩。

```
┌──────────────────────────────────────┐
│●                  ●                  ●│ ← 左上、中上、右上锚点
│                                       │
│                                       │
│●                  ●                  ●│ ← 左中、居中、右中锚点
│                                       │
│                                       │
│●                  ●                  ●│ ← 左下、中下、右下锚点
└──────────────────────────────────────┘
```

### 6.2 锚点的两个作用

锚点同时控制两件事：

**作用1：位置锁定**

- 控件相对于父容器的哪个点保持固定距离
- 例如：锚点设为右上角 → 控件始终在右上角

**作用2：大小伸缩**

- 当父容器大小改变时，控件如何跟着缩放
- 例如：锚点设为上下拉伸 → 控件高度随父容器变化

### 6.3 常用锚点预设

在编辑器中选择控件后，可以在Details面板的Anchors下拉菜单中看到这些预设：

| 预设           | 图示   | 行为                     | 典型用途         |
| -------------- | ------ | ------------------------ | ---------------- |
| 左上角固定     | ●⬜⬜  | 固定在左上角，大小不变   | Logo图标         |
| 右上角固定     | ⬜⬜●  | 固定在右上角，大小不变   | 小地图、关闭按钮 |
| 居中固定       | ⬜●⬜  | 固定在屏幕中央，大小不变 | 暂停菜单         |
| 水平拉伸（顶） | ●⬜→●  | 固定顶部，左右拉伸       | 顶部状态栏       |
| 水平拉伸（底） | ●⬜→●  | 固定底部，左右拉伸       | 底部技能栏       |
| 全屏拉伸       | ●→●→●  | 四角和四边都拉伸         | 全屏背景图       |
| 比例定位       | 自定义 | 按百分比定位和缩放       | 精确响应式布局   |

### 6.4 直观理解锚点

```
假设一个控件锚点设置为"右上角"：

窗口 1920x1080               窗口 1280x720
┌─────────────┐              ┌─────────┐
│         ┌──┐│              │     ┌──┐│
│         │控││              │     │控││  ← 始终在右上角
│         └──┘│              │     └──┘│
│             │              │         │
└─────────────┘              └─────────┘
控件距右边100px              控件距右边100px
控件距上边50px               控件距上边50px
```

### 6.5 手动设置锚点（精确控制）

在Details面板中，找到 `Anchors` 属性：

```
Anchors:
  Minimum: X=0.0, Y=0.0    ← 锚点最小位置（左上角参考点）
  Maximum: X=1.0, Y=1.0    ← 锚点最大位置（右下角参考点）

如果Minimum和Maximum相同 → 只控制位置，不控制大小（点锚点）
如果Minimum和Maximum不同 → 同时控制位置和拉伸大小（区域锚点）
```

**示例1：固定在左上角，大小不变**

```
Minimum: (0.0, 0.0)
Maximum: (0.0, 0.0)
→ 锚点是一个点，控件大小不变
```

**示例2：横向拉伸，纵向固定**

```
Minimum: (0.0, 0.0)
Maximum: (1.0, 0.0)
→ 锚点是一条水平线，控件宽度随父容器变化
```

### 6.6 新手常见锚点问题

```
✅ 血条应该用水平拉伸锚点，这样不同分辨率下宽度都合适
❌ 血条用固定位置锚点 → 分辨率变了血条可能跑到屏幕外面去
✅ 小地图用右上角固定锚点
✅ 对话框用居中固定锚点
✅ 背景图用全屏拉伸锚点
```

## 7. ZOrder层级管理

### 7.1 什么是ZOrder

**一句话理解：** ZOrder决定了控件之间的"前后"关系，ZOrder越大的控件显示在越前面（越靠近观众）。

```
想象一叠扑克牌：
        ┌───┐
        │ A │  ZOrder=2  ← 最上面（看得最清楚）
    ┌───┼───┤
    │ B │   │  ZOrder=1  ← 中间
┌───┼───┤   │
│ C │   │   │  ZOrder=0  ← 最下面（可能被遮挡）
└───┘   │   │
        └───┘
```

### 7.2 ZOrder的使用方法

**在编辑器中设置：**

1. 选中一个控件
2. 在Details面板中找到 `ZOrder` 属性
3. 输入数值（默认是0）

**在Hierarchy面板中调整：**

在Hierarchy面板中，列表从上到下的顺序就是ZOrder从大到小的顺序：

- 列表顶部的控件 → ZOrder最大 → 显示在最前面
- 列表底部的控件 → ZOrder最小 → 显示在最后面

**拖拽调整：** 直接在Hierarchy面板中上下拖拽控件名字来调整顺序

### 7.3 ZOrder的实际应用

```
游戏暂停界面的ZOrder设计：

ZOrder 100  ← 暂停菜单面板（最上面，确保可见）
ZOrder 50   ← 设置子面板（弹出时在菜单之上）
ZOrder 10   ← 对话框面板
ZOrder 0    ← HUD（最底层，菜单打开后被遮挡）
```

### 7.4 CanvasPanel中的ZOrder

在CanvasPanel中，ZOrder的值直接决定渲染顺序：

```cpp
// C++中设置CanvasPanel子控件的ZOrder
UCanvasPanelSlot* Slot = Cast<UCanvasPanelSlot>(MyCanvas->AddChild(MyWidget));
if (Slot)
{
    Slot->SetZOrder(10); // 较大的ZOrder值 → 显示在前面
}
```

### 7.5 注意事项

```
✅ 重要的弹窗/对话框应该设置较大的ZOrder
✅ 始终可见的HUD元素可以设较小的ZOrder
❌ 不要给所有控件都随机设ZOrder → 会导致显示混乱
✅ 使用有意义的ZOrder命名区间（例如：0-9背景层，10-49游戏UI层，50-99弹出层，100+系统层）
```

## 8. DPI缩放处理

### 8.1 什么是DPI缩放

**一句话理解：** DPI缩放就是让你的UI在不同分辨率和不同大小的屏幕上看起来都一样好。

```
问题场景：
- 你做了一个UI，在1920x1080的屏幕上很好看
- 但在4K屏幕（3840x2160）上，UI变得非常小
- 或者在1366x768的笔记本上，UI又变得特别大

DPI缩放就是解决这个问题的！让UI在任何屏幕上看起来大小都合适。
```

### 8.2 UE中的DPI缩放设置

**项目设置中的DPI曲线：**

1. 打开 `Edit → Project Settings`
2. 搜索 `DPI`
3. 找到 `Engine → User Interface → DPI Scaling`
4. 设置 `DPI Scale Rule` 和 `DPI Curve`

```
主要设置选项：
┌──────────────────────────────────────────────┐
│ DPI Scale Rule：                             │
│   - Shortest Side：基于屏幕短边计算缩放       │
│   - Longest Side：基于屏幕长边计算缩放        │
│   - Horizontal：基于水平分辨率计算缩放        │
│   - Vertical：基于垂直分辨率计算缩放          │
│   - Custom：使用自定义DPI曲线                 │
└──────────────────────────────────────────────┘
```

### 8.3 推荐设置（开箱即用）

对于新手，推荐使用以下设置：

```
DPI Scale Rule: Shortest Side
→ 这意味着以1080p（1920x1080，短边1080）为基准
→ 在720p屏幕上，UI会等比缩小
→ 在4K屏幕上，UI会等比放大
```

### 8.4 DPI对实际开发的影响

**UI素材准备：**

```
✅ UI图片素材建议以1080p为基准分辨率制作
✅ 矢量图形比位图更适合做UI（缩放不失真）
❌ 不要以720p做UI素材然后在4K下显示（会模糊）
✅ Slate Brush的DrawAs属性可以设置缩放行为
```

**字体处理：**

```
✅ 选择合适的字号（有意识考虑不同屏幕）
✅ 使用TextBlock的Auto Wrap功能适应不同宽度
✅ Font的Size属性支持根据DPI自动调整
```

**代码中处理DPI：**

```cpp
// 获取当前的DPI缩放比例
float DPIScale = GetGameInstance()
    ? GetGameInstance()->GetDPIScaleFactorForScreenResolution()
    : 1.0f;

// 根据DPI调整UI元素大小
float AdjustedSize = BaseSize * DPIScale;
```

### 8.5 测试不同分辨率

在编辑器中可以直接测试不同分辨率的显示效果：

1. 在UMG编辑器右上角找到分辨率预览下拉框
2. 选择不同的分辨率（如1280x720, 1920x1080, 3840x2160）
3. 观察你的UI在不同分辨率下的表现
4. 调整锚点和布局直到所有分辨率都满意

```
建议测试的分辨率：
1. 1280 x 720   ← 最小常见分辨率
2. 1920 x 1080  ← 标准分辨率（开发基准）
3. 2560 x 1440  ← 2K
4. 3840 x 2160  ← 4K
```
