## 1. 必看的官方资源

官方资源质量最高、更新最及时，应该是你的**第一信息源**。

### 1.1 UE5官方文档

| 资源                             | 链接                        | 说明                                |
| -------------------------------- | --------------------------- | ----------------------------------- |
| **官方文档**                     | docs.unrealengine.com       | 所有API、系统、教程的总入口         |
| **C++ API Reference**            | docs.unrealengine.com/API   | 每个类的详细说明                    |
| **Gameplay Ability System 文档** | docs.unrealengine.com → GAS | GAS的官方文档，虽然偏简略但是最权威 |

**学习建议**：

- 遇到不熟悉的API，第一反应应该是查官方文档而不是搜索引擎
- 阅读文档时注意版本标签（UE 5.0 / 5.1 / 5.2 / 5.3 / 5.4），不同版本API可能有变化
- 官方文档的"Quick Start"和"Programming Guide"是必读的

### 1.2 UE5示例项目

Epic官方提供的示例项目是**最好的学习资料**，因为它们是正确的、完整的、经过了大规模验证的。

| 示例项目              | 重点学习内容                                             | 适合阶段  |
| --------------------- | -------------------------------------------------------- | --------- |
| **Lyra Starter Game** | GAS完整实战、Enhanced Input、模块化GameFeature、CommonUI | 中级/高级 |
| **Action RPG**        | GAS基础、Ability/Effect/Tag/Cue的完整使用                | 初级/中级 |
| **City Sample**       | 大规模开放世界、Mass Entity、World Partition             | 高级      |

**Lyra项目是"UE5最佳实践的百科全书"**，建议这样学习：

1. 首先运行起来，体验一下有哪些功能
2. 看 `ShooterCore` 插件中的技能实现
3. 看 `GameFeatures` 插件如何实现模块化
4. 研究 `CommonUI` 的使用方式
5. 理解 `ModularGameplayActors` 的组件化Actor设计

- 不要一次买太多书，读完一本再买下一本

## 4. 职业发展路径

### 4.1 全景图

```
初级工程师 ──→ 中级工程师 ──→ 高级工程师 ──→ 技术专家/架构师
  (1-3年)       (3-5年)        (5-8年)         (8年+)
     │              │              │                │
  执行功能      独立负责       技术方案         行业影响力
    模块         子系统         决策             技术方向制定
```

### 4.2 初级UE开发工程师（1-3年）

**定位**：在指导下完成具体功能模块的开发。

**日常工作**：

- 实现具体玩法功能的C++/蓝图代码（例如：实现一个可拾取的道具系统）
- 使用UI框架（UMG）开发游戏界面
- 实现基本的角色控制和动画逻辑
- 修复日常bug
- 参与Code Review，接受反馈

**核心技能要求**：

- UE C++编程基础：Actor/Component架构、UPROPERTY/UFUNCTION的使用
- 蓝图基础：能理解和使用蓝图，合理划分蓝图与C++的职责边界
- UMG界面开发
- 动画蓝图基础（AnimBlueprint、State Machine、Blend Space）
- 基本的调试能力（断点、日志、Unreal Insights入门）
- 版本控制（Git/Perforce）

**薪资范围（国内参考）**：

- 一线城市（北上广深）：12k-22k/月
- 新一线城市（杭州、成都、武汉等）：10k-18k/月
- 其他城市：8k-14k/月

> 注：薪资受公司规模、项目阶段、个人学历、面试表现等多因素影响，以上仅为大致参考范围。同一城市内不同公司的差距可能非常大。

### 4.3 中级UE开发工程师（3-5年）

**定位**：独立负责一个子系统（如战斗系统、AI系统、UI框架）的设计与实现。

**日常工作**：

- 独立负责子系统的整体设计与实现
- 指导初级工程师
- 参与技术方案评审，评估可行性和风险
- 性能和内存优化
- 跨部门协作（与策划、美术确定方案）

**核心技能要求**：

- **GAS（Gameplay Ability System）**：能独立搭建完整的技能/属性/状态系统
- **网络同步**：理解RPC、属性同步、Relevancy、Dormancy
- **性能优化**：能用Unreal Insights定位瓶颈并给出优化方案
- **架构设计**：能合理拆分模块，设计扩展性良好的系统
- **AI系统**：Behavior Tree、EQS、AI Perception的综合运用
- **编辑器工具**：能开发简单的编辑器工具提升团队效率

**薪资范围（国内参考）**：

- 一线城市：20k-35k/月
- 新一线城市：16k-28k/月
- 核心项目组/大厂可能更高

**这一阶段的"分水岭"**：

- ❌ 停留在"能用就行"：功能实现了但逻辑混乱、性能差、难以扩展
- ✅ 追求"设计先行"：写代码前先画类图，思考扩展性，考虑网络同步，写技术文档

### 4.4 高级UE开发工程师（5-8年）

**定位**：做出技术方案决策，推动团队技术能力提升。

**日常工作**：

- 负责核心系统的架构设计和技术选型
- 解决团队遇到的疑难技术问题
- 制定代码规范和技术标准
- 性能评审和架构评审
- 跨团队技术协调
- 招聘面试和技术人才引进

**核心技能要求**：

- **引擎底层理解**：能看懂UE引擎源码，定位引擎层问题并能修改/扩展
- **图形学基础**：渲染管线、Shader语言、材质系统
- **工具链开发**：能开发复杂的编辑器工具、自动化流水线
- **技术领导力**：能推动技术决策在团队中落地
- **性能优化专家级**：CPU/GPU/内存/IO各维度的深度优化

**薪资范围（国内参考）**：

- 一线城市：35k-55k/月，加项目奖金/年终
- 核心项目组/知名大作项目：上限更高
- 部分公司有股票/期权

### 4.5 技术专家/架构师（8年+）

**定位**：行业内有技术影响力，可能负责一个工作室的技术方向。

**薪资范围**：50k-80k+/月，加股权/期权激励。（此阶段薪资个体差异极大，不适合用范围来描述）

### 4.6 其他发展方向

并不是所有人都会走到技术专家这条路，还有这些方向：

| 方向               | 说明                                                     |
| ------------------ | -------------------------------------------------------- |
| **技术美术（TA）** | 连接美术和程序，擅长Shader、材质、渲染管线、工具开发     |
| **技术策划（TD）** | 擅长编辑器工具和流水线开发，提升策划的工作效率           |
| **独立开发者**     | 自己做游戏，一人包揽技术、美术、设计                     |
| **技术管理**       | 从技术Leader转向团队管理（但通常仍然需要保持技术敏感度） |
| **外包创业**       | 成立UE技术服务公司，为其他公司提供开发服务               |

## 5. 持续学习的建议

### 5.1 每周至少读一篇引擎源码

这是从"能用UE"到"理解UE"的关键一步。

**具体做法**：

1. 从你常用的类开始读：`AActor`、`UActorComponent`、`APlayerController`
2. 带着问题读源码：例如"GC到底是在哪一步触发标记的？"然后去源码里找答案
3. 不必读懂每一行，目标是理解**核心流程**和**设计思路**
4. 读完一个类后，写一篇简短的笔记（哪怕只是内部给自己看的）

**推荐第一个深入阅读的类**：

- `AActor::BeginPlay()` → 理解Actor的初始化流程
- `UAbilitySystemComponent::TryActivateAbility()` → 理解GAS的核心流程
- `UNetDriver::TickFlush()` → 理解网络数据包的发收流程

### 5.2 保持做Side Project的习惯

工作中的项目受限于商业需求和时间压力，Side Project是你**自由探索技术**的空间。

**Side Project的建议**：

- 小而精：不要试图做一个完整游戏，聚焦一个技术点
- 可展示：做完后放到GitHub上，写清楚的README
- 有持续性：一年做3-4个小项目比做一个永远完不成的巨作要好

**Side Project灵感**：

- "用UE5实现一个回合制战斗系统"
- "基于编辑器扩展做一个关卡生成工具"
- "复刻一个经典游戏的核心机制（如《杀戮尖塔》的卡牌战斗、《传送门》的空间穿梭）"
- "为Lyra项目添加一个新玩法（如潜行系统）"

### 5.3 参加Game Jam

Game Jam是限定时间内（24小时、48小时、72小时）完成一个游戏的比赛。

**为什么参加**：

- 高强度压缩的完整开发流程（从设计到可玩）
- 锻炼快速做决策和"完成比完美重要"的心态
- 团队合作（通常是2-5人）
- 产出可以直接放进作品集

**推荐的Game Jam**：

- **Global Game Jam**：全球最大的Game Jam
- **Ludum Dare**：历史最悠久，有Compo（48h独奏）和Jam（72h团队）两种模式
- **Epic MegaJam**：Epic官方举办的以UE为主题的Game Jam
- **国内的线上/线下Game Jam**：关注各地游戏开发者社区的通知

### 5.4 关注行业动态

**保持信息渠道活跃**：

| 渠道                             | 内容                                 |
| -------------------------------- | ------------------------------------ |
| **GDC Vault**                    | 游戏开发者大会的演讲录像（部分免费） |
| **Unreal Engine Blog**           | Epic官方的技术博客                   |
| **80.lv**                        | 游戏技术访谈和案例研究               |
| **Game Developer (原Gamasutra)** | 行业分析和开发经验                   |
| **Twitter/X**                    | 关注Epic的工程师和知名UE技术博主     |

**国内信息源**：

- 游戏行业媒体：GameLook、游戏葡萄、手游那点事
- 知乎上的游戏开发话题
- 各大游戏公司的技术博客（如：网易游戏技术博客、腾讯游戏学堂）

### 5.5 建立知识体系，而非碎片积累

**❌ 碎片化学习的问题**：

- 今天看一个GAS教程，明天看一个渲染教程，后天看一个AI教程
- 每样都懂一点皮毛，但没有一样深入
- 面试时被稍微深入提问就卡住

**✅ 建立知识体系**：

- 选择1-2个方向作为你的核心竞争力（如GAS+网络同步）
- 系统性地深入（而非浅尝辄止）
- 能用简单的语言把一个复杂的技术概念讲清楚（费曼学习法）
- 写技术博客是检验你有没有真正理解的最好方式

## 6. 求职时间线（校招参考）

如果你是应届生，这是一个参考时间线：

```
大三上学期（9-12月）
  └─ 开始认真学习UE5 C++（你现在可能就在这个阶段）

大三寒假（1-2月）
  └─ 完成第一个完整的UE C++项目
  └─ 把项目放到GitHub上，写好README

大三下学期（3-6月）
  └─ 尝试投递暑期实习
  └─ 参加Game Jam积累项目
  └─ 开始写技术博客

大三暑假（7-9月）
  └─ 暑期实习（最重要的经历背书）
  └─ 如果没有实习，集中精力做Side Project
  └─ 完善作品集

大四上学期（9-11月）★ 秋招黄金期
  └─ 8月开始准备简历
  └─ 9月密集投递 → 大厂的秋招集中在这个阶段
  └─ 9-10月密集面试

大四下学期（3-5月）★ 春招补充期
  └─ 秋招没拿到满意Offer的可以在春招补投
  └─ 部分公司的补录名额
```

**如果不是应届生**（社招）：没有固定的时间线。通常在金三银四（3-4月）和金九银十（9-10月）是招聘旺季。

## 7. 关于薪资谈判的几句话

- **做功课**：面试前了解目标公司/岗位的薪资范围（通过脉脉、知乎、朋友）
- **不要主动说出你的最低期望**：等对方先开口，或者给出一个合理的范围
- **薪资不只是月薪**：年终奖（几个月？）、项目奖金、股票/期权、补贴（房补、餐补、交通补）、五险一金的缴纳基数和比例
- **第一份工作的薪资很重要**：它会影响你后续每次跳槽的基准线
- **但不要只看薪资**：对于初级岗位，**成长空间 > 项目质量 > 团队氛围 > 薪资**。一个好项目中学到的技术，在下一份工作中会转化成更高的薪资

## 8. 最后的话

### 8.1 完成比完美重要

这套手册的16章技术内容加上本章的就业指导，信息量很大。你可能没有全部掌握，你可能觉得自己的地牢探索者项目还不够好，你可能觉得还没准备好投简历。

**这些感觉都很正常。**

但我想告诉你一个事实：**绝大多数拿到Offer的人，在面试的时候也觉得自己"还没准备好"。**

软件开发不是一个"准备好了才能开始的领域"——它是一个"开始了才开始准备的领域"。你的第一个UE C++项目会很粗糙，你的第一次技术面试可能会失败，你的第一年工作会经常遇到不会的东西。但这都是过程的一部分。

关键在于：**开始行动，并且持续行动。**

### 8.3 你的下一步

学完本手册，建议你按照以下优先级行动：

1. **完善你的地牢探索者项目**
   - 确认代码编译通过，游戏可以正常运行
   - 写一份好的README（参考第1章的建议）
   - 录制一个3-5分钟的演示视频
   - 把项目上传到GitHub

2. **做至少一个额外的项目**
   - 不要把鸡蛋放在一个篮子里
   - 建议做一个技术专项Demo（如纯GAS + 网络同步的技术演示）
   - 或者参加一次Game Jam

3. **写一篇技术博客**
   - 记录你在项目中解决的一个技术问题
   - 或者你对某个UE系统的理解
   - 发表到知乎/掘金/个人博客

4. **准备简历和面试**
   - 按照第1章的建议写简历
   - 按照第2章的内容准备面试
   - 找朋友做模拟面试

5. **开始投递**
   - 不要等到"完美"才投
   - 先投几家公司练手，积累面试经验
   - 持续改进简历和面试表现

### 8.4 本手册只是一个起点

如果说游戏开发是一座山，那么这套手册只是给你画了一张登山地图。

真正的攀登，需要你：

- 日复一日的代码编写
- 一次次失败的调试
- 对未知问题的探索和求解
- 从错误中积累的经验

没有捷径，但也正因为没有捷径，这个行业才值得你投入。

**你已经迈出了最重要的一步：开始学习。现在，迈出第二步：开始行动。**
