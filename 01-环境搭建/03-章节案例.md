# 第1章案例：自定义一个会旋转的Cube

> **难度**：⭐（入门级）
> **预计时间**：30分钟
> **目标**：亲手创建第一个C++ Actor类，让一个Cube在场景中自动旋转。

---

## 案例效果

完成后你会得到一个：

- 放在场景中就会**自动绕Z轴旋转**的金色Cube
- 旋转速度可以在编辑器细节面板中**随时调整**
- 可以**拖入任意场景**中直接使用

---

## 第一步：创建C++类

### 1.1 打开你的C++项目

用你在1.2节创建的项目（或任意UE5 C++项目），打开UE编辑器。

### 1.2 创建新的C++类

1. UE编辑器菜单 → **Tools → New C++ Class...**

   > 如果这个选项是灰色的，说明项目本身不是C++项目。确认创建项目时选择了"C++"而不是"Blueprint"。

2. 在弹出的对话框中：

   | 选项     | 选择       |
   | -------- | ---------- |
   | 父类选择 | **Actor**  |
   | 点击     | **Next >** |

3. 命名设置：

   | 选项             | 填入                       |
   | ---------------- | -------------------------- |
   | **Name（类名）** | `RotatingCube`             |
   | **Path（路径）** | 保持默认（Source/项目名/） |
   | 点击             | **Create Class**           |

4. UE会自动：
   - 生成 `.h` 和 `.cpp` 文件
   - 打开Visual Studio（或提示你手动打开）

### 1.3 类名命名规范说明

UE有一套严格的命名前缀规则：

| 前缀 | 表示                    | 示例                          |
| ---- | ----------------------- | ----------------------------- |
| `A`  | 继承自AActor            | `ACharacter`, `ARotatingCube` |
| `U`  | 继承自UObject           | `UActorComponent`, `UTexture` |
| `F`  | 纯数据结构（非UObject） | `FVector`, `FString`          |
| `I`  | 接口类                  | `IInputDevice`                |
| `E`  | 枚举                    | `ECollisionChannel`           |
| `T`  | 模板类                  | `TArray`, `TMap`              |

> **规则**：你创建的类必须遵循这个前缀规则，否则编译会报警告。

## 第二步：编写头文件（.h）

打开VS，找到 `RotatingCube.h`，将内容替换为：

```cpp
// RotatingCube.h
// 一个会自动旋转的Cube Actor

#pragma once

#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "RotatingCube.generated.h"  // ← 必须是最后一个include

UCLASS(Blueprintable)  // ← Blueprintable = 蓝图可以继承
class MYFIRSTUE5PROJECT_API ARotatingCube : public AActor
{
    GENERATED_BODY()   // ← 必须有！

public:
    // 构造函数：创建这个Actor时自动调用
    ARotatingCube();

protected:
    // BeginPlay：游戏开始时调用一次（类似"开机自检"）
    virtual void BeginPlay() override;

public:
    // Tick：每一帧调用一次（UE每秒渲染60-120帧，Tick就每秒调用60-120次）
    virtual void Tick(float DeltaTime) override;

    // ===== 我们定义的属性 =====

    // 静态网格体组件：就是Cube的"身体"——它的3D模型
    // VisibleAnywhere = 编辑器中可见但不可编辑（组件本身）
    // BlueprintReadOnly = 蓝图只读
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly)
    class UStaticMeshComponent* CubeMesh;  // ← 组件指针，必须UPROPERTY！

    // 旋转速度：每秒旋转多少度
    // EditAnywhere = 可以在编辑器详情面板直接改数值
    // BlueprintReadWrite = 蓝图也能读写
    // meta = 元数据，ClampMin/ClampMax限制数值范围在0-720度之间
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Rotation Settings",
              meta = (ClampMin = "0.0", ClampMax = "720.0"))
    float RotationSpeed = 90.0f;  // ← 默认每秒旋转90度

    // 是否启用旋转（方便随时开关）
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Rotation Settings")
    bool bEnableRotation = true;  // ← UE惯例：布尔变量以b开头
};
```

### 逐行理解

```cpp
class MYFIRSTUE5PROJECT_API ARotatingCube : public AActor
```

- `MYFIRSTUE5PROJECT_API`：这个宏由UE自动生成（在你的项目名.Build.cs所在文件夹）。它确保这个类可以被导出到其他模块使用。如果你的项目叫 `MyGame`，这个宏就是 `MYGAME_API`。
- `ARotatingCube`：类名，`A` 前缀表示它继承自 Actor。
- `: public AActor`：继承自 `AActor`。Actor 是UE中所有可放入场景的物体的基类。

```cpp
virtual void BeginPlay() override;
virtual void Tick(float DeltaTime) override;
```

- `override`：C++11关键字，明确表示"我在覆写父类的虚函数"。如果你把函数名拼错了（比如写成 `BeginPlayy`），编译器会报错"找不到基类中的对应函数"。这是C++帮你防止低级错误。
- `virtual`：虚函数。子类可以覆写它的行为。
- `BeginPlay()`：游戏开始（或Actor被放入已运行的世界）时调用一次。常用于初始化逻辑。
- `Tick(float DeltaTime)`：**每一帧**调用一次。`DeltaTime` 是距离上一帧的时间（秒），用来做帧率无关的平滑运动。

> **为什么Tick需要DeltaTime？** 假设你的旋转速度是90度/秒。60FPS时每帧转1.5度，120FPS时每帧只转0.75度——用 `DeltaTime` 乘以速度就能确保无论帧率多少，每秒实际转的角度都相同。

```cpp
UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Rotation Settings",
          meta = (ClampMin = "0.0", ClampMax = "720.0"))
float RotationSpeed = 90.0f;
```

- `Category = "Rotation Settings"`：在编辑器Details面板中，这个属性会显示在"Rotation Settings"分组下。
- `meta = (ClampMin = "0.0", ClampMax = "720.0")`：元数据。`ClampMin`/`ClampMax` 告诉编辑器这是一个数值范围，在UI中会显示为带滑条的输入框，且用户不能输入超出范围的值。

## 第三步：编写实现文件（.cpp）

打开 `RotatingCube.cpp`，将内容替换为：

```cpp
// RotatingCube.cpp
// 自动旋转Cube的实现

#include "RotatingCube.h"          // ← 先包含自己的头文件
#include "Components/StaticMeshComponent.h"  // ← UStaticMeshComponent的完整定义

// 构造函数：对象的"出厂设置"
ARotatingCube::ARotatingCube()
{
    // 开启Tick，注意这里一定要开启，不然Tick不会被触发，旋转不会生效
    PrimaryActorTick.bCanEverTick = true;
    // UE_LOG：输出日志到Output Log窗口
    // LogTemp：临时日志类别
    // Warning：警告级别（显示为黄色）
    // TEXT()：把字符串转为UE的宽字符格式（必须用TEXT包裹中文和大多数字符串）
    UE_LOG(LogTemp, Warning, TEXT("RotatingCube 被创建了！"));

    // ===== 创建静态网格体组件 =====
    // CreateDefaultSubobject<T>：在构造函数中创建子对象（组件）
    // TEXT("CubeMesh")：给这个组件起个内部名字（调试用，可随便起）
    CubeMesh = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("CubeMesh"));

    // 把这个组件设为根组件
    // 根组件 = 这个Actor的空间原点，决定Actor在场景中的位置/旋转/缩放
    // 一个Actor必须有一个根组件，否则不能放入场景
    RootComponent = CubeMesh;

    // ===== 加载一个Cube模型 =====
    // ConstructorHelpers：只能在构造函数中使用的辅助类
    // FObjectFinder：按路径查找资产
    // 这个路径指向引擎自带的基础Cube模型（1x1x1米的方块）
    static ConstructorHelpers::FObjectFinder<UStaticMesh> CubeAsset(
        TEXT("/Engine/BasicShapes/Cube")
        //     └──────────┬──────────┘
        //     引擎内置的基础形状文件夹路径
    );

    // 如果找到了资产（通常都会找到，这是引擎自带的）
    if (CubeAsset.Succeeded())
    {
        // SetStaticMesh：设置静态网格体组件的3D模型
        // CubeAsset.Object：FObjectFinder找到的UStaticMesh对象指针
        CubeMesh->SetStaticMesh(CubeAsset.Object);

        // 播放日志确认加载成功
        UE_LOG(LogTemp, Warning, TEXT("Cube模型加载成功！"));
    }
    else
    {
        // 万一没找到，输出错误日志（红色）
        UE_LOG(LogTemp, Error, TEXT("找不到Cube模型！路径是否正确？"));
    }

    // ===== 设置初始旋转 =====
    // FRotator：UE中的旋转数据结构（单位：度）
    // 参数顺序：Pitch(俯仰), Yaw(偏航), Roll(翻滚)
    //        X轴旋转       Z轴旋转    Y轴旋转
    SetActorRotation(FRotator(0.0f, 45.0f, 0.0f));
    //                       ↑     ↑     ↑
    //                    不抬头  转45度  不侧翻
}

// 游戏开始时调用
void ARotatingCube::BeginPlay()
{
    // 先调用父类的BeginPlay（UE规则，必须调用！）
    Super::BeginPlay();

    // 输出当前旋转速度，方便调试
    UE_LOG(LogTemp, Warning, TEXT("RotatingCube BeginPlay! 旋转速度 = %f"), RotationSpeed);
    //                                                                    ↑
    //                                        %f = 浮点数占位符，会被后面的RotationSpeed替换
}

// 每一帧调用 — DeltaTime = 距离上一帧的时间（秒）
void ARotatingCube::Tick(float DeltaTime)
{
    // 必须先调用父类的Tick（就像你吃饭前先要拿碗筷一样，这是规矩）
    Super::Tick(DeltaTime);

    // 如果禁用了旋转，直接返回（不执行后面的旋转代码）
    if (!bEnableRotation)
    {
        return;  // return = 函数到此结束，后面代码不执行
    }

    // ===== 执行旋转 =====
    // AddActorLocalRotation：在自身坐标系中添加旋转
    // 参数是一个FRotator，表示在这一帧要增加多少旋转
    AddActorLocalRotation(
        FRotator(
            0.0f,                                    // Pitch：不绕X轴转
            RotationSpeed * DeltaTime,               // Yaw：绕Z轴转（水平旋转）
            0.0f                                     // Roll：不绕Y轴转
        )
    );
    // 逐行解释旋转计算：
    // RotationSpeed = 90.0（每秒转90度）
    // DeltaTime = 0.016（假设60FPS，每帧约0.016秒）
    // RotationSpeed * DeltaTime = 90 * 0.016 = 1.44度
    // → 这一帧旋转1.44度
    // → 60帧 * 1.44度 = 86.4度/秒 ≈ 90度/秒（微小误差由浮点数精度导致）
}
```

### 逐段详解

#### 构造函数（Constructor）

```cpp
ARotatingCube::ARotatingCube()  // :: 是"类名::成员"的作用域解析运算符
```

构造函数在你把Actor放入场景时自动调用。**构造函数中的代码在编辑器里也会执行**（编辑器中放置Actor时就构造了）。

```cpp
CubeMesh = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("CubeMesh"));
```

- `CreateDefaultSubobject<T>`：UE提供的工厂方法，在构造函数中创建组件。
- `<UStaticMeshComponent>`：尖括号里是模板参数，表示要创建的组件类型。
- `TEXT("CubeMesh")`：给组件起名。这个名字在编辑器中不可见，主要用于内部调试和序列化。
- **限制**：这个函数**只能在构造函数中调用**，不能在BeginPlay或Tick中调用。

```cpp
RootComponent = CubeMesh;
```

Actor必须有一个根组件。根组件决定Actor在场景中的Transform（位置、旋转、缩放）。所有其他组件都挂在根组件下面，形成层级关系。

```cpp
static ConstructorHelpers::FObjectFinder<UStaticMesh> CubeAsset(
    TEXT("/Engine/BasicShapes/Cube")
);
```

- `static`：静态局部变量，确保这段查找代码只在第一次执行时运行。如果构造100个RotatingCube，只有第一个会真正查找资产，后续99个直接复用结果。
- `ConstructorHelpers::FObjectFinder<T>`：构造函数专用的资产查找器。**只能在构造函数中使用。**
- `/Engine/BasicShapes/Cube`：UE的资产路径格式。`/Engine/` 表示引擎自带内容，`/Game/` 表示你自己项目的内容。

#### BeginPlay

```cpp
void ARotatingCube::BeginPlay()
{
    Super::BeginPlay();  // ← 必须调用！
    // ...
}
```

`Super` 是UE的约定，指向父类。`Super::BeginPlay()` 就是调用 `AActor::BeginPlay()`。如果忘记调用，Actor的基本初始化会被跳过，可能导致奇怪的行为。

#### Tick

```cpp
void ARotatingCube::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);
    // DeltaTime 单位是秒
    // 60FPS时 DeltaTime ≈ 0.0167
    // 120FPS时 DeltaTime ≈ 0.0083
    // 用 DeltaTime 做运算 = 帧率无关的平滑运动
}
```

## 第四步：编译并测试

### 4.1 编译

1. 在VS中按 **Ctrl+Shift+B** 编译（或使用Live Coding: **Ctrl+Alt+F11**）
2. 等待底部输出窗口显示"生成: 成功 1 个"
3. 如果有编译错误，仔细看报错信息。常见错误：
   - `"UStaticMeshComponent": 未定义的类` → 检查 `.cpp` 中是否 `#include "Components/StaticMeshComponent.h"`
   - `GENERATED_BODY 相关错误` → 检查 `.generated.h` 是否是最后一个include
   - `链接错误` → 检查 `.Build.cs` 中是否缺少模块依赖

### 4.2 放入场景

1. 在UE编辑器中打开一个关卡（Level）
2. 在 **Place Actors** 面板（左侧）或 **Content Drawer** 中搜索 `RotatingCube`
3. 把 `RotatingCube` 拖入视口中
4. 你应该看到一个45度角的白色Cube
5. 选中它，在右侧 **Details** 面板找到 **"Rotation Settings"** 分类
6. 你可以看到并修改 `Rotation Speed` 和 `Enable Rotation`

### 4.3 运行测试

1. 点击 **Play** 按钮
2. Cube应该开始旋转！
3. 试试在运行前把 `Rotation Speed` 改成 `360`，观察速度变化
4. 试试取消勾选 `Enable Rotation`，Cube应该停止旋转
5. 按 **Esc** 退出

// 在Tick中添加：
float Time = GetGameTimeSinceCreation();  // 获取游戏开始后经过的时间
float FloatOffset = FMath::Sin(Time * 2.0f) * 50.0f;  // 正弦波，振幅50单位
