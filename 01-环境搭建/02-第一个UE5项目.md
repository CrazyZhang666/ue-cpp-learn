# 1.2 第一个UE5 C++项目

> **目标**：从模板创建一个C++项目，理解项目文件结构，学会在VS中编译，在编辑器中运行。

---

## 创建项目

> 作为学习C++的起点，我们推荐直接创建**Blank（空白）C++项目**，然后手动添加自己的C++类。这样你能从零看到每一个文件是怎么来的，不会被模板里大量的代码淹没。

### 1. 选择模板

1. 启动Epic Games Launcher → 点击UE5版本的 **Launch** 按钮
2. 在项目浏览器中：

   | 设置项           | 选择                              |
   | ---------------- | --------------------------------- |
   | **类别**         | `Games`                           |
   | **模板**         | `Blank`（空白项目）               |
   | **项目类型**     | `C++`（不是Blueprint！）          |
   | **目标平台**     | `Desktop / Console`               |
   | **画质预设**     | `Maximum`                         |
   | **初学者内容包** | ✅ 勾选（包含基本素材，方便测试） |
   | **光线追踪**     | 可选，低配电脑建议关掉            |

3. **项目位置**：选择一个有足够空间的路径（建议SSD，至少预留10GB）
4. **项目名称**：输入 `MyFirstUE5Project`（不要用中文和特殊字符）
5. 点击 **"Create"** 按钮

### 2. 等待编译

点击Create后，UE5会做以下事情：

```
1. 根据Blank模板生成最少的C++源代码 ──→ .h和.cpp文件
2. 生成项目解决方案文件            ──→ .sln和.csproj
3. 调用Visual Studio编译器         ──→ 编译所有C++代码
4. 启动UE编辑器                   ──→ 你看到编辑器界面
```

> **首次编译可能需要5-15分钟**，之后修改少量代码的增量编译只需几秒到几十秒。

### 3. 编译成功标志

- UE编辑器自动打开，显示一个空白的默认场景（只有一片地板和默认天空）
- 编辑器底部 **Output Log** 没有红色报错
- 世界大纲视图（World Outliner）中只有最基本的默认对象（如Light Source、Sky Atmosphere等）

## 项目文件结构详解

在文件资源管理器中打开你的项目文件夹，你会看到：

```
MyFirstUE5Project/
│
├── MyFirstUE5Project.sln          ← VS解决方案文件，双击用VS打开项目
├── MyFirstUE5Project.uproject     ← UE项目文件，右键可启动UE
│
├── Config/                        ← 配置文件（引擎设置、输入设置等）
│   ├── DefaultEngine.ini          ← 引擎核心配置
│   ├── DefaultGame.ini            ← 游戏规则配置
│   └── DefaultInput.ini           ← 输入按键配置
│
├── Content/                       ← 游戏资产（模型、贴图、蓝图、材质等）
│   └── ...
│
├── Source/                        ← ★ 你的C++代码都在这！
│   └── MyFirstUE5Project/
│       ├── MyFirstUE5Project.Build.cs        ← 模块构建文件
│       ├── MyFirstUE5Project.h               ← 模块头文件
│       └── MyFirstUE5Project.cpp             ← 模块实现文件
│
├── DerivedDataCache/              ← 缓存文件，可以删除
├── Intermediate/                  ← 编译中间文件，可以删除
└── Saved/                         ← 日志、崩溃报告、自动保存
```

### 关键文件说明

| 文件         | 作用                       | 你需要改吗？     |
| ------------ | -------------------------- | ---------------- |
| `.uproject`  | UE项目入口文件             | 几乎不改         |
| `.sln`       | VS解决方案，双击打开写代码 | 不改             |
| `*.Build.cs` | 声明模块依赖哪些库         | 添加插件时需要改 |

## 你的第一个UE C++类

现在我们来**手动添加第一个自己的C++类**，这也是你以后在实际项目中最常用的操作。

### 在编辑器中创建C++类

1. 确保UE编辑器已打开你的项目
2. 点击顶部菜单 → **Tools → New C++ Class...**
3. 在弹出的对话框中，选择 `Actor` 作为父类（最基础的"场景中的对象"）
4. 点击 **Next**
5. 在 **Name** 一栏输入 `MyFirstActor`（不要用中文和特殊字符）
6. 在 **Path** 一栏保持默认（`Source/MyFirstUE5Project/`）
7. 点击 **Create Class**
8. UE会自动：
   - 生成 `MyFirstActor.h` 和 `MyFirstActor.cpp`
   - 调用编译器进行热编译
   - 完成后在VS中打开新文件

> 如果编译失败，请检查VS的C++工作负载是否按1.1节完整安装。

### 生成的代码

现在关闭UE编辑器（一定要关闭）

打开 `Source/MyFirstUE5Project/MyFirstActor.h`，你会看到UE为你生成的最基础的类框架：

```cpp
// MyFirstActor.h
// 这个文件定义了"MyFirstActor是什么、有什么属性、能做什么"

#pragma once  // ← 预处理指令：确保这个头文件只被包含一次，防止重复定义

#include "CoreMinimal.h"              // ← UE核心类型定义（FString, TArray等）
#include "GameFramework/Actor.h"       // ← AActor基类（所有可放入场景的对象的祖先）
#include "MyFirstActor.generated.h"    // ← UHT自动生成的反射代码，必须最后include

// ↓ UCLASS宏：告诉UE反射系统"这是一个可以被蓝图继承的类"
UCLASS()
class MYFIRSTUE5PROJECT_API AMyFirstActor : public AActor
//    ↑                    ↑                      ↑              ↑
//    |                    |                      |              |
//    |                    |                      类名            继承自AActor
//    |                    模块导出宏（让其他模块能使用这个类）
//    类的可见性
{
    // ↓ GENERATED_BODY宏：UHT会为这个类生成反射样板代码
    // 每个UCLASS都必须有，否则编译报错
    GENERATED_BODY()

public:
    // 构造函数：当这个Actor被创建时自动调用
    AMyFirstActor();

protected:
    // 游戏开始时调用一次（相当于Unity的Start）
    virtual void BeginPlay() override;

public:
    // 每帧调用（相当于Unity的Update）
    // DeltaTime = 上一帧到这一帧经过的秒数（如0.016 = 16毫秒）
    virtual void Tick(float DeltaTime) override;
};
```

对应的 `.cpp` 文件：

```cpp
// MyFirstActor.cpp

#include "MyFirstActor.h"  // ← 先包含自己的头文件

// 构造函数实现
AMyFirstActor::AMyFirstActor()
{
    // 开启Tick，如果不需要每帧更新可以设为false节省性能
    PrimaryActorTick.bCanEverTick = true;
}

// 游戏开始时调用
void AMyFirstActor::BeginPlay()
{
    // ← Super::调用父类版本，必须先执行
    Super::BeginPlay();
}

// 每帧调用
void AMyFirstActor::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);
}
```

### 让我们加一点东西：属性和函数

上面是UE生成的"空壳"。现在我们在头文件里加两个**属性**和一个**函数**，这是UE C++最核心的日常操作。

打开 `MyFirstActor.h`，在 `GENERATED_BODY()` 下面、构造函数上面添加：

```cpp
UCLASS()
class MYFIRSTUE5PROJECT_API AMyFirstActor : public AActor
{
    GENERATED_BODY()

public:
    AMyFirstActor();

    // ===== 自定义属性 =====

    /** 生命值 */
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "我的属性")
    //        ↑                ↑                    ↑
    //        |                |                    └─ 在编辑器的哪个分类下显示
    //        |                └─ 蓝图中可读可写
    //        └─ 在编辑器细节面板中可以直接修改数值
    int32 Health;

    /** 移动速度 */
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "我的属性")
    float Speed;

    // ===== 自定义函数 =====
    // UFUNCTION让蓝图系统能看到这些函数

    /** 打印问候语到屏幕和日志 */
    UFUNCTION(BlueprintCallable, Category = "我的函数")
    void SayHello();

protected:
    virtual void BeginPlay() override;

public:
    virtual void Tick(float DeltaTime) override;
};
```

然后在 `MyFirstActor.cpp` 中实现 `SayHello`，并在构造函数中给属性设默认值：

```cpp
#include "MyFirstActor.h"

AMyFirstActor::AMyFirstActor()
{
    PrimaryActorTick.bCanEverTick = true;

    // 给属性设置默认值
    Health = 100;         // ← int32类型的初始值
    Speed = 300.0f;       // ← float类型的初始值，UE中float常量要加f后缀
}

void AMyFirstActor::BeginPlay()
{
    Super::BeginPlay();

    // 游戏开始时在屏幕上打印一条消息（持续5秒，黄色）
    // GEngine 是全局引擎指针，只在开发版本中存在
    // 这里我们使用GEngine去判断是否是开发版本
    if (GEngine)
    {
        // 在屏幕上打印一条消息
        GEngine->AddOnScreenDebugMessage(
            -1,                // ← 消息ID，-1表示不覆盖之前消息
            5.0f,              // ← 显示持续时间（秒）
            FColor::Yellow,    // ← 文字颜色
            TEXT("Hello from MyFirstActor!")  // ← 消息内容，必须用TEXT()宏包裹
        );
    }
}

void AMyFirstActor::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);
}

void AMyFirstActor::SayHello()
{
    // UE_LOG：输出到编辑器的Output Log窗口
    // LogTemp = 日志分类，Log = 日志级别（还有Warning、Error）
    // %d 会被Health的值替换，%f 会被Speed的值替换
    UE_LOG(LogTemp, Log, TEXT("Hello! Health=%d, Speed=%f"), Health, Speed);
}
```

改完后按**关闭UE编辑器**，点击VS生成按钮进行编译。

### 逐段详解

#### ① Include 部分

```cpp
#pragma once
#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "MyFirstActor.generated.h"
```

- `#pragma once`：告诉编译器"这个文件的内容只处理一次"。如果你不小心include了同一个头文件两次，没有这行就会报"重复定义"错误。现代C++项目几乎所有头文件都有这行。
- `#include "CoreMinimal.h"`：引入UE的核心类型定义。`FString`（字符串）、`TArray`（动态数组）、`TMap`（字典）等都定义在这里。相当于学习标准C++时的 `#include <iostream>`，但强大得多。
- `#include "GameFramework/Actor.h"`：引入 `AActor` 类的定义。你的类继承自它，所以必须让编译器知道它长什么样。
- `.generated.h`：这是UE的**头文件反射工具（UHT）**自动生成的文件。**这个include必须放在所有include的最后一行**，否则编译报错。它的作用是让UE的反射系统（蓝图、序列化、垃圾回收等）能"看到"你定义的类、属性和函数。

#### ② UCLASS宏

```cpp
UCLASS()
class MYFIRSTUE5PROJECT_API AMyFirstActor : public AActor
```

- `UCLASS()`：括号内为空表示使用默认设置。常用的说明符有：
  - `Blueprintable`：允许在蓝图中创建这个类的子类
  - `BlueprintType`：允许这个类作为蓝图中的变量类型
- `MYFIRSTUE5PROJECT_API`：模块导出宏。UE项目中每个模块会生成一个这样的宏，它告诉链接器"这个类可以被其他模块使用"。
- 类名以 `A` 开头：UE命名规范规定，继承自 `AActor` 的类必须以 `A` 开头。

#### ③ GENERATED_BODY()

```cpp
GENERATED_BODY()
```

这行必须出现在类体最前面。它是UE反射系统的入口。UHT会为标了这个宏的类自动生成：

- 类型信息（类名、父类、属性列表）
- 序列化代码
- 网络复制支持代码
- 垃圾回收标记代码

如果你忘了写这行，编译时有上百条错误，但根本原因就是缺了它。

#### ④ UPROPERTY 宏

```cpp
UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "我的属性")
int32 Health;
```

`UPROPERTY(...)` 宏告诉UE反射系统管理这个成员变量。括号里的参数叫**属性说明符（Property Specifiers）**：

| 说明符               | 含义                                               |
| -------------------- | -------------------------------------------------- |
| `VisibleAnywhere`    | 在编辑器的细节面板中显示这个属性，但是灰色不可编辑 |
| `EditAnywhere`       | 在编辑器细节面板中可以修改这个属性的值             |
| `BlueprintReadOnly`  | 蓝图中可以读取这个值，但不能修改                   |
| `BlueprintReadWrite` | 蓝图中既可以读取也可以修改这个值                   |
| `Category = "XXX"`   | 在细节面板中归到"XXX"分类下                        |

> **为什么必须加UPROPERTY？** 如果不加且属性是指针类型（如 `UStaticMeshComponent*`），UE的垃圾回收系统不知道这个对象"正在被使用"，就可能把对象回收掉。然后你的指针就变成了**野指针**，访问它会导致崩溃。这是新手最容易犯的致命错误！

#### ⑤ UFUNCTION 宏

```cpp
UFUNCTION(BlueprintCallable, Category = "我的函数")
void SayHello();
```

`UFUNCTION()` 告诉UE反射系统这个函数可以被蓝图系统调用。常用的函数说明符：

| 说明符                        | 含义                                       |
| ----------------------------- | ------------------------------------------ |
| `BlueprintCallable`           | 蓝图中可以调用这个函数                     |
| `BlueprintPure`               | 纯函数（无副作用），在蓝图中有特殊显示样式 |
| `BlueprintImplementableEvent` | 在C++中声明，在蓝图中实现（事件）          |
| `BlueprintNativeEvent`        | C++提供默认实现，蓝图可以选择覆写          |
| `Category = "XXX"`            | 在蓝图右键菜单中归到"XXX"分类下            |

## 在VS中打开和编译项目

### 方法1：从UE编辑器打开（推荐）

1. UE编辑器菜单 → **Tools → Open Visual Studio**
2. VS会自动打开当前项目的.sln
3. 对于没有C++类的蓝图项目，也可以用 **Tools → New C++ Class** 来添加第一个C++类

### 方法2：从文件管理器打开

1. 在项目文件夹中，**右键点击 `.uproject` 文件**
2. 选择 **"Generate Visual Studio project files"**
3. 如果没看到这个选项，说明之前没关联好，回到1.1节检查VS安装
4. 生成完成后，双击 `.sln` 文件打开VS

### 在VS中编译

1. 在VS中，确保顶部的配置是：
   - **Solution Configuration**：`Development Editor`
   - **Solution Platform**：`Win64`
2. 按 **Ctrl+Shift+B** 或在菜单栏选择 **生成 → 生成解决方案**
3. 等待编译完成，底部输出窗口显示 **"生成: 成功 X 个"**

### 编译热重载（Live Coding）

UE5支持在编辑器运行时编译C++代码而不关闭编辑器：

1. 在UE编辑器中打开你的项目
2. 在VS中修改C++代码
3. 按 **Ctrl+Alt+F11** 触发 Live Coding 编译
4. 编译完成后，改动立即在编辑器中生效

> **注意**：Live Coding适用于修改函数实现，但如果你改了头文件（添加新属性、新函数签名等），还是需要关闭编辑器重新编译。
>
**强烈建议** 当需要编译任何代码时，请务必关闭编辑器再编译，不要使用Live Coding，避免出现奇奇怪怪的问题。

## 运行游戏

### 在编辑器中运行

点击UE编辑器顶部工具栏的 **"Play"** 按钮（绿色三角形）：

| 模式                        | 说明                           |
| --------------------------- | ------------------------------ |
| **Selected Viewport**       | 在当前编辑器视口中运行（常用） |
| **New Editor Window (PIE)** | 打开新窗口运行                 |
| **Standalone Game**         | 独立窗口运行，不依赖编辑器     |
| **Simulate**                | 模拟模式，只运行逻辑不接收输入 |

Blank空项目没有角色和控制器，点击Play后会进入游戏模式但看不到可控制的角色。我们会在后续章节学习如何添加可玩角色。

按 **Esc** 或再次点击 **Play** 按钮退出游戏模式。

### 独立运行（打包测试）

1. UE编辑器菜单 → **Platforms → Windows → Package Project**
2. 选择输出文件夹
3. 等待打包完成
4. 双击输出文件夹中的 `.exe` 文件运行
