# 1.2 第一个UE5 C++项目

> **目标**：从模板创建一个C++项目，理解项目文件结构，学会在VS中编译，在编辑器中运行。

---

## 创建项目

### 1. 选择模板

1. 启动Epic Games Launcher → 点击UE5版本的 **Launch** 按钮
2. 在项目浏览器中：

   | 设置项 | 选择 |
   |--------|------|
   | **类别** | `Games` |
   | **模板** | `Third Person`（第三人称） |
   | **项目类型** | `C++`（不是Blueprint！） |
   | **目标平台** | `Desktop` |
   | **画质预设** | `Maximum` |
   | **初学者内容包** | ✅ 勾选（包含基本素材） |
   | **光线追踪** | 可选，低配电脑建议关掉 |

3. **项目位置**：选择一个有足够空间的路径（建议SSD，至少预留10GB）
4. **项目名称**：输入 `MyFirstUE5Project`（不要用中文和特殊字符）
5. 点击 **"Create"** 按钮

### 2. 等待编译

点击Create后，UE5会做以下事情：

```
1. 根据模板生成C++源代码文件 ──→ .h和.cpp文件
2. 生成项目解决方案文件      ──→ .sln和.csproj
3. 调用Visual Studio编译器   ──→ 编译所有C++代码
4. 启动UE编辑器             ──→ 你看到编辑器界面
```

> **首次编译可能需要5-15分钟**，之后修改少量代码的增量编译只需几秒到几十秒。

### 3. 编译成功标志

- UE编辑器自动打开，能在视口中看到一个站立的人物模型
- 编辑器底部 **Output Log** 没有红色报错
- 点击编辑器顶部的 **Play** 按钮（绿色三角形），能控制角色跑动

## 项目文件结构详解

在文件资源管理器中打开你的项目文件夹，你会看到：

```
MyFirstUE5Project/
│
├── MyFirstUE5Project.sln          ← VS解决方案文件，双击用VS打开项目
├── MyFirstUE5Project.uproject     ← UE项目文件，右键可启动UE
│
├── Config/                        ← 配置文件（引擎设置、输入设置等）
│   ├── DefaultEngine.ini          ← 引擎核心配置
│   ├── DefaultGame.ini            ← 游戏规则配置
│   └── DefaultInput.ini           ← 输入按键配置
│
├── Content/                       ← 游戏资产（模型、贴图、蓝图、材质等）
│   ├── Characters/                ← 角色相关资产
│   ├── LevelPrototyping/          ← 关卡原型素材
│   └── ...
│
├── Source/                        ← ★ 你的C++代码都在这！
│   └── MyFirstUE5Project/
│       ├── MyFirstUE5Project.Build.cs    ← 模块构建文件
│       ├── MyFirstUE5Project.h           ← 模块头文件
│       ├── MyFirstUE5Project.cpp         ← 模块实现文件
│       ├── MyFirstUE5ProjectCharacter.h  ← 玩家角色头文件 ★
│       ├── MyFirstUE5ProjectCharacter.cpp← 玩家角色实现 ★
│       ├── MyFirstUE5ProjectGameMode.h   ← 游戏模式头文件
│       └── MyFirstUE5ProjectGameMode.cpp ← 游戏模式实现
│
├── DerivedDataCache/              ← 缓存文件，可以删除
├── Intermediate/                  ← 编译中间文件，可以删除
└── Saved/                         ← 日志、崩溃报告、自动保存
```

### 关键文件说明

| 文件 | 作用 | 你需要改吗？ |
|------|------|-------------|
| `.uproject` | UE项目入口文件 | 几乎不改 |
| `.sln` | VS解决方案，双击打开写代码 | 不改 |
| `*.Build.cs` | 声明模块依赖哪些库 | 添加插件时需要改 |
| `*Character.h/.cpp` | 玩家角色的逻辑 | 经常改 |
| `*GameMode.h/.cpp` | 游戏规则 | 经常改 |

## 你的第一段UE C++代码

打开 `Source/MyFirstUE5Project/MyFirstUE5ProjectCharacter.h`，你会看到：

```cpp
// MyFirstUE5ProjectCharacter.h
// 这个文件定义了"玩家角色是什么、有什么属性、能做什么动作"

#pragma once  // ← 预处理指令：确保这个头文件只被包含一次，防止重复定义

#include "CoreMinimal.h"            // ← UE核心类型定义（FString, TArray等）
#include "GameFramework/Character.h" // ← ACharacter基类（会走会跳会碰撞的人形Actor）
#include "MyFirstUE5ProjectCharacter.generated.h"  // ← UHT自动生成的反射代码，必须最后include

// ↓ UCLASS宏：告诉UE反射系统"这是一个可以被蓝图继承的类"
// Blueprintable = 可以在蓝图中继承这个类
UCLASS(config=Game, Blueprintable)
class MYFIRSTUE5PROJECT_API AMyFirstUE5ProjectCharacter : public ACharacter
//    ↑                    ↑                          ↑              ↑
//    |                    |                          |              |
//    |                    |                          类名            继承自ACharacter
//    |                    模块导出宏（让其他模块能使用这个类）
//    类的可见性
{
    // ↓ GENERATED_BODY宏：UHT会为这个类生成反射样板代码
    // 每个UCLASS都必须有，否则编译报错
    GENERATED_BODY()

public:
    // 构造函数：当这个角色被创建时自动调用
    AMyFirstUE5ProjectCharacter();

    // ===== 组件指针 =====
    // UPROPERTY的作用：告诉UE垃圾回收系统"这个指针指向的对象正在被使用，不要回收它"
    
    /** 弹簧臂组件：让摄像机可以平滑跟随角色 */
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = Camera)
    //        ↑                ↑                    ↑
    //        |                |                    └─ 在编辑器的哪个分类下显示
    //        |                └─ 蓝图中可读但不可写
    //        └─ 在编辑器中可见但不可编辑
    class USpringArmComponent* CameraBoom;

    /** 摄像机组件：玩家的眼睛 */
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = Camera)
    class UCameraComponent* FollowCamera;

    // ===== 移动相关属性 =====
    
    /** 基础转身速率（鼠标左右晃动时的旋转速度） */
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = Camera)
    //        ↑                ↑                    ↑
    //        |                |                    └─ 在哪个分类下
    //        |                └─ 蓝图可读可写
    //        └─ 在编辑器详情面板中可以直接修改数值
    float BaseTurnRate;

    /** 基础抬头速率（鼠标上下晃动时的旋转速度） */
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = Camera)
    float BaseLookUpRate;

protected:
    // ===== 输入处理函数 =====
    // UFUNCTION让蓝图系统能看到这些函数

    /** 前后移动（W/S键或手柄左摇杆上下） */
    UFUNCTION()
    void MoveForward(float Value);  // Value: 正=前进, 负=后退, 范围[-1, 1]

    /** 左右移动（A/D键或手柄左摇杆左右） */
    UFUNCTION()
    void MoveRight(float Value);    // Value: 正=右移, 负=左移, 范围[-1, 1]

    /** 鼠标左右看 */
    void TurnAtRate(float Rate);

    /** 鼠标上下看 */
    void LookUpAtRate(float Rate);

protected:
    // ↓ APawn::SetupPlayerInputComponent 的覆写版本
    // 这个函数在游戏开始时调用，用来绑定按键到具体的函数
    virtual void SetupPlayerInputComponent(class UInputComponent* PlayerInputComponent) override;
};
```

### 逐段详解

#### ① Include 部分

```cpp
#pragma once
#include "CoreMinimal.h"
#include "GameFramework/Character.h"
#include "MyFirstUE5ProjectCharacter.generated.h"
```

- `#pragma once`：告诉编译器"这个文件的内容只处理一次"。如果你不小心include了同一个头文件两次，没有这行就会报"重复定义"错误。现代C++项目几乎所有头文件都有这行。
- `#include "CoreMinimal.h"`：引入UE的核心类型定义。`FString`（字符串）、`TArray`（动态数组）、`TMap`（字典）等都定义在这里。相当于学习标准C++时的 `#include <iostream>`，但强大得多。
- `#include "GameFramework/Character.h"`：引入 `ACharacter` 类的定义。你的角色继承自它，所以必须让编译器知道它长什么样。
- `.generated.h`：这是UE的**头文件反射工具（UHT）**自动生成的文件。**这个include必须放在所有include的最后一行**，否则编译报错。它的作用是让UE的反射系统（蓝图、序列化、垃圾回收等）能"看到"你定义的类、属性和函数。

#### ② UCLASS宏

```cpp
UCLASS(config=Game, Blueprintable)
class MYFIRSTUE5PROJECT_API AMyFirstUE5ProjectCharacter : public ACharacter
```

- `UCLASS(config=Game, Blueprintable)`：
  - `config=Game`：表示这个类的配置值保存在 `Game.ini` 文件中
  - `Blueprintable`：允许在蓝图中创建这个类的子类
- `MYFIRSTUE5PROJECT_API`：模块导出宏。UE项目中每个模块会生成一个这样的宏，它告诉链接器"这个类可以被其他模块使用"。
- 类名以 `A` 开头：UE命名规范规定，继承自 `AActor` 的类必须以 `A` 开头。

#### ③ GENERATED_BODY()

```cpp
GENERATED_BODY()
```

这行必须出现在类体最前面。它是UE反射系统的入口。UHT会为标了这个宏的类自动生成：
- 类型信息（类名、父类、属性列表）
- 序列化代码
- 网络复制支持代码
- 垃圾回收标记代码

如果你忘了写这行，编译时有上百条错误，但根本原因就是缺了它。

#### ④ UPROPERTY 宏

```cpp
UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = Camera)
class USpringArmComponent* CameraBoom;
```

`UPROPERTY(...)` 宏告诉UE反射系统管理这个成员变量。括号里的参数叫**属性说明符（Property Specifiers）**：

| 说明符 | 含义 |
|--------|------|
| `VisibleAnywhere` | 在编辑器的细节面板中显示这个属性，但是灰色不可编辑 |
| `EditAnywhere` | 在编辑器细节面板中可以修改这个属性的值 |
| `BlueprintReadOnly` | 蓝图中可以读取这个值，但不能修改 |
| `BlueprintReadWrite` | 蓝图中既可以读取也可以修改这个值 |
| `Category = "XXX"` | 在细节面板中归到"XXX"分类下 |

> **为什么必须加UPROPERTY？** 如果不加，UE的垃圾回收系统不知道这个指针指向的对象"正在被使用"，就可能把对象回收掉。然后你的指针就变成了**野指针**，访问它会导致崩溃。这是新手最容易犯的致命错误！

#### ⑤ UFUNCTION 宏

```cpp
UFUNCTION()
void MoveForward(float Value);
```

`UFUNCTION()` 告诉UE反射系统这个函数可以被蓝图系统调用。括号里可以加各种说明符来控制函数行为（第3章会详细讲解）。

## 在VS中打开和编译项目

### 方法1：从UE编辑器打开（推荐）

1. UE编辑器菜单 → **Tools → Open Visual Studio**
2. VS会自动打开当前项目的.sln
3. 对于没有C++类的蓝图项目，也可以用 **Tools → New C++ Class** 来添加第一个C++类

### 方法2：从文件管理器打开

1. 在项目文件夹中，**右键点击 `.uproject` 文件**
2. 选择 **"Generate Visual Studio project files"**
3. 如果没看到这个选项，说明之前没关联好，回到1.1节检查VS安装
4. 生成完成后，双击 `.sln` 文件打开VS

### 在VS中编译

1. 在VS中，确保顶部的配置是：
   - **Solution Configuration**：`Development Editor`
   - **Solution Platform**：`Win64`
2. 按 **Ctrl+Shift+B** 或在菜单栏选择 **生成 → 生成解决方案**
3. 等待编译完成，底部输出窗口显示 **"生成: 成功 X 个"**

### 编译热重载（Live Coding）

UE5支持在编辑器运行时编译C++代码而不关闭编辑器：

1. 在UE编辑器中打开你的项目
2. 在VS中修改C++代码
3. 按 **Ctrl+Alt+F11** 触发 Live Coding 编译
4. 编译完成后，改动立即在编辑器中生效

> **注意**：Live Coding适用于修改函数实现，但如果你改了头文件（添加新属性、新函数签名等），还是需要关闭编辑器重新编译。

## 运行游戏

### 在编辑器中运行

点击UE编辑器顶部工具栏的 **"Play"** 按钮（绿色三角形）：

| 模式 | 说明 |
|------|------|
| **Selected Viewport** | 在当前编辑器视口中运行（常用） |
| **New Editor Window (PIE)** | 打开新窗口运行 |
| **Standalone Game** | 独立窗口运行，不依赖编辑器 |
| **Simulate** | 模拟模式，不控制角色，只运行物理 |

用 **WASD** 移动角色，**鼠标**旋转视角，**空格**跳跃。

按 **Esc** 或再次点击 **Play** 按钮退出游戏模式。

### 独立运行（打包测试）

1. UE编辑器菜单 → **Platforms → Windows → Package Project**
2. 选择输出文件夹
3. 等待打包完成
4. 双击输出文件夹中的 `.exe` 文件运行
