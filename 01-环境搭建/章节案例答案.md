## 练习1：改变旋转方向

### 目标

让Cube绕Y轴旋转（而不是Z轴）。

### 分析

`FRotator` 的三个参数依次是 `Pitch`（X轴）、`Yaw`（Y轴）、`Roll`（Z轴）。原代码中旋转放在 `Yaw` 参数位置，所以是绕Z轴旋转。要绕Y轴旋转，需要把旋转值放到 `Roll` 参数中。

### 正确修改

修改 `RotatingCube.cpp` 中的 `Tick` 函数：

```cpp
void ARotatingCube::Tick(float DeltaTime)
{
    // 必须先调用父类的Tick（UE规则，不可省略）
    Super::Tick(DeltaTime);

    // 如果禁用了旋转，直接返回
    if (!bEnableRotation)
    {
        return;
    }

    // ===== 执行旋转（绕Y轴） =====
    // FRotator(Pitch, Yaw, Roll)
    // 把 RotationSpeed * DeltaTime 从第二个参数（Yaw/Z轴）移到第三个参数（Roll/Y轴）
    AddActorLocalRotation(
        FRotator(
            0.0f,                           // Pitch：不绕X轴转
            0.0f,                           // Yaw：不绕Z轴转（原来是这里）
            RotationSpeed * DeltaTime       // Roll：现在绕Y轴转 ✅
        )
    );
}
```

### 编译测试

1. 按 **Ctrl+Shift+B** 编译
2. 拖入场景运行，Cube应该像车轮一样向前/向后翻滚

### 常见错误

```cpp
// ❌ 错误：直接改Yaw的值符号，只会改变Z轴旋转方向（顺时针/逆时针）
AddActorLocalRotation(FRotator(0.0f, -RotationSpeed * DeltaTime, 0.0f));

// ❌ 错误：把Roll放第一个参数位置
AddActorLocalRotation(FRotator(RotationSpeed * DeltaTime, 0.0f, 0.0f));
//                           ↑ 这是Pitch，会变成"抬头/低头"旋转
```

## 练习2：添加上下浮动

### 目标

让Cube在旋转的同时上下浮动，形成正弦波运动。

### 分析

上下浮动 = 改变Z轴位置。不能用简单的 `Z += speed`，否则会一直往上飞。正弦函数 `FMath::Sin(Time)` 的值在 `-1` 到 `1` 之间周期性变化，适合制作往复的浮动效果。

### 正确实现

**Step 1：在 `RotatingCube.h` 中添加新属性**

```cpp
// 在 class ARotatingCube 的 public 段中添加：

    // 浮动速度：正弦波的周期速度（值越大，上下浮动越快）
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Float Settings",
              meta = (ClampMin = "0.1", ClampMax = "10.0"))
    float FloatSpeed = 2.0f;  // ← 默认每秒完成2个正弦周期

    // 浮动振幅：上下浮动的最大高度（单位：厘米，UE默认单位）
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Float Settings",
              meta = (ClampMin = "0.0", ClampMax = "200.0"))
    float FloatAmplitude = 50.0f;  // ← 默认上下浮动50厘米

    // 记录初始位置（浮动以此位置为基准，避免累积偏移）
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "Float Settings")
    FVector InitialLocation;  // ← FVector = UE的三维向量（X, Y, Z）
```

**Step 2：在 `RotatingCube.cpp` 中保存初始位置并修改Tick**

```cpp
// 构造函数中不需要修改

void ARotatingCube::BeginPlay()
{
    Super::BeginPlay();

    // ===== 保存初始位置 =====
    // GetActorLocation()：获取Actor当前在世界中的位置（XYZ坐标）
    // 在BeginPlay中保存，此时Actor已经被放到场景中，位置是确定的
    InitialLocation = GetActorLocation();  // ✅ 保存基准点
    UE_LOG(LogTemp, Warning, TEXT("初始位置保存: %s"), *InitialLocation.ToString());
    //                                                   ↑
    //                        * 解引用FVector得到值，ToString()转成字符串用于日志
}

void ARotatingCube::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    if (!bEnableRotation)
    {
        return;
    }

    // ===== 原有旋转逻辑（保持不变） =====
    AddActorLocalRotation(
        FRotator(0.0f, RotationSpeed * DeltaTime, 0.0f)
    );

    // ===== 新增：上下浮动逻辑 =====
    // GetGameTimeSinceCreation()：获取这个游戏对象被创建后经过的秒数
    // 注意：这里用"对象创建时间"而不是"世界时间"，确保每个Cube独立计时
    float Time = GetGameTimeSinceCreation();

    // FMath::Sin()：UE封装的正弦函数（输入是弧度，周期约6.28秒）
    // Time * FloatSpeed：加快或减慢正弦波的周期
    // 结果范围：[-1, 1]
    float SineValue = FMath::Sin(Time * FloatSpeed);

    // 振幅缩放：把 [-1, 1] 映射到 [-FloatAmplitude, FloatAmplitude]
    // 例如：振幅50 → 范围变成 [-50, +50] 厘米
    float FloatOffset = SineValue * FloatAmplitude;

    // 构建新的位置向量：
    // X和Y保持初始位置不变，Z = 初始Z + 浮动偏移
    FVector NewLocation = InitialLocation;
    NewLocation.Z += FloatOffset;  // ✅ 只在Z轴上偏移

    // SetActorLocation()：直接设置Actor在世界空间中的位置
    // 参数1：目标位置（FVector）
    // 参数2：是否触发碰撞检测（false = 不检测，性能更好；true = 会被墙挡住）
    // 参数3：碰撞结果输出（我们不需要，传nullptr）
    SetActorLocation(NewLocation, false, nullptr);
    //                            ↑
    //              sweep = false 表示"瞬移"，不考虑中间障碍物
}
```

### 逐段详解

```cpp
float Time = GetGameTimeSinceCreation();
```

- `GetGameTimeSinceCreation()`：返回从Actor被创建到现在经过的秒数。
- **为什么不用 `GetWorld()->GetTimeSeconds()`？**
  - `GetWorld()->GetTimeSeconds()` 是世界全局时间，所有Actor共享同一个时间。
  - `GetGameTimeSinceCreation()` 是每个Actor独立的时间，拖入场景早的Cube和晚的Cube会有相位差，看起来更自然。

```cpp
float SineValue = FMath::Sin(Time * FloatSpeed);
```

- `FMath::Sin` 的输入是**弧度**。
- 正弦函数周期是 `2π ≈ 6.28` 秒。
- 乘以 `FloatSpeed` 可以压缩/拉伸周期：`FloatSpeed = 2` 表示周期变成 `6.28 / 2 ≈ 3.14` 秒。

```cpp
SetActorLocation(NewLocation, false, nullptr);
```

- `SetActorLocation` 是"绝对位置"设置，把Actor直接放到世界坐标系的某个点。
- `AddActorWorldOffset` 是"相对偏移"，在当前位置上累加。
- 这里必须用 `SetActorLocation` 而非 `AddActorWorldOffset`，因为我们要基于**固定的初始位置**计算，避免每帧累积误差。

### 常见错误

```cpp
// ❌ 错误：不用初始位置做基准，会导致偏移累积
AddActorWorldOffset(FVector(0.0f, 0.0f, FloatOffset * DeltaTime));
// 结果：Cube会越飞越高或越沉越低，不会往复浮动

// ❌ 错误：把DeltaTime再乘一次（Sin的结果已经是每帧独立的绝对偏移量）
float FloatOffset = FMath::Sin(Time * FloatSpeed) * FloatAmplitude * DeltaTime;
// 结果：浮动幅度会被帧率稀释，帧率越高浮动越微弱

// ❌ 错误：在构造函数里调用 GetActorLocation()
ARotatingCube::ARotatingCube()
{
    InitialLocation = GetActorLocation();  // ← 此时Actor还没放入场景！
    // 结果：位置可能是原点 (0,0,0)，运行时所有Cube都在原点浮动
}
```

### 编译测试

1. 保存并编译（**Ctrl+Shift+B**）
2. 拖入场景运行
3. 在 **Details** 面板找到 **"Float Settings"** 分类
4. 修改 `Float Speed` 和 `Float Amplitude` 观察变化：
   - `Float Speed = 0.5`：缓慢浮动（像水母）
   - `Float Speed = 5.0`：快速震动（像蜂鸟）
   - `Float Amplitude = 200`：浮动范围很大（像电梯）

## 练习3：颜色随速度变化

### 目标

让Cube的材质颜色随 `RotationSpeed` 变化：速度慢时偏蓝，速度快时偏红。

### 分析

UE中修改材质颜色需要用到 **动态材质实例**（`UMaterialInstanceDynamic`）。原理是：

1. 创建一个可以在运行时修改参数的材质实例
2. 把这个实例赋给静态网格体
3. 在Tick中根据速度改变颜色参数

### 正确实现

**Step 1：在 `RotatingCube.h` 中添加新属性**

```cpp
// 在 class ARotatingCube 的 public 段中添加：

    // 动态材质实例指针：运行时修改材质颜色的"遥控器"
    // BlueprintReadOnly = 蓝图可以读取这个引用
    UPROPERTY(BlueprintReadOnly, VisibleAnywhere, Category = "Color Settings")
    UMaterialInstanceDynamic* DynamicMaterial = nullptr;  // ← 初始化为空指针

    // 颜色参数名称：材质中定义的颜色参数名字（必须和材质里的参数名一致！）
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Color Settings")
    FName ColorParameterName = FName("BaseColor");  // ← 默认参数名"BaseColor"
```

**Step 2：在 `RotatingCube.cpp` 的构造函数中准备材质**

```cpp
ARotatingCube::ARotatingCube()
{
    PrimaryActorTick.bCanEverTick = true;

    // ===== 创建静态网格体组件（原有代码） =====
    CubeMesh = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("CubeMesh"));
    RootComponent = CubeMesh;

    // ===== 加载Cube模型（原有代码） =====
    static ConstructorHelpers::FObjectFinder<UStaticMesh> CubeAsset(
        TEXT("/Engine/BasicShapes/Cube")
    );
    if (CubeAsset.Succeeded())
    {
        CubeMesh->SetStaticMesh(CubeAsset.Object);

        // ===== 创建动态材质实例 =====
        // GetMaterial(0)：获取静态网格体上第0号插槽的材质
        // 返回值是UMaterialInterface*（材质接口基类指针）
        UMaterialInterface* BaseMaterial = CubeMesh->GetMaterial(0);
        //                                                    ↑
        //              0 表示第一个材质插槽（Cube默认只有一个材质插槽）

        if (BaseMaterial)
        {
            // CreateDynamicMaterialInstance()：基于现有材质创建一个可动态修改的副本
            // 参数0：材质插槽索引（和GetMaterial对应）
            // 参数1：基于哪个材质创建（传nullptr表示使用当前插槽的材质）
            // 返回值：UMaterialInstanceDynamic*，可以直接修改参数
            DynamicMaterial = CubeMesh->CreateDynamicMaterialInstance(0, BaseMaterial);
            //                                                    ↑
            //              ✅ 现在DynamicMaterial就是"可编辑的材质遥控器"

            if (DynamicMaterial)
            {
                UE_LOG(LogTemp, Warning, TEXT("动态材质创建成功！"));

                // 设置初始颜色为蓝色（速度最慢时的颜色）
                // FLinearColor：UE的线性颜色结构（R, G, B, A），范围0.0-1.0
                // 蓝色 = R:0, G:0, B:1
                DynamicMaterial->SetVectorParameterValue(ColorParameterName, FLinearColor(0.0f, 0.0f, 1.0f));
                //                              ↑
                //              SetVectorParameterValue = 设置"向量/颜色"类型参数
            }
        }
        else
        {
            UE_LOG(LogTemp, Error, TEXT("Cube没有材质！无法创建动态材质。"));
        }
    }
}
```

**Step 3：在 `RotatingCube.cpp` 的Tick中更新颜色**

```cpp
void ARotatingCube::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    if (!bEnableRotation)
    {
        return;
    }

    // ===== 原有旋转逻辑 =====
    AddActorLocalRotation(
        FRotator(0.0f, RotationSpeed * DeltaTime, 0.0f)
    );

    // ===== 根据旋转速度更新颜色 =====
    if (DynamicMaterial)
    {
        // 速度范围：0 ~ 720（我们在UPROPERTY中用ClampMin/ClampMax限定了）
        // 目标：速度0 = 蓝色，速度720 = 红色，中间 = 过渡色

        // 归一化速度：把 [0, 720] 映射到 [0, 1]
        // Clamp()：把值限制在最小值和最大值之间，防止超出范围
        float NormalizedSpeed = FMath::Clamp(RotationSpeed / 720.0f, 0.0f, 1.0f);
        //                                   ↑
        //              例如：RotationSpeed = 360 → 360/720 = 0.5

        // 线性插值颜色：从蓝色 (0,0,1) 渐变到红色 (1,0,0)
        // FMath::Lerp(A, B, Alpha)：线性插值，Alpha=0时返回A，Alpha=1时返回B
        FLinearColor BlueColor = FLinearColor(0.0f, 0.0f, 1.0f);   // 慢速：蓝色
        FLinearColor RedColor  = FLinearColor(1.0f, 0.0f, 0.0f);   // 快速：红色

        // FLinearColor 的 LerpTo 方法：在当前颜色和目标颜色之间插值
        FLinearColor NewColor = BlueColor + (RedColor - BlueColor) * NormalizedSpeed;
        //                         ↑                                    ↑
        //                      起始颜色                           颜色差值 * 进度
        // 也可以简写为：
        // FLinearColor NewColor = FMath::Lerp(BlueColor, RedColor, NormalizedSpeed);

        // 应用到材质
        DynamicMaterial->SetVectorParameterValue(ColorParameterName, NewColor);
        //                              ↑
        //              把计算出的颜色设置给材质参数"BaseColor"

        // 可选：每60帧打印一次日志（避免日志刷屏）
        // GFrameCounter：全局帧计数器
        if (GFrameCounter % 60 == 0)
        {
            UE_LOG(LogTemp, Warning, TEXT("速度=%.1f, 颜色=%s"),
                RotationSpeed,
                *NewColor.ToString()
            );
        }
    }
}
```

### 材质设置步骤（编辑器中操作）

动态材质需要一个**支持参数的材质**。如果你的Cube使用的是引擎默认材质，可能需要创建一个自定义材质：

1. 在 **Content Drawer** 中右键 → **Material** → 命名为 `M_CubeColor`
2. 双击打开材质编辑器
3. 在图表中：
   - 右键搜索 **"Vector Parameter"** → 命名为 `BaseColor`
   - 把这个节点的输出连接到 **Base Color** 引脚
   - 编译保存（左上角 **Apply**）
4. 回到Cube的蓝图/C++：
   - 在构造函数中加载这个材质：`TEXT("/Game/M_CubeColor")`
   - 或者直接在编辑器中把材质拖到Cube上

### 常见错误

```cpp
// ❌ 错误：试图直接修改原始材质（原始材质是共享的，修改会影响所有使用它的物体）
UMaterial* Material = Cast<UMaterial>(CubeMesh->GetMaterial(0));
Material->SetVectorParameterValue(...);  // ← 这是资产，运行时修改会导致副作用！

// ❌ 错误：参数名拼写错误（UE材质参数名区分大小写）
FName ColorParameterName = FName("basecolor");  // ← 应该是 "BaseColor"
// 结果：颜色不会变化，因为找不到这个参数

// ❌ 错误：没有判断DynamicMaterial是否为空就使用
DynamicMaterial->SetVectorParameterValue(...);  // ← 如果为nullptr会崩溃！

// ❌ 错误：在Tick中每次都创建新的动态材质实例
UMaterialInstanceDynamic* Mat = CubeMesh->CreateDynamicMaterialInstance(0);
// 结果：每秒创建60-120个材质实例，内存泄漏，性能极差
```

### 进阶：使用Hue渐变（更丰富的颜色）

如果想让颜色从蓝→绿→黄→红连续变化，可以用HSL/HSV色空间：

```cpp
// 把归一化速度映射到色相 [0.66(蓝), 0.0(红)]
float Hue = FMath::Lerp(0.66f, 0.0f, NormalizedSpeed);

// HSVToRGB：UE提供的HSV转RGB函数
// H: 0-1, S: 饱和度, V: 明度
FLinearColor NewColor = FLinearColor::FGetHSV(Hue, 1.0f, 1.0f);

DynamicMaterial->SetVectorParameterValue(ColorParameterName, NewColor);
```

### 编译测试

1. 保存并编译
2. 拖入场景，运行
3. 在 **Details** 面板修改 `Rotation Speed`：
   - 设为 `0` → 蓝色
   - 设为 `360` → 紫色（蓝+红混合）
   - 设为 `720` → 红色
4. 运行时修改速度，颜色应实时平滑过渡

## 三个练习整合版

如果想同时实现三个效果（旋转+浮动+变色），下面是整合后的完整代码。

### RotatingCube.h

```cpp
// RotatingCube.h
// 旋转 + 浮动 + 变色 整合版

#pragma once

#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "RotatingCube.generated.h"

UCLASS(Blueprintable)
class MYFIRSTUE5PROJECT_API ARotatingCube : public AActor
{
    GENERATED_BODY()

public:
    ARotatingCube();

protected:
    virtual void BeginPlay() override;

public:
    virtual void Tick(float DeltaTime) override;

    // ===== 旋转相关 =====
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly)
    class UStaticMeshComponent* CubeMesh;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Rotation Settings",
              meta = (ClampMin = "0.0", ClampMax = "720.0"))
    float RotationSpeed = 90.0f;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Rotation Settings")
    bool bEnableRotation = true;

    // ===== 浮动相关 =====
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Float Settings",
              meta = (ClampMin = "0.1", ClampMax = "10.0"))
    float FloatSpeed = 2.0f;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Float Settings",
              meta = (ClampMin = "0.0", ClampMax = "200.0"))
    float FloatAmplitude = 50.0f;

    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "Float Settings")
    FVector InitialLocation;

    // ===== 颜色相关 =====
    UPROPERTY(BlueprintReadOnly, VisibleAnywhere, Category = "Color Settings")
    UMaterialInstanceDynamic* DynamicMaterial = nullptr;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Color Settings")
    FName ColorParameterName = FName("BaseColor");
};
```

### RotatingCube.cpp

```cpp
// RotatingCube.cpp
// 旋转 + 浮动 + 变色 整合版

#include "RotatingCube.h"
#include "Components/StaticMeshComponent.h"

ARotatingCube::ARotatingCube()
{
    PrimaryActorTick.bCanEverTick = true;
    UE_LOG(LogTemp, Warning, TEXT("RotatingCube 被创建了！"));

    // 创建网格体组件
    CubeMesh = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("CubeMesh"));
    RootComponent = CubeMesh;

    // 加载引擎默认Cube模型
    static ConstructorHelpers::FObjectFinder<UStaticMesh> CubeAsset(
        TEXT("/Engine/BasicShapes/Cube")
    );
    if (CubeAsset.Succeeded())
    {
        CubeMesh->SetStaticMesh(CubeAsset.Object);
        UE_LOG(LogTemp, Warning, TEXT("Cube模型加载成功！"));

        // 创建动态材质实例
        UMaterialInterface* BaseMaterial = CubeMesh->GetMaterial(0);
        if (BaseMaterial)
        {
            DynamicMaterial = CubeMesh->CreateDynamicMaterialInstance(0, BaseMaterial);
            if (DynamicMaterial)
            {
                // 初始颜色：蓝色（低速）
                DynamicMaterial->SetVectorParameterValue(ColorParameterName, FLinearColor(0.0f, 0.0f, 1.0f));
            }
        }
    }
    else
    {
        UE_LOG(LogTemp, Error, TEXT("找不到Cube模型！"));
    }

    // 设置初始旋转45度
    SetActorRotation(FRotator(0.0f, 45.0f, 0.0f));
}

void ARotatingCube::BeginPlay()
{
    Super::BeginPlay();

    // 保存初始位置（浮动基准）
    InitialLocation = GetActorLocation();
    UE_LOG(LogTemp, Warning, TEXT("RotatingCube BeginPlay! 初始位置=%s, 旋转速度=%f"),
        *InitialLocation.ToString(), RotationSpeed);
}

void ARotatingCube::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    if (!bEnableRotation)
    {
        return;
    }

    // ===== 1. 执行旋转 =====
    AddActorLocalRotation(
        FRotator(0.0f, RotationSpeed * DeltaTime, 0.0f)
    );

    // ===== 2. 执行上下浮动 =====
    float Time = GetGameTimeSinceCreation();
    float FloatOffset = FMath::Sin(Time * FloatSpeed) * FloatAmplitude;
    FVector NewLocation = InitialLocation;
    NewLocation.Z += FloatOffset;
    SetActorLocation(NewLocation, false, nullptr);

    // ===== 3. 更新材质颜色 =====
    if (DynamicMaterial)
    {
        float NormalizedSpeed = FMath::Clamp(RotationSpeed / 720.0f, 0.0f, 1.0f);
        FLinearColor BlueColor = FLinearColor(0.0f, 0.0f, 1.0f);
        FLinearColor RedColor  = FLinearColor(1.0f, 0.0f, 0.0f);
        FLinearColor NewColor = FMath::Lerp(BlueColor, RedColor, NormalizedSpeed);
        DynamicMaterial->SetVectorParameterValue(ColorParameterName, NewColor);
    }
}
```

## 核心知识点回顾

| 概念                              | 说明                                     |
| --------------------------------- | ---------------------------------------- |
| `FRotator(Pitch, Yaw, Roll)`      | 旋转数据结构，三个轴分别对应             |
| `FMath::Sin()`                    | 正弦函数，范围 `[-1, 1]`，适合做往复运动 |
| `GetGameTimeSinceCreation()`      | Actor创建后的独立计时器                  |
| `SetActorLocation()`              | 直接设置世界空间位置（绝对）             |
| `AddActorWorldOffset()`           | 在世界空间中累加偏移（相对）             |
| `FMath::Lerp(A, B, Alpha)`        | 线性插值：`A + (B-A) * Alpha`            |
| `FMath::Clamp(Value, Min, Max)`   | 限制数值范围，防止越界                   |
| `UMaterialInstanceDynamic`        | 运行时动态修改材质参数的实例             |
| `CreateDynamicMaterialInstance()` | 创建动态材质实例的方法                   |
| `SetVectorParameterValue()`       | 修改材质中的向量/颜色参数                |
| `FLinearColor(R, G, B, A)`        | 线性颜色，范围 `0.0-1.0`                 |
| `FName`                           | UE的字符串标识符类型，用于材质参数名等   |
