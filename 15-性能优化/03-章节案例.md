## 1. 问题描述与初始状态

### 1.1 游戏场景

这是一款弹幕射击游戏（类似"吸血鬼幸存者"类型），玩家的角色在地图上移动，自动攻击周围的敌人。

- **敌人数量**：同屏同时存在 500-1000 个敌人
- **子弹数量**：各类武器每帧发射，同屏活跃子弹约 300-800 发
- **同屏对象总数**：最高可达 1500+
- **初始帧率**：15 FPS（目标 60 FPS，差距巨大）

### 1.2 优化前的代码（"反面教材"）

以下是最初的、没有经过任何优化的实现代码。仔细看代码中的注释理解每一行为什么会导致性能问题。

```cpp
// ================================================================
// 优化前代码 —— 子弹类（Bullet）
// ================================================================

// ---------- Bullet.h ----------
#pragma once
#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "Bullet.generated.h"

UCLASS()
class BULLETHELL_API ABullet : public AActor
{
    GENERATED_BODY()

public:
    ABullet()
    {
        // ❌ 每颗子弹都启用 Tick
        // 800 发子弹 = 800 次 Tick 调用/每帧
        PrimaryActorTick.bCanEverTick = true;
        //                    └没有关掉，每个子弹每帧都有 Tick 开销

        // 创建根组件（最简单的场景组件，没有网格）
        RootComponent = CreateDefaultSubobject<USceneComponent>(TEXT("Root"));
    }

protected:
    virtual void Tick(float DeltaTime) override;

public:
    // 子弹速度（每秒飞行的距离）
    float Speed = 1000.0f;

    // 子弹伤害值
    float Damage = 10.0f;

    // 子弹生命（飞行多少秒后自动销毁）
    float LifeTime = 2.0f;

    // 已过去的时间（用来计算是否超时）
    float ElapsedTime = 0.0f;
};

// ---------- Bullet.cpp ----------
#include "Bullet.h"

void ABullet::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    // 每帧累加已过去的时间
    ElapsedTime += DeltaTime;

    // ❌ 每帧都检查是否超时
    // 800 发子弹 × 60fps = 48000 次超时检查/秒
    // 但大多数时候结果都是"还没超时"
    if (ElapsedTime >= LifeTime)
    {
        Destroy();  // ❌ 每发子弹消失就是一次 Destroy
                    // Destroy 包含：注销组件、清理物理、触发GC检查
        return;
    }

    // ❌ 每帧计算向前移动
    // 这个运算本身很轻量，但被调用了 800 次/帧
    FVector NewLocation = GetActorLocation() + GetActorForwardVector() * Speed * DeltaTime;
    SetActorLocation(NewLocation);

    // ❌ 每帧对所有Actor做碰撞检测
    // 这是一个昂贵的操作：800发子弹 × 500个敌人 = 最坏情况 400,000 次碰撞检测
    TArray<AActor*> OverlappingActors;
    GetOverlappingActors(OverlappingActors, AEnemy::StaticClass());
    //                  └遍历全部Actor，找到所有与自己重叠的敌人
    //                   如果有800发子弹，每帧都要经历这个昂贵的遍历

    for (AActor* Other : OverlappingActors)
    {
        AEnemy* Enemy = Cast<AEnemy>(Other);  // Cast 有一定开销
        if (Enemy)
        {
            Enemy->TakeDamage(Damage);  // 造成伤害
            Destroy();  // ❌ 打中敌人后又是一次 Destroy
            break;      // 一颗子弹只打一个敌人
        }
    }
}


// ================================================================
// 优化前代码 —— 敌人类（Enemy）
// ================================================================

// ---------- Enemy.h ----------
#pragma once
#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "Enemy.generated.h"

UCLASS()
class BULLETHELL_API AEnemy : public AActor
{
    GENERATED_BODY()

public:
    AEnemy()
    {
        // ❌ 每个敌人每帧都 Tick
        // 500 个敌人 = 500 次 Tick 调用/每帧
        PrimaryActorTick.bCanEverTick = true;

        // 创建网格体组件（这才是真正有可视形状的）
        EnemyMesh = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("EnemyMesh"));
        RootComponent = EnemyMesh;  // 网格体作为根组件
    }

protected:
    virtual void Tick(float DeltaTime) override;

public:
    // 敌人的静态网格体
    UPROPERTY(VisibleAnywhere)
    UStaticMeshComponent* EnemyMesh;

    // 生命值
    float Health = 100.0f;

    // 移动速度
    float MoveSpeed = 200.0f;

    // 伤害函数：被子弹击中时调用
    void TakeDamage(float DamageAmount);

private:
    // 寻路用的目标点（向玩家移动）
    FVector TargetLocation;
};

// ---------- Enemy.cpp ----------
#include "Enemy.h"
#include "Kismet/GameplayStatics.h"

void AEnemy::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    // ❌ 每个敌人每帧都查找一次玩家
    // 500 个敌人 = 500 次"找玩家"操作/每帧
    // 但玩家位置一帧内不会变！499 次查找都是冗余的
    APawn* PlayerPawn = UGameplayStatics::GetPlayerPawn(GetWorld(), 0);
    if (PlayerPawn)
    {
        TargetLocation = PlayerPawn->GetActorLocation();
    }

    // 向玩家方向移动
    FVector Direction = TargetLocation - GetActorLocation();
    Direction.Normalize();
    FVector NewLocation = GetActorLocation() + Direction * MoveSpeed * DeltaTime;
    SetActorLocation(NewLocation);

    // ❌ 每个敌人每帧都做旋转计算
    FRotator NewRotation = Direction.Rotation();
    SetActorRotation(NewRotation);

    // ❌ 每个敌人每帧都检查生命值
    if (Health <= 0.0f)
    {
        // 死亡时掉落经验和粒子效果
        // SpawnEmitter 也是一次 Spawn 操作！
        // 如果 500 个敌人每秒死 100 个，这就是每秒 100 次 Spawn
        Destroy();  // ❌ 又是 Destroy，跟子弹一样的开销
    }
}

void AEnemy::TakeDamage(float DamageAmount)
{
    Health -= DamageAmount;

    // ❌ 每次受伤就 Play Sound
    // 800发子弹命中 = 每秒可能播放数百次音效
    // 大量音效同时播放会导致音频缓冲区溢出，CPU 飙升
    // （此处简化，没有写出 PlaySoundAtLocation 的代码）
}


// ================================================================
// 优化前代码 —— 武器发射系统
// ================================================================

// ---------- WeaponComponent.cpp ----------
void UWeaponComponent::Fire()
{
    // 获取枪口位置（子弹从哪里飞出来）
    FVector MuzzleLocation = GetOwner()->GetActorLocation();
    FRotator MuzzleRotation = GetOwner()->GetActorRotation();

    // ❌ 每次开火都 Spawn 新的子弹（每秒可能 Spawn 60+ 发）
    ABullet* NewBullet = GetWorld()->SpawnActor<ABullet>(
        ABullet::StaticClass(),   // 类
        MuzzleLocation,            // 位置
        MuzzleRotation             // 旋转
    );
    // SpawnActor 的完整开销包括：
    //   1. 堆内存分配（new/malloc）
    //   2. 调用构造函数（初始化组件）
    //   3. 注册组件（物理碰撞注册、渲染注册）
    //   4. 通知世界（添加到 Actor 列表）
    // 每秒 60 发 = 每秒 60 次完整 Spawn

    if (NewBullet)
    {
        NewBullet->Speed = BulletSpeed;    // 设置速度
        NewBullet->Damage = BulletDamage;  // 设置伤害
    }
}
```

**关键发现（从 trace 中提取的精确数字）：**

| 函数 | 调用次数/帧 | 单次耗时 | 总耗时/帧 | 占比 |
|------|-----------|---------|----------|------|
| ABullet::Tick | 800 | 35μs (0.035ms) | 28.00 ms | 42% |
| AEnemy::Tick | 500 | 44μs (0.044ms) | 22.00 ms | 33% |
| GetOverlappingActors | 800 | 27.5μs | 22.00 ms | 33% |
| SpawnActor + Destroy | ~120/帧总计 | 200μs | 24.00 ms | 36% |

> 💡 注意：占比加起来超过 100% 是因为有些开销是重叠的（GetOverlappingActors 是在 Bullet::Tick 内部调用的）。

### 2.2 优化策略制定

根据分析结果，我们制定了以下优化计划：

```
优先级1（效果最大）：对象池化子弹
  → 消除频繁的 Spawn/Destroy
  → 预计节省：~24ms → ~2ms

优先级2：禁用子弹的Tick，改用 UProjectileMovementComponent
  → 消除 800 次 Tick 调用 + 碰撞检测
  → 预计节省：~28ms → ~3ms

优先级3：敌人使用 ISM 替代独立 Actor
  → 减少 Draw Call + 减少 Actor 数量
  → 预计节省：~5ms + 50+ Draw Calls

优先级4：距离 Tier 系统
  → 远处敌人降低更新频率
  → 预计节省：~8ms（敌人Tick部分）

优先级5：Timer Manager 统一管理
  → 替代分散的 Tick 轮询
  → 预计节省：~2ms

优先级6：碰撞检测降频
  → 远处敌人每3帧检测一次
  → 预计节省：~3ms
```

## 3. 优化方案1：对象池化所有子弹

这是收益最大的优化。我们把"反复生成/销毁子弹"改成"预生成一堆子弹，用完放回池子，需要时从池子取出"。

### 3.1 对象池模板类

首先写一个通用的对象池模板类。它不依赖任何特定的 Actor 类型，可以用在任何需要池化的对象上。

```cpp
// ================================================================
// TPooledObject.h —— 通用对象池模板类
// ================================================================
#pragma once

#include "CoreMinimal.h"
#include "UObject/NoExportTypes.h"

// 宏：声明这个类的日志分类（用于UE_LOG输出池子的状态信息）
DECLARE_LOG_CATEGORY_EXTERN(LogObjectPool, Log, All);

/**
 * TPooledObject —— 通用对象池模板类
 *
 * 模板参数 T：
 *   必须继承自 UObject，并且需要实现以下接口：
 *   - InitializeFromPool()：从池子取出时的初始化
 *   - ReturnToPool()：    放回池子时的清理
 *
 * 使用方式：
 *   1. 在游戏初始化时，调用 InitializePool 预生成 N 个对象
 *   2. 需要对象时，调用 GetFromPool() 取出一个
 *   3. 用完后，调用 ReturnToPool(object) 放回池子
 *   4. 池空时，可选自动扩容或返回 nullptr
 *
 * 类比：这是一个"对象仓库"，预先把货架（内存）准备好，
 *       需要时直接取货，不需要等"工厂生产（Spawn）"或"退货处理（Destroy）"
 */
template<typename T>
class TPooledObject
{
    // 静态断言：确保 T 继承自 UObject
    // 编译期检查，如果传入了不合法的类型，编译器会报错
    static_assert(TIsDerivedFrom<T, UObject>::IsDerived,
        "TPooledObject 的模板参数 T 必须继承自 UObject");

public:
    TPooledObject()
        : MaxPoolSize(200)           // 默认最大池容量：200 个对象
        , bAutoExpand(true)          // 默认启用自动扩容（池空了自动生成新的）
        , TotalCreated(0)           // 已创建的对象总数（用于统计）
    {
    }

    /**
     * 初始化对象池 —— 预生成指定数量的对象
     *
     * @param WorldContext  世界上下文对象（用于 SpawnActor）
     * @param ObjectClass   要池化的对象类（例如 ABullet::StaticClass()）
     * @param InitialSize   预生成多少个对象（例如 300 发子弹）
     * @param InMaxPoolSize 池子最大容量（0 表示不限制）
     */
    void InitializePool(
        UObject* WorldContext,        // 世界上下文，任意 UObject 都可以
        TSubclassOf<T> ObjectClass,   // 要生成的类
        int32 InitialSize,            // 初始生成多少个
        int32 InMaxPoolSize = 0       // 最大容量，0表示无上限
    )
    {
        // 保存世界上下文和类的引用
        CachedWorldContext = WorldContext;
        CachedClass = ObjectClass;
        MaxPoolSize = InMaxPoolSize;

        // 获取世界指针（用于后续 SpawnActor）
        // UObject::GetWorld() 可以从任意 UObject 获取其所在的 World
        CachedWorld = WorldContext->GetWorld();

        if (!CachedWorld)
        {
            // 如果获取不到世界，说明 WorldContext 已经被销毁或无效
            UE_LOG(LogObjectPool, Error,
                TEXT("TPooledObject::InitializePool - 无法获取 World！"));
            return;
        }

        UE_LOG(LogObjectPool, Log,
            TEXT("开始初始化对象池 <%s>，预生成 %d 个对象..."),
            *ObjectClass->GetName(), InitialSize);

        // 循环预生成对象
        for (int32 i = 0; i < InitialSize; ++i)
        {
            // 创建新对象（跟 SpawnActor 一样，但只在初始化时做一次）
            T* NewObject = CreateNewObject();

            if (NewObject)
            {
                // 放入空闲池（Inactive 池）—— 这些对象是"库存"，等待被使用
                InactivePool.Add(NewObject);
                // Inactive = 未激活 = 在池子里闲着 = 等待被取用
            }
        }

        UE_LOG(LogObjectPool, Log,
            TEXT("对象池初始化完成：空闲 %d | 活跃 0 | 总计创建 %d"),
            InactivePool.Num(), ActiveObjects.Num(), TotalCreated);
    }

    /**
     * 从池子中取出一个对象
     *
     * 工作流程：
     *   1. 如果空闲池有对象 → 直接取出一个
     *   2. 如果空闲池为空 → 根据需要自动扩容（生成新的）
     *   3. 调用对象的 InitializeFromPool() 进行初始化
     *   4. 将对象从"空闲池"移到"活跃列表"
     *
     * @return 可用的对象指针，如果池空且无法扩容则返回 nullptr
     */
    T* GetFromPool()
    {
        T* PooledObject = nullptr;

        // 情况1：空闲池中有可用的对象
        if (InactivePool.Num() > 0)
        {
            // Pop() 从数组末尾取出并移除一个元素
            // 从末尾取出效率最高（O(1)），不需要移动其他元素
            PooledObject = InactivePool.Pop();
            //                          └移除最后一个并返回
        }
        // 情况2：空闲池为空，但允许自动扩容
        else if (bAutoExpand)
        {
            // 检查是否超过最大容量限制
            if (MaxPoolSize > 0 && TotalCreated >= MaxPoolSize)
            {
                // 池子已达上限，无法继续扩容
                UE_LOG(LogObjectPool, Warning,
                    TEXT("对象池已满！MaxSize=%d，当前活跃=%d，无法获取新对象"),
                    MaxPoolSize, ActiveObjects.Num());
                return nullptr;  // 返回空指针，调用者需要处理这种情况
            }

            // 创建新对象（自动扩容）
            UE_LOG(LogObjectPool, Log,
                TEXT("对象池自动扩容：创建第 %d 个对象"), TotalCreated + 1);
            PooledObject = CreateNewObject();
        }
        // 情况3：池空且不允许扩容
        else
        {
            UE_LOG(LogObjectPool, Warning,
                TEXT("对象池为空且不允许自动扩容，无法获取对象"));
            return nullptr;
        }

        // 如果能取出/创建对象，进行初始化
        if (PooledObject)
        {
            // 加入活跃列表（记录哪些对象正在被使用）
            ActiveObjects.Add(PooledObject);

            // 激活对象：对于 Actor，这通常意味着设置可见、启用碰撞等
            PooledObject->InitializeFromPool();
            //             └这个函数由子类实现，做具体的初始化工作
        }

        return PooledObject;
    }

    /**
     * 将对象放回池子
     *
     * @param Object 要归还的对象指针
     */
    void ReturnToPool(T* Object)
    {
        if (!Object)
        {
            return;  // 空指针检查
        }

        // 从活跃列表中移除这个对象
        int32 RemovedCount = ActiveObjects.Remove(Object);
        //                                 └Remove 返回移除的元素数量
        //                                  如果返回0，说明对象不在活跃列表中

        if (RemovedCount == 0)
        {
            // 对象不在活跃列表中，可能已经被归还过了
            UE_LOG(LogObjectPool, Warning,
                TEXT("ReturnToPool: 对象不在活跃列表中，可能已归还"));
            return;
        }

        // 调用对象的清理函数（由子类实现具体的清理逻辑）
        Object->ReturnToPool();
        //      └对于子弹：隐藏、禁用碰撞、停止移动

        // 将对象放入空闲池（等待下次被取出）
        InactivePool.Add(Object);

        // ❌ 注意：这里并没有 Destroy 对象！
        // 对象仍然存在，只是暂时"闲置"了
        // 下次 GetFromPool 时会重新激活它
    }

    /**
     * 获取当前活跃对象数量（正在被使用的对象数）
     */
    int32 GetActiveCount() const { return ActiveObjects.Num(); }

    /**
     * 获取空闲池中的对象数量（可用的库存数）
     */
    int32 GetInactiveCount() const { return InactivePool.Num(); }

    /**
     * 将池子中所有的活跃对象全部归还（清场）
     * 例如关卡切换时，把所有子弹和粒子都回收
     */
    void ReturnAllToPool()
    {
        // 遍历活跃列表的副本（因为 ReturnToPool 会修改数组）
        TArray<T*> ActiveCopy = ActiveObjects;

        for (T* Object : ActiveCopy)
        {
            ReturnToPool(Object);
        }

        UE_LOG(LogObjectPool, Log,
            TEXT("已归还所有活跃对象，当前空闲池：%d 个"), InactivePool.Num());
    }

    /**
     * 清空整个池子（销毁所有对象，释放内存）
     * 适用于关卡结束、游戏退出等场景
     */
    void ClearPool()
    {
        // 清除活跃对象
        for (T* Object : ActiveObjects)
        {
            if (Object)
            {
                Object->ConditionalBeginDestroy();
                //       └标记对象为"待销毁"，触发垃圾回收清理
            }
        }
        ActiveObjects.Empty();  // 清空活跃列表

        // 清除空闲池对象
        for (T* Object : InactivePool)
        {
            if (Object)
            {
                Object->ConditionalBeginDestroy();
            }
        }
        InactivePool.Empty();  // 清空空闲池

        TotalCreated = 0;  // 重置计数器

        UE_LOG(LogObjectPool, Log, TEXT("对象池已完全清空"));
    }

protected:
    /**
     * 创建一个新对象（内部方法）
     */
    T* CreateNewObject()
    {
        if (!CachedWorld || !CachedClass)
        {
            return nullptr;
        }

        // 使用 UE 的 NewObject 在运行时创建对象
        // 这跟 SpawnActor 不同：
        //   NewObject：只创建 UObject，不注册到世界
        //   SpawnActor：创建 Actor 并注册到世界（更重）
        // 对于对象池，我们用 NewObject 更轻量
        T* NewObj = NewObject<T>(
            (UObject*)GetTransientPackage(), // Outer：挂载到瞬态包（不被GC清理）
            CachedClass,                      // 要创建的类
            NAME_None,                        // 对象名（None = 自动生成）
            EObjectFlags::RF_Transient        // 标记为瞬态（不保存、不序列化）
            //             └RF_Transient：这个对象不需要被保存或序列化
            //              这对于池子中的对象是合适的，因为它们不需要持久化
        );

        if (NewObj)
        {
            TotalCreated++;  // 统计创建数量
        }

        return NewObj;
    }

    // ========== 成员变量 ==========

    // 空闲池：存放"闲置的"对象（等待被取用）
    // 这些对象处于"休眠"状态：隐藏、无碰撞、不Tick
    TArray<T*> InactivePool;

    // 活跃列表：存放"正在使用的"对象
    TArray<T*> ActiveObjects;

    // 缓存的世界指针（用于创建新对象）
    UWorld* CachedWorld = nullptr;

    // 缓存的世界上下文
    UObject* CachedWorldContext = nullptr;

    // 缓存的对象类
    TSubclassOf<T> CachedClass;

    // 最大池容量（0 = 无限制）
    int32 MaxPoolSize;

    // 是否自动扩容（池空时自动生成新的）
    bool bAutoExpand;

    // 已创建的对象总数（用于统计和调试）
    int32 TotalCreated;
};

// （在此省略 .cpp 中的 DEFINE_LOG_CATEGORY 定义）
```

```cpp
     * 设置子弹的飞行参数，将其激活

    void InitializeFromPool();

    /**
     * 放回对象池时的清理
     * 停止子弹飞行，隐藏，禁用碰撞

    void ReturnToPool();

// Bullet.cpp —— 改造后的子弹实现

#include "Bullet.h"
#include "Enemy.h"

void ABullet::BeginPlay()
{
    Super::BeginPlay();

    // ✅ 绑定碰撞事件（事件驱动，替代之前的 Tick 轮询）
    // 当子弹的球体碰撞组件与其他 Actor 重叠时，
    // 引擎自动调用 OnBulletOverlap —— 我们不需要每帧检查
    if (CollisionSphere)
    {
        CollisionSphere->OnComponentBeginOverlap.AddDynamic(
            this,                        // 绑定的对象（this = 这颗子弹）
            &ABullet::OnBulletOverlap   // 要调用的函数
        );
    }
}

void ABullet::InitializeFromPool()
{
    // ---------- 从池子取出时的初始化工作 ----------
    // 这个函数在 TPooledObject::GetFromPool() 中被调用

    // 步骤1：设置状态
    BulletState = EBulletState::Active;  // 标记为激活状态

    // 步骤2：使子弹可见
    SetActorHiddenInGame(false);
    //                    └false = 显示！让玩家能看到子弹
    //                     注意：false 表示"不隐藏"，即显示

    // 步骤3：启用碰撞（子弹可以打中敌人了）
    SetActorEnableCollision(true);

    // 步骤4：启用并配置飞行动作
    if (ProjectileMovement)
    {
        ProjectileMovement->SetActive(true);  // 激活移动组件
        // 设置子弹的速度和方向
        ProjectileMovement->Velocity = GetActorForwardVector() * ProjectileMovement->InitialSpeed;
        //                          └世界前方向 ───┘ └初始速度（1000）───┘
        // Velocity = 方向 × 速度
        // 例如：ForwardVector = (1,0,0)，速度1000 → Velocity = (1000, 0, 0)
    }

    // 步骤5：设置超时定时器
    // 飞行 MaxFlightTime 秒后，自动调用 OnBulletTimeout 回收子弹
    GetWorldTimerManager().SetTimer(
        TimeoutTimerHandle,            // 定时器句柄
        this,                           // 回调对象
        &ABullet::OnBulletTimeout,     // 超时回调
        MaxFlightTime,                  // 延迟时间（3秒）
        false                           // 不循环（只执行一次！）
        //      └false = 一次性定时器，到期后自动失效
    );
}

void ABullet::ReturnToPool()
{
    // ---------- 放回池子时的清理工作 ----------
    // 这个函数在 TPooledObject::ReturnToPool() 中被调用

    // 步骤1：设置状态
    BulletState = EBulletState::Inactive;  // 标记为未激活

    // 步骤2：隐藏子弹（池子里的子弹不应该被看到）
    SetActorHiddenInGame(true);
    //                    └true = 隐藏！

    // 步骤3：禁用碰撞（池子里的子弹不应该打中人）
    SetActorEnableCollision(false);

    // 步骤4：停止移动
    if (ProjectileMovement)
    {
        ProjectileMovement->SetActive(false);  // 停用移动组件
        ProjectileMovement->Velocity = FVector::ZeroVector;  // 速度归零
    }

    // 步骤5：清除超时定时器（避免已经回收的子弹触发超时回调）
    GetWorldTimerManager().ClearTimer(TimeoutTimerHandle);

    // 注意：位置不需要重置，下次 InitializeFromPool 时
    // 调用者会把子弹放到新的开火位置
}

void ABullet::OnBulletOverlap(
    UPrimitiveComponent* OverlappedComponent,
    AActor* OtherActor,
    UPrimitiveComponent* OtherComp,
    int32 OtherBodyIndex,
    bool bFromSweep,
    const FHitResult& SweepResult)
{
    // ---------- 事件驱动：只在真正碰撞时调用，无 Tick 开销 ----------

    // 安全检查1：已经失效的子弹不处理碰撞
    // 防止子弹在回收过程中触发的残余碰撞事件
    if (BulletState != EBulletState::Active)
    {
        return;  // 不活跃的子弹不处理碰撞
    }

    // 安全检查2：排除自身和空指针
    if (!OtherActor || OtherActor == this)
    {
        return;
    }

    // 安全检查3：只对敌人造成伤害
    if (!OtherActor->IsA<AEnemy>())
    {
        return;  // 撞到的不是敌人（可能是墙壁、其他子弹等）
    }

    // 转换为敌人类型
    AEnemy* Enemy = Cast<AEnemy>(OtherActor);
    if (Enemy)
    {
        // ✅ 对敌人造成伤害
        Enemy->TakeDamage(Damage);

        // ✅ 把子弹放回对象池（而不是 Destroy！）
        // 获取对象池的引用（通过武器管理器获取，见下文）
        // 这里假设外部系统会调用 ReturnToPool
        // 实际项目中通过委托或接口通知系统回收此子弹
        OnBulletTimeout();  // 简化：直接调用超时回收逻辑
    }
}

void ABullet::OnBulletTimeout()
{
    // ---------- 子弹超时：飞太久了，回收 ----------
    // 这个函数由 Timer 自动调用（不是由 Tick 调用！）

    // ✅ 通知对象池系统回收此子弹
    // 这里使用了"委托"（Delegate）模式：
    //   子弹不需要知道对象池的存在，只需要广播"我超时了"
    //   外部系统（武器管理器）监听这个委托，负责把子弹放回池子

    // 广播委托（如果有外部监听者，它们会收到通知）
    // OnBulletExpired.Broadcast(this);
    // （注：委托的声明在头文件中省略了，实际项目需要完整实现）

    // 简化实现：直接归还
    // 实际项目中，这里是：BulletPool->ReturnToPool(this);
    ReturnToPool();
}

void UWeaponComponent::Fire()
{
    // ---------- ✅ 对象池版本的"开火" ----------

    // 步骤1：获取枪口位置和方向
    FVector MuzzleLocation;   // 声明位置变量
    FRotator MuzzleRotation;  // 声明旋转变量
    GetMuzzleTransform(MuzzleLocation, MuzzleRotation);
    //                  └传引用！MuzzleLocation 和 MuzzleRotation 会被修改

    ABullet* Bullet = BulletPool.GetFromPool();
    //           └这个操作非常快：
    //             1. 从空闲池末尾取出一个子弹（O(1)）
    //             2. 调用 InitializeFromPool() 激活它
    //             3. 以上都是纯内存操作，不涉及内存分配！

    // 步骤3：检查是否成功获取子弹
    if (!Bullet)
    {

        UE_LOG(LogTemp, Warning,
            TEXT("子弹池已满！活跃子弹：%d，空闲：%d"),
            BulletPool.GetActiveCount(), BulletPool.GetInactiveCount());
        return;  // 放弃开火

    Bullet->SetActorLocation(MuzzleLocation);
    Bullet->SetActorRotation(MuzzleRotation);

    // 步骤5：设置子弹参数
    Bullet->Damage = BulletDamage;  // 伤害值
    // 速度已在 ProjectileMovement 组件中设置，这里不需要再设

    // ✅ 对比旧代码：
    // ❌ SpawnActor<ABullet>(...)  ← 每次开火都分配内存
    // ✅ BulletPool.GetFromPool()  ← 直接复用已分配的内存

    // 统计信息（可选，调试用）
    UE_LOG(LogTemp, Verbose,
        TEXT("开火！活跃子弹：%d | 空闲池：%d"),
        BulletPool.GetActiveCount(),
        BulletPool.GetInactiveCount());
}

void UWeaponComponent::GetMuzzleTransform(FVector& OutLocation, FRotator& OutRotation) const
{
    // 获取武器所属的 Actor（拥有这个组件的角色）
    AActor* Owner = GetOwner();
    if (!Owner)
    {
        // 如果没有所有者，返回零值
        OutLocation = FVector::ZeroVector;
        OutRotation = FRotator::ZeroRotator;
        return;
    }

    // 枪口位置 = 角色位置 + 枪口偏移（偏移量会随角色旋转）
    // TransformPosition：将本地坐标转换为世界坐标
    OutLocation = Owner->GetActorTransform().TransformPosition(MuzzleOffset);
    //                                                     └将枪口的本地偏移
    //                                                      转换为世界坐标

    // 枪口方向 = 角色的正前方
    OutRotation = Owner->GetActorRotation();
}
```

```cpp
// ========== ❌ 优化前 ==========
void ABullet::Tick(float DeltaTime)
{
    // 手动计算位置（800发子弹 × 60fps = 48000次/秒）
    FVector NewLocation = GetActorLocation() + GetActorForwardVector() * Speed * DeltaTime;
    SetActorLocation(NewLocation);

    // 手动碰撞检测（800发子弹 × 每帧遍历敌人）
    TArray<AActor*> OverlappingActors;
    GetOverlappingActors(OverlappingActors, AEnemy::StaticClass());
    for (AActor* Other : OverlappingActors)
    {
        // 手动处理伤害...
    }
}

// ========== ✅ 优化后 ==========
ABullet::ABullet()
{
    // Tick 完全关闭，引擎不会为这个 Actor 调用 Tick
    PrimaryActorTick.bCanEverTick = false;

    // 用 UProjectileMovementComponent 自动处理飞行
    // 这个组件在后台利用引擎的物理系统更新位置，不需要手动 Tick
    ProjectileMovement = CreateDefaultSubobject<UProjectileMovementComponent>(TEXT("PM"));
    ProjectileMovement->InitialSpeed = 1000.0f;
    ProjectileMovement->ProjectileGravityScale = 0.0f;  // 直线飞行，无重力
}

void ABullet::BeginPlay()
{
    // 用碰撞事件替代手动轮询
    CollisionSphere->OnComponentBeginOverlap.AddDynamic(this, &ABullet::OnBulletOverlap);
    // 碰撞发生时引擎自动调用 OnBulletOverlap
    // 不需要每帧手动遍历所有 Actor 做碰撞检测
}
```

### 4.2 为何 UProjectileMovementComponent 更快

- **批量处理**：引擎在 `UProjectileMovementComponent` 中批量处理所有弹丸的移动，而不是每个子弹都在自己的 Tick 里单独计算
- **SIMD 优化**：引擎内部使用 SIMD 指令并行计算多个弹丸的位置，比手动 `SetActorLocation` 快很多
- **无函数调用开销**：Tick 方式的 800 次函数调用的开销被完全消除
- **碰撞利用物理系统**：引擎的物理系统使用空间分区（Spatial Hashing）优化碰撞检测，比 `GetOverlappingActors` 的全局遍历快得多

## 5. 优化方案3：敌人使用 ISM 替代独立 Actor

在弹幕射击游戏中，场景里可能有 500 个敌人，每个都是独立的 Actor，各自有一个 StaticMeshComponent。这导致 500 次 Draw Call。

使用 Instanced Static Mesh（ISM），我们可以把所有同类型敌人合并到少数几个 ISM 组件中。

```cpp
// ================================================================
// EnemyManager.h —— 敌人生成管理器（ISM版本）
// ================================================================
#pragma once

#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "Components/InstancedStaticMeshComponent.h"
#include "EnemyManager.generated.h"

/**
 * FEnemyInstanceData —— 每个敌人实例的数据
 *
 * 因为 ISM 中的实例只是 Transform 数据，不包含逻辑
 * 所以我们需要一个结构体来存储每个敌人的"灵魂"（生命值、状态等）
 */
USTRUCT(BlueprintType)
struct FEnemyInstanceData
{
    GENERATED_BODY()

    // 敌人在 ISM 中的实例索引（-1 表示无效/已死亡）
    // 通过这个索引可以更新 ISM 中对应实例的 Transform
    int32 InstanceIndex = -1;
    //     └-1 默认值表示这个槽位是空的（没有敌人）

    // 生命值
    float Health = 100.0f;

    // 最大生命值
    float MaxHealth = 100.0f;

    // 移动速度
    float MoveSpeed = 200.0f;

    // 当前位置
    FVector Position = FVector::ZeroVector;

    // 当前旋转
    FRotator Rotation = FRotator::ZeroRotator;

    // 是否存活
    bool bIsAlive = false;

    // 敌人类型（0=近战, 1=远程, 2=BOSS）
    int32 EnemyType = 0;
};

UCLASS()
class BULLETHELL_API AEnemyManager : public AActor
{
    GENERATED_BODY()

public:
    AEnemyManager()
    {
        // ✅ 管理器不需要 Tick
        PrimaryActorTick.bCanEverTick = false;

        // 创建 ISM 组件作为根组件
        // 所有敌人共用这一个 ISM 组件
        EnemyISM = CreateDefaultSubobject<UInstancedStaticMeshComponent>(TEXT("EnemyISM"));
        RootComponent = EnemyISM;
    }

protected:
    virtual void BeginPlay() override;

public:
    // ========== 公共接口 ==========

    /**
     * 初始化敌人管理器
     * @param MaxEnemies 最大敌人数
     */
    void Initialize(int32 MaxEnemies);

    /**
     * 生成一个敌人
     * @param SpawnLocation 生成位置
     * @param EnemyType 敌人类型
     * @return 敌人的实例索引，-1 表示生成失败
     */
    int32 SpawnEnemy(const FVector& SpawnLocation, int32 EnemyType);

    /**
     * 消灭一个敌人
     * @param InstanceIndex 敌人的实例索引
     */
    void KillEnemy(int32 InstanceIndex);

    /**
     * 获取敌人数据
     * @param InstanceIndex 实例索引
     * @return 敌人数据的引用
     */
    FEnemyInstanceData& GetEnemyData(int32 InstanceIndex);

    /**
     * 更新 ISM 中实例的 Transform（位置和旋转变化后调用）
     * @param InstanceIndex 实例索引
     */
    void UpdateInstanceTransform(int32 InstanceIndex);

    /**
     * 更新所有敌人的位置和旋转（批量操作）
     * 应该在每帧或每隔几帧调用
     */
    void UpdateAllEnemyTransforms();

    // 获取所有存活的敌人数据（用于遍历和处理）
    const TArray<FEnemyInstanceData>& GetAllEnemies() const { return EnemyDataArray; }

    // 获取当前存活敌人数量
    int32 GetAliveCount() const { return AliveCount; }

private:
    // ========== 组件 ==========

    // ISM 组件：所有敌人共用一个 Draw Call
    UPROPERTY(VisibleAnywhere)
    UInstancedStaticMeshComponent* EnemyISM;

    // 敌人要使用的网格体
    UPROPERTY(EditAnywhere, Category = "敌人设置")
    class UStaticMesh* EnemyMesh;

    // ========== 数据 ==========

    // 敌人数据数组（每个元素记录一个敌人的"灵魂"）
    TArray<FEnemyInstanceData> EnemyDataArray;

    // 当前存活敌人数量
    int32 AliveCount = 0;

    // 最大敌人数量
    int32 MaxEnemyCount = 500;
};


// ================================================================
// EnemyManager.cpp —— 敌人生成管理器实现
// ================================================================
#include "EnemyManager.h"
#include "Engine/StaticMesh.h"

void AEnemyManager::BeginPlay()
{
    Super::BeginPlay();

    if (EnemyMesh)
    {
        EnemyISM->SetStaticMesh(EnemyMesh);
    }
}

void AEnemyManager::Initialize(int32 MaxEnemies)
{
    MaxEnemyCount = MaxEnemies;

    // ✅ 预分配敌人数据数组
    // 一次性分配 500 个槽位，避免后续 Add/Remove 导致的内存重新分配
    EnemyDataArray.SetNum(MaxEnemyCount);
    //            └SetNum：设置数组大小为 MaxEnemyCount
    //             所有元素使用默认构造函数初始化
    //             此时每个 FEnemyInstanceData 的 InstanceIndex 都是 -1

    // ✅ 预清除 ISM 中的实例（以防有残留数据）
    EnemyISM->ClearInstances();
    //       └移除所有已注册的 ISM 实例，但不释放内存

    AliveCount = 0;

    UE_LOG(LogTemp, Log, TEXT("敌人管理器已初始化：最大容量 %d"), MaxEnemies);
}

int32 AEnemyManager::SpawnEnemy(const FVector& SpawnLocation, int32 EnemyType)
{
    // ---------- 查找空闲槽位 ----------

    // 遍历所有敌人数，找第一个空闲的（InstanceIndex == -1）
    int32 FreeSlot = -1;  // -1 表示没找到

    for (int32 i = 0; i < EnemyDataArray.Num(); ++i)
    {
        if (EnemyDataArray[i].InstanceIndex == -1)
        {
            // 找到了空闲槽位！
            FreeSlot = i;
            break;  // 停止遍历
        }
    }

    // 如果没找到空闲槽位，说明敌人数量已达上限
    if (FreeSlot == -1)
    {
        UE_LOG(LogTemp, Warning, TEXT("无法生成敌人：已达到最大数量 %d"), MaxEnemyCount);
        return -1;
    }

    // ---------- 在 ISM 中添加一个实例 ----------

    // 构造变换（位置 + 旋转 + 缩放）
    FTransform InstanceTransform(
        FRotator::ZeroRotator,     // 初始旋转（朝向由逻辑设置）
        SpawnLocation,              // 生成位置
        FVector(1.0f)               // 缩放（1.0 = 原始大小）
    );

    // ✅ 在 ISM 中添加一个视觉实例
    // 这只添加一个 Transform 到 ISM 的实例列表
    // 不创建 Actor，不创建 Component，极其轻量
    int32 InstanceIndex = EnemyISM->AddInstance(InstanceTransform);
    //                   └返回新实例在 ISM 中的索引

    // ---------- 登记敌人数据 ----------

    // 获取对应槽位的敌人数据引用
    FEnemyInstanceData& Data = EnemyDataArray[FreeSlot];

    Data.InstanceIndex = InstanceIndex;  // 绑定 ISM 实例索引
    Data.Health = 100.0f;                // 初始生命值
    Data.MaxHealth = 100.0f;
    Data.MoveSpeed = 200.0f;             // 移动速度
    Data.Position = SpawnLocation;       // 当前位置
    Data.Rotation = FRotator::ZeroRotator;
    Data.bIsAlive = true;                // 标记为存活
    Data.EnemyType = EnemyType;          // 记录敌人类型

    AliveCount++;  // 存活计数 +1

    return FreeSlot;  // 返回槽位索引（调用者用这个索引来操作这个敌人）
}

void AEnemyManager::KillEnemy(int32 InstanceIndex)
{
    // 边界检查
    if (InstanceIndex < 0 || InstanceIndex >= EnemyDataArray.Num())
    {
        return;
    }

    FEnemyInstanceData& Data = EnemyDataArray[InstanceIndex];

    // 检查是不是已经死了
    if (!Data.bIsAlive)
    {
        return;  // 防止重复击杀
    }

    // ---------- 清理 ISM 中的视觉实例 ----------

    // 更新这个实例为"不可见状态"
    // 注意：ISM 不支持直接删除单个实例
    // 采用的方式是将实例缩放到 0 或移到屏幕外
    // 更好的方式是使用 UpdateInstanceTransform 隐藏它
    FTransform InvisibleTransform(
        FRotator::ZeroRotator,
        FVector(0, 0, -100000.0f),  // 移到地下很远的地方
        FVector(0.0f)                // 缩放到0（不可见）
    );
    EnemyISM->UpdateInstanceTransform(Data.InstanceIndex, InvisibleTransform, false, true);
    //                                        └世界空间变换───┘
    //                                              └不更新包围盒──┘
    //                                                   └标记为teleport（瞬间移动）┘

    // ---------- 清理敌人数据 ----------

    Data.bIsAlive = false;
    // Data.InstanceIndex 保持原值（标记槽位时不检查这个）
    // 但当需要复用槽位时，会重新分配一个新的 InstanceIndex

    AliveCount--;  // 存活计数 -1

    // TODO: 生成死亡粒子效果（可以从对象池获取粒子）
    // TODO: 掉落经验道具
}

FEnemyInstanceData& AEnemyManager::GetEnemyData(int32 InstanceIndex)
{
    // 返回敌人数据的引用
    // 调用者可以直接修改这个引用，从而修改敌人的属性
    return EnemyDataArray[InstanceIndex];
}

void AEnemyManager::UpdateInstanceTransform(int32 InstanceIndex)
{
    // 边界检查
    if (InstanceIndex < 0 || InstanceIndex >= EnemyDataArray.Num())
    {
        return;
    }

    FEnemyInstanceData& Data = EnemyDataArray[InstanceIndex];

    if (!Data.bIsAlive)
    {
        return;  // 不更新已死亡的敌人
    }

    // 构造新的 Transform
    FTransform NewTransform(Data.Rotation, Data.Position, FVector(1.0f));

    // ✅ 更新 ISM 中这个实例的位置
    // 这个操作很轻量：只修改 ISM 内部数组中的一行数据
    EnemyISM->UpdateInstanceTransform(Data.InstanceIndex, NewTransform);
    //                                └实例索引──────┘ └新的Transform┘
}

void AEnemyManager::UpdateAllEnemyTransforms()
{
    // ✅ 批量更新所有敌人位置
    // 使用 BatchUpdateInstancesTransform 比逐个 UpdateInstanceTransform 更快
    // 因为引擎只需要在最后发一次标记（dirty flag），而不是每次更新都发

    for (int32 i = 0; i < EnemyDataArray.Num(); ++i)
    {
        FEnemyInstanceData& Data = EnemyDataArray[i];

        if (Data.bIsAlive)
        {
            // 构造 Transform
            FTransform NewTransform(Data.Rotation, Data.Position, FVector(1.0f));

            // 更新 ISM 实例
            EnemyISM->UpdateInstanceTransform(Data.InstanceIndex, NewTransform, false, true);
            //                                                    └false = 不更新包围盒（批量更新时最后一起更新）
            //                                                           └true = 这是瞬间移动
        }
    }

    // 所有实例更新完成后，标记渲染数据需要刷新
    EnemyISM->MarkRenderStateDirty();
    //       └通知渲染系统："ISM 的数据变了，重新画一遍"
    //        这只需要一次通知，而不是每个实例都通知
}
```

### 5.1 ISM 版本的敌人 Tick 逻辑

```cpp
// ================================================================
// EnemyAISystem.cpp —— ISM 版本的敌人 AI 逻辑
// ================================================================

/**
 * 更新所有敌人的 AI（替代原来每个 AEnemy::Tick 的逻辑）
 *
 * 这个函数替换了原来的 500 次独立的 AEnemy::Tick
 * 改为在一个循环中集中处理所有敌人的逻辑
 *
 * 优势：
 *   1. 只需要一个 Timer，而不是 500 个 Actor 各自 Tick
 *   2. 循环内没有虚函数调用开销
 *   3. 可以一起做距离判断、分组处理
 */
void AEnemyManager::UpdateAllEnemiesAI()
{
    // 获取玩家位置（只查一次！）
    APawn* PlayerPawn = UGameplayStatics::GetPlayerPawn(GetWorld(), 0);
    FVector PlayerLocation = PlayerPawn ? PlayerPawn->GetActorLocation() : FVector::ZeroVector;

    // ✅ 一次循环处理所有敌人，而不是 500 次函数调用
    for (int32 i = 0; i < EnemyDataArray.Num(); ++i)
    {
        FEnemyInstanceData& Enemy = EnemyDataArray[i];

        // 跳过已死亡和空槽位
        if (!Enemy.bIsAlive)
        {
            continue;  // 死亡了，跳过
        }

        // ---------- 向玩家移动 ----------
        // 计算从敌人指向玩家的方向
        FVector Direction = PlayerLocation - Enemy.Position;
        //                  └目标位置──┘ └当前位置──┘

        // 检查距离（两个向量相减得到方向向量，取长度就是距离）
        float DistanceToPlayer = Direction.Size();

        // 如果已经在玩家旁边了（距离 < 50 单位），不需要移动
        if (DistanceToPlayer > 50.0f)
        {
            Direction.Normalize();  // 归一化为单位向量（长度为1，只保留方向）

            // 移动到新位置
            Enemy.Position += Direction * Enemy.MoveSpeed * 0.016f;
            //                                                    └DeltaTime ≈ 0.016（假设 60fps）
            // 注意：如果使用自定义 Tick 间隔，要用实际的 DeltaTime

            // 朝向玩家
            Enemy.Rotation = Direction.Rotation();
        }

        // ---------- 检查死亡 ----------
        if (Enemy.Health <= 0.0f)
        {
            KillEnemy(i);  // 消灭这个敌人
            continue;
        }
    }

    // ✅ 一次批量更新所有实例的 Transform
    // 不需要 500 次独立的 ISM 更新调用
    UpdateAllEnemyTransforms();
}
```

## 6. 优化方案4：距离 Tier 系统

不是所有敌人对游戏体验同等重要。远处的敌人玩家根本看不清楚，不需要和近处敌人一样的更新频率。

```cpp
// ================================================================
// 距离 Tier 系统（集成在 EnemyManager 中）
// ================================================================

// ---------- 在 EnemyManager.h 中添加 ----------

// 更新频率等级（Tier）枚举
UENUM(BlueprintType)
enum class EUpdateTier : uint8
{
    Tier0_High,      // 近处（< 2000单位）：每帧更新，完整AI
    Tier1_Medium,    // 中距离（2000-5000）：每3帧更新，简化的AI
    Tier2_Low,       // 远处（5000-10000）：每10帧更新，仅移动
    Tier3_Culled     // 超远处（> 10000）：完全不更新，等待玩家靠近
};

// ---------- 在 EnemyManager.cpp 中添加 ----------

/**
 * 根据距离计算更新等级
 *
 * @param DistanceToPlayer 敌人到玩家的距离（单位：厘米/UE单位）
 * @return 对应的更新等级
 */
EUpdateTier AEnemyManager::CalculateUpdateTier(float DistanceToPlayer) const
{
    // 根据渐变距离分配到不同的更新等级
    if (DistanceToPlayer < 2000.0f)
    {
        // 近距离：2000单位以内（约20米）
        // 每帧都更新，使用完整的AI逻辑（寻路、动画、攻击）
        return EUpdateTier::Tier0_High;
    }
    else if (DistanceToPlayer < 5000.0f)
    {
        // 中距离：2000-5000单位
        // 每3帧更新一次，只做移动和基础检测
        return EUpdateTier::Tier1_Medium;
    }
    else if (DistanceToPlayer < 10000.0f)
    {
        // 远处：5000-10000单位
        // 每10帧更新一次，只做向玩家方向的简单移动
        return EUpdateTier::Tier2_Low;
    }
    else
    {
        // 超远处：10000单位以外
        // 完全不更新！玩家看不到，更新也没有意义
        // 当玩家走近时再恢复更新
        return EUpdateTier::Tier3_Culled;
    }
}

/**
 * Tier 化更新版本的 AI 逻辑
 *
 * 替代之前的 UpdateAllEnemiesAI()
 * 不同Tier的敌人使用不同的更新频率和复杂度
 *
 * @param FrameNumber 当前帧号（用于判断是否到了更新时间）
 */
void AEnemyManager::UpdateAllEnemiesAI_Tiered(int32 FrameNumber)
{
    // ✅ 只获取一次玩家位置（之前每个敌人获取一次，浪费！）
    APawn* PlayerPawn = UGameplayStatics::GetPlayerPawn(GetWorld(), 0);
    if (!PlayerPawn)
    {
        return;  // 没有玩家，所有敌人都不需要处理
    }
    FVector PlayerLocation = PlayerPawn->GetActorLocation();

    // ---------- 遍历所有敌人 ----------
    for (int32 i = 0; i < EnemyDataArray.Num(); ++i)
    {
        FEnemyInstanceData& Enemy = EnemyDataArray[i];

        // 跳过死亡的和空槽位
        if (!Enemy.bIsAlive)
        {
            continue;
        }

        // 步骤1：计算距离和更新等级
        float DistSq = FVector::DistSquared(Enemy.Position, PlayerLocation);
        //             └用DistSquared而不是Dist，避免开平方根运算
        //              DistSquared 只做乘法和加法，比 Dist 快很多
        float Distance = FMath::Sqrt(DistSq);  // 只在需要实际距离时才开平方
        EUpdateTier Tier = CalculateUpdateTier(Distance);

        // 步骤2：根据 Tier 决定是否更新以及更新的复杂度

        switch (Tier)
        {
            case EUpdateTier::Tier0_High:
            {
                // 近距离：完整 AI 逻辑
                UpdateEnemyFull(i, PlayerLocation, Distance);
                break;
            }

            case EUpdateTier::Tier1_Medium:
            {
                // 中距离：每3帧更新一次
                // 用 FrameNumber % 3 来判断是否到了第 3 的倍数帧
                if (FrameNumber % 3 == 0)
                {
                    UpdateEnemyMedium(i, PlayerLocation, Distance);
                }
                // 不是第3的倍数帧时，跳过这个敌人
                break;
            }

            case EUpdateTier::Tier2_Low:
            {
                // 远处：每10帧更新一次，只做简单移动
                if (FrameNumber % 10 == 0)
                {
                    UpdateEnemyLow(i, PlayerLocation, Distance);
                }
                break;
            }

            case EUpdateTier::Tier3_Culled:
            {
                // 超远处：完全不更新
                // 敌人"定格"在最后的位置，等玩家靠近后再恢复
                // 也可以在这里把 ISM 实例隐藏起来
                break;
            }
        }
    }

    // 批量更新所有被修改的 ISM 实例位置
    UpdateAllEnemyTransforms();
}

    {
        Direction.Normalize();
        Enemy.Position += Direction * Enemy.MoveSpeed * 0.016f;  // DeltaTime
        Enemy.Rotation = Direction.Rotation();

/**
```

## 8. 优化方案6：碰撞检测降频

碰撞检测是游戏中开销最大的操作之一。在弹幕射击游戏中，碰撞检测尤其重要（子弹打中敌人），但也尤其可以被优化。

```cpp
// ================================================================
// 碰撞检测降频方案
// ================================================================

void AGamePerformanceManager::UpdateCollisionChecks()
{
    // ---------- 碰撞检测的优化策略 ----------

    // 策略1：近处敌人每帧检测（通过 ProjectileMovement 的自动碰撞）
    //   已在方案2中实现 —— 子弹使用 OnComponentBeginOverlap 事件

    // 策略2：远处敌人每 3 帧检测一次
    //   在方案4的 Tier 系统中已体现

    // 策略3：使用空间网格减少碰撞对的数量
    //   而不是每发子弹 vs 每个敌人（N×M 复杂度）

    // ---------- 空间网格碰撞优化 ----------

    // 基本思路：
    // 将世界划分为网格，每个格子记录包含的敌人
    // 子弹只需要检测所在格子及其相邻格子中的敌人
    // 而不是检测全部 500 个敌人

    // 假设世界大小为 20000×20000，网格大小 1000×1000
    // 那么只有 20×20 = 400 个网格
    // 一颗子弹最多只需要检测 9 个相邻网格中的敌人

    UpdateSpatialGrid();       // 更新空间网格
    PerformGridBasedCollision(); // 基于网格的碰撞检测
}

// ---------- 空间网格数据结构 ----------

// 空间网格 —— 将世界划分为格子，快速查找某个区域内的敌人
struct FSpatialGrid
{
    // 网格大小（每个格子的边长，单位：UE单位/cm）
    float CellSize = 1000.0f;

    // 网格数据：键 = 格子的 (X, Y) 坐标, 值 = 该格子中的敌人索引列表
    TMap<FIntPoint, TArray<int32>> Grid;
    //     └FIntPoint：一个简单的 (X, Y) 整数坐标对
    //      例如 (3, 5) 表示第3列第5行的格子

    /**
     * 清空网格（每帧开始时调用）
     */
    void Clear()
    {
        // 注意：这里用 Empty() 而不是 Reset()
        // Empty() 清空元素但保留内存（避免反复分配）
        // Reset() 清空元素并释放内存
        for (auto& Pair : Grid)
        {
            Pair.Value.Empty();  // 清空每个格子里的敌人列表
            // Pair.Value 是 TArray<int32>&，即该格子的敌人索引列表
        }
        // Grid.Empty(); // 可选：如果敌人分布变化很大才清空整个Grid
    }

    /**
     * 将世界坐标转换为网格坐标
     *
     * @param WorldLocation 世界坐标
     * @return 对应的网格坐标 (列, 行)
     */
    FIntPoint WorldToGrid(const FVector& WorldLocation) const
    {
        // 例如：WorldLocation = (3500, 2200, 0), CellSize = 1000
        // GridX = 3500 / 1000 = 3.5 → floor(3.5) = 3
        // GridY = 2200 / 1000 = 2.2 → floor(2.2) = 2
        // 结果：(3, 2) —— 第3列第2行的格子
        return FIntPoint(
            FMath::FloorToInt(WorldLocation.X / CellSize),
            //                └向下取整为整数
            FMath::FloorToInt(WorldLocation.Y / CellSize)
        );
    }

    /**
     * 添加一个敌人到对应的网格中
     *
     * @param EnemyIndex 敌人在 EnemyDataArray 中的索引
     * @param Position   敌人的世界位置
     */
    void AddEnemy(int32 EnemyIndex, const FVector& Position)
    {
        // 计算这个位置属于哪个格子
        FIntPoint CellCoord = WorldToGrid(Position);

        // 查找这个格子的敌人列表（如果不存在则自动创建）
        TArray<int32>& CellEnemies = Grid.FindOrAdd(CellCoord);
        //                              └如果 CellCoord 不存在，自动创建空数组

        // 把敌人索引添加进这个格子的列表
        CellEnemies.Add(EnemyIndex);
    }

    /**
     * 获取子弹所在格子及其相邻格子的所有敌人
     *
     * @param BulletPosition 子弹的世界位置
     * @param OutEnemyIndices 输出：所有相关敌人索引的数组
     */
    void GetNearbyEnemies(const FVector& BulletPosition, TArray<int32>& OutEnemyIndices)
    {
        OutEnemyIndices.Empty();  // 清空输出数组（保留内存）

        // 计算子弹所在的格子
        FIntPoint CenterCell = WorldToGrid(BulletPosition);

        // ✅ 检查 3×3 = 9 个相邻格子（包括自己所在的格子）
        // 子弹可能位于格子边界附近，所以需要检查相邻格子
        for (int32 dx = -1; dx <= 1; ++dx)  // X方向偏移：-1, 0, +1
        {
            for (int32 dy = -1; dy <= 1; ++dy)  // Y方向偏移：-1, 0, +1
            {
                // 计算相邻格子的坐标
                FIntPoint NeighborCell(CenterCell.X + dx, CenterCell.Y + dy);

                // 查找这个格子里有哪些敌人
                TArray<int32>* CellEnemies = Grid.Find(NeighborCell);
                //                                 └Find返回指针：找到了返回数组地址，没找到返回nullptr

                if (CellEnemies)
                {
                    // 把这个格子里的所有敌人添加到输出数组
                    OutEnemyIndices.Append(*CellEnemies);
                    //                 └解引用指针，得到数组引用
                    //                  Append 将数组内容追加到输出数组末尾
                }
            }
        }
    }
};

// ---------- 基于空间网格的碰撞检测实现 ----------
void AGamePerformanceManager::PerformGridBasedCollision()
{
    // 步骤1：清空上一帧的空间网格数据
    SpatialGrid.Clear();

    // 步骤2：将所有存活的敌人注册到空间网格中
    const TArray<FEnemyInstanceData>& Enemies = EnemyManager->GetAllEnemies();
    for (int32 i = 0; i < Enemies.Num(); ++i)
    {
        if (Enemies[i].bIsAlive)
        {
            SpatialGrid.AddEnemy(i, Enemies[i].Position);
        }
    }

    // 步骤3：对每颗活跃的子弹，在空间网格中查找附近的敌人
    for (ABullet* Bullet : BulletPool->GetActiveBullets())
    {
        // 获取子弹附近格子的所有敌人
        TArray<int32> NearbyEnemyIndices;
        SpatialGrid.GetNearbyEnemies(Bullet->GetActorLocation(), NearbyEnemyIndices);

        // 步骤4：只对这些"附近"的敌人做精确碰撞检测
        for (int32 EnemyIndex : NearbyEnemyIndices)
        {
            const FEnemyInstanceData& Enemy = Enemies[EnemyIndex];

            // 精确距离检测（子弹碰撞球 vs 敌人碰撞球）
            float DistSq = FVector::DistSquared(Bullet->GetActorLocation(), Enemy.Position);

            // 子弹碰撞半径 + 敌人碰撞半径（简化值）
            constexpr float CollisionRadiusSum = 16.0f + 30.0f;  // 46 单位
            constexpr float CollisionRadiusSumSq = CollisionRadiusSum * CollisionRadiusSum;

            if (DistSq <= CollisionRadiusSumSq)
            {
                // 碰撞发生！
                HandleBulletEnemyCollision(Bullet, EnemyIndex);
                break;  // 一颗子弹只打一个敌人
            }
        }
    }
}
```

```
│                                                      │
│  ┌─────────────────────────────────────────────┐    │

```

### 9.4 核心经验总结

这六个优化方案背后是几条通用的性能优化原理：

1. **少做**：能不做就不做（禁用 Tick、Tier 省略远处更新）
2. **慢做**：能少做就少做（降频 Tick、Timer 替代每帧轮询）
3. **批量做**：能一起做就别分开做（ISM 合批、批量 UpdateTransform）
4. **复用**：能重用就别新建（对象池替代 Spawn/Destroy）
5. **提前做**：能提前算好的就别运行时算（Reserve 预分配、缓存计算结果）
6. **聪明地做**：能用高效算法就别暴力遍历（空间网格碰撞检测）

记住：**优化的最高境界是"不做"—— 把不需要的计算剔除掉，远比你优化计算本身更有效。**
