## 第一部分：Blackboard 定义

在动手写C++代码之前，先明确我们的AI需要"记住"什么：

```
BB_Monster 黑板中定义的Key：

┌────────────────────────────────────────────────────┐
│  Key名称          │ 类型    │ 谁写入        │ 谁读取      │
├───────────────────┼─────────┼──────────────┼────────────┤
│ TargetActor       │ Object  │ 感知系统      │ Service/    │
│                   │         │ (看到玩家时)  │ Decorator   │
├───────────────────┼─────────┼──────────────┼────────────┤

│ PatrolIndex       │ Int     │ Service:      │ Task:       │
│                   │         │ UpdatePatrol  │ Patrol      │
├───────────────────┼─────────┼──────────────┼────────────┤
│ PatrolLocations   │ Vector  │ AIController  │ Service:    │

│                   │         │ BeginPlay     │ ReturnHome  │

```

## 第二部分：行为树结构设计

在写代码之前，先画出行为树的完整结构：

```
Root (Selector: 优先级 = 战斗 > 巡逻 > 回家)
│
├── Sequence [优先级1: 战斗]              ← 看到玩家时执行
│   ├── Decorator: 目标是否存在?           ← Blackboard: TargetActor != null
│   │                                      （不满足→跳过整个战斗分支）
│   ├── [Service: 更新目标位置]            ← 每0.3秒更新 TargetLocation
│   │
│   ├── Selector [追击或攻击]
│   │   ├── Sequence [攻击]
│   │   │   ├── Decorator: 在攻击范围内?    ← 距离 < 攻击距离
│   │   │   ├── Task: 攻击目标              ← 播放动画+造成伤害
│   │   │   └── Decorator: 永远成功         ← 攻击后返回成功(保持序列)
│   │   │
│   │   └── Task: 追击目标                  ← 不在范围内就追击
│   │       (追击任务通过MoveTo持续执行)
│   │
│   └── Task: 等待(0.1秒)                  ← 每轮战斗决策后短暂休息
│
├── Sequence [优先级2: 巡逻]              ← 没有玩家时执行
│   ├── [Service: 更新巡逻路径]            ← 每1秒更新 PatrolIndex
│   ├── Task: 移动到巡逻点                 ← 移动到下一个巡逻位置
│   └── Task: 等待(2秒)                    ← 在巡逻点停留2秒
│
└── Task: 返回初始位置                     ← 如果巡逻点配置为空
    (回到出生位置等待)
```

> **设计思路**：战斗优先。Selector从左到右，先检查"有没有目标"，有就战斗，没有就巡逻。

## 第四部分：C++ 代码实现

### 4.1 MonsterAIController（AI控制器——总指挥）

```cpp
// ==================== MonsterAIController.h ====================
#pragma once

#include "CoreMinimal.h"
#include "AIController.h"                            // AAIController 父类
#include "MonsterAIController.generated.h"           // ⚠️ 必须最后include！

/**
 * 怪物AI控制器：
 * 职责——
 * 1. 在OnPossess时启动行为树
 * 2. 初始化AI感知系统（视觉）
 * 3. 管理巡逻路径点
 * 4. 提供移动和攻击相关接口
 */
UCLASS()
class MYGAME_API AMonsterAIController : public AAIController
{
    GENERATED_BODY()

public:
    // --- 构造函数 ---
    AMonsterAIController();

protected:
    // --- 生命周期 ---

    // BeginPlay：游戏开始时调用一次
    // 在这里初始化AI感知组件
    virtual void BeginPlay() override;

    // OnPossess：控制器"占领"Pawn时调用（AI出生/召唤时）
    // 这是启动行为树的时机！
    // InPawn：被占领的Pawn（AI的身体）
    virtual void OnPossess(APawn* InPawn) override;

    // OnUnPossess：控制器失去Pawn时调用（AI死亡/销毁时）
    // 在这里清理资源
    virtual void OnUnPossess() override;

public:
    // --- AI感知回调 ---

    // 当AI感知到或丢失目标时由感知系统调用
    // Actor：被感知到的Actor（玩家）
    // Stimulus：感知事件的详细信息（看到/丢失/听到了）
    UFUNCTION()  // 因为要绑定到委托，必须加UFUNCTION
    void OnTargetPerceptionUpdated(AActor* Actor, FAIStimulus Stimulus);

    // --- 攻击相关 ---

    // 让AI攻击目标，返回是否成功
    // 会触发攻击动画和伤害计算
    bool TryAttackTarget(AActor* Target);

    // --- 巡逻相关 ---

    // 获取巡逻路径点数组
    // 返回世界中预设的巡逻位置
    const TArray<FVector>& GetPatrolLocations() const { return PatrolLocations; }

    // 获取巡逻点数量
    int32 GetPatrolLocationCount() const { return PatrolLocations.Num(); }

    // --- 编辑器可配置属性 ---

    // 行为树资产（在蓝图或编辑器中指定）
    UPROPERTY(EditDefaultsOnly, Category = "AI|Behavior Tree")
    class UBehaviorTree* BehaviorTreeAsset;
    // EditDefaultsOnly：只能在类默认值（CDO）中编辑
    // Category = "AI|Behavior Tree"：在AI分类下的Behavior Tree子分类

    // 黑板资产（在蓝图或编辑器中指定）
    UPROPERTY(EditDefaultsOnly, Category = "AI|Blackboard")
    class UBlackboardData* BlackboardAsset;
    // UBlackboardData 是黑板的数据资产类型
    // ⚠️ 如果BehaviorTree已经关联了Blackboard，这个可以留空

    // --- AI参数配置 ---

    // 视觉感知半径（能看到玩家的最远距离）
    UPROPERTY(EditDefaultsOnly, Category = "AI|Perception")
    float SightRadius = 2000.0f;  // 默认20米

    // 失去视觉的距离（比看到远一点，防止"抖"）
    UPROPERTY(EditDefaultsOnly, Category = "AI|Perception")
    float LoseSightRadius = 2500.0f;  // 默认25米

    // 周边视觉半角（度）：AI前方能看到的半角
    // 60°半角 = 前方120°锥形区域为视野
    UPROPERTY(EditDefaultsOnly, Category = "AI|Perception")
    float PeripheralVisionAngle = 60.0f;

    // 攻击范围（cm）：与目标距离小于此值才能攻击
    UPROPERTY(EditDefaultsOnly, Category = "AI|Combat")
    float AttackRange = 200.0f;  // 默认2米

    // 攻击伤害值
    UPROPERTY(EditDefaultsOnly, Category = "AI|Combat")
    float AttackDamage = 20.0f;

    // 攻击冷却时间（秒）
    UPROPERTY(EditDefaultsOnly, Category = "AI|Combat")
    float AttackCooldown = 1.5f;

    // --- 巡逻路径配置 ---

    // 巡逻路径点（世界坐标，在编辑器或蓝图中配置）
    // EditAnywhere：可以在关卡实例中单独设置每个AI的巡逻路线
    UPROPERTY(EditAnywhere, Category = "AI|Patrol", meta = (MakeEditWidget = true))
    TArray<FVector> PatrolLocations;
    // meta = (MakeEditWidget = true)：
    //   在编辑器视口中为每个路径点显示一个可拖拽的小部件！
    //   策划可以在3D视图中直接拖拽巡逻点，非常方便

protected:
    // --- 内部辅助 ---

    // 攻击冷却计时器（-1表示冷却完成，>=0表示剩余冷却时间）
    float AttackCooldownRemaining;

    // 初始化AI感知组件（视觉配置）
    void InitializePerceptionSystem();
};
```

```cpp
// ==================== MonsterAIController.cpp ====================
#include "MonsterAIController.h"
#include "BehaviorTree/BehaviorTree.h"                           // UBehaviorTree
#include "BehaviorTree/BlackboardComponent.h"                    // UBlackboardComponent
#include "BehaviorTree/BehaviorTreeComponent.h"                  // UBehaviorTreeComponent
#include "BehaviorTree/BlackboardData.h"                         // UBlackboardData
#include "Perception/AIPerceptionComponent.h"                    // UAIPerceptionComponent
#include "Perception/AISenseConfig_Sight.h"                      // 视觉配置
#include "Perception/AISense_Sight.h"                            // 视觉Sense
#include "GameFramework/Character.h"                             // ACharacter
#include "Kismet/GameplayStatics.h"                              // GetPlayerPawn

// ==================== 构造函数 ====================
AMonsterAIController::AMonsterAIController()
{
    // --- 创建AI感知组件 ---
    // AAIController默认没有感知组件，需要手动创建
    PerceptionComponent = CreateDefaultSubobject<UAIPerceptionComponent>(
        TEXT("PerceptionComponent")
    );
    // CreateDefaultSubobject：在构造函数中创建子对象
    //   第一个参数：组件的C++类名（UAIPerceptionComponent::StaticClass()不是必须的，
    //     因为UE可以通过模板推导）
    //   第二个参数：组件的内部名称（用于序列化和复制）

    // --- 创建视觉感知配置 ---
    // 每个感官类型需要一个配置对象
    UAISenseConfig_Sight* SightConfig = CreateDefaultSubobject<UAISenseConfig_Sight>(
        TEXT("SightConfig")
    );

    // 配置视觉参数
    SightConfig->SightRadius = SightRadius;           // 最远可见距离
    SightConfig->LoseSightRadius = LoseSightRadius;   // 丢失视野距离
    SightConfig->PeripheralVisionAngleDegrees = PeripheralVisionAngle; // 视野半角
    SightConfig->DetectionByAffiliation.bDetectEnemies = true;    // ✅ 检测敌人
    SightConfig->DetectionByAffiliation.bDetectNeutrals = true;   // ✅ 检测中立
    SightConfig->DetectionByAffiliation.bDetectFriendlies = false; // ❌ 不检测友军
    // DetectionByAffiliation 是 FAISenseAffiliationFilter 结构体
    // 它根据Actor的"队伍"身份来决定是否检测

    // 将配置注册到感知组件
    PerceptionComponent->ConfigureSense(*SightConfig);
    // ConfigureSense 接受一个 AISenseConfig 引用

    // 设置感知组件的主导感官类型
    // 当多个感官都检测到目标时，以这个为准
    PerceptionComponent->SetDominantSense(UAISense_Sight::StaticClass());
    // UAISense_Sight::StaticClass()：获取视觉Sense的UClass

    // --- 攻击冷却初始化 ---
    AttackCooldownRemaining = -1.0f;  // -1 = 冷却完成（可以攻击）
}

// ==================== BeginPlay ====================
void AMonsterAIController::BeginPlay()
{
    Super::BeginPlay();  // ⚠️ 永远先调用父类！

    // --- 存储出生位置到Blackboard ---
    // 用于"返回出生点"功能
    APawn* MyPawn = GetPawn();
    if (MyPawn && GetBlackboardComponent())
    {
        // 获取AI出生时的位置
        FVector SpawnLocation = MyPawn->GetActorLocation();

        // 写入Blackboard
        GetBlackboardComponent()->SetValueAsVector(
            TEXT("HomeLocation"),
            SpawnLocation
        );
    }

    // --- 初始化AI感知系统 ---
    InitializePerceptionSystem();

    UE_LOG(LogTemp, Log, TEXT("[MonsterAI] 初始化完成。出生位置: %s"),
        MyPawn ? *MyPawn->GetActorLocation().ToString() : TEXT("未知"));
}

// ==================== OnPossess（启动行为树）====================
void AMonsterAIController::OnPossess(APawn* InPawn)
{
    // 先让父类做基础初始化（设置Pawn引用等）
    Super::OnPossess(InPawn);

    UE_LOG(LogTemp, Log, TEXT("[MonsterAI] 占领Pawn: %s"), *InPawn->GetName());

    // --- 启动行为树 ---
    // 方法1：直接使用配置的行为树资产（如果设置了的话）
    if (BehaviorTreeAsset != nullptr)
    {
        RunBehaviorTree(BehaviorTreeAsset);
        // RunBehaviorTree内部做的事：
        // 1. 创建/初始化 BlackboardComponent
        // 2. 从行为树资产的BlackboardAsset获取黑板数据
        // 3. 创建 BehaviorTreeComponent
        // 4. 开始从Root节点执行行为树
    }
    else
    {
        // 没有配置行为树！必须给AI配一个
        UE_LOG(LogTemp, Error,
            TEXT("[MonsterAI] ❌ 没有配置BehaviorTree资产！"
                  "请在AI控制器蓝图中设置 BehaviorTreeAsset。"));
    }

    // --- 方法2：手动指定行为树和黑板（如果不用RunBehaviorTree）---
    // 如果你的行为树和黑板是分开的资产，可以这样：
    // if (BlackboardAsset && BehaviorTreeAsset)
    // {
    //     UseBlackboard(BlackboardAsset, GetBlackboardComponent());
    //     RunBehaviorTree(BehaviorTreeAsset);
    // }
    // UseBlackboard 用于使用自定义的Blackboard数据定义
}

void AMonsterAIController::OnUnPossess()

// ==================== 初始化AI感知系统 ====================
void AMonsterAIController::InitializePerceptionSystem()
{
    UAIPerceptionComponent* PercComp = GetAIPerceptionComponent();
    if (!PercComp)
    {
        UE_LOG(LogTemp, Error,
            TEXT("[MonsterAI] 感知组件不存在！请确保构造函数中创建了它。"));
        return;
    }

    // --- 绑定感知更新回调 ---
    // 动态绑定：当目标被看到或丢失时触发
    PercComp->OnTargetPerceptionUpdated.AddDynamic(
        this,    // 绑定到的对象（当前AIController）
        &AMonsterAIController::OnTargetPerceptionUpdated  // 回调函数
    );
    // AddDynamic 是 UE 的委托绑定宏，用于绑定到 UFUNCTION

    // --- 另一种回调绑定方式：感知信息遗忘 ---
    // 当目标彻底从感知系统中"遗忘"时触发
    PercComp->OnTargetPerceptionForgotten.AddDynamic(
        this,
        &AMonsterAIController::OnTargetPerceptionUpdated  // 复用同一个回调
    );

    UE_LOG(LogTemp, Log,
        TEXT("[MonsterAI] AI感知系统初始化完成。视野半径: %.0fcm"),
        SightRadius);
}

// ==================== 感知更新回调 ====================
void AMonsterAIController::OnTargetPerceptionUpdated(
    AActor* Actor,
    FAIStimulus Stimulus)
{
    // --- 安全检查 ---
    if (!Actor)
    {
        return;  // 没有Actor，直接返回
    }

    // --- 获取黑板组件 ---
    UBlackboardComponent* Blackboard = GetBlackboardComponent();
    if (!Blackboard)
    {
        UE_LOG(LogTemp, Warning,
            TEXT("[MonsterAI] 黑板组件未初始化，无法更新目标信息"));
        return;
    }

    // --- 判断是看到目标还是丢失目标 ---
    if (Stimulus.WasSuccessfullySensed())
    {
        // ✅ 成功感知到目标（看到了！）
        // 检查这个Actor是不是玩家（我们只关心玩家）
        // 获取PlayerController索引0的Pawn
        APawn* PlayerPawn = UGameplayStatics::GetPlayerPawn(GetWorld(), 0);
        // GetPlayerPawn(World, PlayerIndex)：
        //   PlayerIndex=0 → 第一个玩家（单机游戏总是0）

        if (Actor == PlayerPawn)
        {
            // 确实是玩家！写入Blackboard
            Blackboard->SetValueAsObject(TEXT("TargetActor"), Actor);
            // 行为树中的Decorator就能读到 TargetActor 了

            // 同时写入玩家的当前位置
            Blackboard->SetValueAsVector(
                TEXT("TargetLocation"),
                Actor->GetActorLocation()
            );

            // 通知AI进入战斗状态
            Blackboard->SetValueAsBool(TEXT("bCanAttack"), true);

            UE_LOG(LogTemp, Log,
                TEXT("[MonsterAI] 🎯 发现玩家！开始追击。"
                      "玩家位置: %s"),
                *Actor->GetActorLocation().ToString());
        }
    }
    else
    {
        // ❌ 目标丢失（玩家跑出视野了）
        // ⚠️ 注意：这里要小心处理！如果AI刚被另一个AI感知到，
        //   而玩家还在视野中，不应该盲目清除 TargetActor

        // 简单处理：直接检查当前存储的目标是不是这个Actor
        UObject* CurrentTarget = Blackboard->GetValueAsObject(TEXT("TargetActor"));
        if (CurrentTarget == Actor)
        {
            // 丢失的确实是当前在追的目标，清空
            Blackboard->ClearValue(TEXT("TargetActor"));
            Blackboard->ClearValue(TEXT("TargetLocation"));

            UE_LOG(LogTemp, Log,
                TEXT("[MonsterAI] 👀 丢失目标！返回巡逻。"));
        }
    }

    // --- 可选：更健壮的处理方式 ---
    // 实际项目中建议：
    // 1. 使用 GetCurrentlyPerceivedActors 检查"当前能看到的所有Actor"
    // 2. 只有当没有任何可感知的目标时才清空黑板
    //
    // 示例：
    // TArray<AActor*> PerceivedActors;
    // GetAIPerceptionComponent()->GetCurrentlyPerceivedActors(
    //     UAISense_Sight::StaticClass(),  // 只看视觉
    //     PerceivedActors
    // );
    // if (PerceivedActors.Num() == 0)
    // {
    //     // 确实任何目标都看不到了
    //     Blackboard->ClearValue(TEXT("TargetActor"));
    // }
}

// ==================== 攻击目标 ====================
bool AMonsterAIController::TryAttackTarget(AActor* Target)
{
    // --- 安全检查 ---
    if (!Target)
    {
        UE_LOG(LogTemp, Warning,
            TEXT("[MonsterAI] 攻击失败：目标为空"));
        return false;
    }

    // --- 检查攻击冷却 ---
    if (AttackCooldownRemaining > 0.0f)
    {
        // 还在冷却中，不能攻击
        // 注意：AttackCooldown的倒计时应该在Monster自身或Controller的Tick中处理
        // 这里简化处理，仅做检查
        return false;
    }

    // --- 检查距离 ---
    APawn* MyPawn = GetPawn();
    if (!MyPawn)
    {
        return false;
    }

    float Distance = FVector::Dist(
        MyPawn->GetActorLocation(),
        Target->GetActorLocation()
    );

    if (Distance > AttackRange)
    {
        UE_LOG(LogTemp, Warning,
            TEXT("[MonsterAI] 攻击失败：目标太远（%.0f cm > %.0f cm）"),
            Distance, AttackRange);
        return false;
    }

    // 1. 让AI面对目标

    // 2. 触发攻击动画（通过ACharacter的动画蓝图）
    if (ACharacter* MyChar = Cast<ACharacter>(MyPawn))
    {
        // PlayAnimMontage 用于播放一次性动画
        // 实际项目中应该在AnimInstance中处理
        // 这里用简单的方式：通过接口或直接调用
    }

    // 3. 对目标造成伤害
    // ApplyDamage 是UE的通用伤害函数
    UGameplayStatics::ApplyDamage(
        Target,         // 受害者
        AttackDamage,   // 伤害值
        this,           // 伤害来源的Controller
        MyPawn,         // 伤害来源的Actor
        UDamageType::StaticClass()  // 伤害类型
```

```cpp
EBTNodeResult::Type UBTTask_ChaseTarget::ExecuteTask(
    UBehaviorTreeComponent& OwnerComp,
    uint8* NodeMemory)
{
    // ===== 步骤1：获取自定义NodeMemory =====
    FBTTaskChaseMemory* MyMemory = reinterpret_cast<FBTTaskChaseMemory*>(NodeMemory);

    // ===== 步骤2：获取AIController和受控Pawn =====
    AAIController* AIController = OwnerComp.GetAIOwner();
    if (!AIController)
    {
        UE_LOG(LogTemp, Error, TEXT("[追擊] AIController 不存在"));
        return EBTNodeResult::Failed;
    }

    APawn* ControlledPawn = AIController->GetPawn();
    if (!ControlledPawn)
    {
        UE_LOG(LogTemp, Error, TEXT("[追擊] 受控Pawn 不存在"));
        return EBTNodeResult::Failed;
    }

    // ===== 步骤3：从黑板获取目标位置 =====
    UBlackboardComponent* Blackboard = OwnerComp.GetBlackboardComponent();
    if (!Blackboard)
    {
        UE_LOG(LogTemp, Error, TEXT("[追擊] 黑板组件不存在"));
        return EBTNodeResult::Failed;
    }

    FVector TargetLocation = Blackboard->GetValueAsVector(
        TargetLocationKey.SelectedKeyName
    );
    // 读取黑板中由 UBTService_UpdateTarget 持续更新的目标位置

    // ===== 步骤4：检查目标位置是否有效 =====
    if (TargetLocation.IsNearlyZero())
    {
        // 位置是零向量=没有目标位置
        UE_LOG(LogTemp, Warning, TEXT("[追擊] 目标位置无效，放弃追击"));
        return EBTNodeResult::Failed;
    }

    // ===== 步骤5：检查是否已经在目标位置 =====
    float DistanceToTarget = FVector::Dist(
        ControlledPawn->GetActorLocation(),
        TargetLocation
    );

    if (DistanceToTarget <= AcceptableRadius)
    {
        // 已经在目标旁边了，直接成功
        UE_LOG(LogTemp, Log, TEXT("[追擊] 已经在目标旁边（距离 %.0f cm）"), DistanceToTarget);
        return EBTNodeResult::Succeeded;
    }

    // ===== 步骤6：发起移动请求 =====
    EPathFollowingRequestResult::Type MoveResult = AIController->MoveToLocation(
        TargetLocation,          // 要去的位置
        AcceptableRadius,        // 距离目标这么远就算到了
        true,                    // bStopOnOverlap：碰到目标就停
        true,                    // bUsePathfinding：使用NavMesh寻路
        false,                   // bProjectDestinationToNavigation
        false,                   // bCanStrafe：不侧移
        nullptr,                 // FilterClass
        false                    // bAllowPartialPath
    );

    // ===== 步骤7：处理移动结果 =====
    if (MoveResult == EPathFollowingRequestResult::RequestSuccessful)
    {
        // 成功发出了移动命令
        MyMemory->StartTime = GetWorld()->GetTimeSeconds();
        MyMemory->LastUpdateTime = MyMemory->StartTime;

        UE_LOG(LogTemp, Log,
            TEXT("[追擊] 开始追击！起点: %s → 目标: %s"),
            *ControlledPawn->GetActorLocation().ToString(),
            *TargetLocation.ToString());

        return EBTNodeResult::InProgress;
        // InProgress = 任务在行进中，每帧会调到 TickTask
    }
    else if (MoveResult == EPathFollowingRequestResult::AlreadyAtGoal)
    {
        UE_LOG(LogTemp, Log, TEXT("[追擊] 已经在目标位置"));
        return EBTNodeResult::Succeeded;
    }
    else
    {
        // 寻路失败（目标不可达/导航网格外）
        UE_LOG(LogTemp, Warning, TEXT("[追擊] 寻路失败！无法到达目标位置"));
        return EBTNodeResult::Failed;
    }
}

void UBTTask_ChaseTarget::TickTask(
    UBehaviorTreeComponent& OwnerComp,
    uint8* NodeMemory,
    float DeltaSeconds)
{
    // ===== 步骤1：获取自定义NodeMemory =====
    FBTTaskChaseMemory* MyMemory = reinterpret_cast<FBTTaskChaseMemory*>(NodeMemory);

    // ===== 步骤2：获取AIController =====
    AAIController* AIController = OwnerComp.GetAIOwner();
    if (!AIController)
    {
        FinishExecute(OwnerComp, EBTNodeResult::Failed);
        return;
    }

    APawn* ControlledPawn = AIController->GetPawn();
    if (!ControlledPawn)
    {
        FinishExecute(OwnerComp, EBTNodeResult::Failed);
        return;
    }

    UBlackboardComponent* Blackboard = OwnerComp.GetBlackboardComponent();
    if (!Blackboard)
    {
        FinishExecute(OwnerComp, EBTNodeResult::Failed);
        return;
    }

    // ===== 步骤3：检查移动状态 =====
    EPathFollowingStatus::Type MoveStatus = AIController->GetMoveStatus();

    if (MoveStatus == EPathFollowingStatus::Idle)
    {
        // 停止了=成功到达
        UE_LOG(LogTemp, Log, TEXT("[追擊] 到达目标！"));
        FinishExecute(OwnerComp, EBTNodeResult::Succeeded);
        return;
    }

    // ===== 步骤4：检查超时 =====
    float CurrentTime = GetWorld()->GetTimeSeconds();
    float ElapsedTime = CurrentTime - MyMemory->StartTime;

    if (ElapsedTime > MaxChaseDuration)
    {
        // 超时了！追太久了，放弃
        UE_LOG(LogTemp, Warning,
            TEXT("[追擊] 超时！追了 %.1f 秒还没追到，放弃"), ElapsedTime);
        AIController->StopMovement();  // 停止移动
        FinishExecute(OwnerComp, EBTNodeResult::Failed);
        return;
    }

    // ===== 步骤5：定期更新移动目标 =====
    // 因为玩家可能在移动，需要更新路径
    float TimeSinceLastUpdate = CurrentTime - MyMemory->LastUpdateTime;
    if (TimeSinceLastUpdate >= PathUpdateInterval)
    {
        // 重新从黑板读取最新位置
        FVector NewTargetLocation = Blackboard->GetValueAsVector(
            TargetLocationKey.SelectedKeyName
        );

        // 检查新位置是否明显变化（和上次比移动了超过50cm才更新路径）
        FVector CurrentTarget = FVector::ZeroVector;
        // ...实际项目中应缓存上次位置以便比较

        // 更新路径
        AIController->MoveToLocation(
            NewTargetLocation,
            AcceptableRadius,
            true,      // bStopOnOverlap
            true,      // bUsePathfinding
            false,     // bProjectDestinationToNavigation
            false,     // bCanStrafe
            nullptr,   // FilterClass
            false      // bAllowPartialPath
        );

        MyMemory->LastUpdateTime = CurrentTime;

        // 让AI面向目标（可选）
        AActor* TargetActor = Cast<AActor>(
            Blackboard->GetValueAsObject(TargetActorKey.SelectedKeyName)
        );
        if (TargetActor)
        {
            AIController->SetFocus(TargetActor);
        }
    }
}

void UBTTask_ChaseTarget::OnTaskFinished(
    UBehaviorTreeComponent& OwnerComp,
    uint8* NodeMemory,
    EBTNodeResult::Type TaskResult)
{
    // 清理：停止移动（如果任务是失败或被中断）
    if (TaskResult != EBTNodeResult::Succeeded)
    {
        AAIController* AIController = OwnerComp.GetAIOwner();
        if (AIController)
        {
            AIController->StopMovement();
        }
    }

    UE_LOG(LogTemp, Log, TEXT("[追擊] 任务结束。结果: %d"),
        static_cast<int32>(TaskResult));
}
```

```cpp
EBTNodeResult::Type UBTTask_AttackTarget::ExecuteTask(

    if (!GenericController)
    {
        UE_LOG(LogTemp, Error, TEXT("[攻击] AIController不存在"));
        return EBTNodeResult::Failed;

    // 转换为我们的自定义AIController
    // 因为 TryAttackTarget 是我们自定义的函数
    AMonsterAIController* MonsterController =
        Cast<AMonsterAIController>(GenericController);
    if (!MonsterController)
    {
        // AI控制器类型不匹配！说明配错了
        UE_LOG(LogTemp, Error,
            TEXT("[攻击] AIController类型错误！"
                  "应为 AMonsterAIController"));
        return EBTNodeResult::Failed;
    }

    // ===== 步骤2：从黑板获取目标Actor =====
    UBlackboardComponent* Blackboard = OwnerComp.GetBlackboardComponent();
    if (!Blackboard)
    {
        return EBTNodeResult::Failed;
    }

    AActor* TargetActor = Cast<AActor>(
        Blackboard->GetValueAsObject(TargetActorKey.SelectedKeyName)
    );
    // ⚠️ GetValueAsObject 返回 UObject*，需要 Cast 到 AActor*

    if (!TargetActor)
    {
        // 目标Actor不存在（可能死了/被销毁了）
        UE_LOG(LogTemp, Warning, TEXT("[攻击] 目标为空，无法攻击"));
        return EBTNodeResult::Failed;
    }

    // ===== 步骤3：让AI面向目标 =====
    GenericController->SetFocus(TargetActor);
    // SetFocus 会让AI的Pawn平滑转向面向目标
    // 这在攻击动画播放时尤其重要

    // ===== 步骤4：执行攻击 =====
    bool bAttackSuccess = MonsterController->TryAttackTarget(TargetActor);

    if (!bAttackSuccess)
    {
        // 攻击失败（冷却中或其他原因）
        UE_LOG(LogTemp, Warning, TEXT("[攻击] 攻击失败"));
        GenericController->ClearFocus(EAIFocusPriority::Gameplay);
        return EBTNodeResult::Failed;
    }

    // ===== 步骤5：记录动画开始时间 =====
    FBTTaskAttackMemory* MyMemory =
        reinterpret_cast<FBTTaskAttackMemory*>(NodeMemory);
    MyMemory->AnimationStartTime = GetWorld()->GetTimeSeconds();

    UE_LOG(LogTemp, Log, TEXT("[攻击] ⚔️ 攻击已触发！等待动画完成..."));

    // ===== 步骤6：返回InProgress，等待动画播放 =====
    return EBTNodeResult::InProgress;
    // InProgress = 动画播放中，还需等待
    // TickTask 会在每帧调用直到我们调用 FinishExecute
}
```

```cpp
#include "BTTask_Patrol.h"

EBTNodeResult::Type UBTTask_Patrol::ExecuteTask(
    UBehaviorTreeComponent& OwnerComp,
    uint8* NodeMemory)
{
    // ===== 步骤1：获取AIController =====
    AMonsterAIController* MonsterController =
        Cast<AMonsterAIController>(OwnerComp.GetAIOwner());
    if (!MonsterController)
    {
        UE_LOG(LogTemp, Error, TEXT("[巡逻] AIController不存在或类型错误"));
        return EBTNodeResult::Failed;
    }

    APawn* ControlledPawn = MonsterController->GetPawn();
    if (!ControlledPawn)
    {
        return EBTNodeResult::Failed;
    }

    // ===== 步骤2：获取巡逻路径点 =====
    const TArray<FVector>& PatrolLocations = MonsterController->GetPatrolLocations();
    // 获取AIController中配置的巡逻路径点数组

    if (PatrolLocations.Num() == 0)
    {
        // 没有配置巡逻路径点！
        UE_LOG(LogTemp, Warning, TEXT("[巡逻] 没有配置巡逻路径点！"));
        return EBTNodeResult::Failed;
    }

    // ===== 步骤3：从黑板获取当前巡逻索引 =====
    UBlackboardComponent* Blackboard = OwnerComp.GetBlackboardComponent();
    if (!Blackboard)
    {
        return EBTNodeResult::Failed;
    }

    int32 CurrentIndex = Blackboard->GetValueAsInt(
        PatrolIndexKey.SelectedKeyName
    );

    // 边界检查：防止索引越界
    if (CurrentIndex < 0 || CurrentIndex >= PatrolLocations.Num())
    {
        // 索引无效，重置为0
        CurrentIndex = 0;
        Blackboard->SetValueAsInt(
            PatrolIndexKey.SelectedKeyName, 0
        );
        UE_LOG(LogTemp, Warning,
            TEXT("[巡逻] 巡逻索引越界，重置为0"));
    }

    // ===== 步骤4：获取目标巡逻点位置 =====
    FVector TargetLocation = PatrolLocations[CurrentIndex];

    // ===== 步骤5：检查是否已经在巡逻点 =====
    float Distance = FVector::Dist(
        ControlledPawn->GetActorLocation(),
        TargetLocation
    );

    if (Distance <= AcceptableRadius)
    {
        // 已经在这个巡逻点了，不需要移动
        UE_LOG(LogTemp, Log,
            TEXT("[巡逻] 已在巡逻点 %d（距离 %.0f cm），跳过移动"),
            CurrentIndex, Distance);
        return EBTNodeResult::Succeeded;
    }

    // ===== 步骤6：开始移动到巡逻点 =====
    EPathFollowingRequestResult::Type MoveResult =
        MonsterController->MoveToLocation(
            TargetLocation,
            AcceptableRadius,
            true,   // bStopOnOverlap
            true,   // bUsePathfinding：使用导航网格
            false,  // bProjectDestinationToNavigation
            false,  // bCanStrafe
            nullptr,
            false   // bAllowPartialPath
        );

    if (MoveResult == EPathFollowingRequestResult::RequestSuccessful)
    {
        UE_LOG(LogTemp, Log,
            TEXT("[巡逻] 🚶 前往巡逻点 %d/%d (位置: %s)"),
            CurrentIndex, PatrolLocations.Num() - 1,
            *TargetLocation.ToString());
        return EBTNodeResult::InProgress;
    }
    else if (MoveResult == EPathFollowingRequestResult::AlreadyAtGoal)
    {
        return EBTNodeResult::Succeeded;
    }
    else
    {
        UE_LOG(LogTemp, Warning,
            TEXT("[巡逻] 无法到达巡逻点 %d"), CurrentIndex);
        return EBTNodeResult::Failed;
    }
}

void UBTTask_Patrol::TickTask(
    UBehaviorTreeComponent& OwnerComp,
    uint8* NodeMemory,
    float DeltaSeconds)
{
    // ===== 检查是否到达目标 =====
    AAIController* AIController = OwnerComp.GetAIOwner();
    if (!AIController)
    {
        FinishExecute(OwnerComp, EBTNodeResult::Failed);
        return;
    }

    EPathFollowingStatus::Type MoveStatus = AIController->GetMoveStatus();

    if (MoveStatus == EPathFollowingStatus::Idle)
    {
        // 移动停止=到达了
        UE_LOG(LogTemp, Log, TEXT("[巡逻] ✅ 到达巡逻点"));
        FinishExecute(OwnerComp, EBTNodeResult::Succeeded);
    }
    // 如果Moving状态，什么都不做，下一帧继续检查
}
```

```cpp
#include "BTService_UpdateTarget.h"
#include "AIController.h"
#include "BehaviorTree/BlackboardComponent.h"
#include "Perception/AIPerceptionComponent.h"
#include "Perception/AISense_Sight.h"

UBTService_UpdateTarget::UBTService_UpdateTarget()
{
    NodeName = TEXT("更新目标信息");

    // 每0.3秒更新一次目标信息
    // 0.3秒对追踪来说已经足够快
    Interval = 0.3f;

    // 添加随机偏移，避免多个AI同时Tick
    RandomDeviation = 0.05f;

    // --- 配置 Blackboard Key 过滤器 ---

void UBTService_UpdateTarget::TickNode(
    UBehaviorTreeComponent& OwnerComp,
    uint8* NodeMemory,
    float DeltaSeconds)
{
    Super::TickNode(OwnerComp, NodeMemory, DeltaSeconds);

    // ===== 步骤1：获取必要的引用 =====
    AAIController* AIController = OwnerComp.GetAIOwner();
    if (!AIController) return;

    UBlackboardComponent* Blackboard = OwnerComp.GetBlackboardComponent();
    if (!Blackboard) return;

    UAIPerceptionComponent* PerceptionComp = AIController->GetAIPerceptionComponent();
    if (!PerceptionComp) return;

    // ===== 步骤2：获取当前所有感知到的Actor =====
    // 只获取"视觉"感知到的Actor
    TArray<AActor*> PerceivedActors;
    PerceptionComp->GetCurrentlyPerceivedActors(
        UAISense_Sight::StaticClass(),  // 只查视觉感知
        PerceivedActors                 // 输出参数：存储结果
    );
    // GetCurrentlyPerceivedActors 返回所有"当前正在被感知"的Actor
    // 不只是"看到的"，可能是"听到的"、"感知到伤害的"等等

    // ===== 步骤3：查找Player角色 =====
    // 在所有被感知的Actor中找到玩家Pawn
    AActor* BestTarget = nullptr;

    for (AActor* Actor : PerceivedActors)
    {
        if (!Actor) continue;  // 安全检查

        // 检查Actor的Tag（一种简单的方式识别玩家）
        // 这是一种轻量级的识别方式
        if (Actor->ActorHasTag(TEXT("Player")))
        {
            BestTarget = Actor;
            break;  // 找到玩家了就停止（后续可以扩展多目标逻辑）
        }

        // 备用：通过Cast检查是否为玩家Pawn类型
        // if (Cast<ACharacter>(Actor) && Actor->GetInstigatorController() &&
        //     Actor->GetInstigatorController()->IsPlayerController)
        // {
        //     BestTarget = Actor;
        //     break;
        // }
    }

    // ===== 步骤4：更新Blackboard =====
    if (BestTarget)
    {
        // ✅ 看到了玩家！更新目标信息
        AActor* CurrentTarget = Cast<AActor>(
            Blackboard->GetValueAsObject(TargetActorKey.SelectedKeyName)
        );

        // 如果是新目标或目标变了
        if (CurrentTarget != BestTarget)
        {
            Blackboard->SetValueAsObject(
                TargetActorKey.SelectedKeyName,
                BestTarget
            );
            // 黑板中的 TargetActor 现在是玩家了
        }

        // 始终更新目标位置（玩家可能在移动）
        Blackboard->SetValueAsVector(
            TargetLocationKey.SelectedKeyName,
            BestTarget->GetActorLocation()
        );
        // 这样追逐Task就能获得最新的玩家位置来更新路径
    }
    else
    {
        // ❌ 没看到任何目标
        // 注意：这里不能直接清空TargetActor！
        // 因为AIPerceptionComponent的 OnTargetPerceptionUpdated
        // 回调已经处理了"丢失目标"的逻辑
        // 这里清空可能导致和感知系统不同步
    }
}
```

```cpp
// ==================== BTService_UpdatePatrolPath.cpp ====================
#include "BTService_UpdatePatrolPath.h"
#include "AIController.h"
#include "BehaviorTree/BlackboardComponent.h"
#include "MonsterAIController.h"  // 获取巡逻路径点数组

UBTService_UpdatePatrolPath::UBTService_UpdatePatrolPath()
{
    NodeName = TEXT("更新巡逻路径");

    // 巡逻点切换不需要很频繁
    // 只有到了当前巡逻点后，才需要切到下一个
    Interval = 1.0f;  // 每1秒检查一次

    RandomDeviation = 0.1f;

    // 配置 Blackboard Key 过滤
    PatrolIndexKey.AddIntFilter(
        this,
        GET_MEMBER_NAME_CHECKED(UBTService_UpdatePatrolPath, PatrolIndexKey)
    );
}

void UBTService_UpdatePatrolPath::OnBecomeRelevant(
    UBehaviorTreeComponent& OwnerComp,
    uint8* NodeMemory)
{
    Super::OnBecomeRelevant(OwnerComp, NodeMemory);

    // 巡逻分支被激活时，确保PatrolIndex从0开始
    // （可选，取决于你是否想从中断点继续巡逻）
    UBlackboardComponent* Blackboard = OwnerComp.GetBlackboardComponent();
    if (Blackboard)
    {
        // 确保索引在有效范围内
        int32 CurrentIndex = Blackboard->GetValueAsInt(
            PatrolIndexKey.SelectedKeyName
        );

        AMonsterAIController* MonsterController =
            Cast<AMonsterAIController>(OwnerComp.GetAIOwner());

        if (MonsterController)
        {
            int32 TotalPoints = MonsterController->GetPatrolLocationCount();
            if (TotalPoints > 0 && CurrentIndex >= TotalPoints)
            {
                // 索引越界，重置
                Blackboard->SetValueAsInt(
                    PatrolIndexKey.SelectedKeyName, 0
                );
            }
        }
    }
}

void UBTService_UpdatePatrolPath::TickNode(
    UBehaviorTreeComponent& OwnerComp,
    uint8* NodeMemory,
    float DeltaSeconds)
{
    Super::TickNode(OwnerComp, NodeMemory, DeltaSeconds);

    // ===== 步骤1：获取必要引用 =====
    AMonsterAIController* MonsterController =
        Cast<AMonsterAIController>(OwnerComp.GetAIOwner());
    if (!MonsterController) return;

    UBlackboardComponent* Blackboard = OwnerComp.GetBlackboardComponent();
    if (!Blackboard) return;

    // ===== 步骤2：获取巡逻路径点总数 =====
    int32 TotalPoints = MonsterController->GetPatrolLocationCount();
    if (TotalPoints == 0)
    {
        // 没有巡逻点，不需要更新
        return;
    }

    // ===== 步骤3：递增巡逻索引（循环）=====
    int32 CurrentIndex = Blackboard->GetValueAsInt(
        PatrolIndexKey.SelectedKeyName
    );

    // 移动到下一个巡逻点（循环到0）
    int32 NextIndex = (CurrentIndex + 1) % TotalPoints;
    // 取模运算确保索引在 0 ~ TotalPoints-1 之间循环
    // 例如：TotalPoints=4，CurrentIndex=3 → NextIndex=0

    Blackboard->SetValueAsInt(
        PatrolIndexKey.SelectedKeyName,
        NextIndex
    );

    // 这样，当巡逻Task再次执行时，就会移动到新的巡逻点了
}
```

```cpp
bool UBTDecorator_HasTarget::CalculateRawConditionValue(
    UBehaviorTreeComponent& OwnerComp,
    uint8* NodeMemory) const
{
    // ===== 从黑板读取 TargetActor =====
    UBlackboardComponent* Blackboard = OwnerComp.GetBlackboardComponent();
    if (!Blackboard)
    {
        return false;  // 没有黑板 → 没有目标
    }

    UObject* TargetObject = Blackboard->GetValueAsObject(
        TargetActorKey.SelectedKeyName
    );

    // ===== 判断 =====
    if (TargetObject == nullptr)
    {
        return false;  // ✅ 没目标 → 该分支不执行
    }

    // 可以进一步检查目标是否还有效
    AActor* TargetActor = Cast<AActor>(TargetObject);
    if (!TargetActor || TargetActor->IsPendingKillPending())
    {
        return false;  // 目标无效或即将被销毁
    }

    return true;  // ✅ 有有效目标 → 执行该分支
}
```

## ❌ 常见错误排查指南

| 症状                 | 可能原因                                    | 排查方法                                                             |
| -------------------- | ------------------------------------------- | -------------------------------------------------------------------- |
| AI站在原处不动       | 行为树没有配置或没有启动                    | 检查BP_MonsterController中Behavior Tree Asset是否设置                |
| AI不巡逻             | PatrolLocations数组为空                     | 在BP_MonsterController中配置巡逻路径点                               |
| AI看到玩家但不追击   | 视觉感知配置不对或TargetActor没有被写入黑板 | 加日志检查 OnTargetPerceptionUpdated 是否触发                        |
| AI追到一半停了       | MaxChaseDuration设得太短                    | 检查BTTask_ChaseTarget的时间参数                                     |
| 攻击动画没播完就停了 | AttackAnimationDuration设置过短             | 使用AnimNotify来同步而不是魔法数字计时                               |
| 编辑器崩溃           | Blackboard Key名拼写错误                    | 使用FBlackboardKeySelector，不要硬编码Key名                          |
| 行为树切换分支后卡住 | 正在InProgress的Task没有被Abort             | 设置Decorator的Abort Mode                                            |
| 编译错误：缺少头文件 | 忘记include对应的模块头文件                 | 在.Build.cs文件中添加"AIModule", "NavigationSystem", "GameplayTasks" |
