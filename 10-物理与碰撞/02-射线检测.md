## 什么是射线检测？

射线检测（Line Trace / Ray Trace）是游戏开发中最常用的"查询"操作之一。它从空间中一个点向另一个点"射出一条不可见的线"，然后告诉你这条线碰到了什么。

### 射线检测的典型应用

```
射击游戏：     枪口 → 准星方向射出射线，判断打中了哪个敌人
交互系统：     摄像机 → 屏幕中心射出射线，检测玩家正在看什么物品
移动系统：     角色脚底 → 向下射出射线，检测地面高度（贴地移动）
AI感知：       敌人眼睛 → 玩家方向射出射线，判断是否能看到玩家
技能系统：     技能起点 → 目标方向射出射线，检测技能路径上的目标
```

### 三维空间中的射线

```cpp
// ==================== 射线的数学定义 ====================
//
// 射线 = 起点(Origin) + 方向(Direction) × 距离(t)
//
// 起点(Origin)：  射线从哪里射出（FVector，三维空间中的一个点）
// 方向(Direction)：射线射向哪里（FVector，一个单位向量）
// 距离(t)：       射线在当前方向上的长度（float，单位厘米）
//
// 终点(End) = 起点(Origin) + 方向(Direction) * 距离(Distance)

// 例如：
FVector Origin(0.0f, 0.0f, 100.0f);       // 从世界坐标(0,0,100)位置射出
FVector Direction(1.0f, 0.0f, 0.0f);      // 向X轴正方向射出
float Distance = 1000.0f;                   // 探测1000cm（10米）远
FVector End = Origin + Direction * Distance; // 终点 = (1000, 0, 100)
```

### UE5中射线检测API的分类

UE5提供了两大类射线检测API：

| API族        | 前缀                 | 命中行为                   | 返回值                             |
| ------------ | -------------------- | -------------------------- | ---------------------------------- |
| **单次检测** | `LineTraceSingle...` | 命中第一个物体就停止       | `bool` + 一个 `FHitResult`         |
| **多次检测** | `LineTraceMulti...`  | 穿透所有物体，返回所有命中 | `bool` + 一个 `TArray<FHitResult>` |

| API族            | 后缀              | 筛选方式                                 |
| ---------------- | ----------------- | ---------------------------------------- |
| **ByChannel**    | `...ByChannel`    | 按碰撞通道筛选（使用通道的碰撞响应设置） |
| **ByObjectType** | `...ByObjectType` | 按物体类型筛选（只检测指定类型的物体）   |
| **ByProfile**    | `...ByProfile`    | 按碰撞预设名称筛选（使用预设中的配置）   |

> 组合起来就是 2×3 = 6 个核心API：`LineTraceSingleByChannel`、`LineTraceSingleByObjectType`、`LineTraceSingleByProfile`、`LineTraceMultiByChannel`、`LineTraceMultiByObjectType`、`LineTraceMultiByProfile`

## LineTraceSingleByChannel

`LineTraceSingleByChannel` 是**最常用**的射线检测API。它按照碰撞通道的响应规则进行检测，命中第一个阻挡（Block）的物体后立即返回。

### 完整函数签名

```cpp
// ==================== LineTraceSingleByChannel 完整定义 ====================
// 位于：Engine/Source/Runtime/Engine/Public/World.h
// 所属类：UWorld

bool LineTraceSingleByChannel(
    // [输出] 命中结果 —— 函数返回后，这个结构里包含了命中点、法线、命中的Actor等所有信息
    FHitResult& OutHit,

    // [输入] 射线起点 —— 世界坐标
    const FVector& Start,

    // [输入] 射线终点 —— 世界坐标
    const FVector& End,

    // [输入] 碰撞通道 —— 使用哪个通道进行检测
    //       如：ECC_Visibility（可见性检测）、ECC_Camera（摄像机检测）
    ECollisionChannel TraceChannel,

    // [输入] 碰撞查询参数 —— 包含更多控制选项（忽略的Actor、是否复杂碰撞等）
    const FCollisionQueryParams& Params = FCollisionQueryParams::DefaultQueryParam,

    // [输入] 碰撞响应参数 —— 高级控制（一般使用默认值）
    const FCollisionResponseParams& ResponseParam = FCollisionResponseParams::DefaultResponseParam
);

// 返回值：bool
// true  → 射线碰到了某物（且被阻挡了），OutHit 包含详细信息
// false → 射线没有碰到任何东西（或只碰到了设为Overlap/Ingore的物体），OutHit 无效
```

### 基础使用示例

```cpp
// ==================== LineTraceSingleByChannel 基础使用 ====================

#include "Engine/World.h"          // 包含 UWorld 类（射线检测的来源）
#include "DrawDebugHelpers.h"      // 包含 DrawDebugLine 等调试绘制函数
#include "Engine/EngineTypes.h"   // 包含 FHitResult, FCollisionQueryParams 等

void AMyCharacter::PerformLineTrace()
{
    // ===== 第1步：定义射线的起点和终点 =====

    // 方法A：从Actor位置向前方发射
    FVector Start = GetActorLocation();                          // 本Actor的世界位置
    FVector ForwardVector = GetActorForwardVector();             // 本Actor的正前方方向向量（单位向量）
    float TraceDistance = 5000.0f;                               // 检测距离5000cm = 50米
    FVector End = Start + (ForwardVector * TraceDistance);      // 终点 = 起点 + 前方向 × 距离

    // 方法B：从摄像机位置向屏幕中心发射（第一人称/第三人称通用）
    // FVector Start = CameraComponent->GetComponentLocation();
    // FVector End = Start + (CameraComponent->GetForwardVector() * TraceDistance);

    // 方法C：从鼠标/准星目标位置发射
    // FVector WorldLocation, WorldDirection;
    // PlayerController->DeprojectScreenPositionToWorld(
    //     ScreenX, ScreenY, WorldLocation, WorldDirection);
    // FVector Start = WorldLocation;
    // FVector End = Start + (WorldDirection * TraceDistance);


    // ===== 第2步：准备碰撞查询参数 =====
    FCollisionQueryParams QueryParams;
    QueryParams.bTraceComplex = false;   // 使用简单碰撞（性能好）—— false用简单碰撞，true用复杂三角面碰撞
    QueryParams.AddIgnoredActor(this);   // 忽略自己（射线不会打到自己）
    // 也可以忽略多个Actor：
    // QueryParams.AddIgnoredActors(IgnoredActorsArray);


    // ===== 第3步：执行射线检测 =====
    FHitResult HitResult;  // 准备一个空的结果容器
    bool bHit = GetWorld()->LineTraceSingleByChannel(
        HitResult,                  // 输出参数：命中结果写入这里
        Start,                      // 射线起点
        End,                        // 射线终点
        ECC_Visibility,             // 使用可见性通道进行检测
        QueryParams                 // 自定义查询参数
    );

    // ===== 第4步：处理检测结果 =====
    if (bHit)  // 如果命中了某物
    {
        // 获取命中的Actor
        AActor* HitActor = HitResult.GetActor();
        if (HitActor)
        {
            // 打印命中的Actor名称
            UE_LOG(LogTemp, Warning, TEXT("射线命中了: %s"), *HitActor->GetName());
        }

        // 获取命中点的世界坐标
        FVector HitLocation = HitResult.Location;
        // 获取命中点的法线方向（表面朝向）
        FVector HitNormal = HitResult.ImpactNormal;
        // 获取命中点到起点的距离
        float HitDistance = HitResult.Distance;
        // 获取射线命中位置相对于物体原点的偏移
        FVector ImpactPoint = HitResult.ImpactPoint;

        UE_LOG(LogTemp, Log,
            TEXT("命中位置: %s, 表面法线: %s, 距离: %.1f cm"),
            *HitLocation.ToString(),
            *HitNormal.ToString(),
            HitDistance
        );
    }
    else  // 没有命中任何东西
    {
        UE_LOG(LogTemp, Log, TEXT("射线没有命中任何物体"));
    }


    // ===== 第5步：调试可视化（开发阶段使用）=====
    // 用可视化线条展示射线轨迹 —— 只在编辑器中可见，打包后不显示
    DrawDebugLine(
        GetWorld(),           // 世界指针
        Start,                // 线起点
        End,                  // 线终点
        FColor::Red,          // 线的颜色（红色）
        false,                // 是否永久显示（false = 只显示一段时间后消失）
        2.0f,                 // 持续时间（秒）
        0,                    // 深度优先级（0 = 默认）
        2.0f                  // 线粗（2.0像素）
    );

    // 如果命中了，在命中点画一个球
    if (bHit)
    {
        DrawDebugSphere(
            GetWorld(),
            HitResult.Location, // 球心位置（命中点）
            10.0f,              // 球半径（10cm）
            12,                 // 分段数（越多越圆，12是性能和美观的平衡点）
            FColor::Green,      // 绿色
            false,              // 不永久显示
            2.0f                // 持续2秒
        );
    }
}
```

### 实战示例：武器的"弹道检测"

```cpp
// ==================== 实战：第一人称射击的子弹射线 ====================

void AMyWeapon::FireWeapon()
{
    // ---- 安全前置检查 ----
    // 确保必要的组件存在
    if (!GetWorld() || !OwningCharacter)
    {
        return;
    }

    // ---- 获取射击起点和方向 ----
    // 起点：枪口位置（从武器网格的Socket获取）
    FVector MuzzleLocation = WeaponMesh->GetSocketLocation(FName(TEXT("MuzzleFlash")));
    // Socket是网格上的一个"插槽/挂点"，设计师在编辑器中预先定义好的位置

    // 终点：从摄像机向准星方向射出很远的距离
    // 用摄像机方向而不是枪口方向，保证"指哪打哪"
    FVector CameraLocation;
    FRotator CameraRotation;
    OwningCharacter->GetController()->GetPlayerViewPoint(CameraLocation, CameraRotation);
    FVector TraceEnd = CameraLocation + (CameraRotation.Vector() * 100000.0f);
    // 100000cm = 1000m = 1km（实际上远远超出可视范围，确保射线足够长）


    // ---- 配置射线参数 ----
    FCollisionQueryParams Params;
    Params.bTraceComplex = false;          // 用简单碰撞，速度快
    Params.AddIgnoredActor(this);          // 忽略武器自己
    Params.AddIgnoredActor(OwningCharacter); // 忽略射击者自己
    // 如果不忽略自己，近距离射击可能会打到自己

    // 声明变量来接收命中结果时的组件
    Params.bReturnPhysicalMaterial = true; // 返回物理材质（用于判断击中表面的类型）
    // 物理材质可以用来：播放不同音效（击中木头vs金属）、产生不同粒子效果


    // ---- 执行射线检测 ----
    FHitResult Hit;
    bool bBlockingHit = GetWorld()->LineTraceSingleByChannel(
        Hit,
        MuzzleLocation,     // 从枪口开始
        TraceEnd,           // 到远处结束
        ECC_Visibility,     // 可见性通道（也可以使用自定义的"武器"通道）
        Params
    );

    // ---- 处理命中结果 ----
    if (bBlockingHit)
    {
        // 命中了某个物体
        AActor* HitActor = Hit.GetActor();
        if (HitActor)
        {
            // 判断是否击中了角色（敌人）
            ACharacter* HitCharacter = Cast<ACharacter>(HitActor);
            if (HitCharacter)
            {
                // 命中敌人 → 应用伤害
                UE_LOG(LogTemp, Warning, TEXT("击中敌人: %s"), *HitCharacter->GetName());
                // 实际项目中在这里调用伤害系统
                // ApplyDamage(HitCharacter, WeaponDamage);
            }
            else
            {
                // 命中环境物体 → 生成弹孔特效
                UE_LOG(LogTemp, Log, TEXT("击中环境: %s"), *HitActor->GetName());
                // SpawnImpactEffect(Hit.Location, Hit.ImpactNormal);
            }
        }
    }

    // ---- 调试可视化（仅在开发阶段启用）----
    // 画出红色射线，显示子弹轨迹
    DrawDebugLine(GetWorld(), MuzzleLocation,
        bBlockingHit ? Hit.Location : TraceEnd,  // 命中则画到命中点，否则画到终点
        FColor::Red, false, 3.0f, 0, 2.0f);
    // 参数解读：红色、存在3秒、默认深度、2像素宽
}
```

## LineTraceSingleByObjectType

`LineTraceSingleByObjectType` 不依赖碰撞通道的响应规则，而是直接指定"我只关心这些类型的物体"。

### 何时使用 ByObjectType？

- **场景1**：你想检测所有的"可拾取物品"，不管它们对可见性通道设置了什么响应
- **场景2**：你想在可见性检测中排除某些特殊类型（如只检测WorldStatic和WorldDynamic，忽略Pawn）
- **场景3**：碰撞通道响应设置很复杂，你想直接指定目标类型简化代码

### 完整使用示例

```cpp
// ==================== LineTraceSingleByObjectType 使用示例 ====================

void AMyActor::LineTraceByObjectType()
{
    // ---- 射线的起点和终点 ----
    FVector Start = GetActorLocation();
    FVector End = Start + (GetActorForwardVector() * 3000.0f);

    // ---- 关键参数：ObjectTypes数组 ----
    // 这个数组定义了"我要检测哪些类型的物体"
    // 只有碰撞通道属于这些类型的物体才会被命中
    TArray<TEnumAsByte<EObjectTypeQuery>> ObjectTypes;
    // 添加想检测的类型：
    ObjectTypes.Add(UEngineTypes::ConvertToObjectType(ECC_WorldStatic));   // 检测静态世界物体
    ObjectTypes.Add(UEngineTypes::ConvertToObjectType(ECC_WorldDynamic));  // 检测动态世界物体
    // 注意：ECC_Pawn 没有加入数组，所以角色会被忽略
    // 这意味着即使射线穿过了角色，也不会命中

    // 也可以用快捷写法：
    // ObjectTypes.Add(EObjectTypeQuery::ObjectTypeQuery1);  // WorldStatic
    // ObjectTypes.Add(EObjectTypeQuery::ObjectTypeQuery2);  // WorldDynamic


    // ---- 碰撞查询参数 ----
    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(this);
    QueryParams.bTraceComplex = false;

    // ---- 执行检测 ----
    FHitResult HitResult;
    bool bHit = GetWorld()->LineTraceSingleByObjectType(
        HitResult,          // 输出：命中结果
        Start,              // 起点
        End,                // 终点
        ObjectTypes,        // 要检测的物体类型数组
        QueryParams         // 查询参数
    );

    // ---- 处理结果 ----
    if (bHit)
    {
        UE_LOG(LogTemp, Warning, TEXT("命中了 %s"),
            *HitResult.GetActor()->GetName());
    }
}
```

## LineTraceSingleByProfile

`LineTraceSingleByProfile` 使用碰撞预设（Profile）名称来过滤，行为与 `LineTraceSingleByChannel` 类似，但使用的是预设中定义的整套响应规则。

### 何时使用 ByProfile？

- **场景1**：你已经在Project Settings中定义了专用的射线检测预设（如"WeaponTrace"预设）
- **场景2**：你想复用与某个碰撞预设完全一致的检测规则
- **场景3**：团队的碰撞策略已通过预设管理，你不想在C++中重复定义规则

### 使用示例

```cpp
// ==================== LineTraceSingleByProfile 使用示例 ====================

void AMyActor::LineTraceByProfile()
{
    FVector Start = GetActorLocation();
    FVector End = Start + (GetActorForwardVector() * 5000.0f);

    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(this);

    FHitResult HitResult;

    // 使用名为 "WeaponTrace" 的碰撞预设
    // 这个预设中定义的碰撞响应规则将被用于这次射线检测
    bool bHit = GetWorld()->LineTraceSingleByProfile(
        HitResult,
        Start,
        End,
        FName(TEXT("WeaponTrace")),  // 碰撞预设的名称（在Project Settings中定义的）
        QueryParams
    );
    // 注意：如果这个预设不存在，检测会失败（bHit = false）

    if (bHit)
    {
        UE_LOG(LogTemp, Warning, TEXT("WeaponTrace预设射线命中了: %s"),
            *HitResult.GetActor()->GetName());
    }
}
```

## LineTraceMulti（穿透检测）

`LineTraceMulti...` 系列API会将射线**穿透**所有（有Block响应）的物体，返回所有命中结果，按距离从近到远排序。

### 典型使用场景

- **穿透子弹**：子弹穿过多个敌人，每个都造成伤害
- **X射线透视**：看到墙壁后面的物体
- **爆破路径检测**：判断爆炸冲击波路径上的所有障碍物
- **技能链**：AOE技能命中路径上的所有目标

### LineTraceMultiByChannel使用示例

```cpp
// ==================== LineTraceMultiByChannel 使用示例 ====================

void AMyCharacter::PenetratingShot()
{
    // ---- 定义射线 ----
    FVector Start = GetActorLocation();
    FVector End = Start + (GetActorForwardVector() * 8000.0f);

    // ---- 配置查询参数 ----
    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(this);  // 忽略自己
    QueryParams.bTraceComplex = false;  // 用简单碰撞


    // ---- 关键区别：TArray<FHitResult> 而不是单个 FHitResult ----
    // Multi版本返回多个命中结果
    TArray<FHitResult> HitResults;  // 数组，存储所有命中结果
    // 每个元素包含：命中的位置、命中的Actor、表面法线等信息

    bool bHit = GetWorld()->LineTraceMultiByChannel(
        HitResults,           // 输出：所有命中结果的数组
        Start,                // 射线起点
        End,                  // 射线终点
        ECC_Visibility,       // 碰撞通道
        QueryParams           // 查询参数
    );
    // 注意：返回值 bHit = true 表示至少命中了一个物体


    if (bHit)
    {
        // ---- 遍历所有命中结果 ----
        // 结果按距离从近到远排序
        UE_LOG(LogTemp, Warning, TEXT("射线穿透命中了 %d 个物体"), HitResults.Num());

        for (int32 i = 0; i < HitResults.Num(); ++i)
        {
            const FHitResult& Hit = HitResults[i];

            // 调试输出：第i个命中物体的信息
            UE_LOG(LogTemp, Log,
                TEXT("  命中[%d]: %s, 距离=%.1f cm"),
                i,                          // 命中序号
                *Hit.GetActor()->GetName(), // 命中Actor名称
                Hit.Distance                // 距起点的距离
            );

            // 在命中点画绿色小球（验证位置）
            DrawDebugSphere(
                GetWorld(),
                Hit.Location,    // 命中点世界坐标
                5.0f,            // 小球半径5cm
                8,               // 分段数
                FColor::Green,   // 绿色
                false,           // 不永久
                5.0f             // 显示5秒
            );
        }
    }
    else
    {
        UE_LOG(LogTemp, Log, TEXT("射线没有命中任何物体"));
    }


    // ---- 调试可视化：画出整条射线 ----
    DrawDebugLine(GetWorld(), Start, End, FColor::Cyan, false, 5.0f, 0, 2.0f);
    // 青色、显示5秒、2像素宽
}
```

### 穿透伤害示例

```cpp
// ==================== 实战：穿透子弹的伤害处理 ====================

void AMyCharacter::FirePenetratingBullet(float BaseDamage, int32 MaxPenetrations)
{
    FVector Start = GetActorLocation();
    FVector End = Start + (GetActorForwardVector() * 10000.0f);  // 100米检测距离

    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(this);
    QueryParams.bTraceComplex = false;  // 简单碰撞，性能好

    TArray<FHitResult> HitResults;
    GetWorld()->LineTraceMultiByChannel(
        HitResults, Start, End, ECC_Visibility, QueryParams);

    int32 PenetrationsRemaining = MaxPenetrations;  // 剩余穿透次数

    for (const FHitResult& Hit : HitResults)
    {
        // 每次穿透，伤害减半（距离衰减用更复杂的公式也可以）
        float PenetrationDamage = BaseDamage * FMath::Pow(0.5f, MaxPenetrations - PenetrationsRemaining);
        // FMath::Pow(0.5f, n) → 第1个目标100%伤害，第2个50%，第3个25%

        // 对命中的Actor造成伤害
        AActor* HitActor = Hit.GetActor();
        if (HitActor)
        {
            // 应用伤害（在这里调用你的伤害系统）
            // ApplyDamage(HitActor, PenetrationDamage);

            UE_LOG(LogTemp, Log,
                TEXT("穿透命中 %s，造成 %.1f 伤害（穿透次数剩余: %d）"),
                *HitActor->GetName(),
                PenetrationDamage,
                PenetrationsRemaining
            );
        }

        // 消耗一次穿透次数
        PenetrationsRemaining--;
        if (PenetrationsRemaining <= 0)
        {
            // 穿透次数用完，射线被"阻挡"
            UE_LOG(LogTemp, Warning, TEXT("穿透次数用完，子弹停止"));
            break;  // 跳出循环，不再处理后续命中
        }
    }
}
```

```cpp
    FVector End = Start + (GetActorForwardVector() * 1000.0f);

    FCollisionQueryParams SimpleParams;
    SimpleParams.bTraceComplex = false;  // 简单碰撞

    FCollisionQueryParams ComplexParams;

    FHitResult SimpleHit, ComplexHit;

    // 两次检测，用不同精度
    bool bSimpleHit = GetWorld()->LineTraceSingleByChannel(
        SimpleHit, Start, End, ECC_Visibility, SimpleParams);

    bool bComplexHit = GetWorld()->LineTraceSingleByChannel(
        ComplexHit, Start, End, ECC_Visibility, ComplexParams);

    // 可能的结果差异：
    // 简单碰撞：命中点在石头的大致边界盒子上
    // 复杂碰撞：命中点在石头实际表面的精确三角面上
    // 两者的命中位置可能相差几厘米到几十厘米

    if (bSimpleHit && bComplexHit)
    {
        float LocationDiff = FVector::Dist(SimpleHit.Location, ComplexHit.Location);
        UE_LOG(LogTemp, Log, TEXT("简单碰撞与复杂碰撞的命中点偏差: %.2f cm"), LocationDiff);
        // 对于规则形状（如方块），偏差很小
        // 对于不规则形状（如岩石），偏差可能很大
    }
}
```

## FHitResult结构详解

`FHitResult` 是射线检测和扫描检测的核心返回结构。理解它的每个字段是掌握碰撞查询的关键。

### 完整结构定义与逐字段解释

```cpp
// ==================== FHitResult 结构体详解 ====================
// 定义在：Engine/Source/Runtime/Engine/Public/Engine/EngineTypes.h

struct FHitResult
{
    // ===== 基本信息 =====

    // [FVector] 命中点的世界坐标
    // 用途：生成粒子特效、播放音效、放置贴花（如弹孔）
    FVector Location;

    // [FVector] 命中点的世界坐标（通常等于Location，但保留为独立变量）
    // 注意：在大多数情况 ImpactPoint == Location
    FVector ImpactPoint;

    // [FVector] 命中表面的法线方向（单位向量，指向碰撞表面的"外面"）
    // 用途：反弹计算（反射方向 = 入射方向 - 2*(入射方向·法线)*法线）
    // 用途：生成与表面对齐的粒子效果
    FVector ImpactNormal;

    // [FVector] 命中表面的法线（更精确的版本，可以考虑三角面的插值）
    // 在简单碰撞中通常等于 ImpactNormal
    FVector Normal;

    // ===== 命中对象信息 =====

    // [weak_ptr] 被命中的物理Body索引对应的Body
    // 多数情况下不需要关心，用于复杂物理模拟场景
    TWeakObjectPtr<UPrimitiveComponent> Component;

    // [weak_ptr] 被命中的Actor的弱指针
    // 用途：Cast后访问Actor的属性和方法
    // 注意：是弱指针，使用前需要检查是否有效
    TWeakObjectPtr<AActor> Actor;

    // ===== 面和材质信息 =====

    // [int32] 被命中的三角面索引（仅在 bTraceComplex=true 且 bReturnFaceIndex=true 时有效）
    int32 FaceIndex;

    // [FName] 命中的骨骼名称（对骨骼网格体有效）
    // 用途：击中角色的"头部"还是"手臂"，针对不同部位做不同伤害
    FName BoneName;

    // [FName] 命中的自定义碰撞体名称（如果使用了自定义简单碰撞体）
    FName MyBoneName;

    // ===== 物理材质 =====

    // [weak_ptr] 命中的物理材质
    // 用途：判断命中表面是金属/木头/水/石头等
    // 基于此选择不同的音效、粒子和贴花
    TWeakObjectPtr<UPhysicalMaterial> PhysMaterial;

    // ===== 距离和方向信息 =====

    // [float] 命中点到射线起点的距离（厘米）
    // 用途：判断哪个命中更近、计算弹道距离衰减
    float Distance;

    // [float] 从射线起点到命中点的"时间"参数（0到1之间，表示射线长度的比例）
    // HitTime = Distance / (End - Start).Length()
    // 用途：在起点和终点之间插值
    float Time;

    // ===== 射线特定信息 =====

    // [FVector] 命中点相对于射线起点的方向（通常可以自己算，这是引擎帮你算好的）
    // 注意：这是被扫描几何体的表面点，不一定等于 Location
    FVector TraceStart;

    // [FVector] 同 TraceStart，但通常是射线终点位置
    FVector TraceEnd;

    // ===== 穿透信息 =====

    // [int32] 被命中的物体相对于扫描方向的"面"
    // 0 = 正面（射线从前面进入），1 = 背面（射线从背面穿入）
    // 对球体扫描等形状扫描特别有用
    int32 Item;

    // ===== 物理模拟特定信息 =====

    // [FVector] 物理推动的方向（用于物理推动计算）
    FVector PhysicsNormal;

    // ===== 状态标志 =====

    // [bool] 是否发生了阻挡（Block）碰撞
    // 这是最重要的标志位！true = 射线被阻挡了
    bool bBlockingHit;

    // [bool] 射线起点是否在碰撞体内
    // true = 射线起点已经在一个物体内部（常见于修正射线起点以防穿透）
    bool bStartPenetrating;
};
```

```cpp
// ==================== 实战：完整的弹孔/贴花生成 ====================

void AMyWeapon::SpawnBulletHole(const FHitResult& Hit)
{
    // ---- 安全检查 ----
    if (!Hit.IsValidBlockingHit() || !Hit.GetComponent())
    {
        return;  // 没有有效命中，什么也不做
    }

    // ---- 获取命中位置 ----
    FVector HoleLocation = Hit.Location;

    // ---- 获取表面法线，稍微偏移以防Z-fighting（表面闪烁）----
    FVector HoleNormal = Hit.ImpactNormal;
    // 沿法线方向偏移0.5cm，这样贴花刚好浮在表面上方
    // 防止贴花和原始表面在同一深度导致的闪烁（Z-fighting）
    HoleLocation += HoleNormal * 0.5f;

    // 使贴花平面垂直于表面法线
    FRotator HoleRotation = HoleNormal.Rotation();
    // FVector::Rotation() 返回将X轴旋转到该方向的旋转角度


    // ---- 根据物理材质选择贴花 ----
    UMaterialInterface* DecalMaterial = nullptr;  // 贴花材质（弹孔贴图）

    if (Hit.PhysMaterial.IsValid())  // 有物理材质信息

        UPhysicalMaterial* PhysMat = Hit.PhysMaterial.Get();

        else if (MatName.ToString().Contains(TEXT("Wood")))

    }
    else
    {
        // 没有物理材质信息，使用默认弹孔
        DecalMaterial = DefaultHitDecal;
    }

    // ---- 生成贴花（弹孔）----
    if (DecalMaterial)

        // 贴花大小
```

## 形状扫描（SphereTrace / BoxTrace / CapsuleTrace）

形状扫描是"射线检测的升级版"——不再是一条无限细的线，而是一个有体积的形状沿着路径扫过。

### 三种形状扫描对比

| 扫描类型       | API函数                 | 形状                            | 适用场景                                     |
| -------------- | ----------------------- | ------------------------------- | -------------------------------------------- |
| **球体扫描**   | `SphereTraceSingle...`  | 半径为R的球体                   | 爆炸范围检测、角色移动碰撞检测、近战攻击范围 |
| **盒子扫描**   | `BoxTraceSingle...`     | 半长为Extent的盒子              | 门框检测、大范围AOE技能、方形区域检测        |
| **胶囊体扫描** | `CapsuleTraceSingle...` | 半径R+半高H的胶囊体             | 角色移动的精确碰撞检测                       |
| **通用扫描**   | `SweepSingle...`        | 任意形状（传入FCollisionShape） | 需要自定义形状尺寸时                         |

### SphereTraceSingleByChannel完整示例

```cpp
// ==================== 球体扫描 ====================

void AMyCharacter::PerformSphereTrace()
{
    // ---- 定义扫描路径 ----
    FVector Start = GetActorLocation();                      // 从角色位置
    FVector End = Start + (GetActorForwardVector() * 500.0f); // 向前500cm

    float SphereRadius = 100.0f;  // 球体半径100cm = 1米
    // 这意味着检测一个直径2米的大球沿着射线轨迹扫描
    // 任何与这个球相交的物体都会被检测到


    // ---- 配置参数 ----
    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(this);
    QueryParams.bTraceComplex = false;

    // ---- 执行球体扫描 ----
    FHitResult HitResult;
    bool bHit = GetWorld()->SweepSingleByChannel(
        HitResult,                  // 输出：命中结果（最近的一个阻挡命中）
        Start,                      // 扫描起点
        End,                        // 扫描终点
        FQuat::Identity,            // 球体的旋转（球体旋转无所谓，盒子和胶囊体才重要）
        ECC_Visibility,             // 碰撞通道
        FCollisionShape::MakeSphere(SphereRadius),  // 关键参数：定义扫描形状为半径100cm的球体
        QueryParams                 // 查询参数
    );

    if (bHit)
    {
        UE_LOG(LogTemp, Warning, TEXT("球体扫描命中: %s"), *HitResult.GetActor()->GetName());
    }

    // ---- 调试可视化 ----
    // 在起点画球
    DrawDebugSphere(GetWorld(), Start, SphereRadius, 16, FColor::Green, false, 3.0f);
    // 在终点画球
    DrawDebugSphere(GetWorld(), End, SphereRadius, 16, FColor::Blue, false, 3.0f);
    // 从起点到终点画线
    DrawDebugLine(GetWorld(), Start, End, FColor::Yellow, false, 3.0f, 0, 1.0f);
    // 如果命中，在命中点画红色球
    if (bHit)
    {
        DrawDebugSphere(GetWorld(), HitResult.Location, SphereRadius, 16, FColor::Red, false, 3.0f);
```

## SweepMultiByChannel

`SweepMultiByChannel` 结合了"形状扫描"和"穿透检测"两种能力——用一个有体积的形状扫描路径，并返回所有命中结果。

### 使用场景

- **爆炸范围检测**：一个球体扫过爆炸范围，检测所有受影响的物体
- **冲刺技能**：一个胶囊体沿冲刺方向扫过，检测路径上所有敌人
- **大范围AOE**：一个盒子扫过技能区域，命中所有目标

### 完整示例

```cpp
// ==================== SweepMultiByChannel 完整示例 ====================

void AMyCharacter::PerformSphereSweepMulti()
{
    FVector Start = GetActorLocation();
    FVector End = Start + (GetActorForwardVector() * 1000.0f);  // 向前10米

    float SweepRadius = 150.0f;  // 半径150cm = 1.5米（一个较大的检测范围）

    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(this);
    QueryParams.bTraceComplex = false;

    // ---- 关键：TArray<FHitResult> 接收多个结果 ----
    TArray<FHitResult> HitResults;

    bool bHit = GetWorld()->SweepMultiByChannel(
        HitResults,                              // 输出：所有命中结果
        Start,                                   // 扫描起点
        End,                                     // 扫描终点
        FQuat::Identity,                         // 球体旋转（球体无所谓）
        ECC_Pawn,                                // 只检测Pawn通道（角色）
        FCollisionShape::MakeSphere(SweepRadius), // 球体形状
        QueryParams                              // 查询参数
    );

    if (bHit)
    {
        UE_LOG(LogTemp, Warning, TEXT("球体扫描命中了 %d 个目标"), HitResults.Num());

        for (const FHitResult& Hit : HitResults)
        {
            AActor* HitActor = Hit.GetActor();
            if (HitActor)
            {
                UE_LOG(LogTemp, Log,
                    TEXT("  命中: %s, 距离: %.1f cm"),
                    *HitActor->GetName(),
                    Hit.Distance
                );
            }
        }
    }

    // ---- 调试可视化 ----
    DrawDebugSphere(GetWorld(), Start, SweepRadius, 16,
        FColor::Green, false, 3.0f);
    DrawDebugSphere(GetWorld(), End, SweepRadius, 16,
        FColor::Blue, false, 3.0f);
    DrawDebugLine(GetWorld(), Start, End,
        FColor::Yellow, false, 3.0f, 0, 1.0f);

    for (const FHitResult& Hit : HitResults)
    {
        DrawDebugSphere(GetWorld(), Hit.Location, 20.0f, 8,
            FColor::Red, false, 3.0f);
    }
}
```

### 实战：技能面前的锥形AOE检测

```cpp
// ==================== 实战：锥形范围AOE技能 ====================

void AMyCharacter::ConeAOESkill(float ConeAngle, float Range, float Damage)
{
    // ---- 获取扫描起点（角色位置，稍微抬高） ----
    FVector Start = GetActorLocation() + FVector(0, 0, 50.0f);
    FVector Forward = GetActorForwardVector();
    FVector End = Start + (Forward * Range);  // 终点

    // ---- 盒子扫描（大范围） ----
    // 使用一个宽大的盒子，模拟锥形范围
    FVector BoxExtent(Range * 0.5f,            // X方向半长（向前）
                      Range * 0.3f,            // Y方向半长（横向范围）
                      100.0f);                 // Z方向半长（高度）

    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(this);
    QueryParams.bTraceComplex = false;

    TArray<FHitResult> HitResults;
    bool bHit = GetWorld()->SweepMultiByChannel(
        HitResults,
        Start,
        End,
        GetActorRotation().Quaternion(),     // 用角色朝向
        ECC_Pawn,                            // 只检测角色
        FCollisionShape::MakeBox(BoxExtent),
        QueryParams
    );

    // ---- 处理命中的目标 ----
    for (const FHitResult& Hit : HitResults)
    {
        AActor* HitActor = Hit.GetActor();
        if (!HitActor) continue;

        // 计算距离衰减
        float Distance = FVector::Dist(Start, Hit.Location);
        float DistanceRatio = FMath::Clamp(1.0f - (Distance / Range), 0.0f, 1.0f);
        // 近距离满伤害，远距离衰减（在Range处衰减为0）

        float ActualDamage = Damage * DistanceRatio;

        // 检查是否在锥形角度内
        FVector DirectionToTarget = (Hit.Location - Start).GetSafeNormal();
        float DotProduct = FVector::DotProduct(Forward, DirectionToTarget);
        // DotProduct = cos(两个向量之间的夹角)
        // cos(0°) = 1.0, cos(45°) ≈ 0.707, cos(90°) = 0.0

        float AngleInRadians = FMath::Acos(DotProduct);     // 转换为弧度
        float AngleInDegrees = FMath::RadiansToDegrees(AngleInRadians);  // 转换为角度

        if (AngleInDegrees <= ConeAngle * 0.5f)  // 在锥形角度内
        {
            UE_LOG(LogTemp, Log,
                TEXT("AOE命中 %s, 角度=%.1f°, 命中角度=%.1f°, 伤害=%.1f"),
                *HitActor->GetName(),
                ConeAngle * 0.5f,
                AngleInDegrees,
                ActualDamage
            );
            // 实际项目中：ApplyDamage(HitActor, ActualDamage);
        }
    }

    // ---- 调试可视化：画出锥形范围 ----
    DrawDebugBox(GetWorld(), Start, BoxExtent,
        GetActorRotation().Quaternion(), FColor::Purple, false, 2.0f);
    DrawDebugBox(GetWorld(), End, BoxExtent,
        GetActorRotation().Quaternion(), FColor::Cyan, false, 2.0f);
}
```

## 性能注意事项

射线检测和形状扫描虽然效率较高，但在大量使用时仍需注意性能。以下是在实际项目中的性能最佳实践。

### 性能对比速查表

| API                   | 每帧调用100次的成本 | 每帧调用1000次的成本 | 建议         |
| --------------------- | :-----------------: | :------------------: | ------------ |
| `LineTraceSingle`     |       ✅ 极低       |        ✅ 低         | 可放心使用   |
| `LineTraceMulti`      |        ✅ 低        |       ⚠️ 中等        | 限制命中数量 |
| `SphereTraceSingle`   |        ✅ 低        |       ⚠️ 中等        | 节制使用     |
| `SweepMultiByChannel` |       ⚠️ 中等       |        ❌ 高         | 减少调用频次 |

### 最佳实践

```cpp
// ==================== 性能优化最佳实践 ====================

void PerformanceBestPractices()
{
    // ===== 优化1：使用简单碰撞（bTraceComplex = false）=====
    // ✅ 正确
    FCollisionQueryParams Params;
    Params.bTraceComplex = false;

    // ❌ 错误：不必要的复杂碰撞
    // Params.bTraceComplex = true;  // 只在需要精确命中点时使用


    // ===== 优化2：尽量使用LineTrace而不是ShapeTrace =====
    // ✅ 正确：能用细射线就不用形状扫描
    // GetWorld()->LineTraceSingleByChannel(...);  // 计算量：检测线与碰撞体的交点

    // ⚠️ 有节制：确实需要体积扫描时才用
    // GetWorld()->SweepSingleByChannel(...);  // 计算量：检测形状扫掠体与碰撞体的交点


    // ===== 优化3：限制Multi的命中数量（如果可以）=====
    // Multi会穿透所有物体，如果场景中有大量物体，开销会很大
    // 可以考虑使用异步射线检测（Async LineTrace）做非关键路径的检测


    // ===== 优化4：缓存射线检测结果 =====
    // 如果同一个射线检测每帧都返回相同结果（如地面检测），
    // 可以每N帧检测一次，而不是每帧检测

    static float LastGroundCheckTime = 0.0f;
    static float GroundCheckInterval = 0.1f;  // 每0.1秒检测一次（10Hz）
    // 而不是每帧60次检测


    // ===== 优化5：使用碰撞通道过滤 =====
    // ✅ 正确：只检测需要的通道
    // 使用 ECC_Pawn 比使用 ECC_Visibility 过滤掉更多无关碰撞体
    // 因为WorldStatic物体通常对Visibility通道有Block响应


    // ===== 优化6：异步射线检测 =====
    // 对于不紧急的查询，使用异步版本
    // 异步检测在后台线程执行，不会阻塞游戏主线程
    // 适用于AI感知、不太紧急的环境查询等
    //
    // 注意：异步检测API在UE5中变化较大，请查阅当前版本文档
    // FTraceHandle TraceHandle = GetWorld()->AsyncLineTraceByChannel(
    //     EAsyncTraceType::Single,
    //     Start, End, ECC_Visibility, Params,
    //     FTraceDelegate::CreateUObject(this, &AMyActor::OnAsyncTraceComplete)
    // );
}
```
