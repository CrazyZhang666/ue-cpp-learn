## 什么是物理模拟？

物理模拟（Physics Simulation）让引擎的物理系统（Chaos Physics）接管物体的运动——不再需要你手动在Tick中计算位置和旋转，引擎会自动根据重力、碰撞、受力计算出物体的运动轨迹。

### 物理模拟 vs 手动移动

```cpp
// ==================== 对比：手动移动 vs 物理模拟 ====================

// ---- 手动移动（你在Tick中实现）----
// ✅ 优点：完全可控，每个细节都可以定制
// ❌ 缺点：需要手写碰撞检测、重力、摩擦力等逻辑
// 适用场景：玩家角色移动、特殊平台运动

void TickManualMovement(float DeltaTime)
{
    // 手动计算位置：距离 = 速度 × 时间
    FVector NewLocation = GetActorLocation() + Velocity * DeltaTime;
    // 手动应用重力：向下加速
    Velocity.Z -= 980.0f * DeltaTime;  // 980 cm/s² 重力加速度
    SetActorLocation(NewLocation);     // 手动设置位置
}
// 问题：上面的代码完全没有碰撞检测！物体会直接穿过墙壁！


// ---- 物理模拟（引擎接管）----
// ✅ 优点：自动处理碰撞、重力、摩擦力、弹力
// ❌ 缺点：不太可控（但可以通过约束和力来引导）
// 适用场景：弹丸、碎片、可拾取物、载具、布娃娃

void EnablePhysicsSimulation()
{
    MyMesh->SetSimulatePhysics(true);
    // 之后引擎自动处理：
    // 1. 每帧根据受力计算加速度
    // 2. 检测碰撞并处理反弹
    // 3. 处理摩擦力、空气阻力
    // 4. 让物体自动"睡觉"（Sleep）以优化性能
}
```

### 物理模拟的典型应用场景

```
弹丸/子弹     → 发射后受重力影响飞行，碰到墙壁反弹或停止
破碎碎片     → 爆炸后碎片四处飞散，撞击地面弹跳
可拾取物品   → 丢在地上后自然落下，堆叠在桌面上
布娃娃       → 角色死亡后身体自然倒地（骨骼物理）
载具         → 车轮与地面的交互，悬挂系统
布料/绳索   → 旗帜飘动、绳索摆动
```


## 开启物理模拟：SetSimulatePhysics

### 基本开启和关闭

```cpp
// ==================== 物理模拟的开启与关闭 ====================

#include "Components/StaticMeshComponent.h"
#include "Components/SkeletalMeshComponent.h"

void AMyActor::ControlPhysicsSimulation()
{
    // ---- 开启物理模拟 ----
    // 语法：Component->SetSimulatePhysics(true)
    // 效果：该组件开始受物理引擎控制
    // 前提：该组件需要有碰撞体
    // 前提：碰撞启用状态必须包含 Physics（如 QueryAndPhysics 或 PhysicsOnly）

    UStaticMeshComponent* Mesh = FindComponentByClass<UStaticMeshComponent>();
    if (Mesh)
    {
        // ✅ 正确：先确保碰撞包含物理，再开启物理模拟
        Mesh->SetCollisionEnabled(ECollisionEnabled::QueryAndPhysics);  // 必须包含Physical
        Mesh->SetSimulatePhysics(true);  // 现在可以安全开启物理模拟

        // 开启后，这个Mesh会：
        // 1. 受重力影响开始下落
        // 2. 碰到地面会弹开（根据物理材质设置）
        // 3. 与其他物理物体碰撞时会互相推开
        // 4. 缓慢减速直到停止（受角阻尼和线阻尼影响）
    }


    // ---- 关闭物理模拟 ----
    // 语法：Component->SetSimulatePhysics(false)
    Mesh->SetSimulatePhysics(false);
    // 关闭后，Mesh回归静态（或受你手动控制）


    // ---- 检查是否正在物理模拟 ----
    bool bIsSimulating = Mesh->IsSimulatingPhysics();
    // 返回 true = 正在受物理引擎控制
    // 返回 false = 静止或手动控制

    if (bIsSimulating)
    {
        UE_LOG(LogTemp, Log, TEXT("组件正在物理模拟中"));
    }
}
```

### 完整示例：可破坏的木箱

```cpp
// ==================== 实战：可破坏木箱的物理设置 ====================

// BreakableCrate.h
#pragma once
#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "BreakableCrate.generated.h"

UCLASS()
class MYGAME_API ABreakableCrate : public AActor
{
    GENERATED_BODY()

public:
    ABreakableCrate();

protected:
    // 完整箱子时的网格体（正常状态）
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "Components")
    UStaticMeshComponent* IntactMesh;

    // 破坏后生成的碎片Actor类
    UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "Breakable")
    TSubclassOf<AActor> DebrisClass;
    // TSubclassOf 限制只能选择AActor或其子类

    // 被攻击时调用的函数（在蓝图中绑定或通过接口调用）
    UFUNCTION(BlueprintCallable, Category = "Breakable")
    void BreakCrate();

    // 破坏时施加给碎片的冲量大小
    UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "Breakable")
    float BreakImpulse = 500.0f;
};

// BreakableCrate.cpp
#include "BreakableCrate.h"
#include "Components/StaticMeshComponent.h"
#include "Engine/World.h"

ABreakableCrate::ABreakableCrate()
{
    PrimaryActorTick.bCanEverTick = false;  // 不需要Tick

    // 创建完整的箱子网格体
    IntactMesh = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("IntactMesh"));
    RootComponent = IntactMesh;

    // 配置碰撞：阻挡所有（像一个固体木箱）
    // ✅ box碰撞使用简单碰撞体——木箱通常是方形，简单盒子碰撞就够了
    IntactMesh->SetCollisionEnabled(ECollisionEnabled::QueryAndPhysics);
    IntactMesh->SetCollisionProfileName(FName(TEXT("BlockAll")));
    // "BlockAll" 预设 → 自身是WorldStatic，阻挡所有通道
}

void ABreakableCrate::BreakCrate()
{
    // ---- 第1步：隐藏完整的箱子 ----
    IntactMesh->SetHiddenInGame(true);          // 隐藏视觉
    IntactMesh->SetCollisionEnabled(ECollisionEnabled::NoCollision);  // 关闭碰撞
    // 关碰撞是必须的，否则碎片会与已经不存在的完整箱子碰撞


    // ---- 第2步：生成碎片Actor ----
    if (DebrisClass && GetWorld())
    {
        AActor* Debris = GetWorld()->SpawnActor<AActor>(
            DebrisClass,                          // 碎片Actor的类
            GetActorLocation(),                   // 在本箱子的位置生成
            GetActorRotation()                    // 保持相同朝向
        );

        if (Debris)
        {
            // ---- 第3步：对碎片施加爆炸力 ----
            // 找到碎片Actor中的所有StaticMesh组件，分别施加力
            TArray<UStaticMeshComponent*> DebrisPieces;
            // 获取碎片Actor中所有的UStaticMeshComponent
            Debris->GetComponents<UStaticMeshComponent>(DebrisPieces);

            for (UStaticMeshComponent* Piece : DebrisPieces)
            {
                if (Piece)
                {
                    // 3.1 开启物理模拟
                    Piece->SetSimulatePhysics(true);

                    // 3.2 施加随机方向的爆炸力
                    FVector RandomDirection = FMath::VRand();
                    // FMath::VRand() 返回一个随机的单位向量（均匀分布在球面上）
                    // 这样每个碎片都向不同方向飞出去

                    float RandomStrength = FMath::FRandRange(0.5f, 1.5f) * BreakImpulse;
                    // 随机力度：50%~150%的基础冲量

                    // 施加冲量（瞬间的力）
                    Piece->AddImpulse(RandomDirection * RandomStrength,
                        NAME_None,      // BoneName（在骨骼网格体中使用，静态网格体填NAME_None）
                        true            // bVelocityChange（true=直接修改速度） // 使用速度变化模式（不受质量影响）
                    );
                    // AddImpulse参数说明：
                    // 参数1：FVector Impulse  —— 冲量的大小和方向
                    // 参数2：FName BoneName     —— 骨骼名（静态网格体用NAME_None）
                    // 参数3：bool bVelChange    —— true = 直接作为速度变化，不受质量影响
                    //                              false = 作为力×时间的冲量，受质量影响（F=ma → 质量越大速度变化越小）

                    UE_LOG(LogTemp, Log,
                        TEXT("碎片 %s 受到 %s 方向的力 %f"),
                        *Piece->GetName(),
                        *RandomDirection.ToString(),
                        RandomStrength
                    );
                }
            }
        }
    }

    // ---- 第4步：延迟销毁完整箱子（可选）----
    // 给碎片一些时间来展示效果
    SetLifeSpan(0.5f);  // 0.5秒后自动销毁本Actor
}
```

## 施加力：AddForce / AddImpulse / AddTorque

### 力的类型对比

| API                  | 作用方式               | 持续时间 |     质量影响     | 适用场景                 |
| -------------------- | ---------------------- | :------: | :--------------: | ------------------------ |
| `AddForce`           | 持续施加力（每帧累计） |   持续   |  ✅ 受质量影响   | 火箭推进、风扇、风力     |
| `AddImpulse`         | 瞬间施加冲量           |   瞬间   | ✅/❌ 取决于参数 | 爆炸、碰撞、弹射         |
| `AddForceAtLocation` | 在特定位置施加力       |   持续   |  ✅ 受质量影响   | 推动物体边缘（产生旋转） |
| `AddTorque`          | 施加旋转力矩           |   持续   |  ✅ 受质量影响   | 旋转门、螺旋桨           |
| `AddRadialForce`     | 从中心点放射状施加力   |   持续   |  ✅ 受质量影响   | 爆炸冲击波               |
| `AddRadialImpulse`   | 从中心点放射状冲量     |   瞬间   |      ✅/❌       | 瞬间爆炸                 |

### AddForce详解

```cpp
// ==================== AddForce 详解 ====================

void AMyActor::ApplyVariousForces()
{
    UStaticMeshComponent* Mesh = FindComponentByClass<UStaticMeshComponent>();
    if (!Mesh) return;

    // 先确保开启了物理模拟
    Mesh->SetSimulatePhysics(true);


    // ===== 1. AddForce：持续施加力 =====
    // 每帧都会累积这个力，物体持续加速
    // 相当于 F = ma 中的 F

    // 示例：向世界坐标上方施加10000单位（达因）的力
    Mesh->AddForce(FVector(0, 0, 10000.0f));
    // 参数：FVector —— 力的方向和大小（达因，dyne）
    // UE使用厘米-克-秒(CGS)单位系统，力的单位是达因
    // 1达因 = 1克·厘米/秒²
    // 粗略理解：10000达因 ≈ 约能够推动一个1kg物体以0.1m/s²加速度运动

    // 如果你想持续加速：
    // 在Tick中调用 Mesh->AddForce(FVector(0, 0, 10000.0f));
    // 物体会一直加速（类似火箭推进）

    // 参数重载版本：
    Mesh->AddForce(
        FVector(0, 0, 10000.0f),  // 力的大小方向
        NAME_None,                  // 骨骼名（SkeletalMesh才需要）
        true                        // bAccelChange（true=加速度变化，忽略质量）
    );
    // bAccelChange = true  → 作为纯加速度（不受质量影响）
    // bAccelChange = false → 作为力（质量越大加速度越小）


    // ===== 2. AddImpulse：瞬间冲量 =====
    // 冲量 = 力 × 时间，只在当前帧生效一次
    // 相当于瞬间变化的速度

    // 示例：向上弹射（瞬间给一个向上速度）
    Mesh->AddImpulse(FVector(0, 0, 5000.0f));
    // 物体瞬间获得向上的速度5000cm/s = 50m/s

    // bVelocityChange参数：
    Mesh->AddImpulse(
        FVector(0, 0, 5000.0f),  // 冲量
        NAME_None,                 // 骨骼名
        false                      // bVelocityChange
    );
    // bVelocityChange = true  → 直接设为速度变化，忽略质量（大质量物体也一样速度）
    // bVelocityChange = false → 作为冲量，受质量影响（F·dt = m·dv → dv = F·dt/m）


    // ===== 3. AddForceAtLocation：在特定位置施加力 =====
    // 力不作用在质心，会产生旋转效果
    // 就像推门把手（力作用在边缘）vs 推门轴（力作用在中心）

    // 获取物体边缘的一个位置
    FVector Center = Mesh->GetComponentLocation();  // 物体中心
    FVector EdgePoint = Center + FVector(50, 0, 0); // 物体右侧50cm处

    // 在边缘位置向上施加力 → 物体会旋转！
    Mesh->AddForceAtLocation(
        FVector(0, 0, 5000.0f),  // 力的大小方向（向上）
        EdgePoint                  // 力的作用点（不在质心，产生旋转力矩）
    );
    // 想象：推一个浮在水面上的盒子的一角，它会边上升边旋转


    // ===== 4. AddTorque：旋转力矩 =====
    // 纯旋转力，不影响平动

    // 绕Z轴旋转（Yaw方向）
    Mesh->AddTorque(FVector(0, 0, 100000.0f));
    // 物体会绕Z轴旋转，但不发生平移

    // 参数重载：
    Mesh->AddTorque(
        FVector(0, 0, 100000.0f),  // 力矩
        NAME_None,                   // 骨骼名
        true                         // bAccelChange（true=角加速度变化）
    );
}
```

### AddRadialImpulse（放射状冲量）详解

```cpp
// ==================== AddRadialImpulse：模拟爆炸冲击波 ====================

void AMyActor::SimulateExplosion(const FVector& ExplosionCenter, float Radius, float Strength)
{
    // ---- 检测爆炸范围内的所有可物理模拟的物体 ----
    TArray<FHitResult> HitResults;

    FCollisionQueryParams QueryParams;
    QueryParams.bTraceComplex = false;

    // 用球体检测找到范围内的物体
    bool bHit = GetWorld()->SweepMultiByChannel(
        HitResults,
        ExplosionCenter,                            // 从爆炸中心
        ExplosionCenter + FVector(0, 0, 1),        // 几乎不变（原地检测）
        FQuat::Identity,
        ECC_PhysicsBody,                            // 只检测物理物体
        FCollisionShape::MakeSphere(Radius),        // 爆炸半径范围
        QueryParams
    );

    if (!bHit) return;

    UE_LOG(LogTemp, Warning, TEXT("爆炸影响了 %d 个物理物体"), HitResults.Num());

    // ---- 对每个受影响的物体施加放射状冲量 ----
    for (const FHitResult& Hit : HitResults)
    {
        UPrimitiveComponent* HitComponent = Hit.GetComponent();
        if (!HitComponent || !HitComponent->IsSimulatingPhysics())
        {
            continue;  // 跳过没有物理模拟的组件
        }

        // 计算距离衰减
        float Distance = FVector::Dist(ExplosionCenter, Hit.Location);
        float Falloff = FMath::Clamp(1.0f - (Distance / Radius), 0.0f, 1.0f);
        // 在爆炸中心：Falloff = 1.0（全额力度）
        // 在爆炸边缘：Falloff = 0.0（没有力度）
        // 中间线性衰减

        float ActualStrength = Strength * Falloff;

        // 施加放射状冲量
        HitComponent->AddRadialImpulse(
            ExplosionCenter,    // 爆炸中心点
            Radius,             // 爆炸半径
            ActualStrength,     // 冲量强度
            ERadialImpulseFalloff::RIF_Linear,  // 衰减方式：线性
            true                // bVelocityChange（直接修改速度）
        );
        // ERadialImpulseFalloff 选项：
        // RIF_Constant → 恒定（范围内不受距离影响）
        // RIF_Linear   → 线性衰减（距离越远力度越小）

        // 也可以手动添加额外的随机旋转力，让效果更自然：
        HitComponent->AddTorque(FMath::VRand() * ActualStrength * 100.0f);
    }

    // ---- 调试可视化 ----
    DrawDebugSphere(GetWorld(), ExplosionCenter, Radius, 32, FColor::Orange, false, 3.0f);
}
```

## 物理约束（Constraint）

物理约束用于限制两个物体之间的相对运动，就像把两个物体用各种"关节"连接起来。

### 约束类型

| 约束类型            | 效果描述               | 现实类比         |
| ------------------- | ---------------------- | ---------------- |
| **Linear（线性）**  | 控制两个物体之间的位移 | 弹簧、绳索、活塞 |
| **Angular（角度）** | 控制两个物体之间的旋转 | 铰链、门轴       |
| **Fixed**           | 完全固定在一起         | 焊接、粘合       |
| **Spring**          | 弹性的线性连接         | 弹簧、橡皮筋     |
| **Hinge**           | 只能绕一个轴旋转       | 门铰链           |
| **Ball-and-Socket** | 绕一个点自由旋转       | 肩关节、髋关节   |

### C++中创建和使用约束

```cpp
// ==================== 物理约束的C++创建 ====================

#include "PhysicsEngine/PhysicsConstraintComponent.h"

// ConstrainedActor.h
UCLASS()
class AConstrainedActor : public AActor
{
    GENERATED_BODY()

public:
    AConstrainedActor();

    // 物理约束组件
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "Physics")
    UPhysicsConstraintComponent* PhysicsConstraint;

    // 需要连接的两个组件
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "Components")
    UStaticMeshComponent* MeshA;

    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "Components")
    UStaticMeshComponent* MeshB;

    // 配置约束
    void ConfigureAsHinge();
    void ConfigureAsSpring();
};

// ConstrainedActor.cpp
#include "ConstrainedActor.h"
#include "PhysicsEngine/PhysicsConstraintComponent.h"
#include "Components/StaticMeshComponent.h"

AConstrainedActor::AConstrainedActor()
{
    PrimaryActorTick.bCanEverTick = false;

    // ---- 创建组件A（门框）----
    MeshA = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("MeshA"));
    RootComponent = MeshA;
    MeshA->SetSimulatePhysics(false);  // 静止不动物
    MeshA->SetCollisionProfileName(TEXT("BlockAll"));

    // ---- 创建组件B（门板）----
    MeshB = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("MeshB"));
    MeshB->SetupAttachment(RootComponent);  // 挂载到A下面
    MeshB->SetRelativeLocation(FVector(100.0f, 0.0f, 0.0f));  // 放在A旁边
    MeshB->SetSimulatePhysics(true);   // 这个要用物理模拟
    MeshB->SetCollisionProfileName(TEXT("BlockAll"));


    // ---- 创建物理约束组件 ----
    PhysicsConstraint = CreateDefaultSubobject<UPhysicsConstraintComponent>(TEXT("Constraint"));

    // 设置约束连接的两个组件
    PhysicsConstraint->SetConstrainedComponents(
        MeshA,                                 // 组件1（参考端，通常是固定的）
        NAME_None,                             // 组件1的骨骼名（静态Mesh填NAME_None）
        MeshB,                                 // 组件2（被约束端，会运动的）
        NAME_None                              // 组件2的骨骼名
    );

    // 设置两个组件在各自本地空间中的约束锚点位置
    // 这是约束的"枢轴点/旋转轴"
    PhysicsConstraint->SetWorldLocation(GetActorLocation());  // 约束的世界位置
}

// ---- 配置为铰链（门）----
void AConstrainedActor::ConfigureAsHinge()
{
    if (!PhysicsConstraint) return;

    // 锁定所有线性运动（不能平移，只能旋转）
    PhysicsConstraint->SetLinearXLimit(ELinearConstraintMotion::LCM_Locked, 0.0f);
    PhysicsConstraint->SetLinearYLimit(ELinearConstraintMotion::LCM_Locked, 0.0f);
    PhysicsConstraint->SetLinearZLimit(ELinearConstraintMotion::LCM_Locked, 0.0f);
    // LCM_Locked  = 完全锁定（不能在这个方向移动）
    // LCM_Limited = 有限制（可以在范围内移动）
    // LCM_Free    = 自由（随便移动）

    // 锁定两个旋转轴，只允许绕Z轴旋转（门轴）
    PhysicsConstraint->SetAngularSwing1Limit(EAngularConstraintMotion::ACM_Locked, 0.0f);
    // Swing1（一般情况下是绕Y轴的摇摆）
    PhysicsConstraint->SetAngularSwing2Limit(EAngularConstraintMotion::ACM_Locked, 0.0f);
    // Swing2（一般情况下是绕X轴的摇摆）

    // 绕Z轴（Twist/扭转）自由旋转 —— 这是门的开关方向
    PhysicsConstraint->SetAngularTwistLimit(
        EAngularConstraintMotion::ACM_Free,  // 自由旋转
        0.0f                                  // 不限制角度
    );
    // 如果想限制开门角度（如只能开90度）：
    // PhysicsConstraint->SetAngularTwistLimit(
    //     EAngularConstraintMotion::ACM_Limited,  // 有限旋转
    //     90.0f                                    // 最大90度
    // );
}

// ---- 配置为弹簧 ----
void AConstrainedActor::ConfigureAsSpring()
{
    if (!PhysicsConstraint) return;

    // 允许线性运动但有弹性限制
    PhysicsConstraint->SetLinearXLimit(ELinearConstraintMotion::LCM_Limited, 50.0f);
    // 允许X方向在±50cm范围内运动

    PhysicsConstraint->SetLinearYLimit(ELinearConstraintMotion::LCM_Limited, 50.0f);
    PhysicsConstraint->SetLinearZLimit(ELinearConstraintMotion::LCM_Limited, 50.0f);

    // 所有旋转轴锁定（不让它乱转）
    PhysicsConstraint->SetAngularSwing1Limit(EAngularConstraintMotion::ACM_Locked, 0.0f);
    PhysicsConstraint->SetAngularSwing2Limit(EAngularConstraintMotion::ACM_Locked, 0.0f);
    PhysicsConstraint->SetAngularTwistLimit(EAngularConstraintMotion::ACM_Locked, 0.0f);

    // 启用弹簧效果
    PhysicsConstraint->SetLinearPositionDrive(true, true, true);
    // 三个参数分别控制X, Y, Z方向是否启用位置驱动（Position Drive）
    // 位置驱动：试图让物体回到最初的位置（像弹簧回复平衡位置）

    // 设置弹簧的"硬度"（Spring Strength）
    PhysicsConstraint->SetLinearDriveParams(
        500.0f,   // 弹簧系数（Spring Stiffness）—— 越大越硬
        50.0f,    // 阻尼系数（Spring Damping）—— 越大振动衰减越快
        0.0f      // 最大力（0 = 无限制）—— 限制弹簧能施加的最大力
    );
}
```

## 物理材质（Physical Material）

物理材质定义了物体表面的物理属性——摩擦系数、弹力、密度等。它和渲染材质（Material）是不同的概念。

### 物理材质的属性

| 属性                        | 含义                           | 典型值                         |
| --------------------------- | ------------------------------ | ------------------------------ |
| **Friction（摩擦系数）**    | 表面有多滑（0=冰面，1=橡胶）   | 冰:0.1 / 木头:0.5 / 橡胶:1.0   |
| **Restitution（弹性系数）** | 反弹能力（0=不弹，1=完全弹性） | 石头:0.1 / 木头:0.3 / 弹球:0.9 |
| **Density（密度）**         | 单位体积的质量（g/cm³）        | 木头:0.5 / 水:1.0 / 铁:7.8     |
| **SurfaceType（表面类型）** | 分类标签（用于音效/粒子选择）  | 金属/木头/石头/血肉            |

### 创建和应用物理材质

```cpp
// ==================== 物理材质的C++操作 ====================

#include "PhysicalMaterials/PhysicalMaterial.h"

void AMyActor::WorkWithPhysicalMaterials()
{
    UStaticMeshComponent* Mesh = FindComponentByClass<UStaticMeshComponent>();
    if (!Mesh) return;


    // ---- 方法1：通过代码覆盖物理材质 ----
    // 创建一个物理材质实例（通常从编辑器加载资源）

    // 加载项目中已有的物理材质
    UPhysicalMaterial* WoodMaterial = LoadObject<UPhysicalMaterial>(
        nullptr,                                          // Outer（一般为空）
        TEXT("/Game/Materials/Physics/PM_Wood.PM_Wood")   // 资源路径
    );
    // LoadObject 从磁盘加载资产，路径格式为 /Game/路径/资产名.资产名


    // ---- 方法2：在代码中动态创建物理材质 ----
    // 注意：动态创建的物理材质不能在编辑器中持久化保存
    // 通常用于测试或程序化生成的内容

    UPhysicalMaterial* DynamicPhysMat = NewObject<UPhysicalMaterial>();
    // NewObject 创建一个UObject实例（运行时临时对象）

    if (DynamicPhysMat)
    {
        // 设置物理属性
        DynamicPhysMat->Friction = 0.8f;        // 摩擦系数（0~1，越大越不滑）
        DynamicPhysMat->Restitution = 0.3f;     // 弹性系数（0~1，越大弹越高）
        DynamicPhysMat->Density = 1.2f;         // 密度（g/cm³）

        // 表面类型通过 SurfaceType 枚举设置
        // 在 Project Settings → Physics → Physical Surfaces 中定义
        DynamicPhysMat->SurfaceType = SurfaceType1;  // 假设SurfaceType1 = 木头
        // SurfaceType 影响：碰撞音效选择、粒子效果选择、贴花选择
    }


    // ---- 方法3：给StaticMesh应用物理材质 ----
    // StaticMesh在其资产中可能已经预设了物理材质
    // 但也可以通过 BodyInstance 在运行时覆盖

    // 获取StaticMesh的BodyInstance
    FBodyInstance* BodyInstance = Mesh->GetBodyInstance();
    // BodyInstance 是一个物理体实例，存储了该组件的所有物理设置
    // 包括：质量、阻尼、碰撞设置、物理材质等

    if (BodyInstance && DynamicPhysMat)
    {
        // 覆盖简单碰撞体的物理材质
        BodyInstance->SetPhysMaterialOverride(DynamicPhysMat);
        // 这会覆盖该组件上所有简单碰撞体的物理材质
        // 适用于临时改变表面属性（如：下雨后地面变滑）
    }


    // ---- 实际应用示例：可破坏物体的物理材质应用 ----
    UStaticMeshComponent* MyMesh = FindComponentByClass<UStaticMeshComponent>();
    if (MyMesh)
    {
        // 检查当前使用的物理材质
        UPhysicalMaterial* CurrentPhysMat = MyMesh->GetBodyInstance()->GetSimplePhysicalMaterial();
        // GetSimplePhysicalMaterial 返回简单碰撞体使用的物理材质

        if (CurrentPhysMat)
        {
            UE_LOG(LogTemp, Log,
                TEXT("当前物理材质: 摩擦=%.2f, 弹性=%.2f, 密度=%.2f"),
                CurrentPhysMat->Friction,
                CurrentPhysMat->Restitution,
                CurrentPhysMat->Density
            );
        }
    }
}
```

## 质量、线性阻尼、角阻尼

这三个属性共同决定了物理物体"怎么动"。

### 属性解释

```cpp
// ==================== 质量、阻尼详解 ====================

// 质量（Mass）：
// - 定义：物体包含的物质的量（千克级，在UE中自动从形状和密度计算）
// - 影响：力 → 加速度（F = ma，质量越大，同样的力产生的加速度越小）
// - 影响：碰撞中的动量交换（质量大的物体撞质量小的，大的几乎不动）

// 线性阻尼（Linear Damping）：
// - 定义：运动过程中受到的"空气阻力"系数
// - 范围：0 ~ 无穷大（但一般 0 ~ 10）
// - 0 = 真空中运动（不受任何阻力，速度不变）
// - 10+ = 在蜂蜜中运动（很快停下）
// - 日常：空气中 0.01~0.1，水中 1~5

// 角阻尼（Angular Damping）：
// - 定义：旋转运动中的"空气阻力"系数
// - 范围：0 ~ 无穷大（一般 0 ~ 10）
// - 0 = 一直旋转不会停
// - 10+ = 旋转很快停止
```

### C++中设置

```cpp
// ==================== 质量与阻尼的C++设置 ====================

void AMyActor::ConfigureMassAndDamping()
{
    UStaticMeshComponent* Mesh = FindComponentByClass<UStaticMeshComponent>();
    if (!Mesh) return;

    // 开启物理模拟
    Mesh->SetSimulatePhysics(true);

    // ---- 获取BodyInstance（物理体实例） ----
    // BodyInstance 包含该组件所有物理设置
    FBodyInstance* Body = Mesh->GetBodyInstance();
    // 注意：GetBodyInstance() 可能返回nullptr（如果组件未初始化）
    if (!Body) return;


    // ---- 设置质量（Mass） ----
    // 方式1：设置质量覆盖（千克级，在UE中单位是kg）
    Body->SetMassOverride(50.0f);  // 设置质量为50kg
    // 默认情况下，质量 = 形状体积 × 物理材质密度
    // SetMassOverride 覆盖自动计算的质量

    // 方式2：不覆盖（使用自动计算的质量）
    Body->SetMassOverride(0.0f);  // 0 = 不覆盖，恢复自动计算
    // 自动计算：Mass = 碰撞体积 × 物理材质的Density
    // 例如：1m³的盒子 × 密度1.0g/cm³ = 1000g/cm³ × 1,000,000cm³ = 约1000kg

    // 获取当前质量（千克）
    float CurrentMass = Body->GetMass();
    UE_LOG(LogTemp, Log, TEXT("当前质量: %.2f kg"), CurrentMass);


    // ---- 设置线性阻尼（Linear Damping） ----
    Body->LinearDamping = 0.5f;
    // 0.5 的线性阻尼意味着物体在运动中受到中等空气阻力
    // 没有其他力的情况下，物体会逐渐减速直到停止
    // 阻尼越大，停下来越快；阻尼为0则永远不停（理想真空）

    // 测试不同阻尼值的效果：
    // 0.01  = 几乎无阻力（像在太空中），物体可以滚动很远
    // 0.5   = 中等阻力（像在空气中），滚几米就停了
    // 2.0   = 较大阻力（像在水中），很快就停
    // 10.0  = 极大阻力（像在蜂蜜中），几乎动不了


    // ---- 设置角阻尼（Angular Damping） ----
    Body->AngularDamping = 1.0f;
    // 角阻尼控制旋转衰减
    // 值越大，旋转停止得越快

    // 实际场景示例：
    // 石头：   LinearDamping=0.1,  AngularDamping=0.5  (不容易停下来)
    // 木箱：   LinearDamping=0.5,  AngularDamping=1.0  (正常)
    // 泡沫：   LinearDamping=2.0,  AngularDamping=3.0  (很快停)
    // 子弹：   LinearDamping=0.01, AngularDamping=0.01 (几乎不停)


    // ---- 设置质心偏移 ----
    // 质心默认在碰撞体的几何中心
    // CenterOfMassOffset 可以偏移质心位置（在物体本地坐标中）
    Body->COMNudge = FVector(0, 0, -20.0f);  // 质心向下偏移20cm
    // COM = Center Of Mass（质心）
    // COMNudge = 质心微调（Nudge = 轻推）
    // 效果：物体变得"头重脚轻"——更容易翻倒
    // 实际应用：不倒翁效果（质心很低，不容易倒）
}
```

## 睡眠与唤醒

物理引擎有一个重要的性能优化机制：**睡眠（Sleep）**。当物理物体的运动速度降到足够低时，引擎会让它"入睡"——不再对该物体执行物理计算，直到它被"唤醒"。

### 睡眠机制详解

```cpp
// ==================== 睡眠与唤醒 ====================

void AMyActor::PutToSleepAndWakeUp()
{
    UStaticMeshComponent* Mesh = FindComponentByClass<UStaticMeshComponent>();
    if (!Mesh) return;


    // ===== 手动让物体入睡 =====
    Mesh->PutRigidBodyToSleep();
    // PutRigidBodyToSleep：强制让该组件的物理模拟进入睡眠状态
    // 睡眠后，物体不再受力、不再运动
    // 节省CPU时间（引擎不再对静止物体进行物理计算）


    // ===== 检查物体是否在睡眠 =====
    bool bIsSleeping = Mesh->RigidBodyIsAwake() == false;
    // RigidBodyIsAwake() 返回 true = 醒着，false = 睡着
    // 注意：RigidBodyIsAwake 不是 IsSleeping！

    if (Mesh->RigidBodyIsAwake())
    {
        UE_LOG(LogTemp, Log, TEXT("物体醒着（正在运动或受力）"));
    }
    else
    {
        UE_LOG(LogTemp, Log, TEXT("物体睡着了（静止不动，节省性能）"));
    }


    // ===== 唤醒物体 =====
    Mesh->WakeRigidBody();
    // WakeRigidBody：强制唤醒该组件的物理模拟
    // 唤醒后，物体会重新接受物理计算
    // 注意：物体不会自动因为AddForce而唤醒！
    // 但 AddImpulse 会自动唤醒

    // ❌ 常见错误：
    // Mesh->PutRigidBodyToSleep();
    // Mesh->AddForce(FVector(0, 0, 1000));  // 物体在睡眠中，不会响应AddForce！
    // ✅ 正确做法：
    // Mesh->WakeRigidBody();  // 先唤醒
    // Mesh->AddForce(FVector(0, 0, 1000));  // 再施加力


    // ===== 设置睡眠阈值 =====
    // 控制物体多慢时自动入睡
    FBodyInstance* Body = Mesh->GetBodyInstance();
    if (Body)
    {
        // 线速度阈值（cm/s）：低于此速度持续一段时间后自动入睡
        Body->SleepThresholdMultiplier = 1.0f;
        // 默认值1.0 → 使用物理项目设置中的默认阈值
        // 增大（如5.0）→ 物体更快入睡（更省性能，但看起来可能不自然）
        // 减小（如0.1）→ 物体更难入睡（物体很慢时才入睡）
        // 设为极大值（如10000）→ 物体几乎不会自动入睡（始终计算物理）
    }
}
```

### 睡眠的唤醒条件

```cpp
// ==================== 什么操作会自动唤醒物体 ====================

void AutoWakeUpConditions()
{
    // 以下操作会**自动**唤醒睡眠中的物体（不需要手动调用 WakeRigidBody）：

    // ✅ AddImpulse          → 自动唤醒
    // ✅ AddForceAtLocation  → 自动唤醒
    // ✅ 被其他物体碰撞      → 自动唤醒
    // ✅ SetPhysicsLinearVelocity → 自动唤醒
    // ✅ 物理约束的激活      → 自动唤醒

    // ❌ AddForce            → 不会自动唤醒
    // ❌ AddTorque           → 不会自动唤醒
    // ❌ SetMassOverride     → 不会自动唤醒

    // ⚠️ 最佳实践：施加力前总是先检查并唤醒
    UStaticMeshComponent* Mesh = /* ... */;
    if (Mesh && Mesh->IsSimulatingPhysics())
    {
        if (!Mesh->RigidBodyIsAwake())
        {
            Mesh->WakeRigidBody();  // 确保物体醒着
        }
        Mesh->AddForce(FVector(0, 0, 1000));
    }
}
```

## 碰撞回调：OnComponentHit / OnComponentBeginOverlap

当物理模拟的物体发生碰撞或重叠时，引擎会触发事件回调。正确绑定和处理这些回调是物理交互的核心。

### OnComponentHit（碰撞阻塞回调）

```cpp
// ==================== OnComponentHit 完整使用示例 ====================

// CollisionActor.h
UCLASS()
class ACollisionActor : public AActor
{
    GENERATED_BODY()

public:
    ACollisionActor();

protected:
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly)
    UStaticMeshComponent* PhysicsMesh;

    // 物理碰撞回调函数
    UFUNCTION()
    void OnMeshHit(
        UPrimitiveComponent* HitComponent,   // 被命中的组件（本组件）
        AActor* OtherActor,                  // 碰撞的对方Actor
        UPrimitiveComponent* OtherComp,      // 碰撞的对方组件
        FVector NormalImpulse,               // 碰撞产生的冲量方向
        const FHitResult& Hit                // 碰撞命中结果详情
    );
};

// CollisionActor.cpp
#include "CollisionActor.h"
#include "Components/StaticMeshComponent.h"

ACollisionActor::ACollisionActor()
{
    PrimaryActorTick.bCanEverTick = false;

    // ---- 创建物理Mesh ----
    PhysicsMesh = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("PhysicsMesh"));
    RootComponent = PhysicsMesh;

    // 关键配置：物理 + 碰撞事件
    PhysicsMesh->SetSimulatePhysics(true);                       // 开启物理模拟
    PhysicsMesh->SetCollisionEnabled(ECollisionEnabled::QueryAndPhysics);  // 物理+查询
    PhysicsMesh->SetCollisionProfileName(TEXT("PhysicsActor"));  // 使用物理Actor预设

    // 🔑 关键：开启物理碰撞事件通知！
    PhysicsMesh->SetNotifyRigidBodyCollision(true);
    // 不设置这个，OnComponentHit 永远不会触发！
    // 默认值：false（出于性能考虑）
    // 设为true后，每次物理碰撞都会触发 OnComponentHit 委托

    // 绑定碰撞事件
    PhysicsMesh->OnComponentHit.AddDynamic(this, &ACollisionActor::OnMeshHit);
    // AddDynamic 是UE委托系统绑定方法
    // 使用 Dynamic 是因为通过蓝图也可以绑定这个委托


    // ---- 设置物理属性 ----
    PhysicsMesh->GetBodyInstance()->SetMassOverride(10.0f);  // 10kg
    PhysicsMesh->GetBodyInstance()->LinearDamping = 0.3f;
    PhysicsMesh->GetBodyInstance()->AngularDamping = 0.5f;
}

// ---- 碰撞回调实现 ----
void ACollisionActor::OnMeshHit(
    UPrimitiveComponent* HitComponent,
    AActor* OtherActor,
    UPrimitiveComponent* OtherComp,
    FVector NormalImpulse,
    const FHitResult& Hit)
{
    // ---- 参数说明 ----
    // HitComponent  → 被撞的是哪个组件（通常是 PhysicsMesh）
    // OtherActor    → 撞到了哪个Actor
    // OtherComp     → 对方的具体哪个组件
    // NormalImpulse → 碰撞产生的冲量方向（可以用来判断撞击力度）
    // Hit           → 碰撞详细信息（命中点、法线等）


    // ===== 判断碰撞力度 =====
    // NormalImpulse 的大小反映了碰撞的剧烈程度
    float ImpactForce = NormalImpulse.Size();
    // 冲量的大小（粗略理解为撞击力度）
    // 轻轻碰一下：~100
    // 用力扔过去：~5000
    // 高速撞击：  ~50000

    if (ImpactForce > 5000.0f)  // 撞击力度超过阈值
    {
        // 剧烈碰撞！触发特效、音效、伤害等
        UE_LOG(LogTemp, Warning,
            TEXT("剧烈碰撞！力度: %.1f, 对方: %s"),
            ImpactForce,
            OtherActor ? *OtherActor->GetName() : TEXT("None")
        );

        // 在碰撞点生成火花粒子
        // SpawnImpactParticle(Hit.Location, Hit.ImpactNormal);

        // 播放碰撞音效（音量与撞击力度成正比）
        float VolumeMultiplier = FMath::Clamp(ImpactForce / 10000.0f, 0.1f, 1.0f);
        // PlayImpactSound(Hit.Location, VolumeMultiplier);
    }
    else
    {
        // 轻微碰撞
        UE_LOG(LogTemp, Log, TEXT("轻微碰撞，力度: %.1f"), ImpactForce);
    }


    // ===== 根据物理材质选择效果 =====
    if (Hit.PhysMaterial.IsValid())
    {
        UPhysicalMaterial* PhysMat = Hit.PhysMaterial.Get();

        // 根据表面类型选择不同效果
        switch (PhysMat->SurfaceType)
        {
            case SurfaceType1:  // 假设=金属
                // 播放金属撞击音
                // SpawnMetalImpactEffect(Hit.Location, Hit.ImpactNormal);
                break;

            case SurfaceType2:  // 假设=木头
                // 播放木头撞击音
                // SpawnWoodImpactEffect(Hit.Location, Hit.ImpactNormal);
                break;

            default:
                // 默认撞击效果
                break;
        }
    }
}
```

### OnComponentBeginOverlap / OnComponentEndOverlap（重叠事件）

```cpp
// ==================== 重叠事件完整示例（带物理模拟的物体）====================

// OverlapTrigger.h
UCLASS()
class AOverlapTrigger : public AActor
{
    GENERATED_BODY()

public:
    AOverlapTrigger();

protected:
    // 检测区域的碰撞体（只查询，不物理模拟）
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly)
    USphereComponent* OverlapSphere;

    // 重叠开始回调
    UFUNCTION()
    void OnTriggerEnter(
        UPrimitiveComponent* OverlappedComponent,
        AActor* OtherActor,
        UPrimitiveComponent* OtherComp,
        int32 OtherBodyIndex,
        bool bFromSweep,
        const FHitResult& SweepResult
    );

    // 重叠结束回调
    UFUNCTION()
    void OnTriggerExit(
        UPrimitiveComponent* OverlappedComponent,
        AActor* OtherActor,
        UPrimitiveComponent* OtherComp,
        int32 OtherBodyIndex
    );

    // 当前在范围内的Actor计数
    int32 ActorsInside = 0;
};

AOverlapTrigger::AOverlapTrigger()
{
    PrimaryActorTick.bCanEverTick = false;

    // ---- 创建检测球体 ----
    OverlapSphere = CreateDefaultSubobject<USphereComponent>(TEXT("OverlapSphere"));
    RootComponent = OverlapSphere;
    OverlapSphere->SetSphereRadius(200.0f);  // 半径2米

    // 🔑 触发器配置：
    // 1. 仅查询模式（不需要物理反应）
    OverlapSphere->SetCollisionEnabled(ECollisionEnabled::QueryOnly);
    // 2. 只对Pawn产生重叠响应
    OverlapSphere->SetCollisionResponseToAllChannels(ECR_Ignore);
    OverlapSphere->SetCollisionResponseToChannel(ECC_Pawn, ECR_Overlap);
    // 3. 🔑 必须开启重叠事件生成！
    OverlapSphere->SetGenerateOverlapEvents(true);

    // 绑定事件
    OverlapSphere->OnComponentBeginOverlap.AddDynamic(this, &AOverlapTrigger::OnTriggerEnter);
    OverlapSphere->OnComponentEndOverlap.AddDynamic(this, &AOverlapTrigger::OnTriggerExit);
}

// ---- 重叠开始回调 ----
void AOverlapTrigger::OnTriggerEnter(
    UPrimitiveComponent* OverlappedComponent,
    AActor* OtherActor,
    UPrimitiveComponent* OtherComp,
    int32 OtherBodyIndex,
    bool bFromSweep,
    const FHitResult& SweepResult)
{
    // 安全检查
    if (!OtherActor || OtherActor == this) return;

    ActorsInside++;
    UE_LOG(LogTemp, Warning,
        TEXT("%s 进入了触发区域（当前%d个Actor在内）"),
        *OtherActor->GetName(),
        ActorsInside
    );

    // 实际应用示例：
    // - 玩家进入 → 开门
    // - 敌人进入 → 激活警报
    // - 物体进入 → 给予Buff/效果
}

// ---- 重叠结束回调 ----
void AOverlapTrigger::OnTriggerExit(
    UPrimitiveComponent* OverlappedComponent,
    AActor* OtherActor,
    UPrimitiveComponent* OtherComp,
    int32 OtherBodyIndex)
{
    if (!OtherActor || OtherActor == this) return;

    ActorsInside = FMath::Max(0, ActorsInside - 1);  // 防止变成负数
    UE_LOG(LogTemp, Warning,
        TEXT("%s 离开了触发区域（当前%d个Actor在内）"),
        *OtherActor->GetName(),
        ActorsInside
    );
}

// ⚠️ 重要提示：
// OnComponentBeginOverlap 和 OnComponentEndOverlap 与物理模拟之间的关系：
//
// - 如果两个组件都在物理模拟，且碰撞响应为Block
//   它们将触发 OnComponentHit（碰撞回调），而不是 OnComponentBeginOverlap
//
// - 如果碰撞响应为Overlap
//   触发 OnComponentBeginOverlap/OnComponentEndOverlap
//   物体可以互相穿过
//
// - OnComponentHit 和 OnComponentBeginOverlap 是互斥的
//   同一个碰撞事件不会同时触发二者
```

### 碰撞事件触发条件总结表

| 碰撞响应设置 | 物理模拟 | 重叠事件 | 触发的事件                                          |
| :----------- | :------: | :------: | --------------------------------------------------- |
| Block        |    ✅    |    ❌    | `OnComponentHit`                                    |
| Block        |    ❌    |    ❌    | 无事件（只会阻挡）                                  |
| Overlap      |  ✅/❌   |    ✅    | `OnComponentBeginOverlap` + `OnComponentEndOverlap` |
| Ignore       |  ✅/❌   |    ❌    | 无事件                                              |

> **记忆口诀**：
>
> - Block → Hit事件（阻挡了，给你一个Hit通知）
> - Overlap → Overlap事件（让你穿过去了，但告诉你"有东西进来了"）
> - Ignore → 无声无息（完全不理你）

## 物理子步（Sub-stepping）

物理子步是物理引擎内部的一个高级设置，它控制物理模拟的时间精度。

### 什么是子步？

```cpp
// ==================== 物理子步的概念 ====================

// 默认情况下，物理引擎每帧执行一次物理步进（step）
// 如果帧率是60FPS，则物理步进频率 = 60Hz
// 每步的时间步长 ≈ 1/60 ≈ 0.016秒

// 但这可能导致问题：
// - 快速移动的物体可能"穿过"薄墙（因为每步移动的距离超过墙的厚度）
// - 高速碰撞可能计算不准确
// - 复杂的约束系统可能在低步进频率下抖动

// 子步（Sub-stepping）将每帧的物理计算切分成多个更小的子步：
// 如果 MaxSubSteps = 4，则：
//   每步时间 = 1/60 / 4 = 1/240 ≈ 0.004秒
//   物理精度提高了4倍，但计算量也增加了4倍
```

### 子步设置

```cpp
// ==================== 子步的C++设置 ====================

void AMyActor::ConfigureSubstepping()
{
    UStaticMeshComponent* Mesh = FindComponentByClass<UStaticMeshComponent>();
    if (!Mesh) return;

    FBodyInstance* Body = Mesh->GetBodyInstance();
    if (!Body) return;


    // ---- 启用物理子步 ----
    // 注意：子步通常在项目级别设置，但在某些版本中也可以按Body设置

    // 在 DefaultEngine.ini 或代码中：
    // [/Script/Engine.PhysicsSettings]
    // bSubstepping=true
    // MaxSubstepDeltaTime=0.016667
    // MaxSubsteps=4


    // ---- 高速运动物体的建议子步设置 ----
    // 场景：子弹（速度 100000 cm/s = 1000 m/s）
    // 60FPS下每步移动 = 100000/60 = 1667cm ≈ 16米！
    // 没有子步的话，子弹可能"跳过"中间的墙壁

    // 解决方案：
    // 1. 增加子步数量
    // 2. 开启CCD（连续碰撞检测，见下节）
    // 3. 组合使用：子步 + CCD = 最可靠的高速碰撞检测


    // ---- 在项目设置中配置子步 ----
    // Project Settings → Engine → Physics:
    //   ☑ Enable Substepping
    //   Max Substep Delta Time: 0.016667  (60Hz基础步时)
    //   Max Substeps: 4                    (最多子步数)
    //
    // Async Substepping:
    //   ☑ Use Async Scene (异步物理场景)
}
```

## CCD（连续碰撞检测）

CCD（Continuous Collision Detection，连续碰撞检测）解决了快速移动物体"穿透"薄物体的问题。

### 离散碰撞 vs 连续碰撞

```cpp
// ==================== 离散碰撞检测 vs 连续碰撞检测 ====================
//
// 离散碰撞检测（Discrete CCD，默认）：
//   每帧检查物体当前是否与另一个物体相交
//
//   问题：如果物体移动速度很快，在某一帧它在墙的一边，
//        下一帧已经在墙的另一边了——两帧都不与墙相交，但实际穿了过去！
//
//   帧1: [物体] |墙|     ← 在墙左边，没碰到
//   帧2: |墙| [物体]     ← 在墙右边，没碰到！
//   结果：物体穿墙了！
//
//
// 连续碰撞检测（CCD）：
//   不是检查"现在是否相交"，而是检查"在这两帧之间是否扫过了另一个物体"
//
//   帧1→帧2: [物体]===扫掠路径===|墙|
//   检查扫掠路径是否与墙相交 → 会！→ 检测到碰撞！
//   结果：物体会在墙的位置停下来

void AMyActor::EnableCCD()
{
    UStaticMeshComponent* Mesh = FindComponentByClass<UStaticMeshComponent>();
    if (!Mesh) return;

    // ⚠️ 重要前提：
    // CCD 只对开启了物理模拟的组件生效
    // CCD 消耗更多性能，只给真正需要高速运动的物体用
    Mesh->SetSimulatePhysics(true);

    // ---- 开启CCD ----
    // 通过BodyInstance设置CCD
    FBodyInstance* Body = Mesh->GetBodyInstance();
    if (Body)
    {
        // 设置CCD模式
        Body->bUseCCD = true;  // 为该物体启用CCD

        // 在较新版本UE5中，CCD API可能略有不同：
        // 注意：CCD要求物体的碰撞形状是简单碰撞体（球体、盒子、胶囊体）
        // 复杂三角面碰撞不支持CCD
    }

    UE_LOG(LogTemp, Log, TEXT("CCD已启用：高速运动物体不会被穿透"));
}

// ==================== 什么时候使用CCD ====================
//
// ✅ 应该开启CCD的场景：
//   - 子弹/射弹（速度极快，可能穿墙）
//   - 从高处掉落的物体（下落速度很快）
//   - 载具（高速行驶可能穿透路障）
//   - 关键物品（不能掉出世界的道具）
//
// ❌ 不需要CCD的场景：
//   - 慢速移动物体（玩家行走速度）
//   - 静止物体
//   - 不需要精确碰撞的装饰物
//   - 已经通过子步确保了精度的物体
//
// ⚠️ 性能提示：
//   - CCD比普通碰撞检测消耗更多CPU
//   - 只在必要的物体上开启CCD
//   - 场景中开启CCD的物体最好不超过几十个
```
