## 什么是碰撞？

在UE5中，"碰撞"有两个层面的含义：

1. **物理碰撞（Physics Collision）**：物体之间真实的物理反应——推开、弹开、受力反弹。
2. **查询碰撞（Query Collision）**：用程序"问"引擎"这里有没有东西？"——射线检测、范围检测、重叠检测。

UE5的碰撞系统基于 **PhysX物理引擎**（UE5中已改为Chaos物理引擎，但上层API保持一致），使用分层架构：

```
游戏层（C++/蓝图）→ 碰撞查询API → 碰撞通道/响应 → 物理引擎（Chaos）
```

> **傻瓜式理解**：把碰撞系统想象成一个"门禁系统"。
> - **碰撞通道** = 身份卡的类型（员工卡、访客卡、VIP卡）
> - **碰撞响应** = 门禁规则（某些卡可以通过，某些卡被阻挡，某些卡只记录不阻挡）
> - 你通过API"刷卡"时，系统根据规则决定放行（穿透）、阻挡（Block）还是仅记录（Overlap）。

## 碰撞的三种模式

在UE5中，每个有碰撞的组件都有一个**碰撞启用状态（Collision Enabled）**，它是一个枚举值 `ECollisionEnabled::Type`：

### 完整枚举定义

```cpp
// 来自 Engine/Source/Runtime/Engine/Classes/Components/PrimitiveComponent.h
// 碰撞启用状态的枚举定义

enum class ECollisionEnabled : uint8
{
    // ❌ 无碰撞 —— 完全不参与任何碰撞检测
    // 物体像一个"幽灵"，可以穿过任何东西
    NoCollision,

    // ✅ 仅查询 —— 只参与射线检测、重叠检测等程序查询
    // 可以被"问到"（比如："这个位置有东西吗？"）
    // 但不会产生物理推开效果
    QueryOnly,

    // ✅ 仅物理 —— 只参与物理模拟
    // 物体会被推开、会受力、会下落
    // 但不会被射线检测命中（除非射线检测设置了特殊参数）
    PhysicsOnly,

    // ✅✅ 查询和物理 —— 同时参与查询和物理模拟
    // 这是最常用的模式：
    // - 可以被射线检测命中
    // - 可以产生物理碰撞反应
    QueryAndPhysics
};
```

### 四种模式详解

| 模式 | 射线检测命中 | 重叠检测触发 | 物理推开效果 | 典型使用场景 |
|------|:---:|:---:|:---:|-------------|
| `NoCollision` | ❌ | ❌ | ❌ | 纯装饰物、背景天空球、UI组件 |
| `QueryOnly` | ✅ | ✅ | ❌ | 触发器区域、可互动物体（不需要物理） |
| `PhysicsOnly` | ❌ | ❌ | ✅ | 纯物理碎片（不需要被检测） |
| `QueryAndPhysics` | ✅ | ✅ | ✅ | 玩家角色、敌人、可拾取物品、子弹 |

### C++设置示例

```cpp
// ==================== 碰撞模式设置示例 ====================

#include "Components/BoxComponent.h"
#include "Components/SphereComponent.h"

// ===== 示例1：设置不同碰撞模式 =====

void AMyActor::SetupCollisionModes()
{
    // --- 创建一个盒子碰撞体，设置为"仅查询"模式 ---
    // 这个盒子可以被射线检测到，可以触发重叠事件
    // 但其他物体穿过它时不会有物理阻挡
    UBoxComponent* TriggerBox = CreateDefaultSubobject<UBoxComponent>(TEXT("TriggerBox"));
    TriggerBox->SetCollisionEnabled(ECollisionEnabled::QueryOnly);  // 仅查询模式
    // 参数解释：ECollisionEnabled::QueryOnly 表示只参与查询，不参与物理模拟
    TriggerBox->SetBoxExtent(FVector(100.0f, 100.0f, 100.0f));     // 设置盒子大小 100x100x100

    // --- 创建一个球体碰撞体，设置为"查询和物理"模式 ---
    // 这是最完整的碰撞模式——物理+查询都生效
    // 适合作为玩家的碰撞体
    USphereComponent* PlayerCollision = CreateDefaultSubobject<USphereComponent>(TEXT("PlayerCollider"));
    PlayerCollision->SetCollisionEnabled(ECollisionEnabled::QueryAndPhysics);  // 查询+物理模式
    PlayerCollision->SetSphereRadius(40.0f);  // 设置球体半径40厘米

    // --- 关闭碰撞 ---
    // 这个组件将完全透明于碰撞系统
    UStaticMeshComponent* DecorationMesh = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("Decoration"));
    DecorationMesh->SetCollisionEnabled(ECollisionEnabled::NoCollision);  // 完全无碰撞
}

// ===== 示例2：运行时动态切换碰撞模式 =====

void AMyActor::EnableCollisionAtRuntime()
{
    // 获取当前组件（假设已经在Blueprint或构造函数中创建好了）
    UStaticMeshComponent* MeshComp = FindComponentByClass<UStaticMeshComponent>();
    if (MeshComp)
    {
        // 运行时开启碰撞（比如：角色死亡后身体变成物理模拟）
        MeshComp->SetCollisionEnabled(ECollisionEnabled::QueryAndPhysics);
        // 同时开启物理模拟
        MeshComp->SetSimulatePhysics(true);
    }
}

void AMyActor::DisableCollisionAtRuntime()
{
    UStaticMeshComponent* MeshComp = FindComponentByClass<UStaticMeshComponent>();
    if (MeshComp)
    {
        // 运行时关闭碰撞（比如：拾取物品后让它消失但先关碰撞防止被其他东西撞击）
        MeshComp->SetCollisionEnabled(ECollisionEnabled::NoCollision);
        // 同时关闭物理模拟
        MeshComp->SetSimulatePhysics(false);
    }
}
```

> **❌ 常见错误**：把触发器区域的碰撞模式设置为 `PhysicsOnly`，然后期待能触发 `OnBeginOverlap`。
> - `PhysicsOnly` 不产生重叠事件！需要用 `QueryOnly` 或 `QueryAndPhysics`。
>
> **✅ 正确做法**：触发器用 `QueryOnly`，物理物体用 `QueryAndPhysics`。

## 碰撞通道（Collision Channel）

碰撞通道是UE5碰撞系统的核心概念。每个碰撞通道代表一种"交互类别"。

### 内置碰撞通道

UE5默认提供了以下碰撞通道（在 `Engine/Config/BaseEngine.ini` 中定义）：

| 通道名称（枚举值） | 枚举常量 | 用途说明 |
|----------|----------|----------|
| **WorldStatic** | `ECC_WorldStatic` | 静态世界物体：墙壁、地板、建筑（不会移动的东西） |
| **WorldDynamic** | `ECC_WorldDynamic` | 动态世界物体：可移动的箱子、门、平台 |
| **Pawn** | `ECC_Pawn` | 可被玩家控制的角色/生物 |
| **Visibility** | `ECC_Visibility** | 可见性检测：判断视线是否被遮挡 |
| **Camera** | `ECC_Camera** | 摄像机通道：用于摄像机的碰撞避让 |
| **PhysicsBody** | `ECC_PhysicsBody` | 物理模拟的刚体 |
| **Vehicle** | `ECC_Vehicle` | 载具（车辆、飞行器等） |
| **Destructible** | `ECC_Destructible` | 可破坏物体（碎片、易碎品） |

```cpp
// ==================== 碰撞通道的C++定义 ====================
// 这些枚举定义在 Engine/Source/Runtime/Engine/Classes/Engine/EngineTypes.h

// 碰撞通道的完整枚举定义（UE内置部分）
// UE5中碰撞通道本质上是一个uint8整数，范围0-31
// 其中0-8为引擎预留通道
namespace ECollisionChannel
{
    enum Type
    {
        ECC_WorldStatic      = 0,   // 静态世界物体通道
        ECC_WorldDynamic     = 1,   // 动态世界物体通道
        ECC_Pawn             = 2,   // 角色通道（玩家和AI角色）
        ECC_Visibility       = 3,   // 可见性检测通道
        ECC_Camera           = 4,   // 摄像机通道
        ECC_PhysicsBody      = 5,   // 物理刚体通道
        ECC_Vehicle          = 6,   // 载具通道
        ECC_Destructible     = 7,   // 可破坏物体通道

        // ... 你可以通过 Project Settings 继续添加自定义通道（8-31）
        // 引擎预留最多18个自定义通道（ECC_GameTraceChannel1 ~ ECC_GameTraceChannel18）
    };
}
```

### 碰撞通道的工作原理

每个碰撞组件在引擎内部维护两个关键信息：
1. **自身所属的通道**（`ObjectType`）："我是什么类型的东西？"
2. **我对其他通道的响应**（`CollisionResponses`）："遇到各种类型的东西时我该做什么？"

```cpp
// ==================== 查询/修改碰撞通道的C++代码 ====================

#include "Engine/EngineTypes.h"
#include "Components/PrimitiveComponent.h"

void AMyActor::WorkWithCollisionChannels()
{
    // --- 获取本组件属于哪个通道（ObjectType）---
    // 这回答了"我是什么类型"的问题
    ECollisionChannel MyChannel = MyComponent->GetCollisionObjectType();
    // 返回值举例：ECC_WorldDynamic（说明这个组件被标记为"动态世界物体"）

    // --- 设置本组件的通道类型 ---
    // 比如：把玩家控制器的碰撞体设置为Pawn通道
    MyComponent->SetCollisionObjectType(ECC_Pawn);
    // 调用后，其他物体就可以通过碰撞预设判断"这是一个Pawn"
    // 然后根据他们对Pawn的响应规则来处理交互

    // --- 检查碰撞响应 ---
    // 查询"我对Pawn类型的物体是什么态度？"
    ECollisionResponse Response = MyComponent->GetCollisionResponseToChannel(ECC_Pawn);
    // 返回值：ECR_Block（阻挡）/ ECR_Overlap（重叠）/ ECR_Ignore（忽略）

    // --- 常见使用场景 ---
    // 场景1：玩家的胶囊体碰撞应该是Pawn通道
    UCapsuleComponent* PlayerCapsule = GetCapsuleComponent();
    PlayerCapsule->SetCollisionObjectType(ECC_Pawn);  // 我是Pawn

    // 场景2：可拾取物品应该是WorldDynamic通道
    USphereComponent* PickupSphere = CreateDefaultSubobject<USphereComponent>(TEXT("PickupSphere"));
    PickupSphere->SetCollisionObjectType(ECC_WorldDynamic);  // 我是动态物体
}
```

> **傻瓜式理解**：
> - 每个Actor上贴了一个"身份证"，上面写着它的类型（WorldStatic/WorldDynamic/Pawn...）
> - 同时每个Actor还有一个"行为手册"，规定遇到各种身份的人该怎么反应
> - 当两个Actor相遇时，引擎查手册，做出对应反应

## 碰撞响应（Collision Response）

碰撞响应定义了"当一个物体遇到另一个物体时，该做什么"。

### 三种响应类型

```cpp
// ==================== 碰撞响应枚举 ====================
// 定义于 Engine/Source/Runtime/Engine/Classes/Engine/EngineTypes.h

// 碰撞响应只有三种可能：
// ECR_Ignore（忽略）  —— "当作没看见，直接穿过去"
// ECR_Overlap（重叠）  —— "我知道你进来了，但我让你穿过去，只是通知一下"
// ECR_Block  （阻挡）  —— "你进不来的，我会把你挡住"
```

| 响应类型 | 视觉效果 | 重叠事件触发 | 物理阻挡 | 典型用途 |
|----------|----------|:---:|:---:|----------|
| **Ignore** | 两个物体互相穿过 | ❌ | ❌ | 不同阵营的子弹互相穿过；队友之间不阻挡 |
| **Overlap** | 两个物体互相穿过 | ✅ `OnBeginOverlap` | ❌ | 触发器区域；可拾取物品的检测范围 |
| **Block** | 两个物体互相弹开/停住 | ❌（但触发 `OnComponentHit`） | ✅ | 墙壁挡人；子弹击中目标 |

### C++中设置碰撞响应

```cpp
// ==================== 碰撞响应设置示例 ====================

#include "Components/BoxComponent.h"
#include "Engine/EngineTypes.h"

void AMyActor::ConfigureCollisionResponses()
{
    // --- 方法1：设置对单个通道的响应 ---
    // 语法：SetCollisionResponseToChannel(通道, 响应类型)
    // 含义：设置"我"对"通道代表的类型"的碰撞响应

    // 示例：让本组件阻挡Pawn来碰撞（即角色会被这个组件挡住）
    MyComponent->SetCollisionResponseToChannel(ECC_Pawn, ECR_Block);
    // 参数1：ECC_Pawn —— "我对Pawn类型物体的响应"
    // 参数2：ECR_Block —— "阻挡它们，不让它们穿过"

    // 示例：让玩家可以穿过这个组件（但触发重叠事件）
    MyComponent->SetCollisionResponseToChannel(ECC_Pawn, ECR_Overlap);
    // 结果：玩家进入这个区域时触发 OnBeginOverlap 事件
    // 但玩家不会撞到这个组件 —— 适合做拾取检测区域

    // 示例：完全忽略某一类型的碰撞
    MyComponent->SetCollisionResponseToChannel(ECC_WorldStatic, ECR_Ignore);
    // 结果：这个组件会直接穿过墙壁、地面等静态物体


    // --- 方法2：设置对所有通道的响应 ---
    // 一次性把所有通道都设为同样的响应

    // 阻挡一切（有点像"石墙"）
    MyComponent->SetCollisionResponseToAllChannels(ECR_Block);
    // 注意：这会覆盖所有通道，包括Visibility和Camera
    // 通常不会完全阻挡所有通道，因为会干扰摄像机等功能

    // 忽略一切（彻底透明）
    MyComponent->SetCollisionResponseToAllChannels(ECR_Ignore);
    // 之后可以单独打开需要的通道


    // --- 方法3：检查当前响应 ---
    // 查询我对某个通道目前的响应状态
    ECollisionResponse CurrentResponse = MyComponent->GetCollisionResponseToChannel(ECC_Pawn);
    if (CurrentResponse == ECR_Block)
    {
        // 说明目前会阻挡Pawn
        UE_LOG(LogTemp, Warning, TEXT("当前会阻挡Pawn"));
    }
    else if (CurrentResponse == ECR_Overlap)
    {
        // 说明目前会与Pawn发生重叠
        UE_LOG(LogTemp, Warning, TEXT("当前会与Pawn重叠"));
    }
    else // ECR_Ignore
    {
        // 说明目前完全忽略Pawn
        UE_LOG(LogTemp, Warning, TEXT("当前完全忽略Pawn"));
    }
}
```

## 碰撞预设（Collision Presets）

碰撞预设是一组**预配置好的碰撞设置**，让你不用手动调每个通道的响应。在编辑器中选中组件后，`Collision Presets` 下拉菜单就是预设在起作用。

### 如何理解碰撞预设

```cpp
// ==================== 碰撞预设的本质 ====================
// 碰撞预设 = CollisionEnabled + ObjectType + 所有通道的CollisionResponses
//
// 它是一次性帮你把"碰撞模式、自身通道、对所有通道的响应"全部设置好
// 类似于手机的"静音模式""飞行模式"——一键切换整套设置
```

### 内置预设列表

UE5提供了以下内置碰撞预设（在 `Engine/Config/BaseEngine.ini` 的 `[/Script/Engine.CollisionProfile]` 节中定义）：

| 预设名称 | 碰撞启用 | 自身通道 | 对所有通道的默认响应 | 适用场景 |
|----------|:---:|:---:|:---:|----------|
| `NoCollision` | NoCollision | WorldStatic | Ignore | 完全不需要碰撞的东西：天空球、水面平面 |
| `BlockAll` | QueryAndPhysics | WorldStatic | Block | 墙壁、地面、柱子（一切都会被挡住） |
| `BlockAllDynamic` | QueryAndPhysics | WorldStatic | Block | 和BlockAll相同 |
| `OverlapAll` | QueryOnly | WorldStatic | Overlap | 触发器区域（一切穿过但触发事件） |
| `OverlapAllDynamic` | QueryOnly | WorldStatic | Overlap | 只对动态物体触发的触发器 |
| `IgnoreOnlyPawn` | QueryAndPhysics | WorldDynamic | 大部分Block | 在BlockAll基础上忽略Pawn |
| `OverlapOnlyPawn` | QueryAndPhysics | WorldDynamic | 大部分Ignore | 只对Pawn产生Overlap响应 |
| `Pawn` | QueryAndPhysics | Pawn | Block | 玩家/敌人角色的默认预设 |
| `Spectator` | QueryOnly | Pawn | Ignore | 观战模式（可以穿墙） |
| `CharacterMesh` | QueryAndPhysics | PhysicsBody | Block | 角色网格体的碰撞 |
| `PhysicsActor` | QueryAndPhysics | PhysicsBody | Block | 受物理模拟的Actor |
| `DestructibleActor` | QueryAndPhysics | Destructible | Block | 可破碎的物体 |
| `Vehicle` | QueryAndPhysics | Vehicle | Block | 载具 |
| `UI` | QueryOnly | WorldDynamic | Ignore | UI交互射线检测（只检测不阻挡） |
| `Custom` | 自定义 | 自定义 | 自定义 | 自定义配置（最灵活） |

### C++中使用碰撞预设

```cpp
// ==================== 碰撞预设的C++操作 ====================

#include "Components/StaticMeshComponent.h"
#include "BodySetupEnums.h"  // 包含 ECollisionEnabled 定义

void AMyActor::UseCollisionPresets()
{
    UStaticMeshComponent* Mesh = FindComponentByClass<UStaticMeshComponent>();
    if (!Mesh) return;

    // --- 方法1：按名称设置预设（推荐）---
    // 使用引擎预定义的预设名称字符串
    // 这种方式最稳定，与编辑器中的预设下拉菜单对应

    // 设置为"阻挡所有"预设 —— 这个物体会阻挡一切
    Mesh->SetCollisionProfileName(FName(TEXT("BlockAll")));
    // FName是UE中高效字符串标识符
    // TEXT宏确保跨平台字符串兼容

    // 设置为"Pawn"预设 —— 适用于角色
    Mesh->SetCollisionProfileName(FName(TEXT("Pawn")));

    // 设置为"仅重叠所有" —— 适用于触发区域
    Mesh->SetCollisionProfileName(FName(TEXT("OverlapAll")));

    // 设置为"无碰撞" —— 完全透明
    Mesh->SetCollisionProfileName(FName(TEXT("NoCollision")));


    // --- 方法2：使用UPROPERTY宏设置碰撞预设名称 ---
    // 在类的声明中直接指定默认预设
    // 这样在构造函数中就会自动应用

    // （这是一个示例，应该在类的声明中写，不要放在普通函数里）
    // UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "Collision")
    // UStaticMeshComponent* MyMesh;
    // ...
    // 在构造函数中：
    // MyMesh->SetCollisionProfileName(UCollisionProfile::Pawn_ProfileName);

    // 使用UE提供的静态FName常量（比手写字符串更安全）：
    // UCollisionProfile::BlockAll_ProfileName       // "BlockAll"
    // UCollisionProfile::BlockAllDynamic_ProfileName
    // UCollisionProfile::NoCollision_ProfileName    // "NoCollision"
    // UCollisionProfile::OverlapAll_ProfileName     // "OverlapAll"
    // UCollisionProfile::Pawn_ProfileName           // "Pawn"
    // UCollisionProfile::Vehicle_ProfileName        // "Vehicle"
    // UCollisionProfile::PhysicsActor_ProfileName   // "PhysicsActor"
}
```

### 碰撞预设文件的位置

碰撞预设的配置存储在以下位置：
- **引擎预设**（所有项目通用）：`Engine/Config/BaseEngine.ini` 的 `[/Script/Engine.CollisionProfile]` 节
- **项目预设**（仅当前项目）：`<你的项目>/Config/DefaultEngine.ini` 的 `[/Script/Engine.CollisionProfile]` 节

```
# 引擎预设示例（BaseEngine.ini 片段）
[/Script/Engine.CollisionProfile]
+Profiles=(Name="NoCollision",CollisionEnabled=NoCollision,ObjectTypeName="WorldStatic",...)
+Profiles=(Name="BlockAll",CollisionEnabled=QueryAndPhysics,ObjectTypeName="WorldStatic",...)
+Profiles=(Name="OverlapAll",CollisionEnabled=QueryOnly,ObjectTypeName="WorldStatic",...)
# ... 更多预设定义
```

## 自定义碰撞通道

### 为什么要自定义碰撞通道？

内置的8个通道在大型项目中经常不够用。比如你需要：
- 一个专门的"弹幕子弹"通道（让玩家子弹和敌人子弹走不同通道）
- 一个"技能区域"通道（让技能范围的射线检测不与普通物体混淆）
- 一个"UI射线"通道（和可见性检测分开）

### 在Project Settings中创建自定义通道

**操作步骤（图文级详细）**：

1. 打开UE编辑器，点击顶部菜单栏的 **"Edit"（编辑）** → **"Project Settings..."（项目设置）**
2. 在弹出的设置窗口中，左侧导航栏找到 **"Engine"（引擎）** 分类 → 点击 **"Collision"（碰撞）**
3. 你会看到两个关键区域：
   - **"Object Channels"（物体通道）**：在这里新建自定义的碰撞通道类别
   - **"Trace Channels"（射线检测通道）**：在这里新建用于射线检测的专用通道
4. 点击 **"New Object Channel..."**（新建物体通道）按钮
5. 输入通道名称（例如 `Collectible`），选择默认响应（如 `Block`）
6. 点击 **"New Trace Channel..."**（新建射线检测通道）按钮
7. 输入通道名称（例如 `WeaponTrace`），选择默认响应（如 `Ignore`）
8. 创建完毕后，回到任意组件的碰撞预设编辑界面，你应该能在下拉菜单中看到新通道
9. 同时，之前的碰撞预设项中会自动出现该通道的响应配置列

### 在C++中使用自定义通道

```cpp
// ==================== 使用自定义碰撞通道 ====================

// ---- 方法1：通过名称字符串获取通道（编辑器定义的名称） ----
// 前提：你在 Project Settings → Collision 中创建了名为 "Collectible" 的物体通道

// 获取自定义通道的枚举值
// COLLISION_OBJECT_TYPE 返回 ECollisionChannel 类型
ECollisionChannel MyCustomChannel = UEngineTypes::ConvertToCollisionChannel(
    ECC_GameTraceChannel1  // 自定义通道1（对应编辑器创建的第一个自定义通道）
);
// ECC_GameTraceChannel1 ~ ECC_GameTraceChannel18
// 对应编辑器中的自定义Object Channel的1~18号


// ---- 方法2：使用自定义Profile名称 ----
// 前提：你在 DefaultEngine.ini 或 Project Settings 中定义了名为 "MyCustomProfile" 的预设

// 直接用Profile名称设置，引擎会自动查找对应的预设配置
MyComponent->SetCollisionProfileName(FName(TEXT("MyCustomProfile")));


// ---- 方法3：完全手动配置碰撞（不使用预设） ----
// 适合需要在运行时动态改变碰撞行为的场景

void AMyActor::ManualCollisionConfig()
{
    UStaticMeshComponent* Mesh = FindComponentByClass<UStaticMeshComponent>();
    if (!Mesh) return;

    // 第1步：开启碰撞（查询+物理）
    Mesh->SetCollisionEnabled(ECollisionEnabled::QueryAndPhysics);

    // 第2步：设置自身通道（我是什么类型？）
    // 使用第一个自定义物体通道
    Mesh->SetCollisionObjectType(ECC_GameTraceChannel1);
    // 假设这个自定义通道是"Collectible"（可拾取物）

    // 第3步：设置对所有通道的默认响应
    // 先全部忽略，再逐个打开需要的
    Mesh->SetCollisionResponseToAllChannels(ECR_Ignore);

    // 第4步：单独设置需要的响应
    // 被世界静态物体阻挡（不会穿墙）
    Mesh->SetCollisionResponseToChannel(ECC_WorldStatic, ECR_Block);

    // 被角色穿过但触发重叠事件（玩家过来拾取）
    Mesh->SetCollisionResponseToChannel(ECC_Pawn, ECR_Overlap);

    // 忽略其他可拾取物（两个拾取物不会互相碰撞）
    Mesh->SetCollisionResponseToChannel(ECC_GameTraceChannel1, ECR_Ignore);
}
```

### 完整配置示例：一个可拾取物品的碰撞设置

```cpp
// ==================== 完整实战示例：可拾取物品 ====================

// PickupItem.h
#pragma once
#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "PickupItem.generated.h"

UCLASS()
class MYGAME_API APickupItem : public AActor
{
    GENERATED_BODY()

public:
    APickupItem();

protected:
    // 静态网格体 —— 物品的视觉表现
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "Components")
    UStaticMeshComponent* ItemMesh;

    // 球体碰撞体 —— 检测玩家是否走进拾取范围
    // 注意：用球体比用盒子更符合"圆形拾取范围"的直觉
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "Components")
    USphereComponent* PickupTrigger;

    // 重叠事件回调 —— 玩家进入拾取范围时调用
    UFUNCTION()
    void OnPickupRangeEntered(
        UPrimitiveComponent* OverlappedComponent,
        AActor* OtherActor,
        UPrimitiveComponent* OtherComp,
        int32 OtherBodyIndex,
        bool bFromSweep,
        const FHitResult& SweepResult
    );
};

// PickupItem.cpp
#include "PickupItem.h"
#include "Components/StaticMeshComponent.h"
#include "Components/SphereComponent.h"

APickupItem::APickupItem()
{
    // 允许每帧Tick（以便做旋转动画等）
    PrimaryActorTick.bCanEverTick = true;

    // ===== 第1步：创建根组件（球体碰撞）=====
    // 根组件使用球体碰撞，因为拾取范围通常是圆形的
    PickupTrigger = CreateDefaultSubobject<USphereComponent>(TEXT("PickupTrigger"));
    RootComponent = PickupTrigger;  // 设为根组件，物品的位置由这个球体决定

    // 设置球体半径：80厘米（约一步的距离即可拾取）
    PickupTrigger->SetSphereRadius(80.0f);

    // ===== 第2步：配置碰撞 =====
    // 碰撞模式：仅查询（只需要检测重叠，不需要物理碰撞）
    PickupTrigger->SetCollisionEnabled(ECollisionEnabled::QueryOnly);

    // 自身类型标记：WorldDynamic（我是动态世界物体）
    PickupTrigger->SetCollisionObjectType(ECC_WorldDynamic);

    // 默认忽略所有通道的碰撞响应
    // 然后单独打开Pawn的重叠响应
    PickupTrigger->SetCollisionResponseToAllChannels(ECR_Ignore);
    // 只对Pawn（玩家/AI角色）产生重叠响应
    PickupTrigger->SetCollisionResponseToChannel(ECC_Pawn, ECR_Overlap);
    // 注意：ECR_Overlap 表示"不阻挡但触发事件"

    // 开启重叠事件生成（这是关键！不开启的话上面的Overlap就白设置了）
    PickupTrigger->SetGenerateOverlapEvents(true);


    // ===== 第3步：创建物品的视觉网格体 =====
    ItemMesh = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("ItemMesh"));
    // 把网格体挂载到根组件（球体碰撞）下面
    ItemMesh->SetupAttachment(RootComponent);

    // 视觉网格体不需要碰撞（碰撞检测由PickupTrigger负责）
    ItemMesh->SetCollisionEnabled(ECollisionEnabled::NoCollision);
    // 这种"碰撞和视觉分离"的设计模式非常常见：
    // 碰撞用简单的球体/盒子/胶囊体（性能好）
    // 视觉用复杂的StaticMesh（好看）


    // ===== 第4步：绑定重叠事件 =====
    // 当有Actor进入球体范围时，自动调用OnPickupRangeEntered函数
    PickupTrigger->OnComponentBeginOverlap.AddDynamic(
        this,                           // 绑定的对象（this = APickupItem实例）
        &APickupItem::OnPickupRangeEntered  // 要调用的函数指针
    );
}

// ===== 第5步：实现重叠回调 =====
void APickupItem::OnPickupRangeEntered(
    UPrimitiveComponent* OverlappedComponent,  // 被重叠的组件（这里是PickupTrigger）
    AActor* OtherActor,                        // 进入范围的Actor（可能是玩家）
    UPrimitiveComponent* OtherComp,            // 对方具体是哪个组件
    int32 OtherBodyIndex,                      // 对方组件的BodyIndex（通常不关心）
    bool bFromSweep,                          // 是否由扫描触发（vs 直接放置触发）
    const FHitResult& SweepResult             // 扫描结果（包含命中点、法线等）
)
{
    // 判断进入范围的是不是玩家
    // 在实际项目中你会用 Cast<AMyPlayerCharacter>(OtherActor) 来判断
    if (OtherActor && OtherActor->ActorHasTag(FName(TEXT("Player"))))
    {
        // 玩家进入了拾取范围！
        // 在这里处理拾取逻辑：播放音效、增加道具、销毁物品等
        UE_LOG(LogTemp, Warning, TEXT("玩家进入拾取范围，准备拾取物品！"));

        // 典型处理：
        // 1. 给玩家增加道具/属性
        // 2. 播放拾取动画/音效
        // 3. 销毁本物品（Destroy()）
        Destroy();  // 从世界中移除这个物品
    }
}
```

## 碰撞体形状

UE5提供三种基本碰撞体形状。每种形状都有对应的组件类。

### 三种碰撞体对比

| 形状 | 组件类 | 参数 | 适用场景 | 性能 |
|------|--------|------|----------|:---:|
| **球体 (Sphere)** | `USphereComponent` | 半径(Radius) | 角色碰撞、拾取范围、爆炸范围 | ⭐⭐⭐ 最快 |
| **盒子 (Box)** | `UBoxComponent` | 半长(Extent X/Y/Z) | 触发器区域、建筑物碰撞 | ⭐⭐⭐ 最快 |
| **胶囊体 (Capsule)** | `UCapsuleComponent` | 半径(Radius) + 半高(HalfHeight) | 人形角色碰撞 | ⭐⭐ 较快 |

> **性能提示**：球体碰撞计算最快，其次是盒子，胶囊体稍慢（但仍比复杂三角面碰撞快得多）。尽量使用简单碰撞体替代复杂网格碰撞。

### 形状创建与配置

```cpp
// ==================== 碰撞体形状完整示例 ====================

#include "Components/BoxComponent.h"
#include "Components/SphereComponent.h"
#include "Components/CapsuleComponent.h"

void AMyActor::CreateCollisionShapes()
{
    // ===== 1. 球体碰撞体（Sphere Collision）=====
    // 最适合：圆形检测范围、爆炸范围、球状物体的碰撞
    USphereComponent* MySphere = CreateDefaultSubobject<USphereComponent>(TEXT("MySphere"));
    MySphere->SetupAttachment(RootComponent);

    // 设置球体半径（厘米）
    MySphere->SetSphereRadius(50.0f);
    // 半径50cm = 直径1米的球体

    // 球体碰撞检测的数学原理（底层）：
    // 判断点是否在球体内：(Point - Center).Length() <= Radius
    // 判断两个球体是否相交：(Center1 - Center2).Length() <= (R1 + R2)
    // 计算量极小，性能最优


    // ===== 2. 盒子碰撞体（Box Collision）=====
    // 最适合：方形触发器区域、建筑物、方形容器
    UBoxComponent* MyBox = CreateDefaultSubobject<UBoxComponent>(TEXT("MyBox"));
    MyBox->SetupAttachment(RootComponent);

    // 设置盒子半长（Half Extent，即从中心到每个面的距离）
    MyBox->SetBoxExtent(FVector(100.0f, 50.0f, 30.0f));
    // 这个盒子实际大小 = 200cm x 100cm x 60cm（每个方向是半长的两倍）
    // X方向(前/后) = 100*2 = 200cm
    // Y方向(左/右) = 50*2  = 100cm
    // Z方向(上/下) = 30*2  = 60cm

    // ⚠️ 常见错误：把SetBoxExtent的参数当成完整尺寸
    // ❌ 错误：MyBox->SetBoxExtent(FVector(200, 100, 60)); // 实际变成了400x200x120！
    // ✅ 正确：MyBox->SetBoxExtent(FVector(100, 50, 30));   // 得到200x100x60的盒子


    // ===== 3. 胶囊体碰撞体（Capsule Collision）=====
    // 最适合：人形角色（上下是半球+中间是圆柱体）
    UCapsuleComponent* MyCapsule = CreateDefaultSubobject<UCapsuleComponent>(TEXT("MyCapsule"));
    MyCapsule->SetupAttachment(RootComponent);

    // 设置胶囊体尺寸：
    // 参数1：半径（厘米）—— 胶囊体的"宽度"
    // 参数2：半高（厘米）—— 胶囊体圆柱部分的一半高度（不含两端的半球）
    MyCapsule->SetCapsuleSize(34.0f, 88.0f);
    // 半径34cm ≈ 标准人体宽度
    // 半高88cm → 总高度 = 88*2 + 34*2 = 244cm（含两端半球）
    //   - 圆柱部分高度 = 88*2 = 176cm
    //   - 上端半球半径 = 34cm，下端半球半径 = 34cm
    //   - 总高度 = 176 + 34 + 34 = 244cm

    // 也可以分别修改半径和半高：
    MyCapsule->SetCapsuleRadius(34.0f);       // 只改半径
    MyCapsule->SetCapsuleHalfHeight(88.0f);   // 只改半高


    // ===== 4. 使用已有StaticMesh的简单碰撞体 =====
    // StaticMesh自带简单碰撞体（在建模软件或UE编辑器中设置的）
    UStaticMeshComponent* StaticMesh = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("StaticMesh"));
    StaticMesh->SetupAttachment(RootComponent);

    // 设置碰撞响应时使用的是StaticMesh自身绑定的简单碰撞几何体
    StaticMesh->SetCollisionEnabled(ECollisionEnabled::QueryAndPhysics);
    StaticMesh->SetCollisionProfileName(TEXT("BlockAll"));

    // 注意：StaticMesh的碰撞精度取决于它的"简单碰撞体(Simple Collision)"
    // 在StaticMesh编辑器中可以看到/编辑这些简单碰撞体
    // 可以使用多个碰撞体组合来近似表示复杂形状
}
```

### 复杂碰撞 vs 简单碰撞

```cpp
// ==================== 简单碰撞 vs 复杂碰撞 ====================

// 简单碰撞（Simple Collision）：
// - 由球体、盒子、胶囊体等基本形状组成
// - 计算速度快（适合物理模拟和一般碰撞检测）
// - 不精确，只是一个近似形状
// - 在StaticMesh编辑器中通过"Collision"菜单添加
// - 是默认的碰撞检测方式

// 复杂碰撞（Complex Collision）：
// - 使用模型的实际三角面进行碰撞检测
// - 计算速度慢（每个三角面都要检测）
// - 完全精确（用于精确的射线命中点判断等）
// - 仅在射线检测的 bTraceComplex=true 时启用
// - ❌ 不能用于物理模拟（物理模拟必须用简单碰撞）

// 射线检测中启用复杂碰撞的示例：
// 详见第2章"射线检测"中的 bTraceComplex 参数说明
```

## Generate Overlap Events设置

`bGenerateOverlapEvents`（生成重叠事件）是一个经常被新手忽略但极其重要的设置。

### 什么是Generate Overlap Events？

```cpp
// ==================== 重叠事件的开关 ====================

// 核心概念：
// 即使你把碰撞响应设置为"Overlap"
// 如果 GenerateOverlapEvents = false
// 那么 OnBeginOverlap 和 OnEndOverlap 事件也永远不会触发！

// 这是一个双重开关机制：
// 第1层：碰撞响应 → 必须设为 Overlap（告诉引擎"允许重叠"）
// 第2层：GenerateOverlapEvents → 必须设为 true（告诉引擎"重叠时通知我"）

// ❌ 常见错误配置：
MyComponent->SetCollisionResponseToChannel(ECC_Pawn, ECR_Overlap);  // 响应设为Overlap
// 但是！GenerateOverlapEvents 默认为 false
// 结果：玩家可以穿过，但 OnBeginOverlap 不会被调用！
// 新手常常因此困惑："明明设置了Overlap为什么没反应？"

// ✅ 正确配置：
MyComponent->SetCollisionResponseToChannel(ECC_Pawn, ECR_Overlap);  // 响应设为Overlap
MyComponent->SetGenerateOverlapEvents(true);  // 必须开启事件生成！
```

### 使用碰撞预设时的注意事项

```cpp
// ==================== 预设与重叠事件的关系 ====================

// 好消息：大多数内置碰撞预设会自动设置 GenerateOverlapEvents
// - "OverlapAll" 预设 → GenerateOverlapEvents 自动为 true
// - "OverlapAllDynamic" 预设 → GenerateOverlapEvents 自动为 true
// - "Trigger" 相关的预设 → GenerateOverlapEvents 自动为 true

// 但如果手动配置碰撞响应，就必须手动开启 GenerateOverlapEvents

// ---- 完整的安全配置示例 ----
void AMyActor::SafeTriggerConfiguration()
{
    USphereComponent* Trigger = CreateDefaultSubobject<USphereComponent>(TEXT("Trigger"));

    // 方法A：使用预设（最安全，不会漏掉设置）
    Trigger->SetCollisionProfileName(FName(TEXT("OverlapAll")));
    // "OverlapAll" 预设内部已经：
    //   - 设置了 CollisionEnabled = QueryOnly
    //   - 设置了所有通道为 Overlap 响应
    //   - 设置了 GenerateOverlapEvents = true

    // 方法B：手动配置（注意不要漏掉 GenerateOverlapEvents）
    Trigger->SetCollisionEnabled(ECollisionEnabled::QueryOnly);
    Trigger->SetCollisionResponseToAllChannels(ECR_Overlap);
    Trigger->SetGenerateOverlapEvents(true);  // ← 这行不要漏！
}
```

### 重叠事件性能提示

```cpp
// ==================== 性能优化建议 ====================

// ⚠️ 性能陷阱：不要给所有物体都开 GenerateOverlapEvents
// 引擎每一帧都需要检查开启重叠事件的物体是否与其他物体重叠
// 如果场景中有1000个开启重叠事件的物体，每一帧都要做大量的碰撞检测

// ✅ 最佳实践：
// 1. 只在需要触发事件的组件上开启 GenerateOverlapEvents
// 2. 只对需要的通道设置 Overlap 响应（不要用 OverlapAll 除非确实需要）
// 3. 不需要时关闭重叠事件：

void AMyActor::DisableOverlapEventsWhenIdle()
{
    // 假设这个门只在玩家靠近时才有交互
    // 平时可以关闭重叠事件省性能
    UBoxComponent* InteractionZone = FindComponentByClass<UBoxComponent>();
    if (InteractionZone)
    {
        // 关闭重叠事件检测
        InteractionZone->SetGenerateOverlapEvents(false);
        // 注意：这样OnBeginOverlap就不会再触发了
        // 适合"暂时不需要检测"的场景（如门已打开）
    }
}
```

### 重叠事件API完整绑定示例

```cpp
// ==================== 重叠事件绑定完整流程 ====================

// ---- 头文件声明 ----
// 声明重叠事件回调函数（必须用 UFUNCTION 宏标记）
UFUNCTION()
void OnOverlapBegin(
    UPrimitiveComponent* OverlappedComponent,  // 本组件
    AActor* OtherActor,                        // 进入范围的Actor
    UPrimitiveComponent* OtherComp,            // 对方的具体组件
    int32 OtherBodyIndex,                      // Body实例索引（物理用）
    bool bFromSweep,                          // 是否由移动扫描触发
    const FHitResult& SweepResult             // 扫描命中结果
);

UFUNCTION()
void OnOverlapEnd(
    UPrimitiveComponent* OverlappedComponent,  // 本组件
    AActor* OtherActor,                        // 离开范围的Actor
    UPrimitiveComponent* OtherComp,            // 对方的具体组件
    int32 OtherBodyIndex                       // Body实例索引
);

// ---- 构造函数中绑定事件 ----
AMyActor::AMyActor()
{
    MyCollision->SetGenerateOverlapEvents(true);  // 第1步：开启重叠事件

    // 第2步：绑定开始重叠事件
    MyCollision->OnComponentBeginOverlap.AddDynamic(this, &AMyActor::OnOverlapBegin);
    // AddDynamic 是UE的委托绑定机制
    // 参数1: this → 回调对象是当前Actor实例
    // 参数2: &AMyActor::OnOverlapBegin → 回调函数的指针

    // 第3步：绑定结束重叠事件
    MyCollision->OnComponentEndOverlap.AddDynamic(this, &AMyActor::OnOverlapEnd);
}

// ---- 重叠回调实现 ----
void AMyActor::OnOverlapBegin(
    UPrimitiveComponent* OverlappedComponent,
    AActor* OtherActor,
    UPrimitiveComponent* OtherComp,
    int32 OtherBodyIndex,
    bool bFromSweep,
    const FHitResult& SweepResult)
{
    // 安全检查：确保OtherActor有效
    if (OtherActor && (OtherActor != this))
    {
        UE_LOG(LogTemp, Log, TEXT("%s 进入了 %s 的范围"),
            *OtherActor->GetName(),    // 进入者的名字
            *GetName()                  // 本Actor的名字
        );
    }
}

void AMyActor::OnOverlapEnd(
    UPrimitiveComponent* OverlappedComponent,
    AActor* OtherActor,
    UPrimitiveComponent* OtherComp,
    int32 OtherBodyIndex)
{
    if (OtherActor && (OtherActor != this))
    {
        UE_LOG(LogTemp, Log, TEXT("%s 离开了 %s 的范围"),
            *OtherActor->GetName(),
            *GetName()
        );
    }
}
```
