## 3.2 连击系统

### 3.2.1 连击设计

我们的连击系统支持以下连段：

```
轻攻击连段（L=Light Attack, 轻攻击）：
  L1 → L2 → L3          （轻→轻→轻，三连击，伤害递增：1.0x → 1.2x → 1.5x）

混合连段（H=Heavy Attack, 重攻击）：
  L1 → L2 → H            （轻→轻→重，触发上挑终结技，1.8x伤害 + 浮空）
  L1 → H → H             （轻→重→重，横斩+纵劈，2.0x最终伤害）

重攻击起手：
  H → L → L              （重→轻→轻，破防起手，后续伤害提升20%）

终结规则：
  - 任何连段的第三击都附带额外效果（硬直/浮空/破防）
  - 连击窗口：在前一击结束前0.3秒到结束后0.5秒之间按键有效
```

// ============================================================================
// DungeonCombatComponent.cpp — 战斗组件实现
// ============================================================================

#include "DungeonCombatComponent.h"
#include "DungeonCharacter.h"      // 玩家角色类
#include "DungeonDamageSystem.h"   // 伤害系统（本草后面实现）

#include "Animation/AnimMontage.h"

// ── 构造函数 ──
UDungeonCombatComponent::UDungeonCombatComponent()
{
    // 允许Tick更新
    PrimaryComponentTick.bCanEverTick = true;
    // 不需要每帧都更新，但有些连击窗口检查需要
    PrimaryComponentTick.TickInterval = 0.05f;  // 每秒20次检查，足够精确
}

// ── BeginPlay ──
void UDungeonCombatComponent::BeginPlay()
{
    Super::BeginPlay();

    // 获取拥有者角色引用（方便后续频繁使用）
    OwnerCharacter = Cast<ADungeonCharacter>(GetOwner());
    if (!OwnerCharacter)
    {
        UE_LOG(LogTemp, Error, TEXT("DungeonCombatComponent 附加到了非 ADungeonCharacter 的Actor上！"));
    }
}

// ── TickComponent ──
void UDungeonCombatComponent::TickComponent(float DeltaTime, ELevelTick TickType,
    FActorComponentTickFunction* ThisTickFunction)
{
    Super::TickComponent(DeltaTime, TickType, ThisTickFunction);

    // ── 清理过期的输入缓冲 ──
    if (InputBuffer.bIsValid())
    {
        float CurrentTime = GetWorld()->GetTimeSeconds();
        // 如果缓冲的输入已经超过有效期，清除它
        if (CurrentTime - InputBuffer.BufferedTime > InputBufferValidityDuration)
        {
            InputBuffer.BufferedAttack = EAttackType::None;
        }
    }

    // ── 格挡体力消耗 ──
    if (bIsBlocking && OwnerCharacter)
    {
        float StaminaCost = BlockStaminaCostPerSecond * DeltaTime;
        // 如果体力不足，强制停止格挡
        if (OwnerCharacter->GetMaxStamina() - OwnerCharacter->CurrentStamina < StaminaCost)
        {
            StopBlocking();
        }
        else
        {
            OwnerCharacter->CurrentStamina -= StaminaCost;
        }
    }
}

// ============================================================================
// 攻击请求 — 面向玩家的输入接口
// ============================================================================

// ── RequestLightAttack ──
// 玩家按下轻攻击键时调用
// 这个函数处理输入缓冲逻辑：
//   如果当前在连击窗口内 → 直接接续下一段
//   如果当前不在窗口内 → 发起新连击的第一段
//   如果当前在攻击动画中 → 缓冲输入，等动画结束自动接续
void UDungeonCombatComponent::RequestLightAttack()
{
    if (!OwnerCharacter || OwnerCharacter->CurrentState == ADungeonCharacter::ECharacterState::Dead)
    {
        return;
    }

    // ── 情况1：当前正在攻击动画中，缓冲此输入 ──
    if (bIsAttacking)
    {
        // 将轻攻击存入缓冲
        InputBuffer.BufferedAttack = EAttackType::Light;
        InputBuffer.BufferedTime = GetWorld()->GetTimeSeconds();
        UE_LOG(LogTemp, Verbose, TEXT("输入缓冲：轻攻击已缓冲"));
        return;
    }

    // ── 情况2：当前在连击窗口中，接续下一段 ──
    if (CurrentComboStep >= 0)
    {
        // 检查当前段是否允许接轻攻击
        int32* NextStepPtr = NextStepMap.Find(EAttackType::Light);
        if (NextStepPtr && *NextStepPtr >= 0)
        {
            ExecuteComboStep(*NextStepPtr);
            return;
        }
    }

    // ── 情况3：不在任何连击中，开启新连击（轻攻击起手第一段）──
    ResetCombo();                 // 先清空之前的连击状态
    ExecuteComboStep(0);          // 索引0 = 轻攻击第一段
}

// ── RequestHeavyAttack ──
// 玩家按下重攻击键时调用
// 逻辑与轻攻击相同，只是攻击类型为Heavy
void UDungeonCombatComponent::RequestHeavyAttack()
{
    if (!OwnerCharacter || OwnerCharacter->CurrentState == ADungeonCharacter::ECharacterState::Dead)
    {
        return;
    }

    // ── 缓冲输入 ──
    if (bIsAttacking)
    {
        InputBuffer.BufferedAttack = EAttackType::Heavy;
        InputBuffer.BufferedTime = GetWorld()->GetTimeSeconds();
        UE_LOG(LogTemp, Verbose, TEXT("输入缓冲：重攻击已缓冲"));
        return;
    }

    // ── 尝试接续连击 ──
    if (CurrentComboStep >= 0)
    {
        int32* NextStepPtr = NextStepMap.Find(EAttackType::Heavy);
        if (NextStepPtr && *NextStepPtr >= 0)
        {
            ExecuteComboStep(*NextStepPtr);
            return;
        }
    }

    // ── 重攻击起手 ──
    ResetCombo();

    // 索引1 = 重攻击起手段
    // 实际项目中，应该通过查找表确定重攻击起手段的索引
    ExecuteComboStep(1);
}

// ── ResetCombo ──
// 重置连击状态（窗口过期或连击完成时调用）

{

    bIsAttacking = false;
    bIsInHitWindow = false;

    InputBuffer.BufferedAttack = EAttackType::None;

}

// ── OnAttackAnimEnd ──
// 动画通知触发：攻击动画播放完毕
void UDungeonCombatComponent::OnAttackAnimEnd()
{
    // 标记攻击结束
    bIsAttacking = false;
    ActiveComboStep = -1;

    // ── 检查是否有缓冲的输入 ──
    if (InputBuffer.bIsValid())
    {
        EAttackType BufferedType = InputBuffer.BufferedAttack;
        InputBuffer.BufferedAttack = EAttackType::None;  // 消费缓冲

        // 检查缓冲的攻击类型是否可以接续连击
        int32* NextStepPtr = NextStepMap.Find(BufferedType);
        if (NextStepPtr && *NextStepPtr >= 0)
        {
            // 可以接续！
            ExecuteComboStep(*NextStepPtr);
            return;  // 不重置连击
        }
    }

    // ── 开启连击窗口计时器 ──
    // 在 ComboWindow 时间内，玩家还可以按键接续连击
    // 通过定时器来管理窗口，超时后自动重置
    float ComboWindowDuration = 0.5f;  // 实际应从配置读取
    GetWorld()->GetTimerManager().SetTimer(
        ComboWindowTimerHandle,
        this,
        &UDungeonCombatComponent::ResetCombo,  // 回调
        ComboWindowDuration,
        false                                   // 不循环
    );

    // 恢复到Idle状态
    if (OwnerCharacter)
    {
        OwnerCharacter->SetCharacterState(ADungeonCharacter::ECharacterState::Idle);
    }
}

// ── StartBlocking ──
// 开始格挡（按住右键）
void UDungeonCombatComponent::StartBlocking()
{
    if (!OwnerCharacter || bIsAttacking)
    {
        return;  // 攻击中不能格挡
    }

    // 检查体力（至少要有10点体力才能开始格挡）
    if (OwnerCharacter->CurrentStamina < 10.0f)
    {
        return;
    }

    bIsBlocking = true;
    BlockStartTime = GetWorld()->GetTimeSeconds();
    OwnerCharacter->SetCharacterState(ADungeonCharacter::ECharacterState::Blocking);

    UE_LOG(LogTemp, Log, TEXT("开始格挡！时间戳: %f"), BlockStartTime);
}

// ── StopBlocking ──
// 停止格挡（松开右键）
void UDungeonCombatComponent::StopBlocking()
{
    if (!bIsBlocking) return;

    bIsBlocking = false;
    OwnerCharacter->SetCharacterState(ADungeonCharacter::ECharacterState::Idle);

    UE_LOG(LogTemp, Log, TEXT("停止格挡"));
}

// ── TryPerfectBlock ──
// 尝试完美格挡判定
// 在受到攻击时调用，检查格挡开始时间是否在完美格挡窗口内
// 返回 true = 完美格挡成功（0伤害 + 弹反效果）
// 返回 false = 普通格挡（减伤）或没有格挡（正常受伤）
bool UDungeonCombatComponent::TryPerfectBlock()
{
    if (!bIsBlocking)
    {
        return false;  // 根本没格挡
    }

    float CurrentTime = GetWorld()->GetTimeSeconds();
    float TimeSinceBlockStart = CurrentTime - BlockStartTime;

    // 检查是否在完美格挡窗口内（300ms内）
    if (TimeSinceBlockStart <= PerfectBlockWindow)
    {
        UE_LOG(LogTemp, Log, TEXT("★★★ 完美格挡！★★★ 反应时间: %f 秒"), TimeSinceBlockStart);

        // 完美格挡成功后的效果：
        // 1. 不受伤害
        // 2. 使攻击者进入短暂硬直
        // 3. 下一次反击伤害+50%
        // 这些效果在 HandleIncomingDamage 中处理

        return true;
    }

    // 在完美窗口之外 = 普通格挡（正常减伤）
    return false;
}

// ============================================================================
// 伤害处理 — 角色受到伤害的入口
// ============================================================================

// ── HandleIncomingDamage ──
// 当角色被击中时调用此函数
// RawDamage: 未经处理的原始伤害
// DamageCauser: 造成伤害的Actor（用于反击/仇恨等逻辑）
// bIsUnblockable: 不可格挡的攻击（Boss大招等）
// Element: 伤害元素类型（物理/火焰/冰霜/雷电/毒素）
void UDungeonCombatComponent::HandleIncomingDamage(
    float RawDamage,
    AActor* DamageCauser,
    bool bIsUnblockable,
    EDamageElement Element)
{
    if (!OwnerCharacter) return;

    // 如果角色已死亡，忽略伤害
    if (OwnerCharacter->CurrentState == ADungeonCharacter::ECharacterState::Dead)
    {
        return;
    }

    float FinalDamage = RawDamage;

    // ── 闪避无敌帧检查 ──
    if (bIsDodging)
    {
        UE_LOG(LogTemp, Log, TEXT("闪避无敌！完全免疫伤害"));
        return;  // 直接无视所有伤害
    }

    // ── 完美格挡检查 ──
    if (!bIsUnblockable && TryPerfectBlock())
    {
        // 完美格挡：不受伤害，并触发反击效果
        // 使攻击者硬直
        if (DamageCauser)
        {
            // 假设攻击者有 DungeonCombatComponent，使其硬直
            // 实际项目中使用接口或父类统一处理
        }
        // 标记下一次反击伤害加成
        // bHasPerfectBlockCounterBonus = true;

        UE_LOG(LogTemp, Log, TEXT("完美格挡成功！伤害免疫 + 反击加成"));
        return;  // 不受伤害
    }

    // ── 普通格挡减伤 ──
    if (bIsBlocking && !bIsUnblockable)
    {
        // 普通格挡减少70%伤害
        FinalDamage *= 0.3f;  // 只受到30%的伤害

        // 格挡消耗额外体力（与伤害量相关）
        float BlockStaminaCost = FinalDamage * 0.5f;
        OwnerCharacter->CurrentStamina = FMath::Max(
            OwnerCharacter->CurrentStamina - BlockStaminaCost, 0.0f);

        // 如果体力耗尽，自动停止格挡
        if (OwnerCharacter->CurrentStamina <= 0.0f)
        {
            StopBlocking();
            // 体力耗尽时格挡破防，受到全额伤害
            FinalDamage = RawDamage;
            UE_LOG(LogTemp, Warning, TEXT("体力耗尽，格挡被破防！"));
        }

        UE_LOG(LogTemp, Log, TEXT("格挡减伤: %f → %f"), RawDamage, FinalDamage);
    }

    // ── 防御力减伤 ──
    float Defense = OwnerCharacter->GetTotalDefense();
    // 伤害公式：最终伤害 = 原始伤害 * (100 / (100 + 防御力))
    // 这个公式确保了防御力不会导致0伤害，同时递减收益
    if (Defense > 0.0f)
    {
        FinalDamage = FinalDamage * (100.0f / (100.0f + Defense));
    }

    // 保底伤害（无论如何至少造成1点伤害）
    FinalDamage = FMath::Max(FinalDamage, 1.0f);

    // ── 扣除生命值 ──
    OwnerCharacter->CurrentHealth = FMath::Clamp(
        OwnerCharacter->CurrentHealth - FinalDamage,
        0.0f,
        OwnerCharacter->GetMaxHealth()
    );

    UE_LOG(LogTemp, Log, TEXT("%s 受到 %f 点伤害（原始: %f），剩余生命: %f / %f"),
        *OwnerCharacter->GetName(),
        FinalDamage,
        RawDamage,
        OwnerCharacter->CurrentHealth,
        OwnerCharacter->GetMaxHealth());

    // ── 检查死亡 ──
    if (OwnerCharacter->CurrentHealth <= 0.0f)
    {
        OwnerCharacter->Die();
    }
    else
    {
        // 被击中后短暂硬直
        OwnerCharacter->SetCharacterState(ADungeonCharacter::ECharacterState::Stunned);

        // 0.3秒后恢复
        FTimerHandle ResetStunTimer;
        GetWorld()->GetTimerManager().SetTimer(
            ResetStunTimer,
            [this]()
            {
                if (OwnerCharacter &&
                    OwnerCharacter->CurrentState == ADungeonCharacter::ECharacterState::Stunned)
                {
                    OwnerCharacter->SetCharacterState(ADungeonCharacter::ECharacterState::Idle);
                }
            },
            0.3f,
            false
        );
    }
}

// ============================================================================
// 造成伤害 — 攻击命中敌人时调用
// ============================================================================

// ── DealDamageToTarget ──
// 当玩家的武器碰撞体碰到敌人时调用此函数
void UDungeonCombatComponent::DealDamageToTarget(
    AActor* Target,
    float BaseDamage,
    EAttackType AttackType)
{
    if (!OwnerCharacter || !Target)
    {
        return;
    }

    // ── 获取攻击力 ──
    float AttackPower = OwnerCharacter->GetTotalAttack();

    // ── 获取攻击倍率（根据攻击类型）──
    float AttackMultiplier = 1.0f;
    switch (AttackType)
    {
    case EAttackType::Light:
        AttackMultiplier = 1.0f;    // 轻攻击：100%攻击力
        break;
    case EAttackType::Heavy:
        AttackMultiplier = 2.0f;    // 重攻击：200%攻击力
        break;
    case EAttackType::Special:
        AttackMultiplier = 2.5f;    // 终结技：250%攻击力
        break;
    }

    // ── 暴击判定 ──
    float CriticalChance = OwnerCharacter->GetCriticalChance();  // 如 0.15 = 15%
    bool bIsCritical = FMath::FRand() < CriticalChance;  // 随机数判定

    float CriticalMultiplier = bIsCritical ? 2.0f : 1.0f;  // 暴击伤害 = 2倍

    // ── 伤害浮动（±10%的随机浮动，让战斗不那么"精确"）──
    float RandomFactor = FMath::RandRange(0.9f, 1.1f);

    // ── 计算最终伤害 ──
    float FinalDamage = BaseDamage * AttackPower * AttackMultiplier * CriticalMultiplier * RandomFactor;

    // 处理完美格挡的反击加成
    // if (bHasPerfectBlockCounterBonus)
    // {
    //     FinalDamage *= PerfectBlockCounterDamageMultiplier;
    //     bHasPerfectBlockCounterBonus = false;
    // }

    // ── 将伤害传递给目标 ──
    // 使用静态工具函数 DungeonDamageSystem 来计算最终伤害和施加效果
    UDungeonDamageSystem::ApplyDamageToActor(
        Target,
        FinalDamage,
        OwnerCharacter,    // DamageCauser
        bIsCritical
    );

    // ── 日志 ──
    if (bIsCritical)
    {
        UE_LOG(LogTemp, Warning, TEXT("💥 暴击！造成 %f 点伤害"), FinalDamage);
    }
    else
    {
        UE_LOG(LogTemp, Log, TEXT("造成 %f 点伤害 (%s, 倍率: %.1fx)"),
            FinalDamage,
            bIsCritical ? TEXT("暴击") : TEXT("普通"),
            AttackMultiplier);
    }
}

### 3.3.2 DungeonDamageSystem.cpp — 伤害计算实现

```cpp
// ============================================================================
// DungeonDamageSystem.cpp — 伤害计算实现
// ============================================================================

#include "DungeonDamageSystem.h"
#include "DungeonCombatComponent.h"
#include "GameFramework/Actor.h"
#include "Engine/World.h"

// ============================================================================
// CalculateFinalDamage — 伤害计算核心算法
// ============================================================================
// 伤害计算顺序：
// 1. 基础伤害 × 攻击力倍率
// 2. 暴击判定
// 3. 防御力减伤
// 4. 元素相克加成
// 5. 随机浮动
// 6. 最小值保底（1点）
// ============================================================================
float UDungeonDamageSystem::CalculateFinalDamage(
    float BaseDamage,
    float AttackerAttack,
    float DefenderDefense,
    float CriticalChance,
    float& OutCriticalMultiplier,
    EDamageElement Element,
    EDamageElement DefenderElement)
{
    // ── 第1步：攻击力加成 ──
    // 最终基础伤害 = 基础伤害 × (1 + 攻击力/100)
    float AttackBoost = 1.0f + (AttackerAttack / 100.0f);
    float Damage = BaseDamage * AttackBoost;

    // ── 第2步：暴击判定 ──
    bool bCritical = RollCritical(CriticalChance, OutCriticalMultiplier);
    if (bCritical)
    {
        Damage *= OutCriticalMultiplier;  // 暴击伤害×2（默认）
    }

    // ── 第3步：防御力减伤 ──
    // 防御力减伤公式：减伤比例 = Defense / (Defense + 200)
    // 例如：100防御 = 33%减伤，200防御 = 50%减伤，400防御 = 66%减伤
    float DefenseReduction = GetDefenseReductionFactor(DefenderDefense);
    Damage *= (1.0f - DefenseReduction);

    // ── 第4步：元素相克 ──
    float ElementMultiplier = GetElementalMultiplier(Element, DefenderElement);
    Damage *= ElementMultiplier;

    // ── 第5步：随机浮动（±15%）──
    Damage *= GetRandomDamageVariance();

    // ── 第6步：最小值保底（至少造成1点伤害，防止完全无效）──
    Damage = FMath::Max(Damage, 1.0f);

    // ── 四舍五入为整数（伤害数字显示整数更好看）──
    Damage = FMath::RoundToFloat(Damage);

    return Damage;
}

// ============================================================================
// ApplyDamageToActor — 一站式伤害施加
// ============================================================================
FDamageResult UDungeonDamageSystem::ApplyDamageToActor(
    AActor* Target,
    float Damage,
    AActor* DamageCauser,
    bool bForceCritical,
    EDamageElement Element)
{
    FDamageResult Result;
    Result.FinalDamage = Damage;
    Result.Element = Element;

    if (!Target || !Target->IsValidLowLevel())
    {
        UE_LOG(LogTemp, Warning, TEXT("ApplyDamageToActor: 目标无效！"));
        return Result;
    }

    // ── 检查目标是否有战斗组件 ──
    // 使用 FindComponentByClass 查找组件
    UDungeonCombatComponent* TargetCombat =
        Target->FindComponentByClass<UDungeonCombatComponent>();

    if (TargetCombat)
    {
        // 目标有战斗组件，使用组件的 HandleIncomingDamage 处理
        // 这样格挡、闪避等防御逻辑会被正确触发
        TargetCombat->HandleIncomingDamage(Damage, DamageCauser, false, Element);
    }
    else
    {
        // 目标没有战斗组件（可能是可破坏物、陷阱等）
        // 使用UE原生的伤害系统
        FPointDamageEvent DamageEvent;  // 点伤害事件（传递伤害信息）
        Target->TakeDamage(
            Damage,
            DamageEvent,
            nullptr,            // EventInstigator（没有控制器）
            DamageCauser        // DamageCauser
        );
    }

    // ── 判定结果 ──
    // 这里简化处理，实际应该从回调中获取结果
    Result.FinalDamage = Damage;

    // 检查目标是否死亡（生命值为0）
    // 实际需要从目标获取生命值判断
    // Result.bIsKillingBlow = (TargetHealth <= 0.0f);

    return Result;
}

// ============================================================================
// GetElementalMultiplier — 元素相克计算
// ============================================================================
float UDungeonDamageSystem::GetElementalMultiplier(
    EDamageElement AttackElement,
    EDamageElement DefenderWeakness)
{
    // 物理伤害不受元素克制影响
    if (AttackElement == EDamageElement::Physical)
    {
        return 1.0f;
    }

    // ── 元素相克表 ──
    // 火焰克冰霜、冰霜克雷电、雷电克火焰
    // 这是一个简化的相克关系
    if ((AttackElement == EDamageElement::Fire     && DefenderWeakness == EDamageElement::Ice) ||
        (AttackElement == EDamageElement::Ice      && DefenderWeakness == EDamageElement::Lightning) ||
        (AttackElement == EDamageElement::Lightning && DefenderWeakness == EDamageElement::Fire))
    {
        return 1.5f;  // 克制，伤害+50%
    }

    // 反过来被克制
    if ((AttackElement == EDamageElement::Fire     && DefenderWeakness == EDamageElement::Lightning) ||
        (AttackElement == EDamageElement::Ice      && DefenderWeakness == EDamageElement::Fire) ||
        (AttackElement == EDamageElement::Lightning && DefenderWeakness == EDamageElement::Ice))
    {
        return 0.5f;  // 被克制，伤害-50%
    }

    // 无毒克和被克关系时正常伤害
    return 1.0f;
}

// ============================================================================
// RollCritical — 暴击判定
// ============================================================================
bool UDungeonDamageSystem::RollCritical(float CriticalChance, float& OutCriticalMultiplier)
{
    // FMath::FRand() 返回 [0.0, 1.0) 的随机浮点数
    if (FMath::FRand() < CriticalChance)
    {
        OutCriticalMultiplier = 2.0f;  // 默认暴击倍率200%
        // 可以扩展为根据装备改变暴击倍率
        return true;
    }

    OutCriticalMultiplier = 1.0f;  // 普通伤害
    return false;
}

// ============================================================================
// GetRandomDamageVariance — 伤害浮动
// ============================================================================
float UDungeonDamageSystem::GetRandomDamageVariance()
{
    // 返回0.85到1.15之间的随机值
    // 使每次伤害都有±15%的浮动，避免伤害数字过于"精准"
    return FMath::RandRange(0.85f, 1.15f);
}

// ============================================================================
// GetDefenseReductionFactor — 防御力减伤
// ============================================================================
float UDungeonDamageSystem::GetDefenseReductionFactor(float Defense)
{
    // 减伤公式：DamageReduction = Defense / (Defense + 200)
    // 特点：
    //   - Defense=0   时，减伤=0%
    //   - Defense=100 时，减伤=33%
    //   - Defense=200 时，减伤=50%
    //   - Defense=400 时，减伤=66%
    //   - 边际收益递减：防御越高，每点防御提供的减伤越少
    if (Defense <= 0.0f)
    {
        return 0.0f;  // 没有防御力，不减伤
    }

    return Defense / (Defense + 200.0f);
}

// ============================================================================
// GetDamageNumberColor — 伤害数字颜色
// ============================================================================
FLinearColor UDungeonDamageSystem::GetDamageNumberColor(const FDamageResult& Result)
{
    // 暴击 = 金色
    if (Result.bIsCritical)
    {
        return FLinearColor(1.0f, 0.84f, 0.0f);  // Gold
    }

    // 被格挡 = 灰色
    if (Result.bWasBlocked)
    {
        return FLinearColor(0.6f, 0.6f, 0.6f);  // Gray
    }

    // 根据元素返回颜色
    switch (Result.Element)
    {
    case EDamageElement::Fire:
        return FLinearColor(1.0f, 0.3f, 0.0f);     // 橙红色
    case EDamageElement::Ice:
        return FLinearColor(0.2f, 0.6f, 1.0f);     // 冰蓝色
    case EDamageElement::Lightning:
        return FLinearColor(1.0f, 0.8f, 0.0f);     // 亮黄色
    case EDamageElement::Poison:
        return FLinearColor(0.4f, 1.0f, 0.2f);     // 毒绿色
    default:
        return FLinearColor(1.0f, 1.0f, 1.0f);     // 白色（物理）
    }
}

USTRUCT(BlueprintType)
struct FSkillCooldownInfo
{
    GENERATED_BODY()

    // 冷却总时长（秒）
    UPROPERTY(EditAnywhere, BlueprintReadWrite)
    float CooldownDuration = 5.0f;

    // 冷却剩余时间（秒）→ 0 = 可用，>0 = 冷却中
    UPROPERTY(BlueprintReadOnly)
    float CooldownRemaining = 0.0f;

    // 是否在冷却中
    bool IsOnCooldown() const { return CooldownRemaining > 0.0f; }

    // 开始冷却

        if (Duration > 0.0f) CooldownDuration = Duration;
        CooldownRemaining = CooldownDuration;

    void TickCooldown(float DeltaTime)
    {
        if (CooldownRemaining > 0.0f)

            CooldownRemaining = FMath::Max(0.0f, CooldownRemaining - DeltaTime);
        }
    }

    // 获取冷却进度（0.0 = 刚进入冷却, 1.0 = 冷却完毕）
    float GetCooldownProgress() const
    {
        return CooldownDuration > 0.0f ? (1.0f - CooldownRemaining / CooldownDuration) : 1.0f;
    }
};

// Activate — 激活技能
// ============================================================================
bool UDungeonSkillBase::Activate(ADungeonCharacter* Caster)
{
    // 1. 检查施法者是否有效
    if (!Caster)
    {

        return false;

        return false;

    float ManaCostTotal = GetCurrentManaCost();
    Caster->CurrentMana -= ManaCostTotal;

    // 4. 消耗体力
    Caster->CurrentStamina = FMath::Max(Caster->CurrentStamina - StaminaCost, 0.0f);

    // 5. 开始冷却
    CooldownInfo.StartCooldown(GetCurrentCooldown());

    // FVector CasterLocation = Caster->GetActorLocation();

// ============================================================================
// CanActivate — 检查技能是否可用
// ============================================================================
bool UDungeonSkillBase::CanActivate(ADungeonCharacter* Caster) const
{
    if (!Caster)
    {
        return false;
    }

    // 检查1：冷却中？
    if (CooldownInfo.IsOnCooldown())
    {
        return false;
    }

    // 检查2：法力够不够？
    if (Caster->CurrentMana < GetCurrentManaCost())
    {
        UE_LOG(LogTemp, Warning, TEXT("法力不足！需要: %f，当前: %f"),
            GetCurrentManaCost(), Caster->CurrentMana);
        return false;
    }

    // 检查3：体力够不够？
    if (Caster->CurrentStamina < StaminaCost)
    {
        UE_LOG(LogTemp, Warning, TEXT("体力不足！需要: %f，当前: %f"),
            StaminaCost, Caster->CurrentStamina);
        return false;
    }

    // 检查4：角色状态？死亡/硬直中不能释放技能
    if (Caster->CurrentState == ADungeonCharacter::ECharacterState::Dead ||
        Caster->CurrentState == ADungeonCharacter::ECharacterState::Stunned)
    {
        return false;
    }

    // 所有检查通过！
    return true;
}

// ============================================================================
// Upgrade — 升级技能
// ============================================================================
bool UDungeonSkillBase::Upgrade()
{
    if (SkillLevel >= MaxSkillLevel)
    {
        UE_LOG(LogTemp, Warning, TEXT("技能 [%s] 已达最大等级！"), *SkillName.ToString());
        return false;
    }

    SkillLevel++;
    UE_LOG(LogTemp, Log, TEXT("技能 [%s] 升级到 Lv.%d！"), *SkillName.ToString(), SkillLevel);
    return true;
}

// ── GetCurrentDamage — 获取当前等级的伤害 ──
float UDungeonSkillBase::GetCurrentDamage() const
{
    // 伤害公式：基础伤害 + 每级伤害 × (当前等级 - 1)
    return BaseDamage + DamagePerLevel * (SkillLevel - 1);
}

// ── GetCurrentManaCost — 获取当前等级的法力消耗 ──
float UDungeonSkillBase::GetCurrentManaCost() const
{
    return ManaCost + ManaCostPerLevel * (SkillLevel - 1);
}

// ── GetCurrentCooldown — 获取当前等级的冷却时间 ──
float UDungeonSkillBase::GetCurrentCooldown() const
{
    // 冷却缩减（每级减少一定秒数，但最低不低于1秒）
    float ReducedCooldown = CooldownInfo.CooldownDuration -
        CooldownReductionPerLevel * (SkillLevel - 1);
    return FMath::Max(ReducedCooldown, 1.0f);
}

// ── TickCooldown — 更新冷却 ──
void UDungeonSkillBase::TickCooldown(float DeltaTime)
{
    CooldownInfo.TickCooldown(DeltaTime);
}

## 3.6 实战操作指南

### 3.6.1 添加武器碰撞体

```
1. 打开 BP_DungeonCharacter 蓝图
2. 在 Mesh 组件下面添加一个 CapsuleCollision 组件
   - 命名为 "WeaponCollision"
   - 附着到武器的Socket（如 hand_r_Socket）
   - 设置碰撞半径为 20
   - 设置碰撞高度为 80（剑的长度）
3. 设置碰撞预设为 "OverlapAllDynamic"（只触发重叠，不产生物理阻挡）
4. 在角色蓝图中添加 OnComponentBeginOverlap 事件
   - 当武器碰撞体与敌人重叠时 → 调用 DealDamageToTarget
```

### 3.6.2 配置动画通知（AnimNotify）

连击系统依赖动画通知来精确控制攻击判定窗口：

```
1. 打开攻击动画蒙太奇（如 AM_LightAttack_01）
2. 在动画通知轨道上添加自定义AnimNotify：
   - 在武器挥出时 → "OnAttackHitWindowStart"
   - 在武器收回时 → "OnAttackHitWindowEnd"
   - 在动画结束时 → "OnAttackAnimEnd"
3. AnimNotify 的实现类命名：
   - AnimNotify_AttackHitWindowStart
   - AnimNotify_AttackHitWindowEnd
   - AnimNotify_AttackAnimEnd
   每个实现类中调用 UDungeonCombatComponent 的对应方法
```

```

2. 运行游戏

   □ 轻攻击中途按重攻击：是否能接上上挑终结技？
