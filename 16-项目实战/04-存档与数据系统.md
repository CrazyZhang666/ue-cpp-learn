# 16.4 存档与数据系统

> **目标**：实现游戏数据的持久化——存档系统、数据表系统和物品实例系统。这三个系统构成了游戏的数据层，让玩家的进度可以保存、游戏内容可以配置、物品可以被唯一追踪。

---

## 4.1 存档系统（SaveGame）

### 4.1.1 存档系统架构

```
┌─────────────────────────────────────────────────┐
│                   存档系统架构                     │
│                                                   │
│  UDungeonSaveGame (USaveGame子类)                  │
│  ├── 玩家数据（等级/经验/属性/装备/背包）          │
│  ├── 进度数据（当前楼层/击败的Boss/收集的钥匙）     │
│  ├── 设置数据（音量/灵敏度/画质）                  │
│  └── 元数据（存档时间/游戏时长/存档槽名称）        │
│                                                   │
│  UDungeonGameInstance (UGameInstance子类)            │
│  ├── 选择存档槽位                                  │
│  ├── 调用 SaveGameToSlot / LoadGameFromSlot       │
│  └── 保存/加载时传递数据给角色和游戏状态           │
│                                                   │
│  ADungeonCharacter (玩家角色)                      │
│  ├── SaveData() → 将自己的数据导出                   │
│  └── LoadData() → 从存档数据恢复自己                │
└─────────────────────────────────────────────────┘
```

```cpp
// 1. 自动序列化所有 UPROPERTY 标记的成员

#pragma once

#include "CoreMinimal.h"
#include "GameFramework/SaveGame.h"

// ── 物品存档结构体 ──
// 不能直接存 UObject 指针（那是内存概念），必须用一个结构体存储物品的"身份信息"
USTRUCT(BlueprintType)
struct FItemSaveData
{
    GENERATED_BODY()

    // 物品的唯一标识ID（FGuid = 全局唯一标识符，128位）
    // 用于追踪"这把剑"在世界中的唯一身份
    UPROPERTY()
    FGuid ItemGuid;

    // 物品的数据资产ID（用于查表获取属性）
    // 例如：ItemDataID = "Sword_Iron_01" → 从DataTable查找这把剑的属性
    UPROPERTY()
    FName ItemDataID;

    // 物品数量（可堆叠物品有用，比如"药水×5"）
    UPROPERTY()
    int32 Quantity = 1;

    // 物品等级（装备可能被强化过，等级会提升）
    UPROPERTY()
    int32 ItemLevel = 1;

    // 装备的强化等级（+0 ~ +10）
    UPROPERTY()
    int32 EnhancementLevel = 0;

    // ── 构造函数（FGuid自动生成）──
    FItemSaveData()
    {
        ItemGuid = FGuid::NewGuid();  // 每个物品实例都有唯一的全局ID
    }
};

// ── 装备槽存档结构体 ──
// 用 槽位类型（FName）→ 物品ID（FGuid）的映射来保存装备状态
USTRUCT(BlueprintType)
struct FEquipmentSaveData
{
    GENERATED_BODY()

    // 槽位名称 → 物品GUID
    // 例如："Weapon" → {1234-5678-...}
    UPROPERTY()
    TMap<FName, FGuid> Slots;

    // 判断指定槽位是否有装备
    bool HasEquipment(FName SlotName) const
    {
        return Slots.Contains(SlotName) && Slots[SlotName].IsValid();
    }

    // 获取指定槽位的装备GUID
    FGuid GetEquipment(FName SlotName) const
    {
        const FGuid* Found = Slots.Find(SlotName);
        return Found ? *Found : FGuid();
    }
};
```

```cpp
// ============================================================================
// DungeonSaveGame.cpp — 存档类实现
// ============================================================================

#include "DungeonSaveGame.h"
#include "Misc/DateTime.h"

// ── 构造函数 ──
UDungeonSaveGame::UDungeonSaveGame()
{
    // 设置默认存档槽名称
    SaveSlotName = TEXT("Slot_01");

    // 记录当前时间作为存档时间
    SaveTimestamp = FDateTime::Now();

    // 默认玩家名称
    PlayerName = TEXT("冒险者");
}

// ── GetFormattedSaveTime ──
// 将存档时间戳格式化为 "2026-05-10 14:30:00" 这样的文本
FText UDungeonSaveGame::GetFormattedSaveTime() const
{
    // FDateTime::ToString 输出 ISO8601 格式: "2026.05.10-14.30.00"
    FString TimeString = SaveTimestamp.ToString(TEXT("%Y-%m-%d %H:%M:%S"));
    return FText::FromString(TimeString);
}

// ── GetFormattedPlayTime ──
// 将总游戏时长（秒）格式化为 "12小时35分钟" 这样的文本
FText UDungeonSaveGame::GetFormattedPlayTime() const
{
    // 转换秒为 时/分/秒
    int32 TotalSeconds = FMath::FloorToInt(TotalPlayTimeSeconds);
    int32 Hours   = TotalSeconds / 3600;
    int32 Minutes = (TotalSeconds % 3600) / 60;
    int32 Seconds = TotalSeconds % 60;

    FString Formatted;
    if (Hours > 0)
    {
        Formatted = FString::Printf(TEXT("%d小时%d分"), Hours, Minutes);
    }
    else if (Minutes > 0)
    {
        Formatted = FString::Printf(TEXT("%d分%d秒"), Minutes, Seconds);
    }
    else
    {
        Formatted = FString::Printf(TEXT("%d秒"), Seconds);
    }

    return FText::FromString(Formatted);
}

// ── IsBossDefeated ──
bool UDungeonSaveGame::IsBossDefeated(FName BossID) const
{
    // TArray::Contains 检查数组中是否包含指定元素
    return DefeatedBosses.Contains(BossID);
}

// ── HasKey ──
bool UDungeonSaveGame::HasKey(FGuid KeyID) const
{
    return CollectedKeys.Contains(KeyID);
}
```

```cpp
// ── LoadGameFromSlot ──
bool UDungeonGameInstance::LoadGameFromSlot(const FString& SlotName)
{
    // 1. 检查存档是否存在
    if (!DoesSaveSlotExist(SlotName))
    {
        UE_LOG(LogTemp, Warning, TEXT("存档槽 %s 不存在！"), *SlotName);
        return false;
    }

    // 2. 从磁盘加载存档对象
    UDungeonSaveGame* LoadedSave = Cast<UDungeonSaveGame>(
        UGameplayStatics::LoadGameFromSlot(SlotName, 0)
    );

    if (!LoadedSave)
    {
        UE_LOG(LogTemp, Error, TEXT("加载存档失败！槽位: %s"), *SlotName);
        return false;
    }

    // 3. 将加载的数据存储到 GameInstance（等待关卡加载后传递给角色）
    PlayerLevel             = LoadedSave->PlayerLevel;
    PlayerExperience        = LoadedSave->PlayerExperience;
    CurrentFloor            = LoadedSave->CurrentFloor;

    // ... 保存其他数据到 GameInstance 的缓存变量 ...

    // 4. 标记当前存档槽
    CurrentSaveSlot = SlotName;

    // 5. 加载关卡（切换到保存时的楼层）
    FString LevelName = FString::Printf(TEXT("Dungeon_Floor%d"), CurrentFloor);
    UGameplayStatics::OpenLevel(GetWorld(), FName(*LevelName));

    UE_LOG(LogTemp, Log, TEXT("存档加载成功！槽位: %s, 等级: %d, 楼层: %d"),
        *SlotName, PlayerLevel, CurrentFloor);

    return true;
}

// ── DoesSaveSlotExist ──
bool UDungeonGameInstance::DoesSaveSlotExist(const FString& SlotName) const
{
    // UGameplayStatics::DoesSaveGameExist 检查磁盘上是否有存档文件
    return UGameplayStatics::DoesSaveGameExist(SlotName, 0);
}

// ── DeleteSaveSlot ──
bool UDungeonGameInstance::DeleteSaveSlot(const FString& SlotName)
{
    if (!DoesSaveSlotExist(SlotName))
    {
        return false;
    }

    return UGameplayStatics::DeleteGameInSlot(SlotName, 0);
}

// ── GetAllSaveSlots ──
TArray<FString> UDungeonGameInstance::GetAllSaveSlots() const
{
    TArray<FString> Slots;

    // 检查常见的存档槽名称
    // 注意：UE没有内置"列出所有存档"的API，需要我们自己管理槽位列表
    // 做法一：固定槽位（Slot_01 ~ Slot_05）
    for (int32 i = 1; i <= 5; i++)
    {
        FString SlotName = FString::Printf(TEXT("Slot_%02d"), i);
        if (DoesSaveSlotExist(SlotName))
        {
            Slots.Add(SlotName);
        }
    }

    // 做法二：额外检查自动存档
    FString AutoSaveSlot = TEXT("AutoSave");
    if (DoesSaveSlotExist(AutoSaveSlot))
    {
        Slots.Add(AutoSaveSlot);
    }

    return Slots;
}
```

### 4.1.5 存档时机

| 保存类型 | 触发时机 | 实现方式 |
|----------|----------|----------|
| 手动保存 | 玩家在暂停菜单点击"保存" | 按钮事件 → SaveGameToSlot |
| 自动保存 | 每次进入新楼层 | GameMode::TransitionToNextFloor 中调用 |
| Boss前保存 | 进入Boss房间前 | 在Boss房间的触发器Volume中调用 |
| 退出保存 | 玩家退出游戏时 | 在GameInstance::Shutdown中调用 |

```cpp
// ── 自动保存调用示例（在 ADungeonGameMode 中）──
void ADungeonGameMode::TransitionToNextFloor()
{
    // 1. 自动保存当前进度
    AutoSave();

    // 2. 加载下一层
    FString NextLevelName = FString::Printf(TEXT("Dungeon_Floor%d"), CurrentFloor + 1);
    UGameplayStatics::OpenLevel(GetWorld(), FName(*NextLevelName));
}

void ADungeonGameMode::AutoSave()
{
    // 获取GameInstance并调用存档
    UDungeonGameInstance* GI = Cast<UDungeonGameInstance>(GetGameInstance());
    if (GI)
    {
        GI->SaveGameToSlot(TEXT("AutoSave"));
    }
}
```

```cpp
{
    if (!User)
    {
        UE_LOG(LogTemp, Warning, TEXT("物品 [%s] 使用失败：用户为空"), *ItemName.ToString());
        return false;
    }

    // 检查战斗限制

    {
        UE_LOG(LogTemp, Warning, TEXT("物品 [%s] 不能在战斗中使用"), *ItemName.ToString());
        return false;
    }

    // 子类会覆盖此方法实现具体效果（如药水恢复生命、钥匙开门）
    // 这里只是记录使用日志
    UE_LOG(LogTemp, Log, TEXT("使用了物品 [%s] (UID: %s)"),
        *ItemName.ToString(), *ItemUID.ToString());

    // 减少堆叠数量
    CurrentStackSize--;
    // 如果堆叠为0，物品会被移除

    // 触发蓝图事件（用于UI反馈、音效等）
    OnItemUsed(User);

    return true;
```

```cpp
// ── DungeonPickup.cpp 实现 ──
ADungeonPickup::ADungeonPickup()
{
    // 创建碰撞球体
    PickupCollision = CreateDefaultSubobject<USphereComponent>(TEXT("PickupCollision"));
    RootComponent = PickupCollision;
    PickupCollision->SetSphereRadius(PickupRange);
    // 设置碰撞为 OverlapOnly（不产生物理阻挡，只触发重叠事件）
    PickupCollision->SetCollisionResponseToAllChannels(ECR_Ignore);
    PickupCollision->SetCollisionResponseToChannel(ECC_Pawn, ECR_Overlap);

    // 创建网格体组件
    ItemMesh = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("ItemMesh"));
    ItemMesh->SetupAttachment(RootComponent);
    // 网格体不需要碰撞（碰撞球体已经处理了）
    ItemMesh->SetCollisionEnabled(ECollisionEnabled::NoCollision);

    // 绑定重叠事件
    PickupCollision->OnComponentBeginOverlap.AddDynamic(this, &ADungeonPickup::OnOverlapBegin);
}

// ── OnOverlapBegin ──
void ADungeonPickup::OnOverlapBegin(UPrimitiveComponent* OverlappedComponent,
    AActor* OtherActor, UPrimitiveComponent* OtherComp,
    int32 OtherBodyIndex, bool bFromSweep,
    const FHitResult& SweepResult)
{
    // 判断重叠的是否是玩家
    ADungeonCharacter* Player = Cast<ADungeonCharacter>(OtherActor);
    if (Player)
    {
        // 显示拾取提示
        ShowPickupPrompt(true);
        // 实际项目中，按E键才拾取（不是在重叠时就自动拾取）
    }
}

// ── OnPickedUp ──
void ADungeonPickup::OnPickedUp(ADungeonCharacter* Picker)
{
    if (!Picker || !Item) return;

    // 将物品添加到玩家背包
    bool bSuccess = Picker->AddToInventory(Item);

    if (bSuccess)
    {
        // 拾取成功，销毁Pickup Actor
        UE_LOG(LogTemp, Log, TEXT("玩家拾取了: %s"), *Item->ItemName.ToString());
        Destroy();
    }
    else
    {
        // 背包已满 — 显示提示或保留物品
        UE_LOG(LogTemp, Warning, TEXT("背包已满，无法拾取！"));
    }
}
```

❌ **错误做法**：直接在UE编辑器中手打大量数据——容易打错且难以批量修改。

## 4.5 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 存档文件找不到 | `DoesSaveGameExist`返回false | 检查存档槽名称是否正确，检查是否正确调用了`SaveGameToSlot` |
| 加载后数据丢失 | SaveGame的UPROPERTY未标记 | 所有需要序列化的成员必须加`UPROPERTY()`宏 |
| DataTable查询失败 | 行名称不匹配 | 使用`FindRow`时行名称必须与DataTable中完全一致 |
| UDataAsset配置无效 | DataAsset类未正确继承 | 检查类是否正确继承自`UDataAsset`，是否有`UCLASS(BlueprintType)` |
| 物品UID重复 | 存档加载时重新生成了UID | 从存档加载物品时必须调用`SetItemUID`恢复原始UID，不要用NewGuid |
| 背包数据保存后加载重复 | 加载时没有清空旧数据 | 加载前先清空Inventory数组 |
| CSV导入后中文乱码 | CSV文件编码问题 | CSV文件保存为UTF-8编码 |
