## UE命名前缀规则（强制性的）

每个类名、结构体名、枚举名等都有固定的前缀，这是**强制性的编码规范**：

| 前缀 | 表示                  | 基类       | 示例                               |
| ---- | --------------------- | ---------- | ---------------------------------- |
| `A`  | Actor（场景中的物体） | AActor     | `ACharacter`, `APlayerController`  |
| `U`  | UObject及其子类       | UObject    | `UActorComponent`, `UTexture2D`    |
| `F`  | 纯数据结构/值类型     | -          | `FVector`, `FRotator`, `FString`   |
| `I`  | 接口类                | UInterface | `IInputDevice`, `ITargetDevice`    |
| `E`  | 枚举                  | -          | `ECollisionChannel`, `EWeaponType` |
| `T`  | 模板类/容器           | -          | `TArray`, `TMap`, `TSharedPtr`     |
| `S`  | Slate UI控件          | SWidget    | `SButton`, `STextBlock`            |
| `G`  | 全局变量前缀          | -          | `GEngine`, `GWorld`                |

### 命名违规后果

```cpp
// ❌ 编译警告！
class MyActor : public AActor { };     // 缺少A前缀
class UComponent : public UActorComponent { };  // 正确

// ✓ 正确
class AMyActor : public AActor { };
class FMyDataStruct { };  // 结构体用F前缀
enum class EMyEnum { };
```

> 如果你的类没有遵循前缀规则，UE会自动发出编译警告。面试时会直接问你这个。

## UE字符串类型完整对比

```cpp
// FString — 通用可变字符串
FString Name = TEXT("Hello");
Name += TEXT(" World");          // 拼接
Name.ToUpper();                   // 转大写
int32 Len = Name.Len();          // 长度
FString Formatted = FString::Printf(TEXT("%s: %d"), *Name, 42);  // 格式化

// FName — 不可变标识符（超级快）
FName AssetName = FName(TEXT("MyAsset"));
// 内部是哈希表查找，比较两个FName只比较哈希值（O(1)）
// 不区分大小写！FName("hello") == FName("HELLO")
// 一度创建永不删除，适合资源名、标签、Socket名等

// FText — 可本地化的显示文本
FText DisplayText = FText::FromString(TEXT("按W键前进"));
FText LocalizedText = LOCTEXT("Key", "Press W to move forward");
// LOCTEXT会在编辑器中收集所有文本，用于翻译工具
// 游戏需要多语言支持时，FText是必须的
```

### 选择决策流程图

```
需要一个字符串？
├─ 用于资源名称/标识符/标签？ → FName
├─ 显示给玩家的文本？ → FText（支持翻译）
├─ 通用字符串操作（拼接、格式化等）？ → FString
└─ 临时操作后转为FText或FName？ → FString
```

## UE数值类型

```cpp
// UE推荐使用明确位数类型（而不是int/long/unsigned等）
int8     TinyValue = 127;         // 8位有符号，范围[-128, 127]
int16    ShortValue = 32767;      // 16位有符号
int32    NormalValue = 42;        // 32位有符号（最常用）
int64    BigValue = 999999999LL;  // 64位有符号

uint8    ByteValue = 255;         // 8位无符号，范围[0, 255]（UE中byte用这个）
uint16   UShortValue = 65535;    // 16位无符号
uint32   UNormalValue = 100u;    // 32位无符号
uint64   UBigValue = 999999ULL;  // 64位无符号

float    FValue = 3.14f;           // 32位浮点（最常用）
double   DValue = 3.1415926535;   // 64位浮点

// UE也提供了语义化的类型别名
typedef int32 int32;  // 这其实就是int
typedef uint8 uint8;  // 这就是unsigned char
```

> **为什么UE不用 `int` 而用 `int32`？**
> `int` 的大小在不同平台上可能不同（16位/32位/64位）。`int32` 明确保证是32位，避免跨平台问题。

## UE数学类型

```cpp
#include "Math/Vector.h"       // FVector, FVector2D, FVector4
#include "Math/Rotator.h"      // FRotator
#include "Math/Transform.h"    // FTransform
#include "Math/Color.h"        // FColor, FLinearColor
#include "Math/Matrix.h"       // FMatrix
#include "Math/Quat.h"         // FQuat（四元数）

// 二维向量
FVector2D ScreenPos(1920.0f, 1080.0f);
FVector2D UV(0.5f, 0.5f);

// 三维向量
FVector Position(100.0f, 200.0f, 50.0f);
FVector Forward = FVector::ForwardVector;     // (1, 0, 0)
FVector Up = FVector::UpVector;               // (0, 0, 1)
float Dist = FVector::Dist(PosA, PosB);        // 两点距离

// 四维向量（常用于颜色和齐次坐标）
FVector4 Vec4(1.0f, 0.0f, 0.0f, 1.0f);

// 旋转
FRotator Rotation(0.0f, 90.0f, 0.0f);        // Pitch, Yaw, Roll

// 变换（位置+旋转+缩放）
FTransform Transform(Rotation, Position, FVector(1.0f));
FVector ExtractedPos = Transform.GetLocation();

// 颜色
FColor IntColor(255, 0, 0, 255);              // RGBA, 0-255整数
FLinearColor LinearColor(1.0f, 0.0f, 0.0f, 1.0f);  // RGBA, 0.0-1.0浮点

// 四元数（平滑旋转插值）
FQuat QuatRotation(FVector::UpVector, PI/2);  // 绕Z轴旋转90度
```

## UE容器类型速查

| UE类型              | C++标准库等效        | 特点               |
| ------------------- | -------------------- | ------------------ |
| `TArray<T>`         | `vector<T>`          | 动态数组           |
| `TMap<K,V>`         | `unordered_map<K,V>` | 哈希表字典         |
| `TSet<T>`           | `unordered_set<T>`   | 哈希集合           |
| `TQueue<T>`         | `queue<T>`           | 先进先出队列       |
| `TLinkedList<T>`    | `list<T>`            | 双向链表（很少用） |
| `TStaticArray<T,N>` | `array<T,N>`         | 固定大小数组       |
| `TSparseArray<T>`   | -                    | UE特有稀疏数组     |

## UE智能指针类型

| 类型                | 特点                          | 使用场景                  |
| ------------------- | ----------------------------- | ------------------------- |
| `TSharedPtr<T>`     | 共享所有权，引用计数          | 非UObject对象             |
| `TSharedRef<T>`     | 共享引用（不能为空）          | 确定非空的共享所有权      |
| `TUniquePtr<T>`     | 独占所有权                    | 唯一所有者的资源          |
| `TWeakPtr<T>`       | 弱引用（不影响计数）          | 打破循环引用              |
| `TWeakObjectPtr<T>` | 弱引用UObject                 | 安全引用可能被GC的UObject |
| `TObjectPtr<T>`     | UE5.1+推荐替代原始UObject指针 | 新版UObject指针包装       |

## 常见代码注释规范

```cpp
/**
 * 多行注释：用于函数/类的文档说明
 * @param DamageAmount 伤害值（原始数值，会经过护甲减免）
 * @return 实际造成的伤害值
 */
float CalculateDamage(float DamageAmount);

// 单行注释：用于解释某行代码的意图
float FinalDamage = RawDamage * ArmorReduction;  // 护甲减免50%

// HACK: 暂时用硬编码值，等配置系统做好后替换
```

## 代码组织规范

```cpp
// .h 文件结构顺序
#pragma once

// 1. 引擎头文件
#include "CoreMinimal.h"
#include "GameFramework/Actor.h"

// 2. 项目头文件
#include "MyProject/SomeClass.h"

// 3. 最后的generated.h
#include "MyClass.generated.h"

// 4. 前置声明（减少include依赖）
class USkeletalMeshComponent;
class UDamageType;

// 5. 委托声明
DECLARE_DYNAMIC_MULTICAST_DELEGATE(FOnDeathSignature);

// 6. 类定义
UCLASS()
class MYPROJECT_API AMyClass : public AActor
{
    GENERATED_BODY()

public:
    AMyClass();

    // 先公开属性、再公开方法

protected:
    // 受保护的成员

private:
    // 私有成员（UE中较少用private，因为子类通常需要访问）
};
```
