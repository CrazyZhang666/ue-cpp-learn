## 如何使用本手册

### 如果你是完全零基础

1. **按顺序从头开始**，不要跳章节
2. **每个案例都要亲手敲一遍代码**，不要复制粘贴
3. 遇到报错先自己读错误信息，锻炼排查能力
4. 每个章节的案例完成后再进入下一章
5. 学完第16章后，试着不看书做一个自己的小游戏

### 如果你有其他语言基础

1. 第1章照做（环境必须配好）
2. 第2章快速浏览（重点看指针和STL部分）
3. 第3章起正常学习
4. 第13-14章（GAS）要重点花时间

### 如果你会UE蓝图

1. 第2章重点学习
2. 第3章是关键——理解"C++怎么写蓝图能干的事"
3. 第6章重点关注蓝图与C++的互操作

## 代码约定

本手册所有代码示例遵循以下约定：

```cpp
// ✅ 正确示例 — 用 ✅ 标记
UCLASS(Blueprintable)
class MYGAME_API AMyActor : public AActor
{
    GENERATED_BODY()

public:
    // 属性：暴露给蓝图
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "My Actor")
    float Health = 100.0f;

    // 函数：可在蓝图中调用
    UFUNCTION(BlueprintCallable, Category = "My Actor")
    void TakeDamage(float DamageAmount);
};
```

```cpp
// ❌ 错误示例 — 用 ❌ 标记，提醒你这是常见错误
class AMyActor : public AActor  // 缺少 UCLASS 和 GENERATED_BODY
{
    float Health;  // 缺少 UPROPERTY，会被垃圾回收！
};
```

## 常见问题

**Q: 我电脑配置低，能学吗？**
A: 最低要求16GB内存、4核CPU、GTX1060。低于此配置跑UE会很痛苦，建议先升级硬件。

**Q: 一定要用Visual Studio吗？**
A: 是的，UE5在Windows上只能用VS2022编译。Rider付费版也可但不在本手册范围。

**Q: 学完能找到工作吗？**
A: 认真学完+做出第16章的项目+准备好第17章的面试内容，具备初级UE开发工程师的竞争力。

**Q: 为什么不教蓝图？**
A: 蓝图在第6章会涉及（因为要和C++交互），但本手册聚焦C++。实际工作中蓝图和C++都重要，建议额外学习蓝图。
