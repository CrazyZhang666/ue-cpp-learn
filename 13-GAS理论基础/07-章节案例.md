# 第13章案例：GAS角色基础框架

> **难度**：⭐⭐⭐⭐（综合实战级）
> **预计时间**：2-3小时
> **目标**：从零搭建一个完整的GAS角色框架，涵盖ASC、AS、GA、GE、GT的完整实现。本案例是你学完本章后的综合检验。

---

## 新手先读：先跑通一个最小GAS角色

这个案例会一次出现ASC、AttributeSet、GameplayAbility、GameplayEffect和GameplayTag。第一遍建议只追踪一个最小闭环：

```
角色拥有ASC
    ↓
ASC持有AttributeSet
    ↓
角色获得火球术Ability
    ↓
释放火球术应用Damage GameplayEffect
    ↓
Health属性变化并同步/刷新
```

缩写对照：

| 缩写 | 完整名称 | 新手先理解成 |
| ---- | -------- | ------------ |
| ASC | AbilitySystemComponent | GAS总管 |
| AS | AttributeSet | 属性表 |
| GA | GameplayAbility | 技能 |
| GE | GameplayEffect | 改属性/状态的效果 |
| GT | GameplayTag | 标签 |

> **阅读建议**：先照做火球术和伤害效果，Buff堆叠、暴击和完整网络同步可以作为第二轮学习重点。

## 案例目标

构建一个名为 `AMyGASCharacter` 的角色，具备：

1. **完整的ASC架构**（PlayerState方案）
2. **10个属性**（生命/法力/体力/攻击力/防御力/暴击率/暴击伤害/移动速度 及最大值）
3. **一个火球术技能**（消耗法力、冷却时间、受标签限制）
4. **一个伤害效果**（使用AttributeBased计算、支持暴击）
5. **一个攻击力Buff效果**（持续10秒、可堆叠3层）

## 第一步：创建项目标签系统

先在项目中定义我们需要的GameplayTag。这些标签在 `DefaultGameplayTags.ini` 中定义，或在编辑器 `Project Settings → GameplayTags` 中添加。

```
需要创建的标签列表：

状态标签：
  State.Dead                 — 角色死亡
  State.CC                   — 所有控制效果的父标签
  State.CC.Stun              — 眩晕
  State.CC.Root              — 定身
  State.CC.Silence           — 沉默
  State.Combat.Casting       — 正在施法
  State.Combat.InCombat      — 战斗中
  State.Invincible           — 无敌

技能标签：
  Ability.Type.Active        — 主动技能
  Ability.Element.Fire       — 火系技能

Buff标签：
  Buff.Offensive.AttackUp    — 攻击力提升Buff

冷却标签：
  Cooldown.Skill.Fireball    — 火球术冷却中

数据标签（SetByCaller用）：
  Data.Damage.Base           — 基础伤害值
  Data.SkillCoeff            — 技能系数

GameplayCue标签：
  Cue.Combat.Fireball.Cast   — 火球术施法表现
  Cue.Combat.Fireball.Impact — 火球术命中表现
  Cue.Combat.Hit             — 受击表现
  Cue.Combat.Death           — 死亡表现
  Cue.Buff.AttackUp          — 攻击力Buff表现
```

## 第二步：创建属性集 (MyAttributeSet)

### MyAttributeSet.h

```cpp
// ===== MyAttributeSet.h =====
#pragma once

#include "CoreMinimal.h"
#include "AttributeSet.h"
#include "AbilitySystemComponent.h"
#include "MyAttributeSet.generated.h"

/**
 * 角色属性集——包含角色所有可被GAS管理的数值属性
 *
 * 使用 ATTRIBUTE_ACCESSORS 宏自动为每个属性生成：
 *   static FGameplayAttribute GetXxxAttribute() — 获取属性的FGameplayAttribute引用
 *   float GetXxx() const                        — 获取属性当前值
 *   void SetXxx(float NewVal)                   — 设置属性当前值
 *   void InitXxx(float NewVal)                  — 初始化属性当前值
 */
UCLASS()
class MYGAME_API UMyAttributeSet : public UAttributeSet
{
    GENERATED_BODY()

public:
    UMyAttributeSet();

    // =============================================================
    // 生命值属性
    // =============================================================
    // FGameplayAttributeData: GAS提供的属性数据类型
    // 内部维护 BaseValue（基础值）和 CurrentValue（当前值=基础值+临时GE修改）
    // ReplicatedUsing: 属性同步到客户端时触发OnRep函数

    UPROPERTY(BlueprintReadOnly, ReplicatedUsing = OnRep_Health,
              Category = "Attributes|Vital")
    FGameplayAttributeData Health;      // 当前生命值
    ATTRIBUTE_ACCESSORS(UMyAttributeSet, Health)

    UPROPERTY(BlueprintReadOnly, ReplicatedUsing = OnRep_MaxHealth,
              Category = "Attributes|Vital")
    FGameplayAttributeData MaxHealth;   // 最大生命值
    ATTRIBUTE_ACCESSORS(UMyAttributeSet, MaxHealth)

    // =============================================================
    // 法力值属性
    // =============================================================
    UPROPERTY(BlueprintReadOnly, ReplicatedUsing = OnRep_Mana,
              Category = "Attributes|Vital")
    FGameplayAttributeData Mana;         // 当前法力值
    ATTRIBUTE_ACCESSORS(UMyAttributeSet, Mana)

    UPROPERTY(BlueprintReadOnly, ReplicatedUsing = OnRep_MaxMana,
              Category = "Attributes|Vital")
    FGameplayAttributeData MaxMana;      // 最大法力值
    ATTRIBUTE_ACCESSORS(UMyAttributeSet, MaxMana)

    // =============================================================
    // 体力值属性
    // =============================================================
    UPROPERTY(BlueprintReadOnly, ReplicatedUsing = OnRep_Stamina,
              Category = "Attributes|Vital")
    FGameplayAttributeData Stamina;      // 当前体力值
    ATTRIBUTE_ACCESSORS(UMyAttributeSet, Stamina)

    UPROPERTY(BlueprintReadOnly, ReplicatedUsing = OnRep_MaxStamina,
              Category = "Attributes|Vital")
    FGameplayAttributeData MaxStamina;   // 最大体力值
    ATTRIBUTE_ACCESSORS(UMyAttributeSet, MaxStamina)

    // =============================================================
    // 战斗属性
    // =============================================================
    UPROPERTY(BlueprintReadOnly, ReplicatedUsing = OnRep_AttackPower,
              Category = "Attributes|Combat")
    FGameplayAttributeData AttackPower;  // 攻击力（影响造成伤害）
    ATTRIBUTE_ACCESSORS(UMyAttributeSet, AttackPower)

    UPROPERTY(BlueprintReadOnly, ReplicatedUsing = OnRep_Defense,
              Category = "Attributes|Combat")
    FGameplayAttributeData Defense;      // 防御力（减伤）
    ATTRIBUTE_ACCESSORS(UMyAttributeSet, Defense)

    UPROPERTY(BlueprintReadOnly, ReplicatedUsing = OnRep_CriticalChance,
              Category = "Attributes|Combat")
    FGameplayAttributeData CriticalChance;  // 暴击率 (0.0 ~ 1.0)
    ATTRIBUTE_ACCESSORS(UMyAttributeSet, CriticalChance)

    UPROPERTY(BlueprintReadOnly, ReplicatedUsing = OnRep_CriticalDamage,
              Category = "Attributes|Combat")
    FGameplayAttributeData CriticalDamage;  // 暴击伤害倍率 (如 1.5 = 150%)
    ATTRIBUTE_ACCESSORS(UMyAttributeSet, CriticalDamage)

    // =============================================================
    // 移动属性
    // =============================================================
    UPROPERTY(BlueprintReadOnly, ReplicatedUsing = OnRep_MoveSpeed,
              Category = "Attributes|Movement")
    FGameplayAttributeData MoveSpeed;    // 移动速度（影响实际行走速度）
    ATTRIBUTE_ACCESSORS(UMyAttributeSet, MoveSpeed)

    // =============================================================
    // 属性变化回调（覆写父类方法）
    // =============================================================

    /**
     * PreAttributeChange: 属性被修改前的回调
     *
     * 调用时机：在任何GE修改属性之前（在服务器端调用）
     * 主要用途：Clamp属性值，确保不超出合理范围
     *
     * @param Attribute  被修改的属性引用
     * @param NewValue   将要被设置的新值（引用，可以修改它来Clamp）
     */
    virtual void PreAttributeChange(
        const FGameplayAttribute& Attribute,
        float& NewValue
    ) override;

    /**
     * PostGameplayEffectExecute: GE执行完成后的回调
     *
     * 调用时机：GE的所有Modifier计算完成、属性修改完成之后（仅服务器端）
     * 主要用途：处理"属性变化后的逻辑"——如死亡检测、连锁属性更新
     *
     * @param Data 包含修改的详细信息（哪个GE、修改了什么属性、修改了多少）
     */
    virtual void PostGameplayEffectExecute(
        const FGameplayEffectModCallbackData& Data
    ) override;

    // 基础属性变化回调（用的较少）
    virtual void PreAttributeBaseChange(
        const FGameplayAttribute& Attribute,
        float& NewValue
    ) const override;

    virtual void PostAttributeChange(
        const FGameplayAttribute& Attribute,
        float OldValue,
        float NewValue
    ) override;

protected:
    // =============================================================
    // 网络同步回调函数（OnRep）
    //
    // 当服务器同步属性到客户端时触发
    // 在客户端UI更新中使用
    // =============================================================

    UFUNCTION()
    virtual void OnRep_Health(const FGameplayAttributeData& OldHealth);

    UFUNCTION()
    virtual void OnRep_MaxHealth(const FGameplayAttributeData& OldMaxHealth);

    UFUNCTION()
    virtual void OnRep_Mana(const FGameplayAttributeData& OldMana);

    UFUNCTION()
    virtual void OnRep_MaxMana(const FGameplayAttributeData& OldMaxMana);

    UFUNCTION()
    virtual void OnRep_Stamina(const FGameplayAttributeData& OldStamina);

    UFUNCTION()
    virtual void OnRep_MaxStamina(const FGameplayAttributeData& OldMaxStamina);

    UFUNCTION()
    virtual void OnRep_AttackPower(const FGameplayAttributeData& OldAttackPower);

    UFUNCTION()
    virtual void OnRep_Defense(const FGameplayAttributeData& OldDefense);

    UFUNCTION()
    virtual void OnRep_CriticalChance(const FGameplayAttributeData& OldCriticalChance);

    UFUNCTION()
    virtual void OnRep_CriticalDamage(const FGameplayAttributeData& OldCriticalDamage);

    UFUNCTION()
    virtual void OnRep_MoveSpeed(const FGameplayAttributeData& OldMoveSpeed);

private:
    // 处理死亡逻辑的内部函数
    void HandleDeath(const FGameplayEffectModCallbackData& Data);
};
```

```cpp
// ============================================================================
// GetLifetimeReplicatedProps —— 注册需要网络同步的属性
// ============================================================================
// 这个函数告诉UE的网络系统：哪些属性需要从服务器复制到客户端
// 如果不写这个函数，属性值只存在于服务器上，客户端无法获取

void UMyAttributeSet::GetLifetimeReplicatedProps(
    TArray<FLifetimeProperty>& OutLifetimeProps) const
{
    // 必须调用父类
    Super::GetLifetimeReplicatedProps(OutLifetimeProps);

    // DOREPLIFETIME_CONDITION_NOTIFY 的三个组成部分：
    //
    // DOREPLIFETIME:      声明此属性需要网络复制
    // CONDITION:          复制条件
    //   COND_None:        无条件复制到所有客户端
    //   COND_OwnerOnly:   只复制到拥有此属性的客户端
    //   COND_SkipOwner:   复制到除了拥有者之外的所有客户端
    // NOTIFY:             使用ReplicatedUsing指定的OnRep回调函数
    //   REPNOTIFY_Always: 每次值变化都触发OnRep
    //   REPNOTIFY_OnChanged: 只有值真的改变了才触发（默认）

    // ----- 生命/法力/体力——所有人都需要看到（用于UI显示）-----
    DOREPLIFETIME_CONDITION_NOTIFY(UMyAttributeSet, Health, COND_None, REPNOTIFY_Always);
    DOREPLIFETIME_CONDITION_NOTIFY(UMyAttributeSet, MaxHealth, COND_None, REPNOTIFY_Always);
    DOREPLIFETIME_CONDITION_NOTIFY(UMyAttributeSet, Mana, COND_None, REPNOTIFY_Always);
    DOREPLIFETIME_CONDITION_NOTIFY(UMyAttributeSet, MaxMana, COND_None, REPNOTIFY_Always);
    DOREPLIFETIME_CONDITION_NOTIFY(UMyAttributeSet, Stamina, COND_None, REPNOTIFY_Always);
    DOREPLIFETIME_CONDITION_NOTIFY(UMyAttributeSet, MaxStamina, COND_None, REPNOTIFY_Always);

    // ----- 战斗属性——只同步给拥有者（防止其他玩家看到你的数值）-----
    DOREPLIFETIME_CONDITION_NOTIFY(UMyAttributeSet, AttackPower, COND_OwnerOnly, REPNOTIFY_Always);
    DOREPLIFETIME_CONDITION_NOTIFY(UMyAttributeSet, Defense, COND_OwnerOnly, REPNOTIFY_Always);
    DOREPLIFETIME_CONDITION_NOTIFY(UMyAttributeSet, CriticalChance, COND_OwnerOnly, REPNOTIFY_Always);
    DOREPLIFETIME_CONDITION_NOTIFY(UMyAttributeSet, CriticalDamage, COND_OwnerOnly, REPNOTIFY_Always);

    // ----- 移动速度——所有人都需要看到（影响动画等）-----
    DOREPLIFETIME_CONDITION_NOTIFY(UMyAttributeSet, MoveSpeed, COND_None, REPNOTIFY_Always);
}

// ============================================================================
// PreAttributeChange —— 属性被修改前的回调
// ============================================================================
//
// 作用：在GE修改属性之前，给你机会"修正"即将被设置的值
// 最常用的场景是 Clamp（限制值域）：确保属性值不超出合法范围
//
// 调用时机：在GE计算完Modifier、准备应用之前（仅服务器端）
//
// 重要：NewValue 是引用参数，修改它就能改变最终被设置的值

void UMyAttributeSet::PreAttributeChange(
    const FGameplayAttribute& Attribute,
    float& NewValue)
{
    // 如果代码执行到这里，说明有一个GE正在尝试修改某个属性
    // 我们在这里对修改的值做Clamp处理

    // ─── 生命值不能小于0，不能大于最大生命值 ───
    if (Attribute == GetHealthAttribute())
    {
        // FMath::Clamp(值, 最小值, 最大值)
        // 如果NewValue < 0，返回0
        // 如果NewValue > MaxHealth，返回MaxHealth
        // 否则返回NewValue本身
        NewValue = FMath::Clamp(NewValue, 0.0f, GetMaxHealth());
    }
    // ─── 法力值不能小于0，不能大于最大法力值 ───
    else if (Attribute == GetManaAttribute())
    {
        NewValue = FMath::Clamp(NewValue, 0.0f, GetMaxMana());
    }
    // ─── 体力值不能小于0，不能大于最大体力值 ───
    else if (Attribute == GetStaminaAttribute())
    {
        NewValue = FMath::Clamp(NewValue, 0.0f, GetMaxStamina());
    }
    // ─── 最大生命值不能小于1（防止除以0）───
    else if (Attribute == GetMaxHealthAttribute())
    {
        NewValue = FMath::Max(NewValue, 1.0f);
    }
    // ─── 最大法力值不能小于1 ───
    else if (Attribute == GetMaxManaAttribute())
    {
        NewValue = FMath::Max(NewValue, 1.0f);
    }
    // ─── 移动速度不能为负数 ───
    else if (Attribute == GetMoveSpeedAttribute())
    {
        NewValue = FMath::Max(NewValue, 0.0f);
    }
    // ─── 暴击率在 0~1 之间 ───
    else if (Attribute == GetCriticalChanceAttribute())
    {
        NewValue = FMath::Clamp(NewValue, 0.0f, 1.0f);
    }
}

// ============================================================================
// PostGameplayEffectExecute —— GE执行完成后的回调（最重要！）
// ============================================================================
//
// 作用：GE的所有Modifier已经应用完毕，属性值已经更新
// 在这里处理"属性变化引发的结果"：
//   - 生命值<=0 → 角色死亡
//   - 移动速度变化 → 更新CharacterMovement组件的速度
//   - 某种属性达到阈值 → 触发特殊状态
//
// 调用时机：GE修改完成后（仅服务器端）
//
// Data 参数包含：
//   Data.Target              → 目标的信息（Actor + ASC）
//   Data.EffectSpec          → 哪个GE的Spec导致了这次修改
//   Data.EvaluatedData       → 修改了哪个属性、修改了多少
//   Data.EffectSpec.GetEffectContext() → 谁造成这个效果、怎么造成的

void UMyAttributeSet::PostGameplayEffectExecute(
    const FGameplayEffectModCallbackData& Data)
{
    Super::PostGameplayEffectExecute(Data);

    // 获取被修改的属性
    const FGameplayAttribute& ModifiedAttribute = Data.EvaluatedData.Attribute;
    // 获取修改量（注意：这是Magnitude，不是最终值）
    float Magnitude = Data.EvaluatedData.Magnitude;

    // ─── 处理生命值变化 ───
    if (ModifiedAttribute == GetHealthAttribute())
    {
        // 再次Clamp（双保险——虽然没有PreAttributeChange已经Clamp）
        // 但 PostGameplayEffectExecute 中对CurrentValue可能再次操作
        SetHealth(FMath::Clamp(GetHealth(), 0.0f, GetMaxHealth()));

        // 检查是否死亡
        if (GetHealth() <= 0.0f)
        {
            HandleDeath(Data);
        }
    }
    // ─── 处理法力值变化 ───
    else if (ModifiedAttribute == GetManaAttribute())
    {
        SetMana(FMath::Clamp(GetMana(), 0.0f, GetMaxMana()));
    }
    // ─── 处理体力值变化 ───
    else if (ModifiedAttribute == GetStaminaAttribute())
    {
        SetStamina(FMath::Clamp(GetStamina(), 0.0f, GetMaxStamina()));
    }
    // ─── 处理移动速度变化 ───
    // 当MoveSpeed被GE修改时，需要同步更新CharacterMovement组件的实际速度
    else if (ModifiedAttribute == GetMoveSpeedAttribute())
    {
        // 获取Avatar Actor（物理角色）
        AActor* Avatar = Data.Target.AvatarActor.Get();
        if (Avatar)
        {
            // 尝试转换为Character
            ACharacter* Character = Cast<ACharacter>(Avatar);
            if (Character)
            {
                // 获取角色移动组件
                UCharacterMovementComponent* Movement =
                    Character->GetCharacterMovement();
                if (Movement)
                {
                    // 同步GAS属性到实际移动速度
                    Movement->MaxWalkSpeed = GetMoveSpeed();
                }
            }
        }
    }
}

// ============================================================================
// HandleDeath —— 处理死亡逻辑
// ============================================================================

void UMyAttributeSet::HandleDeath(const FGameplayEffectModCallbackData& Data)
{
    // 获取目标的AbilitySystemComponent（角色的"大脑"）
    UAbilitySystemComponent* TargetASC = Data.Target.AbilitySystemComponent.Get();
    if (!TargetASC)
    {
        return;  // 没有ASC，不处理
    }

    // ===== 步骤1：给角色的ASC添加"死亡"标签 =====
    // 添加State.Dead标签后，所有配置了ActivationBlockedTags(State.Dead)的技能
    // 都会被自动阻止激活，不需要手动编写判断代码
    TargetASC->AddLooseGameplayTag(
        FGameplayTag::RequestGameplayTag(FName("State.Dead"))
    );
    // 注意：这里用的是 AddLooseGameplayTag，而不是通过GE添加
    // 因为"死亡"是一个直接的状态，不是持续时间效果
    // 复活时需要手动 RemoveLooseGameplayTag

    // ===== 步骤2：触发死亡事件（如果需要）=====
    // 可以通过GameplayEvent系统通知其他系统
    FGameplayEventData DeathEventData;
    // 谁造成了死亡（EffectContext记录了谁施放了这次效果）
    DeathEventData.Instigator = Data.EffectSpec.GetEffectContext().GetInstigator();
    // 谁死了
    DeathEventData.Target = Data.Target.AvatarActor.Get();
    // 事件携带的上下文
    DeathEventData.ContextHandle = Data.EffectSpec.GetEffectContext();
```

```cpp
    if (AttributeSet)
    {
        return AttributeSet;
    }

    // 备用方案：从ASC中查找
    return AbilitySystemComponent ?
        const_cast<UMyAttributeSet*>(
            AbilitySystemComponent->GetAttributeSet<UMyAttributeSet>()
        ) : nullptr;

void AMyPlayerState::BeginPlay()
{
    // ===== 重要：先调用Super =====
    Super::BeginPlay();

    // ===== 安全检查 =====
    if (!AbilitySystemComponent)
    {
        UE_LOG(LogTemp, Error,
            TEXT("AMyPlayerState::BeginPlay - AbilitySystemComponent is null!"));
        return;
    }

    // ===== 步骤1：创建AttributeSet并注册到ASC =====
    // 使用NewObject动态创建（因为AttributeSet不是Component，是纯UObject）
    AttributeSet = NewObject<UMyAttributeSet>(this);
    AbilitySystemComponent->AddAttributeSetSubobject(AttributeSet);

    // ===== 步骤2：初始化ASC的ActorInfo =====
    // OwnerActor: PlayerState本身（逻辑持有者，跨关卡持久）
    // AvatarActor: PlayerState的Pawn（物理代表，可能被销毁和重建）
    // 这里用this作为临时Avatar，后续在Character的PossessedBy中更新
    AbilitySystemComponent->InitAbilityActorInfo(this, GetPawn());

    // ===== 步骤3：初始化默认属性值 =====
    InitDefaultAttributes();

    // ===== 步骤4：授予默认技能 =====
    GiveDefaultAbilities();

    UE_LOG(LogTemp, Log, TEXT("AMyPlayerState::BeginPlay - GAS系统初始化完成"));
}

void AMyPlayerState::InitDefaultAttributes()
{
    // ===== 通过GE初始化属性默认值 =====
    // 为什么用GE而不是直接SetHealth(100)？
    // 1. GE是数据驱动的——策划可以在蓝图中调整初始值
    // 2. GE可以同时设置多个属性
    // 3. GE可以触发GameplayCue（初始化的特效等）

    if (!AbilitySystemComponent || !DefaultAttributeGE)
    {
        UE_LOG(LogTemp, Warning,
            TEXT("InitDefaultAttributes - ASC or DefaultAttributeGE is null"));
        return;
    }

    // 创建EffectContext（效果上下文——包含来源信息）
    FGameplayEffectContextHandle Context = AbilitySystemComponent->MakeEffectContext();
    Context.AddSourceObject(this);  // 来源是这个PlayerState

    // 创建Spec（GE的实例）
    FGameplayEffectSpecHandle SpecHandle =
        AbilitySystemComponent->MakeOutgoingSpec(DefaultAttributeGE, 1, Context);

    if (SpecHandle.IsValid())
    {
        // 应用Spec到自身
        // ApplyGameplayEffectSpecToSelf 会在服务器端执行：
        //   1. 计算所有Modifier
        //   2. 修改AttributeSet中的属性值
        //   3. 触发PreAttributeChange/PostGameplayEffectExecute
        AbilitySystemComponent->ApplyGameplayEffectSpecToSelf(
            *SpecHandle.Data.Get()
        );
    }

    // 应用二级属性初始化（如果有）
    if (DefaultSecondaryAttributeGE)
    {
        FGameplayEffectSpecHandle SpecHandle2 =
            AbilitySystemComponent->MakeOutgoingSpec(DefaultSecondaryAttributeGE, 1, Context);
        if (SpecHandle2.IsValid())
        {
            AbilitySystemComponent->ApplyGameplayEffectSpecToSelf(
                *SpecHandle2.Data.Get()
            );
        }
    }
}

void AMyPlayerState::GiveDefaultAbilities()
{
    // ===== 给予默认技能 =====
    // GiveAbility ≠ 激活技能
    // GiveAbility = 把技能"安装"到角色身上，让它"可用"
    // 技能等待触发条件（按键输入、事件等）才能真正激活

    if (!AbilitySystemComponent)
    {
        return;
    }

    // 遍历默认技能列表
    for (int32 i = 0; i < DefaultAbilities.Num(); ++i)
    {
        TSubclassOf<UGameplayAbility> AbilityClass = DefaultAbilities[i];

        if (!AbilityClass)
        {
            continue;  // 跳过空的配置
        }

        // 创建技能规格（Skill Spec）
        FGameplayAbilitySpec AbilitySpec(
            AbilityClass,   // 技能类
            1,              // 技能等级（可以通过升级提升等级）
            i               // InputID（可选的输入绑定ID）
        );

        // 给予技能——返回Handle，后续用于激活/移除
        FGameplayAbilitySpecHandle Handle =
            AbilitySystemComponent->GiveAbility(AbilitySpec);

        UE_LOG(LogTemp, Log, TEXT("给予技能: %s (InputID: %d)"),
            *AbilityClass->GetName(), i);
    }
}
```

```cpp
bool AMyGASCharacter::TryActivateAbilityByInputID(int32 InputID)
{
    UAbilitySystemComponent* ASC = GetAbilitySystemComponent();
    if (!ASC)
    {
        return false;
    }

    // ─── 通过InputID查找并激活技能 ───
    // 遍历ASC中所有已给予的、可激活的技能
    TArray<FGameplayAbilitySpecHandle> HandlesToActivate;

    for (const FGameplayAbilitySpec& Spec : ASC->GetActivatableAbilities())
    {
        // 找到InputID匹配的技能
        if (Spec.InputID == InputID && Spec.Ability)
        {
            // 尝试激活
            // TryActivateAbility 会：
            // 1. 检查技能是否已给予
            // 2. 检查所有标签条件（ActivationRequiredTags/BlockedTags）
            // 3. 检查冷却
            // 4. 检查消耗
            // 5. 如果全部通过 → 调用GA::ActivateAbility
            bool bSuccess = ASC->TryActivateAbility(Spec.Handle);

            if (bSuccess)
            {
                UE_LOG(LogTemp, Log, TEXT("成功激活技能 InputID=%d: %s"),
                    InputID, *Spec.Ability->GetName());
            }
            return bSuccess;
        }
    }

    UE_LOG(LogTemp, Warning, TEXT("没有找到InputID=%d的技能"), InputID);
    return false;
}
```

```cpp
    // 客户端按键 → 服务器收到请求 → 服务器执行 → 结果同步回客户端
    NetExecutionPolicy = EGameplayAbilityNetExecutionPolicy::ServerOnly;

    // ===== 3. 技能自身的标签（用于被其他技能识别）=====
    // 这个技能是"主动技能"
    AbilityTags.AddTag(
        FGameplayTag::RequestGameplayTag(FName("Ability.Type.Active"))
    );
    // 这个技能是"火系技能"
    AbilityTags.AddTag(
        FGameplayTag::RequestGameplayTag(FName("Ability.Element.Fire"))
    );

    // ===== 4. 激活时取消哪些标签的技能 =====
    // 火球术是瞬时技能，不打断其他技能
    // CancelAbilitiesWithTag 留空

    // ===== 5. 激活期间阻止哪些标签的技能的激活 =====
    // 火球术是瞬时的（释放瞬间结束），不阻塞其他技能
    // BlockAbilitiesWithTag 留空

    // ===== 6. 激活时给ASC临时加哪些标签 =====
    // 释放火球时，角色处于"施法中"状态
    // 这个标签会阻止移动（如果移动逻辑检查这个标签的话）
    // 技能结束时GAS自动移除这些标签
    ActivationOwnedTags.AddTag(
        FGameplayTag::RequestGameplayTag(FName("State.Combat.Casting"))
    );

    // ===== 7. 需要哪些标签才能激活 =====
    // 火球术不需要特定状态标签
    // ActivationRequiredTags 留空

    // ===== 8. 有这些标签时不能激活 =====
    // 死亡、被任何控制效果影响时不能释放
    ActivationBlockedTags.AddTag(
        FGameplayTag::RequestGameplayTag(FName("State.Dead"))
    );
    // 注意：使用父级标签 State.CC 会自动匹配所有控制效果！
    // State.CC → 匹配 State.CC.Stun, State.CC.Root, State.CC.Silence...
    ActivationBlockedTags.AddTag(
        FGameplayTag::RequestGameplayTag(FName("State.CC"))
    );
}

void UMyFireballAbility::ActivateAbility(
    const FGameplayAbilitySpecHandle Handle,
    const FGameplayAbilityActorInfo* ActorInfo,
    const FGameplayAbilityActivationInfo ActivationInfo,
    const FGameplayEventData* TriggerEventData)
{
    // ═══════════════════════════════════════════════════════════
    // 步骤1：提交技能 ——检查消耗并应用消耗
    // ═══════════════════════════════════════════════════════════
    //
    // CommitAbility 内部执行：
    // ① CommitAbilityCost()  → CheckCost() + ApplyCost()
    //    默认实现：检查CostGE是否可应用（法力够不够？）
    //             应用CostGE（扣除法力）
    // ② CommitAbilityCooldown()
    //    应用CooldownGE（启动冷却计时）
    //
    // CommitAbility返回false的情况：
    //   - 法力不足（CheckCost失败）
    //   - 冷却中（CheckCooldown失败）
    //   - 其他防止技能提交的条件

    if (!CommitAbility(Handle, ActorInfo, ActivationInfo))
    {
        // 提交失败 → 结束技能（法力不足或冷却中）
        // EndAbility的bWasCancelled=true表示技能被取消
        EndAbility(Handle, ActorInfo, ActivationInfo, true, true);
        return;
    }

    // 提交成功后：
    // ✅ 法力值已经扣除（CostGE已应用）
    // ✅ 冷却GE已应用（冷却计时已启动）
    // ✅ ActivationOwnedTags已自动添加到ASC上
    //    → ASC上现在有 State.Combat.Casting 标签

    // ═══════════════════════════════════════════════════════════
    // 步骤2：获取必要的引用
    // ═══════════════════════════════════════════════════════════

    UAbilitySystemComponent* ASC = ActorInfo->AbilitySystemComponent.Get();
    AActor* Avatar = ActorInfo->AvatarActor.Get();
    ACharacter* Character = Cast<ACharacter>(Avatar);

    if (!ASC || !Avatar)
    {
        EndAbility(Handle, ActorInfo, ActivationInfo, true, true);
        return;
    }

    // ═══════════════════════════════════════════════════════════
    // 步骤3：播放施法动画
    // ═══════════════════════════════════════════════════════════
    if (Character && CastingAnimMontage)
    {
        // 使用AbilityTask播放蒙太奇（异步）
        // 这个Task会在蒙太奇完成、中断、混合结束时触发回调
        UAbilityTask_PlayMontageAndWait* MontageTask =
            UAbilityTask_PlayMontageAndWait::CreatePlayMontageAndWaitProxy(
                this,                       // 此任务的拥有者GA
                TEXT("FireballCast"),      // 任务实例名称（用于调试）
                CastingAnimMontage,        // 要播放的动画蒙太奇
                1.0f,                      // 播放速率
                NAME_None,                 // 起始Section名称
                true,                      // bStopWhenAbilityEnds：技能结束时自动停止
                1.0f                       // 动画根节点缩放
            );

        // 绑定回调
        MontageTask->OnBlendOut.AddDynamic(this, &UMyFireballAbility::OnMontageCompleted);
        MontageTask->OnInterrupted.AddDynamic(this, &UMyFireballAbility::OnMontageInterrupted);

        // 激活异步任务
        MontageTask->ReadyForActivation();
    }

    // ═══════════════════════════════════════════════════════════
    // 步骤4：延迟生成火球（配合动画"出手"时间点）
    // ═══════════════════════════════════════════════════════════
    UAbilityTask_WaitDelay* DelayTask = UAbilityTask_WaitDelay::WaitDelay(
        this,
        ProjectileSpawnDelay    // 0.5秒后（动画中角色"抛出"火球的时刻）
    );
    DelayTask->OnFinish.AddDynamic(this, &UMyFireballAbility::OnSpawnDelayFinished);
    DelayTask->ReadyForActivation();

    // 注意：这里不立即调用 EndAbility
    // 技能会在动画结束或火球生成后结束
    // GA的异步任务会自动管理生命周期
}

void UMyFireballAbility::OnSpawnDelayFinished()
{
    // 延迟结束 → 动画应该到了"出手"帧 → 生成火球
    SpawnFireballProjectile();
}

void UMyFireballAbility::SpawnFireballProjectile()
{
    // 获取角色信息
    const FGameplayAbilityActorInfo* ActorInfo = GetActorInfo();
    if (!ActorInfo || !ActorInfo->AvatarActor.IsValid())
    {
        return;
    }

    AActor* Avatar = ActorInfo->AvatarActor.Get();
    UWorld* World = Avatar->GetWorld();

    if (!World || !ProjectileClass)
    {
        UE_LOG(LogTemp, Error, TEXT("Fireball: World or ProjectileClass is null"));
        return;
    }

    // ─── 计算生成位置和旋转 ───
    // 在角色前方偏移位置生成火球
    FVector SpawnLocation = Avatar->GetActorLocation() +
                            Avatar->GetActorRotation().RotateVector(ProjectileSpawnOffset);
    FRotator SpawnRotation = Avatar->GetActorRotation();

    // ─── 配置生成参数 ───
    FActorSpawnParameters SpawnParams;
    SpawnParams.Owner = ActorInfo->OwnerActor.Get();     // 技能拥有者
    SpawnParams.Instigator = Cast<APawn>(Avatar);         // 触发者
    SpawnParams.SpawnCollisionHandlingOverride =
        ESpawnActorCollisionHandlingMethod::AlwaysSpawn;  // 忽略碰撞直接生成

    // ─── 生成投射物 ───
    AActor* Projectile = World->SpawnActor<AActor>(
        ProjectileClass,
        SpawnLocation,
        SpawnRotation,
        SpawnParams
    );

    if (Projectile)
    {
        // 通知蓝板火球已释放
        OnFireballCast.Broadcast();

        // 注意：实际项目中，投射物需要：
        // 1. 持有DamageEffectClass和来源ASC的引用
        // 2. 在碰撞时通过来源ASC的ApplyGameplayEffectSpecToTarget造成伤害
        // 3. 使用SetByCaller设置动态伤害值
        //
        // 这里省略了投射物的具体实现，因为在章节案例中
        // 我们关注的是GA如何与GE配合，而非投射物的物理移动
    }
}
```

## 第六步：创建伤害GE的MMC (MMC_FireballDamage)

```cpp
// ===== MMC_FireballDamage.h =====
#pragma once

#include "CoreMinimal.h"
#include "GameplayModMagnitudeCalculation.h"
#include "MMC_FireballDamage.generated.h"

/**
 * 火球术伤害计算 MMC
 *
 * 计算公式：
 * 伤害 = (攻击力 * 技能系数 - 目标防御力 * 0.5) * 暴击修正
 *
 * 捕获的属性：
 * - 来源攻击力 (AttackPower)
 * - 目标防御力 (Defense)
 * - 来源暴击率 (CriticalChance)
 * - 来源暴击伤害 (CriticalDamage)
 */
UCLASS()
class MYGAME_API UMMC_FireballDamage : public UGameplayModMagnitudeCalculation
{
    GENERATED_BODY()

public:
    UMMC_FireballDamage();

    // 核心计算函数
    virtual float CalculateBaseMagnitude_Implementation(
        const FGameplayEffectSpec& Spec
    ) const override;

private:
    // 从来源捕获的属性
    FGameplayEffectAttributeCaptureDefinition AttackPowerDef;
    FGameplayEffectAttributeCaptureDefinition CriticalChanceDef;
    FGameplayEffectAttributeCaptureDefinition CriticalDamageDef;

    // 从目标捕获的属性
    FGameplayEffectAttributeCaptureDefinition DefenseDef;
};
```

```cpp
// ===== MMC_FireballDamage.cpp =====
#include "MMC_FireballDamage.h"
#include "MyAttributeSet.h"  // 你的属性集，定义了GetXxxAttribute

UMMC_FireballDamage::UMMC_FireballDamage()
{
    // ===== 定义需要从来源（施法者）捕获的属性 =====

    AttackPowerDef.AttributeToCapture = UMyAttributeSet::GetAttackPowerAttribute();
    AttackPowerDef.AttributeSource = EGameplayEffectAttributeCaptureSource::Source;
    AttackPowerDef.bSnapshot = true;  // ⭐ 快照模式——伤害基于释放时的攻击力
    // 这样即使火球飞行过程中攻击力变化，伤害不变

    CriticalChanceDef.AttributeToCapture = UMyAttributeSet::GetCriticalChanceAttribute();
    CriticalChanceDef.AttributeSource = EGameplayEffectAttributeCaptureSource::Source;
    CriticalChanceDef.bSnapshot = true;

    CriticalDamageDef.AttributeToCapture = UMyAttributeSet::GetCriticalDamageAttribute();
    CriticalDamageDef.AttributeSource = EGameplayEffectAttributeCaptureSource::Source;
    CriticalDamageDef.bSnapshot = true;

    // ===== 定义需要从目标捕获的属性 =====

    DefenseDef.AttributeToCapture = UMyAttributeSet::GetDefenseAttribute();
    DefenseDef.AttributeSource = EGameplayEffectAttributeCaptureSource::Target;
    DefenseDef.bSnapshot = false;  // 实时模式——使用目标当前的防御力

    // ===== 注册所有捕获定义 =====
    // GAS在计算GE时会根据这个列表自动获取对应属性的值
    RelevantAttributesToCapture.Add(AttackPowerDef);
    RelevantAttributesToCapture.Add(CriticalChanceDef);
    RelevantAttributesToCapture.Add(CriticalDamageDef);
    RelevantAttributesToCapture.Add(DefenseDef);
}

float UMMC_FireballDamage::CalculateBaseMagnitude_Implementation(
    const FGameplayEffectSpec& Spec) const
{
    // ===== 步骤1：获取捕获的属性值 =====
    // GetCapturedAttributeMagnitude 会从对应的ASC获取属性的当前值
    // 如果bSnapshot=true，获取的是GE创建时的快照值

    // 获取来源的攻击力（快照值）
    float AttackPower = 0.0f;
    GetCapturedAttributeMagnitude(
        AttackPowerDef,
        Spec,
        FAggregatorEvaluateParameters(),
        AttackPower
    );

    // 获取目标的防御力（实时值）
    float Defense = 0.0f;
    GetCapturedAttributeMagnitude(
        DefenseDef,
        Spec,
        FAggregatorEvaluateParameters(),
        Defense
    );

    // 获取暴击率和暴击伤害
    float CriticalChance = 0.0f;
    GetCapturedAttributeMagnitude(
        CriticalChanceDef,
        Spec,
        FAggregatorEvaluateParameters(),
        CriticalChance
    );

    float CriticalDamage = 1.5f;  // 默认1.5倍
    GetCapturedAttributeMagnitude(
        CriticalDamageDef,
        Spec,
        FAggregatorEvaluateParameters(),
        CriticalDamage
    );

    // ===== 步骤2：获取SetByCaller动态参数 =====
    // 技能系数（在GA中通过Spec设置）
    float SkillCoefficient = Spec.GetSetByCallerMagnitude(
        FGameplayTag::RequestGameplayTag(FName("Data.SkillCoeff")),
        false,  // 如果没有设置，不输出警告
        1.0f    // 默认系数1.0
    );

    // ===== 步骤3：计算基础伤害 =====
    // 伤害 = (攻击力 * 技能系数 - 防御力 * 0.5)
    float BaseDamage = (AttackPower * SkillCoefficient) - (Defense * 0.5f);

    // 伤害至少为1（不能为负或0）
    BaseDamage = FMath::Max(BaseDamage, 1.0f);

    // ===== 步骤4：暴击判定 =====
    bool bIsCritical = (FMath::FRand() <= CriticalChance);  // 随机判定
    if (bIsCritical)
    {
        BaseDamage *= CriticalDamage;  // 暴击伤害倍率
    }

    // ===== 步骤5：调试日志 =====
    UE_LOG(LogTemp, Log,
        TEXT("火球伤害计算: 攻击力=%.1f 防御=%.1f 系数=%.2f 暴击=%s → 最终伤害=%.1f"),
        AttackPower, Defense, SkillCoefficient,
        bIsCritical ? TEXT("是") : TEXT("否"),
        BaseDamage
    );

    return BaseDamage;  // 返回的值被GE的Modifier使用
}
```

## 第八步：完整的伤害应用流程

### 在GA中创建并应用伤害GE

```cpp
// ===== 在技能中应用伤害GE的完整流程 =====

void ApplyDamageToTarget(
    UAbilitySystemComponent* SourceASC,      // 施法者的ASC
    AActor* TargetActor,                     // 目标Actor
    TSubclassOf<UGameplayEffect> DamageGE,   // 伤害GE模板
    float SkillCoefficient)                  // 技能系数
{
    // ─── 1. 获取目标的ASC ───
    if (!TargetActor)
    {
        return;
    }

    // 使用IAbilitySystemInterface获取目标的ASC
    IAbilitySystemInterface* GASInterface =
        Cast<IAbilitySystemInterface>(TargetActor);
    if (!GASInterface)
    {
        UE_LOG(LogTemp, Warning, TEXT("目标没有实现IAbilitySystemInterface!"));
        return;
    }

    UAbilitySystemComponent* TargetASC = GASInterface->GetAbilitySystemComponent();
    if (!TargetASC || !SourceASC)
    {
        return;
    }

    // ─── 2. 创建效果上下文 ───
    // EffectContext 记录了"谁、怎样、在哪里"造成了这个效果
    FGameplayEffectContextHandle EffectContext = SourceASC->MakeEffectContext();
    // SourceASC的OwnerActor会自动设为Instigator
    // 如果需要设置其他信息：
    // EffectContext.AddHitResult(HitResult);  // 从碰撞产生的效果
    // EffectContext.AddOrigin(Origin);        // 伤害来源位置

    // ─── 3. 创建Spec ───
    FGameplayEffectSpecHandle SpecHandle = SourceASC->MakeOutgoingSpec(
        DamageGE,        // 伤害GE的模板（包含Modifier配置）
        1,               // GE等级
        EffectContext    // 上下文
    );

    if (!SpecHandle.IsValid())
    {
        UE_LOG(LogTemp, Error, TEXT("创建伤害Spec失败!"));
        return;
    }

    // ─── 4. 设置SetByCaller动态参数 ───
    // 这些参数会在MMC的CalculateBaseMagnitude中被读取

    // 设置技能系数
    SpecHandle.Data->SetSetByCallerMagnitude(
        FGameplayTag::RequestGameplayTag(FName("Data.SkillCoeff")),
        SkillCoefficient
    );

    // ─── 5. 应用Spec到目标 ───
    // ApplyGameplayEffectSpecToTarget 会：
    // ① 检查Application Tag Requirements（目标是否符合条件）
    // ② 计算所有Modifier（调用MMC获取最终值）
    // ③ 调用PreAttributeChange（Clamp属性值）
    // ④ 修改属性（Health BaseValue减少）
    // ⑤ 调用PostGameplayEffectExecute（处理死亡等后续逻辑）
    // ⑥ 触发GameplayCue（播放受击特效）

    SourceASC->ApplyGameplayEffectSpecToTarget(
        *SpecHandle.Data.Get(),  // 解引用获取Spec
        TargetASC                // 目标ASC
    );
}
```

## 调试检查清单

完成本案例后，请按以下步骤验证：

- [ ] **基础架构检查**
  - [ ] AMyPlayerState 创建了 ASC 吗？（在构造函数中 CreateDefaultSubobject）
  - [ ] AMyPlayerState 创建了 AttributeSet 吗？（在 BeginPlay 中 NewObject）
  - [ ] AMyPlayerState 实现了 IAbilitySystemInterface 吗？
  - [ ] AMyGASCharacter 实现了 IAbilitySystemInterface 吗？
  - [ ] InitAbilityActorInfo 被正确调用了？（在 PossessedBy 和 OnRep_PlayerState 中）

  - [ ] 所有属性使用了 ATTRIBUTE_ACCESSORS 宏吗？
  - [ ] GetLifetimeReplicatedProps 中注册了所有属性吗？
  - [ ] OnRep 函数中调用了 GAMEPLAYATTRIBUTE_REPNOTIFY 吗？
  - [ ] PreAttributeChange 正确地 Clamp 了值域吗？
  - [ ] PostGameplayEffectExecute 处理了生命值变化和死亡吗？

  - [ ] GA 的 NetExecutionPolicy 设置正确吗？
  - [ ] GA 的六种标签配置正确吗？
  - [ ] ActivateAbility 中调用了 CommitAbility 吗？
