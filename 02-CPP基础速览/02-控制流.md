# 2.2 控制流：if/else、循环与Switch

> **目标**：理解程序的执行分支逻辑——根据条件走不同的路，以及如何重复执行代码。

---

## if / else if / else — 条件判断

### 基础语法

```cpp
// 如果括号里的条件为true，就执行花括号里的代码
if (条件)
{
    // 条件为true时执行这里
}

// ===== 最简单的if =====
int32 Health = 50;

if (Health > 0)
{
    UE_LOG(LogTemp, Warning, TEXT("玩家还活着"));
}

// ===== if + else：二选一 =====
if (Health > 0)
{
    UE_LOG(LogTemp, Warning, TEXT("玩家还活着"));
}
else  // ← 上面条件不成立时走这里
{
    UE_LOG(LogTemp, Warning, TEXT("玩家已死亡"));
}

// ===== if + else if + else：多选一 =====
if (Health >= 80)
{
    UE_LOG(LogTemp, Warning, TEXT("血量健康"));
}
else if (Health >= 30)  // ← 只有上面的if条件不成立时才会检查这个
{
    UE_LOG(LogTemp, Warning, TEXT("血量偏低"));
}
else
{
    UE_LOG(LogTemp, Warning, TEXT("血量危险！"));  // 都不成立才到这
}
```

### 比较运算符

| 运算符 | 含义     | 示例                 |
| ------ | -------- | -------------------- |
| `==`   | 等于     | `if (Health == 100)` |
| `!=`   | 不等于   | `if (Health != 0)`   |
| `>`    | 大于     | `if (Health > 50)`   |
| `<`    | 小于     | `if (Health < 20)`   |
| `>=`   | 大于等于 | `if (Health >= 100)` |
| `<=`   | 小于等于 | `if (Health <= 0)`   |

```cpp
// ⚠️ 常见错误：把 == 写成 =
if (Health = 100)  // ❌ 这是赋值，不是比较！会把100赋给Health，且条件永远为true
if (Health == 100) // ✓ 这才是比较
```

### 逻辑运算符

```cpp
// && = 并且（AND）：两边都为true，结果才为true
if (bIsAlive && bHasShield)
//   ────┬───   ─────┬────
//   玩家活着      有护盾
{
    // 只有当"活着"且"有护盾"时才执行
}

// || = 或者（OR）：只要一边为true，结果就为true
if (Health <= 0 || Stamina <= 0)
//   ────┬────   ─────┬─────
//    没血了         没体力了
{
    // 没血或没体力任一满足就执行
}

// ! = 取反（NOT）：true变false，false变true
if (!bIsAlive)
//  ↑
//  "!活着" = "死了"
{
    // 玩家死了
}

// 组合使用
if ((Health > 50 && HasArmor) || bIsInvincible)
//   ─────────┬─────────    ──────┬──────
//      血量>50且有护甲        或者无敌状态
{
    // 复杂条件：要么血量高且有护甲，要么开了无敌
    // 括号保证优先级和可读性
}
```

### 实际游戏示例

```cpp
// 一个伤害处理的例子
void AMyCharacter::TakeDamage(float Damage)
{
    // 1. 先检查是否无敌
    if (bIsInvincible)
    {
        UE_LOG(LogTemp, Warning, TEXT("无敌状态，不受伤害"));
        return;  // ← 直接返回，后面的代码不执行
    }

    // 2. 检查是否有护盾
    if (CurrentShield > 0.0f)
    {
        // 护盾先吸收伤害
        if (CurrentShield >= Damage)
        {
            // 护盾够厚，完全吸收
            CurrentShield -= Damage;
            Damage = 0.0f;
        }
        else
        {
            // 护盾不够，剩下的扣血
            Damage -= CurrentShield;
            CurrentShield = 0.0f;
        }
    }

    // 3. 剩余的伤害扣血量
    CurrentHealth -= Damage;

    // 4. 检查是否死亡
    if (CurrentHealth <= 0.0f)
    {
        CurrentHealth = 0.0f;
        Die();
    }
}
```

## 三元运算符 — 简洁版if/else

```cpp
// 语法：条件 ? 为true的值 : 为false的值

// 普通if/else写法
FString Status;
if (Health > 0)
{
    Status = TEXT("存活");
}
else
{
    Status = TEXT("死亡");
}

// 三元运算符写法（一行搞定）
FString Status = (Health > 0) ? TEXT("存活") : TEXT("死亡");
//               ─────┬─────    ────┬───    ────┬───
//                   条件         true时的值   false时的值
```

> **使用建议**：只在简单赋值时用三元运算符。如果逻辑复杂，还是用if/else更清晰。

## for循环 — 做固定次数的事情

```cpp
// ===== 基础for循环 =====
// for(初始值; 条件; 每次循环后的操作)
for (int32 i = 0; i < 10; i++)   // i从0开始，每次+1，i<10时继续
//      ──┬───  ──┬─  ──┬──
//      起始   条件   步进
{
    UE_LOG(LogTemp, Warning, TEXT("i = %d"), i);
}
// 输出：i = 0, i = 1, i = 2 ... i = 9
// 总共循环10次（0到9）

// i++ 是 "i = i + 1" 的简写
// i-- 是 "i = i - 1" 的简写
// i += 2 是 "i = i + 2" 的简写

// ===== 倒序循环 =====
for (int32 i = 9; i >= 0; i--)
{
    // 从9数到0
}

// ===== 遍历数组 =====
TArray<int32> Scores = {10, 20, 30, 40, 50};

// 方法1：索引遍历（需要知道位置时用）
for (int32 i = 0; i < Scores.Num(); i++)
{
    UE_LOG(LogTemp, Warning, TEXT("第%d个分数: %d"), i, Scores[i]);
}

// 方法2：范围for（最简洁，推荐）
for (int32 Score : Scores)
{
    UE_LOG(LogTemp, Warning, TEXT("分数: %d"), Score);
}

// 方法3：带引用的范围for（需要修改原数组元素时用）
for (int32& Score : Scores)  // & = 引用，修改Score会影响原数组
{
    Score += 5;  // 每个分数+5
}
```

## while循环 — 做"直到条件不满足"的事情

```cpp
// ===== while：先检查条件，再执行 =====
int32 Count = 0;
while (Count < 5)  // 只要Count<5就继续循环
{
    UE_LOG(LogTemp, Warning, TEXT("Count = %d"), Count);
    Count++;  // ← 别忘了改变条件！否则死循环
}
// 输出：Count = 0, 1, 2, 3, 4

// ===== do...while：先执行一次，再检查条件 =====
int32 Num = 0;
do
{
    UE_LOG(LogTemp, Warning, TEXT("这行至少执行一次"));
    Num++;
} while (Num < 0);  // 条件一开始就为false，但do里的代码已经执行了一次

// ===== 游戏中的while示例：寻找最近的敌人 =====
TArray<AActor*> Enemies = GetAllEnemies();
int32 Index = 0;
bool bFoundCloseEnemy = false;

while (Index < Enemies.Num() && !bFoundCloseEnemy)
//     ─────────┬─────────   ─────┬─────
//        还没找完所有敌人      还没找到
{
    float Distance = FVector::Dist(GetActorLocation(), Enemies[Index]->GetActorLocation());
    if (Distance < 500.0f)  // 500单位（5米）以内
    {
        bFoundCloseEnemy = true;
        UE_LOG(LogTemp, Warning, TEXT("找到近处的敌人！"));
    }
    Index++;
}
```

## break与continue — 循环控制

```cpp
// ===== break：立刻退出整个循环 =====
for (int32 i = 0; i < 100; i++)
{
    if (i == 5)
    {
        break;  // ← 到第5次时直接跳出，不再执行后面的循环
    }
    UE_LOG(LogTemp, Warning, TEXT("i = %d"), i);  // 输出：0,1,2,3,4
}

// ===== continue：跳过本次循环，继续下一次 =====
for (int32 i = 0; i < 5; i++)
{
    if (i == 2)
    {
        continue;  // ← i=2时跳过下面代码，直接进入i=3
    }
    UE_LOG(LogTemp, Warning, TEXT("i = %d"), i);  // 输出：0,1,3,4（跳过了2）
}
```

## switch — 多路分支

```cpp
// 当if/else if链太长时，用switch更清晰
int32 WeaponType = 1;  // 0=剑, 1=弓, 2=法杖

switch (WeaponType)  // 检查WeaponType的值
{
    case 0:  // WeapnType == 0
        UE_LOG(LogTemp, Warning, TEXT("使用剑攻击：近距离+10伤害"));
        break;  // ← 必须有break！否则继续执行下一个case

    case 1:  // WeaponType == 1
        UE_LOG(LogTemp, Warning, TEXT("使用弓攻击：远距离+7伤害"));
        break;

    case 2:  // WeaponType == 2
        UE_LOG(LogTemp, Warning, TEXT("使用法杖攻击：魔法+15伤害"));
        break;

    default:  // 以上case都不匹配时走这里
        UE_LOG(LogTemp, Warning, TEXT("未知道武器类型！"));
        break;
}
```

> **⚠️ 注意**：case里忘记写 `break` 会导致 **"fall through"**——程序会继续执行下一个case的代码。大多数时候这不是你想要的，UE会给出编译警告。

## 实际游戏中的控制流示例

```cpp
// 一个完整的Tick函数示例：根据玩家状态做不同的事
void AMyCharacter::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    // 检查各种状态
    if (bIsDead)
    {
        // 死了就不要执行后面的逻辑了
        return;
    }

    // 根据移动状态更新动画
    FVector Velocity = GetVelocity();
    float Speed = Velocity.Size();

    if (Speed > 600.0f)  // 快速移动（冲刺）
    {
        UpdateMovementState(EMovementState::Sprinting);
    }
    else if (Speed > 10.0f)  // 正常移动
    {
        UpdateMovementState(EMovementState::Walking);
    }
    else  // 静止
    {
        UpdateMovementState(EMovementState::Idle);
    }

    // 检查是否掉出地图
    if (GetActorLocation().Z < -1000.0f)  // Z坐标低于-1000（掉入深渊）
    {
        RespawnCharacter();  // 重生角色
        return;
    }

    // 更新Buff计时器
    for (int32 i = ActiveBuffs.Num() - 1; i >= 0; i--)  // 倒序：方便删除
    {
        ActiveBuffs[i].RemainingTime -= DeltaTime;
        if (ActiveBuffs[i].RemainingTime <= 0.0f)
        {
            ActiveBuffs[i].OnExpired();  // 效果结束
            ActiveBuffs.RemoveAt(i);     // 从列表中移除
        }
    }
}
```

> **倒序遍历技巧**：删除数组元素时，**倒序遍历**（从最后一个到第一个），这样删除元素不会影响其他元素的索引。如果正序删除，索引会错位。
