# 2.3 函数

> **目标**：理解函数的定义、参数、返回值、重载，以及UE中特殊的函数标记。

---

## 什么是函数？

**函数 = 把一段代码打包成一个可重复使用的名字**。调用这个名字就等于执行那段代码。

```cpp
// 日常比喻：
// "泡咖啡"是一个函数
// 参数 = 加糖吗、加奶吗（输入）
// 函数体 = 磨豆→烧水→冲泡的过程
// 返回值 = 泡好的咖啡（输出）

// 在代码中：
FString MakeCoffee(bool bAddSugar, bool bAddMilk)  // 函数签名
// │      │         │              │
// │      │         函数名         参数列表
// │      返回值类型
// ↓
{
    // 函数体：具体做什么
    FString Coffee = TEXT("黑咖啡");

    if (bAddSugar)
    {
        Coffee += TEXT("+糖");  // += 是字符串拼接
    }
    if (bAddMilk)
    {
        Coffee += TEXT("+牛奶");
    }

    return Coffee;  // 返回值
}

// 调用函数
FString MyDrink = MakeCoffee(true, false);  // 要糖不要奶 → "黑咖啡+糖"
```

## 函数的基本结构

```cpp
//         ┌── 返回值类型（void = 不返回任何东西）
//         │    ┌── 函数名（驼峰命名，首字母大写）
//         │    │        ┌── 参数列表（类型+名字）
//         │    │        │          ┌── 函数体开始
//         │    │        │          │
void PrintHealth(float Current, float Max)
{
    // 局部变量：只在函数内部存在
    float Percent = (Current / Max) * 100.0f;

    UE_LOG(LogTemp, Warning, TEXT("血量: %.0f/%.0f (%.0f%%)"), Current, Max, Percent);

    // void函数不需要return（也可以写return;直接退出）
}
// └── 函数体结束
```

### 参数传递方式

```cpp
// ===== 1. 值传递（传副本）=====
void ModifyValue(int32 X)  // X是原始变量的副本
{
    X = 100;  // 只改了副本，不影响原始变量
}

int32 MyNumber = 5;
ModifyValue(MyNumber);  // MyNumber 仍然是 5！

// ===== 2. 引用传递（传本身，可以修改原变量）=====
void ModifyReference(int32& X)  // & = 引用，是原变量的别名
{
    X = 100;  // 真的改了原始变量
}

int32 MyNumber = 5;
ModifyReference(MyNumber);  // MyNumber 现在是 100

// ===== 3. 常量引用传递（传本身，但不能修改）=====
void PrintLargeData(const FString& Name)  // const& = 只读引用，高效
{
    // Name = TEXT("New"); // ❌ 编译错误！const不能修改
    UE_LOG(LogTemp, Warning, TEXT("名字: %s"), *Name);  // ✅ 读取没问题
}

// ===== 4. 指针传递 =====
void ModifyPointer(int32* Ptr)  // * = 指针，指向原变量的地址
{
    if (Ptr != nullptr)  // 检查指针是否有效
    {
        *Ptr = 100;  // *Ptr = 解引用，修改指针指向的值
    }
}
```

> **选择规则**：
>
> - 小类型（int, float, bool）：直接值传递即可
> - 大类型（FString, FVector, struct）：用 `const &` 常量引用（避免拷贝开销）
> - 需要修改原变量：用 `&` 引用
> - 可能是空值的：用指针

### 返回值

```cpp
// 返回一个值
int32 Add(int32 A, int32 B)
{
    return A + B;  // return = 输出这个值，并立即退出函数
}
int32 Result = Add(3, 5);  // Result = 8

// 返回FVector
FVector GetSpawnPosition()
{
    return FVector(100.0f, 200.0f, 0.0f);
}

// 返回bool（表示成功/失败）
bool TryPurchaseItem(int32 ItemID)
{
    if (PlayerGold < ItemPrice)
    {
        return false;  // 钱不够，购买失败
    }

    PlayerGold -= ItemPrice;
    AddItemToInventory(ItemID);
    return true;  // 购买成功
}

// void = 不返回值
void PlayDamageSound()
{
    // 播放音效，不需要返回值
}
```

## 函数重载（Overloading）

**同名函数，参数不同 = 函数重载**。编译器根据你传的参数自动选择最匹配的版本。

```cpp
// 三个同名函数，参数类型/个数不同
void SpawnEnemy(FVector Location)
{
    // 在指定位置生成默认敌人
}

void SpawnEnemy(FVector Location, int32 Level)
{
    // 在指定位置生成指定等级的敌人
}

void SpawnEnemy(FVector Location, FString EnemyType)
{
    // 在指定位置生成指定类型的敌人
}

// 调用时编译器自动匹配：
SpawnEnemy(FVector(0,0,100));                          // 调用第1个
SpawnEnemy(FVector(0,0,100), 5);                       // 调用第2个
SpawnEnemy(FVector(0,0,100), TEXT("Boss"));             // 调用第3个
```

## 默认参数

```cpp
// 给参数设置默认值，调用时可以省略
void SpawnEnemy(FVector Location, int32 Level = 1, FString Type = TEXT("Normal"))
//                                   ──────┬──────    ─────────┬─────────
//                                   默认等级=1             默认类型="Normal"
{
    // 生成逻辑
}

// 调用时很灵活：
SpawnEnemy(FVector(0,0,100));                          // 等级1，类型Normal
SpawnEnemy(FVector(0,0,100), 5);                       // 等级5，类型Normal
SpawnEnemy(FVector(0,0,100), 5, TEXT("Boss"));         // 等级5，类型Boss

// ⚠️ 规则：有默认值的参数必须放在参数列表的末尾
// void Bad(int32 A = 1, int32 B);      // ❌ 错误！默认参数后面还有非默认参数
// void Good(int32 A, int32 B = 1);     // ✓ 正确
```

## 内联函数（Inline）

```cpp
// 短小的函数可以用inline提示编译器"把函数体内嵌到调用处，省的来回跳转"
inline int32 Square(int32 X)
{
    return X * X;
}
// inline只是一个建议，编译器可能忽略。UE宏里用的FORCEINLINE会强制内联。
```

## 递归函数

函数自己调用自己。

```cpp
// 经典例子：阶乘
int32 Factorial(int32 N)
{
    if (N <= 1)
    {
        return 1;  // 递归终止条件（必须有，否则无限递归→栈溢出→崩溃）
    }
    return N * Factorial(N - 1);  // 自己调用自己，参数缩小
}
// Factorial(5) = 5 * Factorial(4)
//               = 5 * 4 * Factorial(3)
//               = 5 * 4 * 3 * Factorial(2)
//               = 5 * 4 * 3 * 2 * Factorial(1)
//               = 5 * 4 * 3 * 2 * 1 = 120
```

> **游戏开发中很少用递归**（因为调用栈可能很深，影响性能）。涉及树的遍历时偶尔使用。

## UE中函数的特殊标记

UE通过宏给函数添加了额外的"元信息"：

```cpp
// BlueprintCallable：蓝图可以调用这个函数
UFUNCTION(BlueprintCallable, Category = "Combat")
void Attack();

// BlueprintPure：蓝图可以调用，且不会改变任何状态（类似const函数）
UFUNCTION(BlueprintPure, Category = "Combat")
float GetHealthPercent() const;  // const = 函数内不修改任何成员变量

// BlueprintImplementableEvent：函数签名在C++，实现在蓝图
UFUNCTION(BlueprintImplementableEvent)
void OnDamageReceived(float Damage);  // 没有实现！在蓝图中写逻辑

// BlueprintNativeEvent：C++有默认实现，蓝图可以覆盖
UFUNCTION(BlueprintNativeEvent)
void OnDeath();
// 需要实现：void AMyActor::OnDeath_Implementation() { ... }

// Server/Client/NetMulticast：网络相关（第12章详细讲）
UFUNCTION(Server, Reliable)
void ServerDoAction();

// CallInEditor：可以在编辑器细节面板中手动点击调用
UFUNCTION(CallInEditor, Category = "Debug")
void DebugResetValues();
```

这个表格会经常用到，先有个印象，第3章和第6章会深入讲解。

## 函数指针与Lambda（先了解一下）

```cpp
// ===== 函数指针：把函数像变量一样传来传去 =====
// 适合：回调、事件处理
void PrintSuccess()
{
    UE_LOG(LogTemp, Warning, TEXT("成功！"));
}

// 声明一个函数指针类型
typedef void (*FuncPtr)();  // 指向"返回void、无参数"的函数
FuncPtr MyFunc = &PrintSuccess;  // MyFunc是一个指向PrintSuccess的变量
MyFunc();  // 调用它 → 输出"成功！"

// ===== Lambda表达式（匿名函数）=====
// 需要 C++11+，语法：[捕获](参数) -> 返回类型 { 函数体 }
auto MyLambda = [](int32 X) -> int32
{
    return X * 2;
};
int32 Result = MyLambda(5);  // Result = 10

// Lambda的捕获
int32 BaseValue = 10;
auto AddBase = [BaseValue](int32 X) { return X + BaseValue; };
//             ────┬────
//        [=] 捕获所有外部变量（副本）
//        [&] 捕获所有外部变量（引用）
//        [BaseValue] 只捕获BaseValue（副本）
//        [&BaseValue] 只捕获BaseValue（引用）

// UE中用Lambda的典型场景：定时器
GetWorldTimerManager().SetTimer(
    MyTimerHandle,
    [this]()  // 捕获this指针以便访问成员
    {
        this->Health = 100.0f;  // 恢复血量
        UE_LOG(LogTemp, Warning, TEXT("血量已恢复"));
    },
    3.0f,   // 3秒后执行
    false   // 不循环
);
```
