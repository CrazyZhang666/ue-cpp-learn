## 什么是类？什么是对象？

**类（Class）= 蓝图/模板** — 定义了一种东西"有什么属性"和"能做什么"。
**对象（Object）= 实例** — 根据类这个模板创建出来的具体的东西。

```
类：                "人"
属性：               名字、年龄、身高
行为：               走路、说话、跳跃

对象（实例）：        "张三"（年龄25，身高175）
                    "李四"（年龄30，身高180）
                    "王五"（年龄22，身高168）
```

在代码中：

```cpp
// 定义一个"敌人"类
class AEnemy
{
public:
    // 属性（成员变量）
    FString Name;
    int32 Level;
    float Health;

    // 行为（成员函数/方法）
    void Attack()
    {
        UE_LOG(LogTemp, Warning, TEXT("%s 发动攻击！"), *Name);
    }

    void TakeDamage(float Damage)
    {
        Health -= Damage;
        if (Health <= 0)
        {
            UE_LOG(LogTemp, Warning, TEXT("%s 被击败！"), *Name);
        }
    }
};

// 创建对象（实例化）
AEnemy Goblin;             // 创建一个AEnemy对象，叫Goblin
Goblin.Name = TEXT("哥布林");
Goblin.Level = 3;
Goblin.Health = 50.0f;
Goblin.Attack();           // 输出："哥布林 发动攻击！"

AEnemy Dragon;             // 创建另一个AEnemy对象，叫Dragon
Dragon.Name = TEXT("巨龙");
Dragon.Level = 20;
Dragon.Health = 500.0f;
Dragon.Attack();           // 输出："巨龙 发动攻击！"
```

> 两个对象各自独立。Goblin受伤不会影响Dragon的血量。

## 访问控制符：public / private / protected

```cpp
class ACharacter
{
public:     // ← 公开：类外部可以访问
    FString Name;
    void SayHello();  // 公开的方法

private:    // ← 私有：只有类内部可以访问
    float InternalHealth;  // 外部无法直接访问

protected:  // ← 受保护：类内部和子类可以访问
    int32 PlayerID;         // 子类可以访问

public:
    // Getter/Setter模式：私有成员通过公开接口访问
    float GetHealth() const { return InternalHealth; }  // Getter：获取值
    void SetHealth(float NewValue)                       // Setter：设置值
    {
        // 在Setter里可以做数据验证
        if (NewValue >= 0.0f && NewValue <= MaxHealth)
        {
            InternalHealth = NewValue;
        }
    }
};

// 使用
ACharacter Player;
Player.Name = TEXT("Hero");       // ✅ public成员可以直接访问
Player.SetHealth(100.0f);         // ✅ 通过public方法修改private成员
// Player.InternalHealth = 50.0f; // ❌ 编译错误！private成员不能外部访问
```

> **封装的价值**：把数据藏起来（private），通过公开的函数（public）提供受控的访问。这样你可以换内部的实现方式而不影响外部代码。

## 构造函数与析构函数

```cpp
class AMyActor
{
public:
    // 构造函数：创建对象时自动调用
    AMyActor()
    {
        UE_LOG(LogTemp, Warning, TEXT("一个AMyActor被创建了！"));
        Health = 100.0f;  // 给默认值
    }

    // 带参数的构造函数
    AMyActor(float InitialHealth)
    {
        Health = InitialHealth;
    }

    // 析构函数：对象销毁时自动调用（清理资源）
    ~AMyActor()
    {
        UE_LOG(LogTemp, Warning, TEXT("一个AMyActor被销毁了！"));
    }

private:
    float Health;
};

// 使用
AMyActor Actor1;                  // 调用默认构造函数，Health=100
AMyActor Actor2(200.0f);          // 调用带参数的构造函数，Health=200
// 当 } 结束作用域时，析构函数自动调用
```

> **UE中注意**：UObject派生类的构造函数有特殊规则（不能有参数），详见第3章。

## 继承（Inheritance）

**子类继承父类的所有属性和方法**，然后可以添加自己的东西或覆写父类的行为。

```cpp
// ===== 父类（基类）=====
class ACharacter
{
public:
    FString Name;
    float Health = 100.0f;
    float Speed = 600.0f;

    void Move()
    {
        UE_LOG(LogTemp, Warning, TEXT("%s 在移动"), *Name);
    }

    virtual void Attack()  // virtual = 允许子类覆写
    {
        UE_LOG(LogTemp, Warning, TEXT("普通攻击"));
    }
};

// ===== 子类（派生类）=====
class AWarrior : public ACharacter  // 战士继承自角色
{
public:
    // 战士特有的属性
    int32 ArmorValue = 50;  // 护甲值

    // 战士特有的方法
    void ShieldBlock()
    {
        UE_LOG(LogTemp, Warning, TEXT("举起盾牌格挡！"));
    }

    // 覆写父类的Attack方法
    virtual void Attack() override  // override = 显式声明覆写
    {
        UE_LOG(LogTemp, Warning, TEXT("战士的强力斩击！"));
    }
};

class AMage : public ACharacter  // 法师继承自角色
{
public:
    int32 Mana = 100;  // 法力值

    void CastSpell()
    {
        UE_LOG(LogTemp, Warning, TEXT("施放火球术！"));
    }

    virtual void Attack() override
    {
        UE_LOG(LogTemp, Warning, TEXT("法师释放魔法弹！"));
    }
};

// 使用
AWarrior Warrior;
Warrior.Name = TEXT("亚瑟");
Warrior.Health = 150.0f;       // 继承自ACharacter
Warrior.Move();                 // 继承自ACharacter → "亚瑟 在移动"
Warrior.Attack();              // 覆写的版本 → "战士的强力斩击！"
Warrior.ShieldBlock();         // 战士特有的 → "举起盾牌格挡！"

AMage Mage;
Mage.Name = TEXT("梅林");
Mage.Attack();                 // 覆写的版本 → "法师释放魔法弹！"
Mage.CastSpell();              // 法术特有的 → "施放火球术！"
```

## 多态（Polymorphism）

**通过父类指针或引用调用方法，实际执行的是子类的版本。**

```cpp
// 关键：用父类指针指向子类对象
ACharacter* Ptr1 = new AWarrior();
ACharacter* Ptr2 = new AMage();

// 虽然Ptr1和Ptr2都是"ACharacter*"类型
// 但因为Attack是virtual的，实际调用的是子类版本！
Ptr1->Attack();  // → "战士的强力斩击！"
Ptr2->Attack();  // → "法师释放魔法弹！"

// 如果没有多态，两个都会输出"普通攻击"——这显然不是你想要的

// 如果Attack没有virtual声明
class ABadCharacter
{
public:
    void Attack()  // 没有virtual！
    {
        UE_LOG(LogTemp, Warning, TEXT("普通攻击"));
    }
};
// 那么用父类指针调用时，永远调父类的版本，多态失效
```

多态是UE中最常用的模式：

```cpp
// 所有Actor放在一个TArray里，遍历时不需要知道具体类型
TArray<AActor*> AllCharacters;
AllCharacters.Add(WarriorPtr);
AllCharacters.Add(MagePtr);

// 统一的Attack调用，每个角色自动执行自己的版本
for (AActor* Character : AllCharacters)
{
    // 不需要if(战士)/if(法师)判断！多态自动处理！
    Character->Attack();
}
```

## 虚函数表（vtable）— 多态的实现原理

当你给一个方法加上 `virtual`，编译器会为这个类创建一张**虚函数表（vtable）**：

```
ACharacter 的 vtable:     AWarrior 的 vtable:
┌─────────────┐          ┌─────────────┐
│ Attack → 普通攻击 │      │ Attack → 强力斩击│  ← 指向覆写的版本
└─────────────┘          └─────────────┘

AMage 的 vtable:
┌─────────────┐
│ Attack → 魔法弹│  ← 指向覆写的版本
└─────────────┘
```

每个对象内部有一个隐藏指针指向它所属类的vtable。调用 `Ptr->Attack()` 时：

1. 通过对象找到vtable
2. 在vtable中找到Attack函数的地址
3. 执行该函数

这就是多态的底层实现。编译器自动管理vtable，你不用操心。

## 抽象类与纯虚函数

**纯虚函数 = 只有声明没有实现的虚函数。包含纯虚函数的类就是抽象类，不能实例化。**

```cpp
class AAnimal  // 抽象类
{
public:
    // 纯虚函数：= 0 表示"这个函数没有默认实现，子类必须覆写"
    virtual void MakeSound() = 0;

    // 普通函数
    void Breathe()
    {
        UE_LOG(LogTemp, Warning, TEXT("呼吸中..."));
    }
};

// AAnimal Cat;  // ❌ 编译错误！抽象类不能实例化

class ADog : public AAnimal
{
public:
    virtual void MakeSound() override
    {
        UE_LOG(LogTemp, Warning, TEXT("汪汪！"));
    }
};

ADog Dog;
Dog.MakeSound();  // "汪汪！"
Dog.Breathe();    // "呼吸中..."

// 在UE中，每个继承自AActor的类都不是抽象类（可以实例化）
// 但接口类（以I开头）通常是抽象的
```

## 静态成员

```cpp
class AEnemy
{
public:
    // 静态成员变量：所有对象共享，不属于任何特定对象
    static int32 TotalEnemyCount;  // 总敌人数，所有实例共享一个值

    AEnemy()
    {
        TotalEnemyCount++;  // 每创建一个敌人就+1
    }

    ~AEnemy()
    {
        TotalEnemyCount--;  // 每销毁一个敌人就-1
    }

    // 静态成员函数：不需要对象就能调用
    static int32 GetEnemyCount()
    {
        return TotalEnemyCount;
    }
};

// 静态成员变量必须在类外部定义（不在头文件里初始化）
int32 AEnemy::TotalEnemyCount = 0;

// 使用
UE_LOG(LogTemp, Warning, TEXT("敌人数量: %d"), AEnemy::TotalEnemyCount);  // 0

AEnemy Enemy1;
AEnemy Enemy2;
UE_LOG(LogTemp, Warning, TEXT("敌人数量: %d"), AEnemy::GetEnemyCount());  // 2
// 通过类名::直接访问，不需要对象
```

## struct vs class

在C++中，`struct` 和 `class` **几乎一样**。唯一的区别：

```cpp
struct MyStruct
{
    int32 Value;  // struct中默认是public
};

class MyClass
{
    int32 Value;  // class中默认是private（需要显式加public）
};
```

UE中大量使用 `struct` 定义纯数据结构（如 `FVector`, `FRotator`），用 `class` 定义有行为的对象。这是约定俗成的风格。

## 在UE中使用类与对象

UE在标准C++基础上加了：

1. **UObject** — 所有UE类的基类，提供垃圾回收、序列化、反射等
2. **UCLASS/USTRUCT/UENUM** — 标记类/结构体/枚举，暴露给UE反射系统
3. **UPROPERTY/UFUNCTION** — 标记由UE管理的属性和函数
4. **GENERATED_BODY()** — 自动生成反射样板代码

这些会在第3章详细展开。现在你只需要知道：UE的类体系建立在标准C++的类/继承/多态之上。
