# 2.1 变量与类型

## 什么是变量？

**变量 = 一个带名字的内存盒子**，用来存储数据。

```cpp
int Health = 100;
// │   │       │
// │   │       └─ 存在盒子里的值
// │   └─ 盒子的名字（你起的）
// └─ 盒子的类型（决定了能存什么、占多大内存）
```

用日常比喻：

- `int` = "这个盒子只能装整数"
- `Health` = "盒子上贴的标签是 Health"
- `100` = "盒子里现在装的是100"

## C++基本数据类型

### 整数类型（存整数）

```cpp
// 各种大小的整数——数字越大，占内存越多，能表示的范围越大
int Age = 25;                // 32位整数，最常用。范围：约 -21亿～+21亿
int64 BigNumber = 9999999999LL;  // 64位整数，范围极大。LL后缀表示"这是int64字面量"
int32 SmallNumber = 100;     // 32位整数（与int相同，UE风格中更常用int32）
uint32 Unsigned = 100u;      // 无符号整数：只能>0，但上限更大。u后缀表示unsigned

// 较小范围的整数（省内存但不常用）
int16 Short = 32767;         // 16位整数，范围：-32768～32767
int8 Tiny = 127;             // 8位整数，范围：-128～127
```

> **UE风格建议**：在UE代码中，习惯用 `int32`、`uint32`、`int64` 等明确位数的类型，而不是直接用 `int`、`long`。两者的底层是相同的，但带位数的写法意图更清晰。

### 浮点类型（存小数）

```cpp
float Health = 100.0f;       // 32位浮点数，UE中最常用的数值类型！
//    f后缀表示"这是float字面量"，不加f默认是double

double Precise = 3.14159265358979;  // 64位浮点数，科学计算用
```

> **为什么UE用float而不是double？**
>
> 1. 游戏不需要double的精度（小数点后15位没人看得出）
> 2. float占4字节，double占8字节。100万个数据就是4MB vs 8MB的差距
> 3. GPU专门为float优化，处理float比double快很多
> 4. 大多数游戏物理和图形API都用float

### 字符与字符串

```cpp
char SingleChar = 'A';       // 单个字符（1字节，ASCII编码）

// C风格字符串（不推荐在UE中使用！）
const char* CStyle = "Hello";  // 字符数组指针，古老而危险

// C++标准字符串
std::string StdStr = "Hello";  // 标准C++字符串

// UE字符串（★ UE开发中首选）
FString UEStr = TEXT("你好，UE5！");     // 可变字符串
FName UEStr2 = FName(TEXT("MyName"));   // 不可变字符串（用于标识符/名字）
FText UEStr3 = FText::FromString(TEXT("显示文本")); // 用于UI显示、支持本地化
```

#### FString vs FName vs FText 的区别

| 类型      | 用途             | 特点                       | 示例                    |
| --------- | ---------------- | -------------------------- | ----------------------- |
| `FString` | 通用字符串操作   | 可变，功能最多最慢         | 拼接消息、临时字符串    |
| `FName`   | 标识符/名称      | 不可变，超级快（哈希查表） | 资源名、Socket名、Tag名 |
| `FText`   | 显示给玩家的文本 | 支持翻译/本地化            | UI文本、对话文本        |

```cpp
// 使用场景示例
FName AssetName = FName(TEXT("MyMesh"));        // 资产名称 → 用FName
FString DebugMsg = FString::Printf(TEXT("血量: %d"), Health);  // 调试输出 → 用FString
FText UIMessage = FText::FromString(TEXT("按W键前��"));  // UI文本 → 用FText
```

### 布尔类型

```cpp
bool bIsAlive = true;        // 只有true或false两个值
bool bHasWeapon = false;

// UE命名约定：布尔变量以小写b开头
// bIsAlive ✓
// IsAlive  ✗（缺少b前缀，UE规则要求b开头）
```

## 变量声明方式

```cpp
// ===== 普通变量 =====
int Score = 0;               // 声明 + 初始化（推荐始终给初始值）
float Speed;                 // 声明但不初始化（危险！值是随机的垃圾数据）
float Power = 100.0f;        // 声明 + 初始化

// ===== 常量 =====
const int MaxHealth = 1000;  // const：值不可修改
// MaxHealth = 500;          // ❌ 编译报错！const变量不能被修改

// ===== 静态变量 =====
static int CallCount = 0;    // 静态：值在函数调用间保持
// 普通局部变量：每次进入函数都重新创建
// 静态局部变量：只创建一次，之后每次进入函数保持上次的值

// ===== UE特有的宏 =====
TEXT("Hello");               // 把字符串转为UE的宽字符格式
LOCTEXT("Key", "你好");      // 创建可本地化的FText

// ===== auto关键字（自动推断类型）=====
auto Health2 = 100.0f;       // 编译器自动推断Health2是float类型
auto Name = TEXT("Mario");   // 编译器推断Name是const TCHAR*
// auto很方便但不建议过度使用——阅读代码时看不出类型
```

## UE5常用数据类型（你每天都会用到的）

### FVector — 三维向量

```cpp
#include "Math/Vector.h"  // 通常不需要手动include，CoreMinimal.h已包含

// FVector 存储三个float：X, Y, Z
FVector Position(100.0f, 200.0f, 0.0f);
//               ──┬──   ──┬──   ─┬─
//                 X坐标   Y坐标   Z坐标
//             （前后）  （左右）  （上下）

// 创建向量的几种方式
FVector ZeroVector = FVector::ZeroVector;     // (0, 0, 0)
FVector OneVector = FVector::OneVector;       // (1, 1, 1)
FVector Forward = FVector::ForwardVector;     // (1, 0, 0) — X轴前方向
FVector Right = FVector::RightVector;         // (0, 1, 0) — Y轴右方向
FVector Up = FVector::UpVector;               // (0, 0, 1) — Z轴上方

// 向量运算
FVector A(3.0f, 4.0f, 0.0f);
FVector B(1.0f, 2.0f, 0.0f);

FVector Sum = A + B;           // 加法：(4, 6, 0)
FVector Diff = A - B;          // 减法：(2, 2, 0)
FVector Scaled = A * 2.0f;     // 数乘：(6, 8, 0)
float Length = A.Size();       // 长度：5.0（勾股定理：√(3²+4²)）
FVector Normalized = A.GetSafeNormal();  // 归一化（变成长度为1的方向向量）
float DotProduct = FVector::DotProduct(A, B);  // 点积：3*1+4*2=11
FVector Cross = FVector::CrossProduct(A, B);   // 叉积：得到垂直向量
```

> **向量理解**：FVector既可以表示**位置**（在哪个坐标），也可以表示**方向**（指向哪里），还可以表示**速度**（移动多快朝哪走）。一个结构体，三种用途。

### FRotator — 旋转

```cpp
// FRotator 存储三个float：Pitch(俯仰), Yaw(偏航), Roll(翻滚)
FRotator Rotation(0.0f, 90.0f, 0.0f);
//                ──┬──  ──┬──  ──┬──
//              绕Y轴转 绕Z轴转 绕X轴转
//              (抬头)  (左右转) (侧头)

// 常用旋转
FRotator ZeroRot = FRotator::ZeroRotator;  // (0, 0, 0)
FRotator TurnRight(0.0f, 90.0f, 0.0f);    // 向右转90度
FRotator LookUp(45.0f, 0.0f, 0.0f);       // 抬头45度
```

在UE中：

- **Pitch** = 绕Y轴旋转（抬头/低头）
- **Yaw** = 绕Z轴旋转（左转/右转）
- **Roll** = 绕X轴旋转（侧头/歪头）

想象你是飞机驾驶舱里的飞行员：

- Pitch（俯仰）→ 拉操纵杆，机头上下
- Yaw（偏航）→ 踩脚蹬，机头左右
- Roll（翻滚）→ 侧压操纵杆，飞机侧翻

### FTransform — 位置+旋转+缩放的组合

```cpp
// FTransform 是位置(FVector) + 旋转(FRotator) + 缩放(FVector) 的三合一
FTransform Transform(
    FRotator(0.0f, 90.0f, 0.0f),  // 旋转
    FVector(100.0f, 200.0f, 50.0f),  // 位置
    FVector(2.0f, 2.0f, 2.0f)     // 缩放（放大2倍）
);

// 获取信息
FVector MyLocation = Transform.GetLocation();
FRotator MyRotation = Transform.Rotator();
FVector MyScale = Transform.GetScale3D();
```

### TArray — 动态数组

```cpp
// TArray<元素类型> ：可以动态增长/缩小的数组
TArray<int32> Scores;                    // 创建空数组
Scores.Add(10);                          // 添加元素：[10]
Scores.Add(20);                          // 添加元素：[10, 20]
Scores.Add(30);                          // 添加元素：[10, 20, 30]

int32 FirstScore = Scores[0];            // 访问元素：10（索引从0开始）
int32 Num = Scores.Num();                // 元素个数：3

// 遍历数组
for (int32 Score : Scores)               // 范围for循环（简洁）
{
    UE_LOG(LogTemp, Warning, TEXT("分数: %d"), Score);
}

for (int32 i = 0; i < Scores.Num(); i++) // 传统for循环（需要索引用这个）
{
    UE_LOG(LogTemp, Warning, TEXT("第%d个分数: %d"), i, Scores[i]);
}

// 其他操作
Scores.Remove(20);                       // 删除值为20的元素
Scores.RemoveAt(0);                      // 删除索引0位置的元素
Scores.Empty();                          // 清空数组
bool bEmpty = Scores.Num() == 0;         // 判断是否为空
```

### TMap — 字典/映射/哈希表

```cpp
// TMap<键类型, 值类型> ：键值对容器
TMap<FString, int32> HealthMap;
HealthMap.Add(TEXT("Player"), 100);      // 添加键值对
HealthMap.Add(TEXT("Enemy"), 50);

int32 PlayerHealth = HealthMap[TEXT("Player")];  // 通过键获取值：100

// 安全查找（键可能不存在时用这个）
int32* Found = HealthMap.Find(TEXT("Boss"));
if (Found != nullptr)
{
    // 找到了，*Found 就是值
    UE_LOG(LogTemp, Warning, TEXT("Boss血量: %d"), *Found);
}
else
{
    UE_LOG(LogTemp, Warning, TEXT("没找到Boss"));
}

// 遍历字典
for (auto& Pair : HealthMap)
{
    FString Key = Pair.Key;    // 键
    int32 Value = Pair.Value;  // 值
}
```

## 类型转换

```cpp
// C++风格转换（推荐）
float HealthFloat = 100.5f;
int32 HealthInt = static_cast<int32>(HealthFloat);  // float→int：截断小数，=100

int32 Score = 95;
float ScoreFloat = static_cast<float>(Score);       // int→float：=95.0f

// UE提供的安全转换
int32 Clamped = FMath::Clamp(HealthInt, 0, 100);    // 限制范围：确保在0~100之间
int32 Rounded = FMath::RoundToInt(3.7f);            // 四舍五入：=4
int32 Ceiled = FMath::CeilToInt(3.2f);              // 向上取整：=4
int32 Floored = FMath::FloorToInt(3.8f);            // 向下取整：=3
```

## 打印日志（调试常用技能）

```cpp
// UE_LOG：输出到Output Log窗口
// 格式：UE_LOG(日志类别, 级别, TEXT("格式字符串"), 参数...)

// 不同级别的日志
UE_LOG(LogTemp, Log, TEXT("这是普通日志（灰色）"));
UE_LOG(LogTemp, Warning, TEXT("这是警告日志（黄色）"));
UE_LOG(LogTemp, Error, TEXT("这是错误日志（红色）"));

// 带变量的日志
int32 Health = 75;
FString Name = TEXT("Player1");
UE_LOG(LogTemp, Warning, TEXT("角色 %s 的血量是 %d"), *Name, Health);
//                            ↑       ↑    ↑         ↑       ↑
//                            |       |    |         |       └─ 整数占位符
//                            |       |    |         └─ FString变量（*取数据）
//                            |       |    └─ %d = 整数占位符
//                            |       └─ %s = 字符串占位符
//                            └─ 格式字符串

// 常用占位符
// %d  → 整数（int32）
// %f  → 浮点数（float），%.2f = 保留2位小数
// %s  → 字符串（FString前面要加*）
// %c  → 单个字符
// %p  → 指针地址
```
