# 2.5 指针与引用

> **目标**：真正理解指针和引用——这是新手最头疼但最重要的C++概念。在UE中你每天都会用到它们。

---

## 内存地址的概念

在理解指针之前，先理解**内存地址**。

把电脑内存想象成一个超级大的**快递柜**：

```
内存地址:    0x1000  0x1001  0x1002  0x1003  0x1004  ...
             ┌──────┬──────┬──────┬──────┬──────┐
存储的数据:   │  ...  │  ...  │  ...  │  ...  │  ...  │
             └──────┴──────┴──────┴──────┴──────┘
```

- 每个柜子（字节）有一个唯一的编号，这就是**内存地址**
- 变量就是某个地址开始的一段内存区域
- **指针 = 存储地址的变量**——它告诉你"数据在第几个柜子里"

```cpp
int32 Health = 100;
// ┌─────────────────┐
// │   变量名: Health  │
// │   值: 100        │
// │   地址: 0x00A2F  │  ← 这是内存地址，每次运行可能不同
// │   大小: 4字节     │  ← int32占4个字节
// └─────────────────┘
```

## 什么是指针？

**指针就是一个专门存储"内存地址"的变量。**

```cpp
int32 Health = 100;         // 普通变量：存储值100
int32* HealthPtr = &Health; // 指针变量：存储Health的地址

// Health     = 100        （直接值）
// &Health    = 0x00A2F    （"取地址"——Health在内存中的位置）
// HealthPtr  = 0x00A2F    （指针存的就是地址）
// *HealthPtr = 100        （"解引用"——去这个地址取出值）
```

用图理解：

```
      Health                         HealthPtr
  ┌───────────┐                 ┌───────────┐
  │   100     │  ←───指向────   │  0x00A2F   │
  └───────────┘                 └───────────┘
   地址: 0x00A2F                  存着Health的地址
```

### 核心操作符

| 操作符 | 名称 | 作用 | 示例 |
|--------|------|------|------|
| `&变量` | 取地址 | 获取变量的内存地址 | `&Health` → 地址值 |
| `类型*` | 指针类型 | 声明一个指针 | `int32* Ptr` |
| `*指针` | 解引用 | 去地址处取值 | `*Ptr` → 100 |

```cpp
int32 Value = 42;

int32* Ptr = &Value;   // Ptr现在"指向"Value
//  *Ptr = 取指针指向的值
//  Ptr = 获取指针本身的值（即Value的地址）

UE_LOG(LogTemp, Warning, TEXT("Value的值: %d"), Value);    // 42
UE_LOG(LogTemp, Warning, TEXT("Ptr的值(地址): %p"), Ptr);   // 0x00A2F之类的地址
UE_LOG(LogTemp, Warning, TEXT("*Ptr的值: %d"), *Ptr);       // 42（解引用）

*Ptr = 99;  // 通过指针修改Value的值
UE_LOG(LogTemp, Warning, TEXT("Value的新值: %d"), Value);   // 99！
```

## 为什么需要指针？（三个最重要的原因）

### 原因1：避免拷贝大对象

```cpp
// 不用指针：每次传参都要拷贝整个对象（慢 + 占内存）
void ProcessLargeData(FString Data)  // Data是整个字符串的拷贝（几千个字符！）
{
    // ...
}

// 用指针：只传一个地址（8字节），不拷贝数据本身
void ProcessLargeData(FString* DataPtr)  // 只传了地址
{
    // ...
}

// 更好的方式：用引用（语法糖）
void ProcessLargeData(const FString& Data)  // 不拷贝，且保证不修改
{
    // ...
}
```

### 原因2：在函数内修改外部变量

```cpp
void HealPlayer(float Amount)  // 值传递：改不了调用处的变量
{
    Amount *= 2;  // 只改了副本
}

void HealPlayer(float* Amount)  // 指针传递：可以修改外部变量
{
    *Amount *= 2;  // 真正修改了调用处的变量
}

// 使用
float Health = 50.0f;
HealPlayer(Health);    // Health还是50（值传递，没改）
HealPlayer(&Health);   // Health变成100（指针传递，改了）
```

### 原因3：动态分配内存

```cpp
// 在运行时按需创建对象

// C++原生方式（不推荐在UE中直接使用）
AEnemy* Enemy = new AEnemy();  // new = 在堆上分配内存，返回指针
Enemy->Name = TEXT("Boss");
delete Enemy;  // 用完必须delete，否则内存泄漏！这是C++最大的坑之一

// UE方式（推荐）
AEnemy* Enemy = GetWorld()->SpawnActor<AEnemy>();  // UE管理生命周期
// 不需要手动delete！UE的垃圾回收会自动处理
```

## nullptr与空指针检查

```cpp
int32* Ptr = nullptr;  // nullptr = 空指针，"不指向任何东西"

// 解引用空指针会导致崩溃！所以使用前必须检查
if (Ptr != nullptr)
{
    *Ptr = 100;  // 安全：确认不是空指针后才使用
}
else
{
    UE_LOG(LogTemp, Error, TEXT("指针为空！不能解引用"));
}

// UE提供的便捷检查宏（UE代码中用这些更安全）
if (IsValid(MyActor))  // 检查UObject派生类的指针
{
    MyActor->Destroy();
}

if (ensure(Ptr != nullptr))  // 如果为null，打印调用栈但不崩溃
{
    *Ptr = 100;
}

if (check(Ptr != nullptr))   // 如果为null，直接崩溃（用于"绝不可能为null"的场合）
{
    *Ptr = 100;
}
```

## 指针与数组

```cpp
int32 Scores[5] = {10, 20, 30, 40, 50};

// 数组名本身就是指向第一个元素的指针！
int32* Ptr = Scores;  // 等同于 int32* Ptr = &Scores[0];

// 通过指针+偏移访问数组元素
int32 First = *Ptr;       // 10（解引用=取第一个元素）
int32 Second = *(Ptr + 1); // 20（偏移1个int32 = 偏移4字节）
int32 Third = *(Ptr + 2);  // 30

// *(Ptr + i) 完全等同于 Ptr[i]
int32 Third_Also = Ptr[2]; // 30（更可读的写法）
```

> 在UE中你几乎不需要操作原始数组和指针运算（TArray帮你做了一切），但理解原理很重要。

## 指针与类的箭头操作符

```cpp
AEnemy EnemyStack;           // 栈上的对象
EnemyStack.Name = TEXT("A"); // 用 . 访问成员

AEnemy* EnemyPtr = new AEnemy();  // 堆上的对象（指针）
EnemyPtr->Name = TEXT("B");       // 用 -> 访问成员（等价于 (*EnemyPtr).Name）

//  -> 是 (*ptr). 的语法糖（简写）
(*EnemyPtr).Name = TEXT("B");  // 等价写法，但你永远会写 ->
```

## 引用——指针的"安全版"

引用是指针的语法简化，**引用就是别名**：

```cpp
int32 Original = 10;
int32& Ref = Original;  // & = 引用类型，Ref是Original的别名

Ref = 20;  // 修改Ref就是修改Original
UE_LOG(LogTemp, Warning, TEXT("Original = %d"), Original);  // 20

// 引用 vs 指针的区别
int32* Ptr = &Original;   // 指针：存储地址
int32& Ref = Original;    // 引用：直接是别名

*Ptr = 30;                // 指针需要解引用
Ref = 40;                 // 引用直接使用（更简洁）

Ptr = nullptr;            // ✅ 指针可以是空
// int32& Ref2 = nullptr; // ❌ 引用不能为空！必须初始化为现有变量

Ptr = &OtherVariable;      // ✅ 指针可以改变指向
// Ref = OtherVariable;    // ❌ 引用不能重定向（这行会变成赋值，不是改变指向）
```

| 特性 | 指针 | 引用 |
|------|------|------|
| 声明 | `int32* p` | `int32& r` |
| 访问值 | `*p` | `r`（无需解引用） |
| 可为空 | ✅ | ❌ 必须绑定有效变量 |
| 可重指向 | ✅ | ❌ 绑定后不能改 |
| 取地址 | `p` 本身就是地址 | `&r` 才能取地址 |
| 安全程度 | 中（随时可能空） | 高（不能空） |
| 使用场景 | 可能为空/需要重新指向 | 传递参数/已知有效 |

## 在UE中常见的指针使用场景

```cpp
// ===== 场景1：用UPROPERTY标记的组件指针（UE管理生命周期）=====
UPROPERTY()
class UStaticMeshComponent* MeshComp;  // UE的GC会管理

// ===== 场景2：从世界获取Actor =====
AActor* FoundActor = GetWorld()->SpawnActor<AEnemy>(...);
if (FoundActor != nullptr)  // Spawn可能失败
{
    FoundActor->SetActorLocation(FVector(0,0,100));
}

// ===== 场景3：使用临时指针遍历 =====
for (AActor* Actor : AllActorsArray)
{
    if (Actor != nullptr)
    {
        Actor->Destroy();
    }
}

// ===== 场景4：使用const引用传参（不拷贝 + 保证不修改）=====
void PrintInfo(const FString& Name, const FVector& Position)
{
    // Name和Position都是引用，不拷贝大对象
    // const保证函数内不会修改它们
    UE_LOG(LogTemp, Warning, TEXT("%s 在 (%f, %f, %f)"), *Name,
           Position.X, Position.Y, Position.Z);
}

// ===== 场景5：返回值用指针（可能为空）vs 引用（保证有效）=====
AEnemy* FindNearestEnemy();  // 可能返回nullptr，调用者需要检查
AEnemy& GetPlayer();          // 保证有效，调用者不需要检查
```

## 常见指针错误

```cpp
// ❌ 错误1：解引用空指针（最常见的崩溃原因）
int32* Ptr = nullptr;
*Ptr = 100;  // CRASH! 往不存在的地址写数据

// ❌ 错误2：野指针（指向已释放的内存）
AEnemy* Enemy = new AEnemy();
delete Enemy;
Enemy->Attack();  // CRASH! Enemy指向的内存已经释放了

// ❌ 错误3：返回局部变量的地址
int32* BadFunction()
{
    int32 LocalVar = 42;
    return &LocalVar;  // ❌ LocalVar在函数结束时销毁，返回的指针变成野指针！
}

// ❌ 错误4：UPROPERTY指针没加UPROPERTY
class USkeletalMeshComponent* Mesh;  // ❌ 可能被垃圾回收，导致野指针崩溃
UPROPERTY()
class USkeletalMeshComponent* Mesh;  // ✅ UE的GC知道这个指针在使用

// ❌ 错误5：TArray元素指针在Add后失效
TArray<int32> Arr;
int32* Ptr = &Arr[0];
Arr.Add(100);  // TArray可能重新分配内存，Ptr变成野指针！
```
