# 12.3 服务器权威与网络预测

> **目标**：理解服务器权威架构的设计原则，掌握客户端预测与服务器调和的实现方法。

## 新手先读：先分清“真实”和“感觉”

多人游戏网络最容易混乱的地方，是同一件事在客户端和服务器上各有一份表现。先记住：

| 概念 | 新手先理解成 |
| ---- | ------------ |
| 服务器权威 | 服务器说了算，最终结果以服务器为准 |
| 客户端预测 | 客户端先演给玩家看，减少延迟感 |
| 服务器调和 | 如果预测错了，客户端再修正回服务器结果 |
| 网络相关性 | 服务器只把你需要知道的Actor同步给你 |
| 网络平滑 | 修正位置时不要瞬移，用插值让画面顺滑 |

第一遍只需要掌握一个流程：

```
客户端按下开火
    ↓
本地先播放枪口火焰
    ↓
Server RPC请求服务器开火
    ↓
服务器验证并计算命中
    ↓
属性同步/RPC把结果发回客户端
```

> **阅读建议**：先理解服务器权威，再看预测。预测是为了手感，不是为了让客户端拥有最终决定权。

## 服务器权威原则

### 核心铁律

> **服务器拥有绝对的权威。客户端发送的任何数据都是"请求"，服务器有权拒绝。**

这是多人游戏开发的第一原则。如果你只记住一章的内容，记住这个原则就够了。

### 为什么客户端不可信

```
┌────────────────────────────────────────────────────────────────┐
│  客户端上的代码和数据都在"敌人手中"                               │
│                                                                │
│  玩家可以做以下事情来作弊：                                       │
│  - 修改客户端内存（无限血量、无限弹药）                            │
│  - 注入 DLL 修改游戏逻辑                                        │
│  - 拦截网络包修改发送的数据                                      │
│  - 加速游戏客户端让角色跑得更快                                   │
│  - 移除墙壁、烟雾等阻碍物                                        │
│  - 自动瞄准（Aimbot）                                           │
└────────────────────────────────────────────────────────────────┘
```

唯一的应对策略：**服务器验证一切，不相信任何客户端数据**。

### 什么逻辑必须在服务器执行

## 典型网络流程详解

### 完整案例：玩家A攻击玩家B

让我们逐步追踪"玩家A对玩家B造成50点伤害"的完整网络流程：

```
时间线：
═══════════════════════════════════════════════════════════════════

步骤1【客户端A】：玩家A按下开火键
    - 玩家A的 Pawn 是 ROLE_AutonomousProxy（在客户端A上是本地角色）
    - 调用 ServerFireWeapon()  ——这是一个 Server RPC
    - 注意：客户端A自身不执行任何伤害逻辑
    - 客户端A可以做"预测性"的本地效果（预测枪口火焰，不等服务器确认）

步骤2【网络】：RPC 请求发送到服务器
    - ServerFireWeapon RPC 通过网络发送
    - 如果是 Reliable，连接正常时会重传直到确认
    - 包含客户端A当前的瞄准方向和位置信息

步骤3【服务器】：收到 RPC，开始验证
    - 服务器上的玩家A的 Pawn 是 ROLE_Authority
    - ServerFireWeapon_Implementation() 开始执行
    - _Validate 已经通过了（验证弹药、冷却时间等）

步骤4【服务器】：执行射线检测
    - 服务器用玩家A的瞄准方向做 LineTrace（射线检测）
    - 检测到命中了玩家B的角色
    - 这就是"服务器端的命中判定"——客户端不能干预这个结果

步骤5【服务器】：计算并应用伤害
    - 服务器对玩家B执行 ApplyDamage(50.0f)
    - 玩家B的 Health 从 100 变为 50
    - Health 是 Replicated 属性，变化自动标记需要同步

步骤6【服务器→所有客户端】：同步伤害结果
    - 服务器调用 MulticastPlayHitEffect() — 所有客户端看到受击火花
    - 服务器对玩家B调用 ClientShowDamageNumber(50.0f) — 只有B看到伤害数字
    - 引擎自动检测 Health 变化，将新值 50 发送给相关客户端

步骤7【客户端B】：收到伤害通知
    - 客户端B上的玩家B的 Pawn 收到 OnRep_Health() 回调
    - Health 从 100 更新为 50
    - OnRep_Health() 内部：更新血条UI、播放受击动画
    - ClientShowDamageNumber(50.0f) 在客户端B上显示"50"伤害数字

步骤8【客户端A】：看到效果
    - 客户端A上的玩家B的 Pawn（ROLE_SimulatedProxy）收到 OnRep_Health()
    - 客户端A看到玩家B的血条减少了
    - MulticastPlayHitEffect() 让客户端A看到火花

步骤9【服务器】：检查玩家B是否死亡
    - if (Health <= 0) → 处理死亡逻辑
    - bIsAlive = false → Replicated 属性，自动同步
    - 通知 GameMode 处理击杀统计
    - 触发重生计时器等

═══════════════════════════════════════════════════════════════════
流程结束
```

```cpp
    void OnRep_Health();

private:

    float FireCooldownTime = 0.5f;

// ============================================
// Server RPC 实现
// ============================================
void ACombatCharacter::ServerFireWeapon_Implementation(FVector AimDirection)
{
    // ===== 验证这只能在服务器执行 =====
    // (实际上 Server RPC 的 _Implementation 天然在服务器上执行，
    //  但加上这个检查是好习惯)
    if (!HasAuthority())
    {
        UE_LOG(LogTemp, Error, TEXT("ServerFireWeapon 在非服务器端被调用！这不正常！"));
        return;
    }

    // ===== 验证1：角色是否还活着？ =====
    if (!bIsAlive)
    {
        UE_LOG(LogTemp, Warning, TEXT("开火失败：角色已死亡"));
        return;  // 死人不能开枪
    }

    // ===== 验证2：冷却时间检查（防止射速外挂） =====
    if (!bCanFire)
    {
        UE_LOG(LogTemp, Warning, TEXT("开火失败：射击冷却中"));
        return;  // 拒绝！射击速度太快
    }

    // ===== 验证3：弹药检查 =====
    // 假设有一个 CurrentAmmo 变量
    // if (CurrentAmmo <= 0) { return; }

    // ===== 验证4：瞄准方向合法性 =====
    if (AimDirection.ContainsNaN() || AimDirection.IsNearlyZero())
    {
        UE_LOG(LogTemp, Warning, TEXT("开火失败：无效的瞄准方向"));
        return;  // 拒绝！可能被篡改的数据
    }

    // ===== 验证通过！执行真正的开火逻辑 =====

    // 1. 消耗弹药（如果有弹药系统）
    // CurrentAmmo--;

    // 2. 设置冷却
    bCanFire = false;
    GetWorldTimerManager().SetTimer(
        FireCooldownTimer,
        [this]() { bCanFire = true; },
        FireCooldownTime,
        false
    );

    // 3. 执行服务器端射线检测
    FVector StartLocation = GetActorLocation();
    // 从角色眼睛位置（或枪口位置）发出射线
    FVector EyeLocation;
    FRotator EyeRotation;
    GetActorEyesViewPoint(EyeLocation, EyeRotation);

    FVector EndLocation = EyeLocation + (AimDirection * 10000.0f); // 1万单位约100米

    FHitResult HitResult;
    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(this);  // 忽略自己

    // 执行射线检测
    bool bHit = GetWorld()->LineTraceSingleByChannel(
        HitResult,
        EyeLocation,
        EndLocation,
        ECC_Visibility,     // 使用可见性通道
        QueryParams
    );

    // 4. 处理命中结果
    if (bHit)
    {
        AActor* HitActor = HitResult.GetActor();
        if (HitActor)
        {
            // 尝试对被击中的角色造成伤害
            ACombatCharacter* HitCharacter = Cast<ACombatCharacter>(HitActor);
            if (HitCharacter && HitCharacter->bIsAlive)
            {
                // 计算伤害（可以基于距离衰减等）
                float Damage = 25.0f;  // 基础伤害
                HitCharacter->ServerApplyDamage(Damage, this);

                // 播放命中特效（所有客户端看到）
                MulticastPlayHitEffect(HitResult.Location);
            }
        }
    }

    // 5. 播放枪口火焰（所有客户端看到）
    // MulticastPlayMuzzleFlash();

    UE_LOG(LogTemp, Log, TEXT("服务器：%s 开火，%s"),
        *GetName(),
        bHit ? TEXT("命中！") : TEXT("未命中"));
}

bool ACombatCharacter::ServerFireWeapon_Validate(FVector AimDirection)
{
    // 基础验证：方向向量不能包含 NaN
    if (AimDirection.ContainsNaN())
    {
        return false;
    }
    // 方向向量的长度应该大约为1（标准化后的向量）
    float Length = AimDirection.Size();
    if (Length < 0.5f || Length > 1.5f)
    {
        return false;
    }
    return true;
}

// 服务器端伤害处理
// ============================================
void ACombatCharacter::ServerApplyDamage(float Damage, ACombatCharacter* Instigator)
{
    // 第一步：确保在服务器上
    if (!HasAuthority())
    {
        UE_LOG(LogTemp, Error, TEXT("ServerApplyDamage 在客户端被调用！拒绝执行"));
        return;
    }

    // 第二步：确保目标还活着
    if (!bIsAlive)
    {
        return;  // 已经死了，不要再鞭尸
    }

    // 第三步：应用伤害
    Health = FMath::Max(0.0f, Health - Damage);
    // ↑ Health 是 Replicated 属性
    // 引擎会自动检测到 Health 的变化
    // 并自动将新值同步给所有需要它的客户端

    UE_LOG(LogTemp, Log, TEXT("服务器：%s 受到了 %.0f 点伤害，剩余血量 %.0f"),
        *GetName(), Damage, Health);

    // 第四步：通知拥有者客户端显示伤害数字
    ClientShowDamageNumber(Damage);
    // ↑ 这是 Client RPC，只有被击中的玩家能看到伤害数字
    // 攻击者（Instigator）看不到这个数字（除非攻击者同时是施害者）

    // 第五步：检查是否死亡
    if (Health <= 0.0f)
    {
        bIsAlive = false;  // Replicated 属性，自动同步到所有客户端
        UE_LOG(LogTemp, Log, TEXT("服务器：%s 被 %s 击杀"),
            *GetName(), Instigator ? *Instigator->GetName() : TEXT("Unknown"));

        // 在这里可以：
        // - 通知 GameMode 处理击杀
        // - 给击杀者加分
        // - 播放死亡动画
        // - 启动重生计时器
    }
}
```

## IsLocallyControlled 详解

### 三个重要函数的区别

```cpp
// ===== 函数1：HasAuthority() =====
// 问："这个 Actor 的当前实例是否在服务器上？"
// 服务器上的所有 Actor 都返回 true
// 客户端上的所有 Actor 都返回 false
// 用途：保护服务器端逻辑

if (HasAuthority())
{
    // 安全地修改血量、计算伤害
}

// ===== 函数2：IsLocallyControlled() =====
// 问："这个 Pawn/Character 是否是本地玩家在操控？"
// 客户端上的 AutonomousProxy → true
// Listen Server 上的本地玩家 Pawn → true
// 客户端上的 SimulatedProxy（别人的角色）→ false
// 服务器上的被远程玩家控制的 Pawn → false（服务器不是"本地玩家"）
// 用途：处理本地 UI、输入、音效

if (IsLocallyControlled())
{
    // 这是我的角色！更新本地专用的UI
    // 播放第一人称音效
    // 处理本地输入
}

// ===== 函数3：IsPlayerControlled() =====
// 问："这个 Pawn 是否被任何一个 PlayerController 控制？"
// 被真人玩家控制的 Pawn → true
// 被 AI Controller 控制的 Pawn → false
// 用途：区分玩家和 AI
```

### 实际使用组合

```cpp
void AMyCharacter::SomeFunction()
{
    // ===== 组合1：服务器上的权威逻辑 =====
    if (HasAuthority())
    {
        // 场景：修改血量、计算伤害、判定胜负
        // 适用端：Dedicated Server、Listen Server
        ApplyDamage(10.0f);
    }

    // ===== 组合2：本地玩家的专属逻辑 =====
    if (IsLocallyControlled())
    {
        // 场景：更新本地UI、播放第一人称音效
        // 适用端：纯客户端的 AutonomousProxy
```

## 网络相关性（Net Relevancy）

### 什么是网络相关性

服务器不需要把世界上所有 Actor 都同步给每个客户端。网络相关性决定了一个 Actor 是否对某个特定的客户端"重要"，不重要的 Actor 不同步。

```
┌─────────────────────────────────────────────────────────┐
│                      服务器                              │
│                                                         │
│  Actor列表（1000个）：                                     │
│  ┌──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┐   │
│  │A │B │C │D │E │F │G │H │I │J │K │L │M │N │O │P │...│
│  └──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┘   │
│                                                         │
│  对客户端1（在房间A）而言：                                   │
│  相关▲：A, B, C, D, E (房间A附近的)                        │
│  无关✗：F, G, H, ..., P (太远了)                           │
└─────────────────────────────────────────────────────────┘
```

### 相关性的判断规则

```cpp
// 在 Actor 中覆写这个函数来自定义相关性判断
// 这个函数在服务器上被调用，每个客户端都会调用一次

    const AActor* RealViewer,       // 目标客户端对应的观察者
    const AActor* ViewTarget,       // 观察者正在看的对象
    const FVector& SrcLocation      // 观察者的位置
) const
{
    // ===== 默认行为 =====
    // 引擎默认：距离检测 + 所有者检测
    // 如果 Actor 太远，不相关，不同步
    // 如果 Actor 设置了 Owner，对 Owner 永远相关

    if (!Super::IsNetRelevantFor(RealViewer, ViewTarget, SrcLocation))
    {
        return false;  // 不相关
    }

    // ===== 自定义规则示例1：只对同一队伍的玩家可见 =====
    // AMyCharacter* ViewerCharacter = Cast<AMyCharacter>(RealViewer);
    // if (ViewerCharacter && ViewerCharacter->TeamID != this->TeamID)
    // {
    //     return false;  // 不同队伍，不可见
    // }

    // ===== 自定义规则示例2：特殊道具只对特定玩家可见 =====
    // if (HasCustomVisibilityCondition(RealViewer))
    // {
    //     return true;   // 强制可见
    // }

    return true;
}
```

## 客户端预测（Client-Side Prediction）

### 问题：网络延迟让移动很卡

```
如果没有客户端预测：

玩家按下W键 → 发到服务器 → 服务器移动角色 → 发回新位置 → 客户端更新位置
                   ↑_____网络延迟 50ms_____↑    ↑_____网络延迟 50ms_____↑

从按键到看到角色移动需要 100ms！感觉非常"粘滞"
```

### 解决方案：客户端预测移动

```cpp
// ===== 客户端预测移动的原理 =====
// UE5 的角色移动组件（CharacterMovementComponent）已经内置了客户端预测
// 你通常不需要手动实现移动预测，但你需要理解它的原理

// 原理简述：
// 1. 客户端按下按键 → 立即在本地移动角色（预测！）
// 2. 同时发送移动请求给服务器
// 3. 服务器也执行相同的移动
// 4. 服务器把"权威位置"发回客户端
// 5. 客户端对比"预测位置"和"权威位置"
// 6. 如果一致 → 什么也不做
// 7. 如果不一致 → 用服务器的权威位置"纠正"客户端的预测位置
//    （在严重不一致时会有"瞬移"的感觉）

// 这就是为什么在延迟高的时候，你有时会看到角色"跳回"之前的位置
```

### 客户端预测的其他应用

```cpp
// ===== 开火预测（减少射击延迟） =====
void AMyCharacter::OnFireButtonPressed()
{
    // 1. 立即在本地播放枪口火焰、音效（不等服务器确认）
    PlayLocalMuzzleFlash();     // 本地预测性特效
    PlayLocalFireSound();       // 本地预测性音效

    // 2. 同时发送 Server RPC 给服务器
    ServerFireWeapon();
    // 如果服务器拒绝（比如没弹药），需要在 Client RPC 中"纠正"
}

// ===== 弹药数量的预测 =====
// 当玩家开火时：
void AMyCharacter::OnFireButtonPressed()
{
    // 1. 本地立即减少弹药显示（预测）
    PredictedAmmo--;
    UpdateAmmoUI(PredictedAmmo);

    // 2. 发送请求给服务器
    ServerFireWeapon();

    // 3. 服务器处理完后，通过属性同步发回"真实的弹药数"
    // 4. OnRep_Ammo 会纠正本地的预测值
    //    大多数情况下预测是正确的，UI 不会闪烁
}
```

### 预测的局限性和注意事项

```cpp
// ❌ 无法预测的情况（不要预测这些）：
// - 其他玩家的位置（只能靠服务器同步）
// - 伤害数值（必须等服务器确认）
// - 拾取物品（物品是否存在只能服务器说了算）
// - 命中判定（只能服务器判定）

// ✅ 可以安全预测的情况：
// - 你自己角色的移动（UE已内置）
// - 枪口火焰和音效（纯视觉效果）
// - 本地弹药数（之后用服务器同步来纠正）
// - 第一人称动画

// ⚠️ 需要谨慎预测的情况：
// - 第三方的受击反应（可以预测播放，但可能被服务器纠正）
// - 门的打开/关闭（如果服务器判定门是锁的，需要纠正）
```

## 网络平滑（Interpolation 和 Extrapolation）

### Interpolation（插值/内插）

```
问题：服务器每 100ms（约0.1秒）发一次位置更新给客户端。
      客户端收到的是"过去的位置"。如果直接设置位置，角色会"跳跃"移动。

解决方案：在"当前收到的最新位置"和"之前收到的位置"之间插值。
         角色平滑地从旧位置过渡到新位置。

时间线：
  服务器:   Pos0 ----------> Pos1 ----------> Pos2
  客户端:   收到Pos0 ------ 收到Pos1 ------- 收到Pos2
             |___插值___|    |___插值___|

客户端延迟一个"插值延迟"后渲染：
  客户端渲染: (固定延迟) Pos0 ---> Pos1 ---> Pos2
            平滑！                    平滑！
```


### Extrapolation（外推/预测）

```
问题：如果服务器长时间没有发送数据（丢包、网络波动），
      客户端不能让人物"卡住不动"。

解决方案：根据"上次收到的速度"来推算（外推）当前位置。
         角色即使没收到新数据也会继续"向前滑动"。

时间线：
  服务器:   Pos0 ---[丢包]---[丢包]--- Pos3
  客户端:   收到Pos0 ----x----x------ 收到Pos3
             |___外推___||_外推_|

   根据 Pos0 的速度/方向，推测中间的位置来填补丢失的更新
```

```cpp

{

    // （需要通过 GetCharacterMovement() 访问）

    // 然后当服务器数据到达时，角色会被"拉回"正确位置（这看起来是"瞬移"）

    // 一般来说，引擎的默认设置已经足够好
```

### 实际建议

对于零基础学习者：

1. **不需要手动实现插值/外推** —— UE5 引擎已经内置处理了。
2. **理解这两个概念** —— 帮助你在出现网络问题时进行调试。
3. **调整频率而非算法** —— 如果移动看起来卡顿，优先尝试提高 `NetUpdateFrequency`。

## 防作弊设计

### 核心原则回顾

```cpp
// ╔══════════════════════════════════════════════════════════╗
// ║  黄金法则：永远、永远、永远不要信任客户端发送的数据        ║
// ╚══════════════════════════════════════════════════════════╝
```

### 服务器端必须验证的常见数据

```cpp
void AMyCharacter::ServerUseItem_Implementation(int32 ItemID, FVector UseLocation)
{
    // ===== 验证清单 =====

    // □ 1. 物品ID是否合法？
    if (ItemID < 0 || ItemID >= MaxItemCount)
    {
        UE_LOG(LogTemp, Warning, TEXT("作弊检测：物品ID %d 不合法"), ItemID);
        return;  // 拒绝！
    }

    // □ 2. 角色是否还活着？
    if (!bIsAlive)
    {
        UE_LOG(LogTemp, Warning, TEXT("作弊检测：死亡角色尝试使用物品"));
        return;
    }

    // □ 3. 距离是否合理？（防止远程拾取外挂）
    float Distance = FVector::Dist(GetActorLocation(), UseLocation);
    if (Distance > MaxUseDistance)  // 比如最大使用距离300单位
    {
        UE_LOG(LogTemp, Warning, TEXT("作弊检测：使用距离 %f 超出上限"), Distance);
        return;
    }

    // □ 4. 速度是否合理？（防止加速外挂）
    float CurrentSpeed = GetVelocity().Size();
    if (CurrentSpeed > MaxAllowedSpeed)  // 比如最大允许速度2000
    {
        UE_LOG(LogTemp, Warning, TEXT("作弊检测：速度 %f 超出上限"), CurrentSpeed);
        // 可以选择踢出玩家
        return;
    }

    // □ 5. 冷却时间是否合规？（防止无冷却外挂）
    float TimeSinceLastUse = GetWorld()->GetTimeSeconds() - LastUseTime;
    if (TimeSinceLastUse < CooldownTime)
    {
        UE_LOG(LogTemp, Warning, TEXT("作弊检测：使用间隔 %f 低于冷却时间"), TimeSinceLastUse);
        return;
    }

    // □ 6. 是否拥有该物品？（防止使用未拥有的物品）
    if (!OwnedItems.Contains(ItemID))
    {
        UE_LOG(LogTemp, Warning, TEXT("作弊检测：玩家不拥有物品 %d"), ItemID);
        return;
    }

    // ===== 全部验证通过，执行逻辑 =====
    LastUseTime = GetWorld()->GetTimeSeconds();
    // ... 实际的使用逻辑
}

bool AMyCharacter::ServerUseItem_Validate(int32 ItemID, FVector UseLocation)
{
    // Validate 中的验证可以轻量一些
    // 主要是验证数据格式的合法性（而不是游戏规则的合法性）
    if (ItemID < 0) return false;
    if (UseLocation.ContainsNaN()) return false;
    return true;
}
```

### 防作弊的层次

```
第一层：Validate 函数（数据格式验证）
    ↓ 通过
第二层：_Implementation 开头（游戏规则验证）
    ↓ 通过
第三层：持续的服务器端监控（速度检测、行为分析）
    ↓ 可疑
第四层：处罚机制（警告、回弹、踢出、封禁）
```

```cpp
#include "CoreMinimal.h"

#include "SecureDamageSystem.generated.h"

// 伤害信息结构体（服务器用来追踪每次伤害请求的合法性）
USTRUCT()
struct FSecureDamageInfo
{
    GENERATED_BODY()

    // 谁造成的伤害
    UPROPERTY()
    AActor* Instigator;

    // 伤害基础值
    UPROPERTY()
    float BaseDamage;

    // 伤害类型（物理/火焰/冰冻等）
    UPROPERTY()
    int32 DamageType;

    // 伤害施加时间（用于冷却检测）
    UPROPERTY()
    float Timestamp;

    // 伤害施加位置（用于距离检测）
    UPROPERTY()
    FVector HitLocation;

UCLASS()
class MYGAME_API ASecureDamageableActor : public AActor
{
    GENERATED_BODY()

public:
    ASecureDamageableActor();

    // ============================================
    // 同步属性
    // ============================================
    UPROPERTY(ReplicatedUsing = OnRep_Health)
    float Health;

    UPROPERTY(Replicated)
    bool bIsAlive;

    UPROPERTY(Replicated)
    float Armor;

    // ============================================
    // Server RPC — 客户端请求造成伤害（但服务器验证！）
    // ============================================

    void ServerRequestDamage(float RequestedDamage, AActor* DamageCauser,
                             FVector HitLocation, int32 DamageType);

    // ============================================
    // NetMulticast RPC — 服务器和相关客户端看到受击效果
    // ============================================
    UFUNCTION(NetMulticast, Reliable)
    void MulticastOnDamageReceived(float ActualDamage, FVector HitLocation);

    // ============================================
    // Client RPC — 通知拥有者客户端
    // ============================================
    UFUNCTION(Client, Reliable)
    void ClientShowDamageFeedback(float Damage, bool bWasBlocked);

    // ============================================
    // OnRep 回调
    // ============================================
    UFUNCTION()
    void OnRep_Health();

    // ============================================
    // 复制属性注册
    // ============================================
    virtual void GetLifetimeReplicatedProps(
        TArray<FLifetimeProperty>& OutLifetimeProps
    ) const override;

protected:
    // 服务器端真正的伤害处理（只在服务器上执行）
    void ApplyDamageInternal(const FSecureDamageInfo& DamageInfo);

    // 验证伤害请求的合法性（服务器端）
    bool ValidateDamageRequest(const FSecureDamageInfo& DamageInfo, FString& OutFailReason);

    // 计算最终伤害（考虑护甲、距离衰减等）
    float CalculateFinalDamage(const FSecureDamageInfo& DamageInfo);

private:
    // 配置常量
    static constexpr float MaxHealth = 100.0f;
    static constexpr float MaxArmor = 50.0f;
    static constexpr float MaxValidDistance = 5000.0f;   // 最大有效攻击距离
    static constexpr float MinDamageInterval = 0.1f;      // 最小伤害间隔（防快速攻击外挂）
    static constexpr float MaxDamagePerHit = 1000.0f;     // 单次最大伤害上限

    // 上次受伤害的时间（防冷却绕过）
    float LastDamageTime = 0.0f;

    bool bCanReceiveDamage = true;
```

```cpp
// ============================================
// SecureDamageSystem.cpp
// ============================================
#include "SecureDamageSystem.h"
#include "Net/UnrealNetwork.h"
#include "Engine/Engine.h"

ASecureDamageableActor::ASecureDamageableActor()
{
    // 构造函数设置默认值
    Health = MaxHealth;
    Armor = MaxArmor;
    bIsAlive = true;

    // 设置网络属性
    bReplicates = true;                    // 这个 Actor 需要网络复制
    NetUpdateFrequency = 10.0f;            // 每秒10次网络更新
    NetPriority = 2.0f;                    // 高于普通Actor的优先级
}

void ASecureDamageableActor::GetLifetimeReplicatedProps(
    TArray<FLifetimeProperty>& OutLifetimeProps
) const
{
    Super::GetLifetimeReplicatedProps(OutLifetimeProps);
    DOREPLIFETIME(ASecureDamageableActor, Health);
    DOREPLIFETIME(ASecureDamageableActor, bIsAlive);
    DOREPLIFETIME(ASecureDamageableActor, Armor);
}

// ============================================
// Server RPC：客户端请求伤害（服务器验证）
// ============================================

void ASecureDamageableActor::ServerRequestDamage_Implementation(
    float RequestedDamage,
    AActor* DamageCauser,
    FVector HitLocation,
    int32 DamageType
)
{
    // ===== 第一步：权限检查 =====
    if (!HasAuthority())
    {
        UE_LOG(LogTemp, Error, TEXT("ServerRequestDamage 在非服务器端被执行！"));
        return;
    }

    // ===== 第二步：基本状态检查 =====
    if (!bIsAlive)
    {
        UE_LOG(LogTemp, Warning, TEXT("伤害请求被拒绝：目标已死亡"));
        return;
    }

    // ===== 第三步：构建伤害信息并验证 =====
    FSecureDamageInfo DamageInfo;
    DamageInfo.Instigator = DamageCauser;
    DamageInfo.BaseDamage = RequestedDamage;
    DamageInfo.DamageType = DamageType;
    DamageInfo.Timestamp = GetWorld()->GetTimeSeconds();
    DamageInfo.HitLocation = HitLocation;

    FString FailReason;
    if (!ValidateDamageRequest(DamageInfo, FailReason))
    {
        UE_LOG(LogTemp, Warning, TEXT("伤害请求被拒绝：%s"), *FailReason);
        // ♜ 作弊检测：可以考虑在这里记录可疑玩家的行为
        return;
    }

    // ===== 第四步：验证通过，执行真正的伤害逻辑 =====
    ApplyDamageInternal(DamageInfo);
}

bool ASecureDamageableActor::ServerRequestDamage_Validate(
    float RequestedDamage,
    AActor* DamageCauser,
    FVector HitLocation,
    int32 DamageType
)
{
    // Validate 做轻量验证：数据格式合法性
    if (RequestedDamage < 0.0f) return false;                     // 伤害不能为负
    if (RequestedDamage > MaxDamagePerHit) return false;           // 单次伤害不能太离谱
    if (HitLocation.ContainsNaN()) return false;                   // 位置不能含 NaN
    if (DamageType < 0) return false;                              // 伤害类型不能为负
    return true;
}

// ============================================
// 伤害验证：确保请求合法
// ============================================
bool ASecureDamageableActor::ValidateDamageRequest(
    const FSecureDamageInfo& DamageInfo,
    FString& OutFailReason
)
{
    // 验证1：伤害值是否合理？
    if (DamageInfo.BaseDamage <= 0.0f || DamageInfo.BaseDamage > MaxDamagePerHit)
    {
        OutFailReason = FString::Printf(
            TEXT("伤害值不合理：%.1f（允许范围：0.1 ~ %.1f）"),
            DamageInfo.BaseDamage, MaxDamagePerHit
        );
        return false;
    }

    // 验证2：伤害间隔是否合法？
    float TimeSinceLastDamage = DamageInfo.Timestamp - LastDamageTime;
    if (TimeSinceLastDamage < MinDamageInterval)
    {
        OutFailReason = FString::Printf(
            TEXT("伤害间隔过短：%.3f秒（最小间隔：%.3f秒）"),
            TimeSinceLastDamage, MinDamageInterval
        );
        return false;
    }

    // 验证3：攻击距离是否合理？
    if (DamageInfo.Instigator)
    {
        float Distance = FVector::Dist(
            GetActorLocation(),
            DamageInfo.Instigator->GetActorLocation()
        );
        if (Distance > MaxValidDistance)
        {
            OutFailReason = FString::Printf(
                TEXT("攻击距离过远：%.1f（最大有效距离：%.1f）"),
                Distance, MaxValidDistance
            );
            return false;
        }
    }

    // 验证4：攻击者是否合法？（不能自己打自己，除非是特殊机制）
    if (DamageInfo.Instigator == this)
    {
        OutFailReason = TEXT("不能对自己造成伤害");
        return false;
    }

    // 验证5：目标是否可以受伤？（冷却检测）
    if (!bCanReceiveDamage)
    {
        OutFailReason = TEXT("目标处于伤害冷却中");
        return false;
    }

    // 全部通过
    return true;
}

// ============================================
// 服务器端实际伤害处理
// ============================================
void ASecureDamageableActor::ApplyDamageInternal(const FSecureDamageInfo& DamageInfo)
{
    // ===== 计算最终伤害（考虑护甲等） =====
    float FinalDamage = CalculateFinalDamage(DamageInfo);

    // ===== 先消耗护甲 =====
    if (Armor > 0.0f && FinalDamage > 0.0f)
    {
        // 护甲吸收70%的伤害
        float ArmorAbsorption = FMath::Min(Armor, FinalDamage * 0.7f);
        Armor -= ArmorAbsorption;
        FinalDamage -= ArmorAbsorption;
    }

    // ===== 应用伤害到血量 =====
    Health = FMath::Max(0.0f, Health - FinalDamage);

    // ===== 记录上次受伤害时间 =====
    LastDamageTime = DamageInfo.Timestamp;

    // ===== 设置伤害冷却 =====
    bCanReceiveDamage = false;
    GetWorldTimerManager().SetTimer(
        DamageCooldownTimer,
        [this]() { bCanReceiveDamage = true; },
        MinDamageInterval,
        false
    );

    // ===== 网络同步 =====
    // 1. Health 是 Replicated 属性，引擎自动同步新值到客户端
    // 2. 通过多播RPC通知所有客户端播放特效
    MulticastOnDamageReceived(FinalDamage, DamageInfo.HitLocation);
    // 3. 通过Client RPC通知拥有者客户端显示伤害反馈
    ClientShowDamageFeedback(FinalDamage, Armor > 0.0f);

    // ===== 日志记录 =====
    UE_LOG(LogTemp, Log,
        TEXT("服务器：%s 受到 %.1f 伤害（来自 %s），剩余血量 %.1f，护甲 %.1f"),
        *GetName(),
        FinalDamage,
        DamageInfo.Instigator ? *DamageInfo.Instigator->GetName() : TEXT("Unknown"),
        Health,
        Armor
    );

    // ===== 死亡检测 =====
    if (Health <= 0.0f)
    {
        bIsAlive = false;  // Replicated 属性，自动同步

        // 在这里通知 GameMode 处理击杀...
        UE_LOG(LogTemp, Log, TEXT("服务器：%s 已死亡"), *GetName());
    }
}
```
