# 第7章案例：多输入模式角色控制系统

> **难度**：⭐⭐⭐（中级）
> **预计时间**：90分钟
> **目标**：创建一个完整的支持键盘鼠标和手柄的角色控制器，包含移动、视角、跳跃、冲刺、蹲伏、攻击连击、交互拾取和输入模式切换。

---

## 案例目标

实现一个 `AEnhancedInputCharacter` 类，具备以下功能：

| 功能         | 键盘/鼠标 | 手柄          | 输入类型 |
| ------------ | --------- | ------------- | -------- |
| 移动         | WASD      | 左摇杆        | Axis2D   |
| 视角         | 鼠标      | 右摇杆        | Axis2D   |
| 跳跃         | 空格键    | A键(下方按钮) | Bool     |
| 冲刺（按住） | 左Shift   | 按下左摇杆    | Bool     |
| 蹲伏（切换） | 左Ctrl    | B键(右边按钮) | Bool     |
| 轻攻击       | 鼠标左键  | RB(右肩键)    | Bool     |
| 重攻击       | 鼠标右键  | RT(右扳机)    | Bool     |
| 交互         | E键       | X键(左边按钮) | Bool     |
| 暂停菜单     | ESC       | Start键       | Bool     |

附加功能：

- 连击检测：在轻攻击动画窗口内再次按轻攻击 → 触发连击
- 交互射线检测：E键检测前方的可拾取物
- 输入模式切换：按ESC在游戏模式和UI模式之间切换

---

## 项目准备

### 第一步：创建 InputAction 资产

在编辑器的 Content Browser 中创建以下 InputAction 资产（建议放在 `Content/Input/Actions/` 目录）：

```
InputAction 资产列表:
├── IA_Move            ValueType = Axis2D
├── IA_Look            ValueType = Axis2D
├── IA_Jump            ValueType = Bool
├── IA_Sprint          ValueType = Bool
├── IA_Crouch          ValueType = Bool
├── IA_LightAttack     ValueType = Bool
├── IA_HeavyAttack     ValueType = Bool
├── IA_Interact        ValueType = Bool
└── IA_Pause           ValueType = Bool
```

每个 IA 的 Triggers 配置（直接在资产上配置）：

| InputAction      | Triggers                                               |
| ---------------- | ------------------------------------------------------ |
| IA_Move, IA_Look | 不需要 Triggers（由 BindAction 的 ETriggerEvent 控制） |
| IA_Jump          | `Pressed`                                              |
| IA_Sprint        | `Pressed`, `Released`                                  |
| IA_Crouch        | `Pressed`                                              |
| IA_LightAttack   | `Pressed`, `Released`（用于连击窗口判断）              |
| IA_HeavyAttack   | `Pressed`                                              |
| IA_Interact      | `Pressed`                                              |
| IA_Pause         | `Pressed`                                              |

### 第二步：创建 InputMappingContext 资产

在编辑器的 Content Browser 中创建 `IMC_Default`（放在 `Content/Input/` 目录）。

IMC_Default 的映射配置：

```
IMC_Default 映射表:

▼ [0] IA_Move
    → W (Scale: 1.0)       ← 绑定到 Y 轴（前后）
    → S (Scale: -1.0)      ← 绑定到 Y 轴（前后）
    → A (Scale: -1.0)      ← 绑定到 X 轴（左右）
    → D (Scale: 1.0)       ← 绑定到 X 轴（左右）
    → Gamepad Left Thumbstick 2D

▼ [1] IA_Look
    → Mouse XY 2D
    → Gamepad Right Thumbstick 2D
    [Modifiers] → DeadZone (LowerThreshold=0.1, Radial)
               → Scalar (0.5)  ← 鼠标灵敏度

▼ [2] IA_Jump
    → Space Bar
    → Gamepad Face Button Bottom

▼ [3] IA_Sprint
    → Left Shift
    → Gamepad Left Thumbstick Button

▼ [4] IA_Crouch
    → Left Control
    → Gamepad Face Button Right

▼ [5] IA_LightAttack
    → Left Mouse Button
    → Gamepad Right Shoulder

▼ [6] IA_HeavyAttack
    → Right Mouse Button
    → Gamepad Right Trigger

▼ [7] IA_Interact
    → E
    → Gamepad Face Button Left

▼ [8] IA_Pause
    → Escape
    → Gamepad Special Right
```

// EnhancedInputCharacter.cpp
// 完整实现

#include "EnhancedInputCharacter.h"

// ===== Enhanced Input 头文件 =====
#include "EnhancedInputComponent.h"        // UEnhancedInputComponent
#include "EnhancedInputSubsystems.h"       // UEnhancedInputLocalPlayerSubsystem

// ===== 角色功能头文件 =====
#include "GameFramework/CharacterMovementComponent.h"  // 移动组件

#include "Engine/World.h"                              // 世界（计时器）
#include "TimerManager.h"                              // 计时器管理器
#include "DrawDebugHelpers.h"                          // 调试绘制（射线可视化）

// ========== 构造函数 ==========

AEnhancedInputCharacter::AEnhancedInputCharacter()
{
    // PrimaryActorTick.bCanEverTick = true
    // 允许这个Actor每帧调用 Tick() 函数
    // 需要的理由：我们要在 Tick 中做交互检测的可视化（非必须，但有帮助）
    PrimaryActorTick.bCanEverTick = true;

    // ===== 获取并配置 CharacterMovementComponent =====
    // ACharacter 自动创建了一个 UCharacterMovementComponent
    // 它负责走路、跳跃、蹲伏、飞行等所有移动逻辑
    UCharacterMovementComponent* Movement = GetCharacterMovement();
    if (Movement)
    {
        // 设置默认走路速度
        Movement->MaxWalkSpeed = WalkSpeed;  // 600.0f

        // ===== 蹲伏配置 =====
        // ⚠️ 默认情况下 ACharacter 不允许蹲伏！
        // 必须设置 bCanCrouch = true 才能使用 Crouch()/UnCrouch()
        Movement->GetNavAgentPropertiesRef().bCanCrouch = true;

        // 蹲伏时的高度设为正常高度的一半
        // （这个值需要在 ACharacter 的 CapsuleComponent 上配合设置）
        Movement->CrouchedHalfHeight = 40.0f;  // 默认是40

        // ===== 跳跃配置 =====
        // JumpZVelocity: 跳跃的初始Z轴速度（厘米/秒）
        // 420是默认值，约0.55秒到达跳跃最高点
        Movement->JumpZVelocity = 600.0f;

        // AirControl: 空中移动控制系数（0=完全不能控制，1=和地面一样）
        // 用于控制跳跃时玩家能多大幅度地改变方向
        Movement->AirControl = 0.35f;

        // ===== 旋转配置 =====
        // bOrientRotationToMovement: 角色是否朝向移动方向
        // true = 角色自动面向移动方向（第三人称常见）
        // false = 角色朝向由代码/动画控制
        GetCharacterMovement()->bOrientRotationToMovement = true;

        // RotationRate: 旋转速度（度/秒）
        // 控制角色转身的平滑度
        GetCharacterMovement()->RotationRate = FRotator(0.0f, 540.0f, 0.0f);

        // ===== 弹簧臂和摄像机配置 =====
        // 检查是否通过蓝图或其他方式已经创建了这些组件
        // 如果没有，我们在代码中创建它们（但现在不做，因为Character类
        // 的传感器通常在蓝图中添加——这体现了C++和蓝图的协作）
        //
        // 注意：以下注释代码展示了如何在C++中创建默认的第三人称摄像机设置
        // 在实际项目中，摄像机设置通常放在蓝图子类中，方便美术调整
        //
        // USpringArmComponent* SpringArm = CreateDefaultSubobject<USpringArmComponent>(TEXT("SpringArm"));
        // SpringArm->SetupAttachment(RootComponent);
        // SpringArm->TargetArmLength = 400.0f;     // 摄像机距离
        // SpringArm->bUsePawnControlRotation = true; // 弹簧臂跟随控制器旋转
        //
        // UCameraComponent* Camera = CreateDefaultSubobject<UCameraComponent>(TEXT("Camera"));

        // Camera->bUsePawnControlRotation = false;  // 摄像机不独立旋转
    }

// ========== BeginPlay ==========

void AEnhancedInputCharacter::BeginPlay()
{
    // Super::BeginPlay() 必须首先调用，确保父类的初始化完成
    Super::BeginPlay();

    // ===== 激活输入映射上下文 =====

    // 第1步：获取 PlayerController
    // GetController() 返回当前控制此 Pawn 的 AController
    // 对于玩家控制的角色，它返回 APlayerController
    APlayerController* PC = Cast<APlayerController>(GetController());
    if (!PC)
    {
        // 如果还没被控制，打印警告并退出
        // 这种情况可能发生在此角色作为AI使用，或还没被Possess时
        UE_LOG(LogTemp, Warning,
               TEXT("AEnhancedInputCharacter::BeginPlay - 未找到PlayerController！"));
        return;
    }

    // 第2步：获取 EnhancedInputLocalPlayerSubsystem
    // 这是 Enhanced Input 系统的核心管理器
    // 它负责：
    //   1. 管理所有已激活的 InputMappingContext
    //   2. 处理输入事件的优先级
    //   3. 提供运行时修改映射的方法
    UEnhancedInputLocalPlayerSubsystem* Subsystem =
        ULocalPlayer::GetSubsystem<UEnhancedInputLocalPlayerSubsystem>(
            PC->GetLocalPlayer());

    if (!Subsystem)
    {
        UE_LOG(LogTemp, Warning,
               TEXT("AEnhancedInputCharacter::BeginPlay - 未找到EnhancedInput子系统！"));
        return;
    }

    // 第3步：激活默认游戏输入上下文
    if (IMC_Default)
    {
        // AddMappingContext 参数说明:
        //   IMC_Default: 要激活的Context资产
        //   0: 优先级（最低级别，让UI Context可以覆盖它）
        Subsystem->AddMappingContext(IMC_Default, 0);

        UE_LOG(LogTemp, Log, TEXT("IMC_Default 已激活，优先级=0"));
    }
    else
    {
        UE_LOG(LogTemp, Warning,
               TEXT("AEnhancedInputCharacter::BeginPlay - IMC_Default 为空！请在蓝图中赋值。"));
    }

    // 设置游戏输入模式（确保鼠标被正确捕获）
    SetGameInputMode();
}

// ========== Tick ==========

void AEnhancedInputCharacter::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    // 在编辑器中可视化交互检测范围（仅开发调试用）
    // 打包游戏时这段代码会被编译器优化掉
    #if WITH_EDITOR
    {
        // 获取角色前方 InteractDistance 的位置
        FVector Start = GetActorLocation();
        FVector Forward = GetActorForwardVector();
        FVector End = Start + (Forward * InteractDistance);

        // 绘制检测球体（半透明青色）
        DrawDebugSphere(GetWorld(), End, InteractRadius,
                        8,                          // 分段数（越高越圆）
                        FColor::Cyan,               // 颜色
                        false,                      // 是否持久显示
                        -1.0f,                      // 持续时间（-1=单帧）
                        0,                          // 深度优先级
                        0.5f);                      // 线宽
    }
    #endif
}

// ========== SetupPlayerInputComponent ==========

void AEnhancedInputCharacter::SetupPlayerInputComponent(
    UInputComponent* PlayerInputComponent)
{
    // 调用父类方法（确保任何基类绑定也被执行）
    Super::SetupPlayerInputComponent(PlayerInputComponent);

    // ===== 类型转换为 UEnhancedInputComponent =====
    // CastChecked: 转换失败会触发断言（assert）
    // 在开发阶段，尽早崩溃有利于发现问题
    // 如果你不确定会不会转换失败，用 Cast + if 判断
    UEnhancedInputComponent* EnhancedInput =
        CastChecked<UEnhancedInputComponent>(PlayerInputComponent);

    // ===== 逐个绑定 InputAction 到回调函数 =====
    // 每个绑定都先检查 IA 是否为 nullptr
    // （防止蓝图中没有赋值时崩溃）

    // --- 移动（Axis2D, Triggered）---
    if (IA_Move)
    {
        EnhancedInput->BindAction(IA_Move, ETriggerEvent::Triggered,
                                  this, &AEnhancedInputCharacter::OnMove);
    }

    // --- 视角（Axis2D, Triggered）---
    if (IA_Look)
    {
        EnhancedInput->BindAction(IA_Look, ETriggerEvent::Triggered,
                                  this, &AEnhancedInputCharacter::OnLook);
    }

    // --- 跳跃（Bool, Started + Completed）---
    if (IA_Jump)
    {
        EnhancedInput->BindAction(IA_Jump, ETriggerEvent::Started,
                                  this, &AEnhancedInputCharacter::OnJumpStarted);
        EnhancedInput->BindAction(IA_Jump, ETriggerEvent::Completed,
                                  this, &AEnhancedInputCharacter::OnJumpCompleted);
    }

    // --- 冲刺（Bool, Started + Completed + Canceled）---
    if (IA_Sprint)
    {
        EnhancedInput->BindAction(IA_Sprint, ETriggerEvent::Started,
                                  this, &AEnhancedInputCharacter::OnSprintStarted);
        EnhancedInput->BindAction(IA_Sprint, ETriggerEvent::Completed,
                                  this, &AEnhancedInputCharacter::OnSprintCompleted);
        // ⚠️ Canceled 必须有！否则打开菜单时会"卡冲刺"
        EnhancedInput->BindAction(IA_Sprint, ETriggerEvent::Canceled,
                                  this, &AEnhancedInputCharacter::OnSprintCanceled);
    }

    // --- 蹲伏（Bool, Started）---
    if (IA_Crouch)
    {
        EnhancedInput->BindAction(IA_Crouch, ETriggerEvent::Started,
                                  this, &AEnhancedInputCharacter::OnCrouchToggle);
    }

    // --- 轻攻击（Bool, Started + Completed）---
    if (IA_LightAttack)
    {
        EnhancedInput->BindAction(IA_LightAttack, ETriggerEvent::Started,
                                  this, &AEnhancedInputCharacter::OnLightAttackStarted);
        EnhancedInput->BindAction(IA_LightAttack, ETriggerEvent::Completed,
                                  this, &AEnhancedInputCharacter::OnLightAttackCompleted);
    }

    // --- 重攻击（Bool, Started）---
    if (IA_HeavyAttack)
    {
        EnhancedInput->BindAction(IA_HeavyAttack, ETriggerEvent::Started,
                                  this, &AEnhancedInputCharacter::OnHeavyAttackStarted);
    }

    // --- 交互（Bool, Started）---
    if (IA_Interact)
    {
        EnhancedInput->BindAction(IA_Interact, ETriggerEvent::Started,
                                  this, &AEnhancedInputCharacter::OnInteractStarted);
    }

    // --- 暂停/菜单（Bool, Started）---
    if (IA_Pause)
    {
        EnhancedInput->BindAction(IA_Pause, ETriggerEvent::Started,
                                  this, &AEnhancedInputCharacter::OnPauseStarted);
    }
}

// ========================================
//  移动和视角
// ========================================

void AEnhancedInputCharacter::OnMove(const FInputActionValue& Value)
{
    // ===== 提取2D移动向量 =====
    FVector2D MovementVector = Value.Get<FVector2D>();

    if (IsValid(GetController()) && !MovementVector.IsNearlyZero())
    {
        // ===== 计算世界空间移动方向 =====
        // 取Controller的Yaw（只保留水平旋转）
        const FRotator YawRotation(
            0.0f,                                    // Pitch（上下）= 0
            GetController()->GetControlRotation().Yaw, // 只取偏航
            0.0f);                                    // Roll（翻滚）= 0

        // 从旋转矩阵中获取前方和右方方向向量
        const FVector ForwardDirection =
            FRotationMatrix(YawRotation).GetUnitAxis(EAxis::X);  // X轴 = 前方
        const FVector RightDirection =
            FRotationMatrix(YawRotation).GetUnitAxis(EAxis::Y);  // Y轴 = 右方

        // ===== 添加移动输入 =====
        // AddMovementInput 将输入累积起来，由 CharacterMovementComponent 统一处理
        AddMovementInput(ForwardDirection, MovementVector.Y); // W/S → 前后
        AddMovementInput(RightDirection, MovementVector.X);   // A/D → 左右
    }
}

void AEnhancedInputCharacter::OnLook(const FInputActionValue& Value)
{
    // ===== 提取2D视角向量 =====
    FVector2D LookAxisVector = Value.Get<FVector2D>();

    if (IsValid(GetController()))
    {
        // ===== 应用视角旋转 =====
        // AddControllerYawInput: 左右旋转（绕Z轴）
        // LookAxisVector.X: 鼠标水平移动量 / 右摇杆X轴
        // 乘以灵敏度系数进行缩放
        AddControllerYawInput(LookAxisVector.X * MouseSensitivity);

        // AddControllerPitchInput: 上下旋转（绕Y轴）
        // LookAxisVector.Y: 鼠标垂直移动量 / 右摇杆Y轴
        // ACharacter 自动限制 Pitch 范围到 [-90°, +90°]
        AddControllerPitchInput(LookAxisVector.Y * MouseSensitivity);
    }
}

void AEnhancedInputCharacter::OnJumpStarted(const FInputActionValue& Value)
{
    // ACharacter::Jump():
    //   1. 检查 CanJump()（是否在地面上）
    //   2. 设置 Z 速度为 JumpZVelocity
    //   3. 切换到 Falling 运动模式
    //   4. 通知动画蓝图进入跳跃动画
    //
    // 注意：Jump() 不会在已跳跃的情况下再次跳跃
    Jump();

    UE_LOG(LogTemp, Verbose, TEXT("跳跃 Started"));
}

void AEnhancedInputCharacter::OnJumpCompleted(const FInputActionValue& Value)
{
    // StopJumping():
    //   1. 如果角色正在上升，降低上升速度（实现"可变跳跃高度"）
    //   2. 重置 bPressedJump 标志
    //
    // 如果不调用 StopJumping():
    //   角色会上升到最大跳跃高度（由 JumpMaxHoldTime 控制，默认0秒即立即达到最大高度）
    //
    // 调用 StopJumping() 后:
    //   松开空格时立即停止上升 → 短按=低跳，长按=高跳
    StopJumping();
}

void AEnhancedInputCharacter::OnSprintCanceled(const FInputActionValue& Value)
{
    // Canceled 和 Completed 做同样的事
    // 区别：
    //   Completed: 玩家正常松开了按键
    //   Canceled:  Context被移除（例如打开菜单），按键事件被拦截
    //
    // 如果不处理 Canceled，就会出现：
    //   按住Shift → 打开菜单 → 在菜单中松开Shift → 关闭菜单
    //   角色继续以冲刺速度移动（bIsSprinting和MaxWalkSpeed都没有被重置！）
    UCharacterMovementComponent* Movement = GetCharacterMovement();
    if (Movement)
    {
        Movement->MaxWalkSpeed = WalkSpeed;
    }
    bIsSprinting = false;
}

        //   3. 调整摄像机位置（平滑过渡）
        //   4. 设置 bIsCrouched = true
    }
}

void AEnhancedInputCharacter::OnLightAttackStarted(const FInputActionValue& Value)
{
    // ===== 轻攻击逻辑 =====

    if (bIsAttacking && bInComboWindow)
    {
        // 情况A：正在攻击中，且在连击窗口内
        // → 缓存这次攻击，等当前动画播完后再执行
        bHasQueuedAttack = true;
        QueuedAttackType = EAttackType::Light;
        UE_LOG(LogTemp, Log, TEXT("连击输入已缓存（轻攻击），连击数将变为: %d"),
               CurrentComboCount + 1);
    }
    else if (!bIsAttacking)
    {
        // 情况B：没在攻击 → 正常执行攻击
        PerformAttack(EAttackType::Light);
    }
    else
    {
        // 情况C：正在攻击中但不在连击窗口内 → 忽略这次输入
        // （玩家按太快了，窗口还没开，或者按太慢了，窗口已经关了）
        UE_LOG(LogTemp, Verbose, TEXT("轻攻击输入被忽略（不在连击窗口内）"));
    }
}

void AEnhancedInputCharacter::OnLightAttackCompleted(const FInputActionValue& Value)
{
    // 松开轻攻击键时不需要处理
    // 这个函数的存在只是为了配合 Started/Completed 的对称性
    // 在需要"松开时停止攻击"的设计中会用到
}

void AEnhancedInputCharacter::OnHeavyAttackStarted(const FInputActionValue& Value)
{
    // 重攻击不参与连击系统（简单设计）
    // 重攻击始终可以直接执行，它会打断当前的连击
    if (!bIsAttacking)
    {
        PerformAttack(EAttackType::Heavy);
    }
    else
    {
        // 正在攻击中(包括轻攻击连击中) → 忽略重攻击输入
        // 实际项目中可能希望重攻击打断轻攻击，这里为了简单就不做了
        UE_LOG(LogTemp, Verbose, TEXT("重攻击输入被忽略（正在攻击中）"));
    }
}

{

    // 第1步：设置攻击状态

           AttackType == EAttackType::Light ? TEXT("轻攻击") : TEXT("重攻击"),

    // TODO: 4.2 检测攻击碰撞

    // FHitResult Hit;
    // if (PerformAttackTrace(Hit))
    // {

    // TODO: 4.3 播放攻击音效

    // ===== 第5步：启动连击窗口计时器 =====

    // 重置之前的计时器（如果存在）
    if (GetWorldTimerManager().TimerExists(ComboWindowTimerHandle))
    {
        GetWorldTimerManager().ClearTimer(ComboWindowTimerHandle);
    }

    // 启动新的连击窗口
    // 当 ComboWindowDuration 时间到后，调用 TryCombo() 检查是否有缓存的连击
    // 参数说明:
    //   参数1: FTimerHandle（输出，用于后续操作此计时器）

void AEnhancedInputCharacter::TryCombo()
{
    // ===== 连击窗口到期时调用 =====
    // 检查是否有缓存的攻击输入

    // 关闭连击窗口
    bInComboWindow = false;

    if (bHasQueuedAttack)
    {
        // 有缓存的攻击！执行连击
        UE_LOG(LogTemp, Log, TEXT("连击触发！继续执行攻击"));
        PerformAttack(QueuedAttackType);
    }
    else
    {
        // 没有连击输入 → 攻击序列结束
        UE_LOG(LogTemp, Log, TEXT("连击窗口关闭，无连击输入 → 重置攻击"));
        ResetCombo();
    }
}

void AEnhancedInputCharacter::ResetCombo()
{
    // 清除所有攻击状态
    bIsAttacking = false;
    CurrentAttackType = EAttackType::None;
    CurrentComboCount = 0;
    bInComboWindow = false;
    bHasQueuedAttack = false;
    QueuedAttackType = EAttackType::None;

    // 清除相关计时器
    if (GetWorldTimerManager().TimerExists(ComboWindowTimerHandle))
    {
        GetWorldTimerManager().ClearTimer(ComboWindowTimerHandle);
    }
    if (GetWorldTimerManager().TimerExists(AttackResetTimerHandle))
    {
        GetWorldTimerManager().ClearTimer(AttackResetTimerHandle);
    }

    UE_LOG(LogTemp, Verbose, TEXT("攻击状态已完全重置"));
}

AActor* AEnhancedInputCharacter::FindInteractableActor() const
{
    // ===== 使用射线检测查找前方可交互的物体 =====
    // 这是最常见的"按E交互"实现方式

    // 第1步：准备检测参数
    UWorld* World = GetWorld();
    if (!World)
    {
        return nullptr;
    }

    // 检测起点：角色的位置（通常是胶囊体的中心）
    FVector Start = GetActorLocation();

    // 检测方向：角色面朝的方向
    FVector Forward = GetActorForwardVector();

    // 检测终点：起点 + 方向 * 距离
    FVector End = Start + (Forward * InteractDistance);

    // 第2步：配置碰撞检测参数
    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(this);   // 忽略自己（否则会检测到自己）
    QueryParams.bTraceComplex = false;   // 使用简单碰撞（性能更好）
    QueryParams.bReturnPhysicalMaterial = false; // 不需要物理材质信息

    // 第3步：配置碰撞响应参数
    FCollisionObjectQueryParams ObjectParams;
    // 检测 WorldDynamic 类型的物体（这是交互物的常见碰撞类型）
    ObjectParams.AddObjectTypesToQuery(ECC_WorldDynamic);
    // 同时也检测 PhysicsBody（如果交互物是物理模拟的）
    ObjectParams.AddObjectTypesToQuery(ECC_PhysicsBody);

    // 第4步：执行球体扫描（比射线检测更好，因为有一定的容错范围）
    FHitResult Hit;
    bool bHit = World->SweepSingleByObjectType(
        Hit,                        // 输出：命中结果
        Start,                      // 起点
        End,                        // 终点
        FQuat::Identity,            // 旋转（不需要旋转球体）
        ObjectParams,               // 碰撞对象类型
        FCollisionShape::MakeSphere(InteractRadius), // 检测形状：球体
        QueryParams                 // 碰撞查询参数
    );

    // 第5步：处理结果
    if (bHit && IsValid(Hit.GetActor()))
    {
        // 在屏幕上显示命中的Actor名称（调试用）
        if (GEngine)
        {
            GEngine->AddOnScreenDebugMessage(
                -1,                          // Key=-1表示每次都是新消息
                0.5f,                        // 显示时长（秒）
                FColor::Green,               // 颜色
                FString::Printf(TEXT("命中: %s"), *Hit.GetActor()->GetName())
            );
        }
        return Hit.GetActor();
    }

    // 没命中任何物体
    return nullptr;
}

void AEnhancedInputCharacter::SetGameInputMode()
{
    bIsInUIMode = false;

    // ===== 设置输入模式为"仅游戏" =====
    APlayerController* PC = Cast<APlayerController>(GetController());
    if (!PC)
    {
        return;
    }

    // FInputModeGameOnly: 输入只发送给游戏（不发送给UI）
    //   鼠标: 被锁定在屏幕中心（用于视角控制）
    //   键盘: 全部发送给游戏
    FInputModeGameOnly GameInputMode;
    GameInputMode.SetConsumeCaptureMouseDown(false); // 不消耗鼠标按下事件
    PC->SetInputMode(GameInputMode);

    // 显示鼠标光标？不！游戏模式中隐藏光标
    PC->bShowMouseCursor = false;

    // ===== 激活/停用 Context =====
    UEnhancedInputLocalPlayerSubsystem* Subsystem =
        ULocalPlayer::GetSubsystem<UEnhancedInputLocalPlayerSubsystem>(
            PC->GetLocalPlayer());

    if (Subsystem)
    {
        // 移除UI Context（如果之前添加了）
        if (IMC_UI)
        {
            Subsystem->RemoveMappingContext(IMC_UI);
        }

        // 确保游戏Context是激活的
        // 注意：BeginPlay 中已经添加了，这里是为了防止被意外移除后重新添加
        if (IMC_Default)
        {
            // HasMappingContext 检查是否已存在，避免重复添加
            if (!Subsystem->HasMappingContext(IMC_Default))
            {
                Subsystem->AddMappingContext(IMC_Default, 0);
            }
        }
    }

    UE_LOG(LogTemp, Log, TEXT("切换到游戏输入模式"));
}

void AEnhancedInputCharacter::SetUIInputMode()
{
    bIsInUIMode = true;

    // ===== 设置输入模式为"游戏和UI" =====
    APlayerController* PC = Cast<APlayerController>(GetController());
    if (!PC)
    {
        return;
    }

    // FInputModeGameAndUI: 输入同时发送给游戏和UI
    //   游戏输入: 用于处理关闭菜单（例如再次按ESC关闭）
    //   UI输入: 用于处理UI控件（按钮、滑块等）
    FInputModeGameAndUI UIInputMode;
    UIInputMode.SetLockMouseToViewportBehavior(
        EMouseLockMode::DoNotLock);  // 不锁定鼠标
    UIInputMode.SetHideCursorDuringCapture(false); // 不隐藏光标
    PC->SetInputMode(UIInputMode);

    // 显示鼠标光标（用于UI操作）
    PC->bShowMouseCursor = true;

    // ===== Context 切换 =====
    UEnhancedInputLocalPlayerSubsystem* Subsystem =
        ULocalPlayer::GetSubsystem<UEnhancedInputLocalPlayerSubsystem>(
            PC->GetLocalPlayer());

    if (Subsystem)
    {
        // 激活UI Context（高优先级，拦截游戏输入）
        if (IMC_UI)
        {
            // 优先级100：远高于IMC_Default的0
            // 这样UI模式下，WASD等按键优先被UI Context处理
            Subsystem->AddMappingContext(IMC_UI, 100);
        }

        // 注意：不移除 IMC_Default！
        // 因为在UI模式中，玩家仍然需要能够按ESC退出菜单
        // 而ESC在IMC_Default中绑定到IA_Pause
        // IMC_UI的优先级更高，其他按键被UI拦截，但我们仍需要Pause功能
        //
        // 如果你有 IMC_UI 中的 IA_Pause 绑定，可以移除 IMC_Default
    }

    UE_LOG(LogTemp, Log, TEXT("切换到UI输入模式"));
}

## 连击系统详解

### 连击的时间线

```
连击系统工作流程（以3连击为例）:

时间:  0.0     0.2     0.6     0.8     1.2     1.8
        │       │       │       │       │       │
输入:   按轻攻击    按轻攻击              按轻攻击
        │       │       │       │       │       │
状态:   攻击1   窗口开   攻击2   窗口开   攻击3   (太慢，窗口已关)
        │       │               │       │       │
        └───────┴───────────────┘       │       │
              连击窗口(0.6秒)             │       │
                                      连击窗口(0.6秒)

结果:  连击1          连击2          重置为攻击1
```

### 连击检测状态机

```
                    玩家输入轻攻击
                         │
          ┌──────────────┼──────────────┐
          │              │              │
    正在攻击？      不在攻击        正在攻击但
    且在窗口内                     不在窗口内
          │              │              │
          ▼              ▼              ▼
    缓存攻击输入    执行新攻击      忽略输入
    (bHasQueued    (PerformAttack)  (什么也不做)
     Attack=true)
          │              │
          │              ▼
          │        启动连击窗口计时器
          │        (ComboWindowDuration秒后
          │         调用 TryCombo)
          │              │
          └──────┬───────┘
                 │
          窗口到期 → TryCombo()
                 │
        ┌────────┼────────┐
        │                 │
   有缓存的攻击      没有缓存的攻击
        │                 │
        ▼                 ▼
   执行连击攻击      重置攻击状态
   (PerformAttack)   (ResetCombo)
```

## 常见问题与解决

### ❌ 问题1：按键没有反应

```
可能原因：
  1. 蓝图中没有给 IA_xxx 变量赋值
     → 打开蓝图，检查 Input 分类下的变量是否都有值
  2. IMC_Default 没有激活
     → 检查 BeginPlay 中是否正确添加了 Context
  3. 游戏没有使用这个角色类
     → 检查 GameMode 的 Default Pawn Class 是否指向 BP_EnhancedInputCharacter
  4. Build.cs 中没有加 "EnhancedInput"
     → 检查 Build.cs 的 PrivateDependencyModuleNames
```

### ❌ 问题2：打开菜单后角色卡在冲刺状态

```
可能原因：没有绑定 Canceled 事件

✅ 解决方案（已在代码中实现）:
  EnhancedInput->BindAction(IA_Sprint, ETriggerEvent::Canceled,
                            this, &AEnhancedInputCharacter::OnSprintCanceled);

  这个 Canceled 绑定确保：
  - Context被移除时（打开菜单），冲刺状态被正确重置
  - 高优先级Context抢走输入时，冲刺状态被正确重置
```

### ❌ 问题3：蹲伏没有反应

```
可能原因：没有启用 bCanCrouch

✅ 解决方案（已在构造函数中处理）:
  Movement->GetNavAgentPropertiesRef().bCanCrouch = true;

  ⚠️ 这是最容易被忘记的配置！默认 ACharacter 不允许蹲伏。
```

### ❌ 问题4：手柄摇杆有漂移

```
可能原因：手柄摇杆的微小偏移被当作了有效输入

✅ 解决方案：在 IA_Move 和 IA_Look 的 IMC 条目上添加 DeadZone Modifier
   - LowerThreshold = 0.15（移动摇杆）
   - LowerThreshold = 0.10（视角摇杆）
   - Type = Radial
```

### ❌ 问题5：连击系统不工作

```
可能原因：
  1. ComboWindowDuration 太短（连击窗口时间太短，来不及输入）
     → 尝试增大到 0.8~1.0 秒进行调试
  2. 没有在轻攻击动画中设置"连击窗口开始"的通知
     → 当前简化实现中，PerformAttack 直接开启窗口
  3. 攻击动画还没播完就允许了下次攻击
     → 确认 TryCombo 是在 ComboWindowDuration 之后才被调用
```
