# 第9章案例：第三人称角色动画系统

> **难度**：⭐⭐⭐（综合实战级）
> **预计时间**：90分钟
> **目标**：综合运用动画蓝图、C++ AnimInstance、动画通知和蒙太奇，实现一个生产级别的第三人称角色动画系统。

---

## 案例目标

我们要实现以下功能：

1. **基础移动动画**：待机、走路、跑步、冲刺，通过Blend Space混合
2. **八方向移动**：前后左右及斜向的移动动画
3. **跳跃和坠落**：离地和落地的过渡动画
4. **蹲伏系统**：站立/蹲伏的平滑切换
5. **武器持有系统**：空手、单手武器、双手武器的不同待机/移动动画
6. **上下半身分层**：上半身攻击/瞄准，下半身继续移动
7. **攻击蒙太奇**：轻攻击、重攻击，支持连击
8. **受击蒙太奇**：根据受击方向播放不同动画
9. **伤害检测通知**：在攻击动画的特定帧触发伤害判定

## 第一步：创建武器状态枚举

首先创建一个头文件定义武器状态。这是整个动画系统的"配置中心"。

```cpp
// WeaponTypes.h
#pragma once

#include "CoreMinimal.h"
#include "WeaponTypes.generated.h"  // ⚠️ 必须是最后一个include！

/**
 * 武器持有状态枚举
 * 决定了角色当前使用什么类型的武器，进而影响动画
 */
UENUM(BlueprintType)  // 让这个枚举在蓝图中可用
enum class EWeaponState : uint8  // uint8 = 占用1字节，足够存状态
{
    Unarmed     UMETA(DisplayName = "空手"),     // 没有武器
    OneHanded   UMETA(DisplayName = "单手武器"),  // 剑、手枪等
    TwoHanded   UMETA(DisplayName = "双手武器"),  // 大剑、步枪等
};

/**
 * 攻击类型枚举
 * 用于区分轻攻击和重攻击，播放不同的蒙太奇
 */
UENUM(BlueprintType)
enum class EAttackType : uint8
{
    Light       UMETA(DisplayName = "轻攻击"),   // 快而轻
    Heavy       UMETA(DisplayName = "重攻击"),   // 慢而重
};
```

> **说明**：把枚举单独放在一个头文件是UE的常见做法，方便多个类引用而不会造成循环依赖。

## 第二步：创建自定义AnimNotify（伤害检测）

```cpp
// AttackHitNotify.h
#pragma once

#include "CoreMinimal.h"
#include "Animation/AnimNotifies/AnimNotify.h"  // AnimNotify基类
#include "AttackHitNotify.generated.h"

/**
 * 攻击命中的动画通知
 * 在动画播放到特定帧时触发，执行伤害检测
 */
UCLASS(meta = (DisplayName = "攻击伤害检测"))  // 编辑器中显示中文名
class MYPROJECT_API UAttackHitNotify : public UAnimNotify
{
    GENERATED_BODY()

public:
    // 覆写通知函数
    virtual void Notify(
        USkeletalMeshComponent* MeshComp,     // 播放动画的骨骼网格体
        UAnimSequenceBase* Animation          // 当前动画序列
    ) override;

    // ---- 编辑器可配置参数 ----

    // 伤害值
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "伤害参数",
              meta = (DisplayName = "伤害值"))
    float DamageAmount = 25.0f;  // 默认25点伤害

    // 伤害检测半径（球形检测）
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "伤害参数",
              meta = (DisplayName = "检测半径"))
    float DamageRadius = 150.0f;  // 默认150cm

    // 检测起始位置的前方偏移
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "伤害参数",
              meta = (DisplayName = "前方偏移"))
    float ForwardOffset = 80.0f;  // 在角色前方80cm检测

    // 是否绘制调试球体（开发阶段打开）
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "调试",
              meta = (DisplayName = "绘制调试"))
    bool bDrawDebug = false;
};
```

```cpp
// AttackHitNotify.cpp
#include "AttackHitNotify.h"
#include "Engine/World.h"            // 获取世界对象
#include "Kismet/GameplayStatics.h"  // 球形检测、施加伤害
#include "DrawDebugHelpers.h"        // 调试绘制

void UAttackHitNotify::Notify(USkeletalMeshComponent* MeshComp, UAnimSequenceBase* Animation)
{
    // ===== 第1步：判空 =====
    // 安全检查——在游戏开发中必须时刻注意，预防崩溃
    if (!MeshComp)
    {
        UE_LOG(LogTemp, Error, TEXT("AttackHitNotify：MeshComp 为空！"));
        return;  // 直接返回，不做任何操作
    }

    // 获取拥有者Actor（通常是AMyCharacter）
    AActor* OwnerActor = MeshComp->GetOwner();
    if (!OwnerActor)
    {
        UE_LOG(LogTemp, Error, TEXT("AttackHitNotify：OwnerActor 为空！"));
        return;
    }

    // 获取世界对象（后续检测需要用）
    UWorld* World = OwnerActor->GetWorld();
    if (!World) return;

    // ===== 第2步：计算检测位置 =====
    // 以角色位置为起点，向前偏移
    FVector StartLocation = OwnerActor->GetActorLocation();
    FVector ForwardDirection = OwnerActor->GetActorForwardVector();

    // 检测中心 = 角色位置 + 前方偏移量
    FVector CheckCenter = StartLocation + ForwardDirection * ForwardOffset;

    // ===== 第3步：执行球形碰撞检测 =====
    TArray<FHitResult> HitResults;  // 用于接收检测结果的数组

    // 配置检测参数
    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(OwnerActor);  // 不要检测到自己
    QueryParams.bTraceComplex = false;         // 使用简化碰撞（性能更好）

    // 球形扫描检测
    bool bHit = World->SweepMultiByChannel(
        HitResults,                              // 输出：命中的目标列表
        CheckCenter,                             // 检测起点
        CheckCenter,                             // 检测终点（同起点=原地球形检测）
        FQuat::Identity,                         // 球形不需要旋转
        ECC_Pawn,                                // 只检测Pawn类型的对象
        FCollisionShape::MakeSphere(DamageRadius), // 球形，半径用配置值
        QueryParams                              // 忽略自己的参数
    );

    // ===== 第4步：对每个命中的目标施加伤害 =====
    if (bHit)
    {
        for (const FHitResult& Hit : HitResults)
        {
            AActor* HitTarget = Hit.GetActor();
            if (!HitTarget) continue;  // 跳过空的

            // 使用引擎内置的伤害系统施加伤害
            // 参数：(目标, 伤害值, 伤害来源的Controller, 造成伤害的Actor, 伤害类型)
            UGameplayStatics::ApplyDamage(
                HitTarget,          // 被击中的Actor
                DamageAmount,       // 伤害值（从编辑器配置）
                nullptr,            // Controller（AI或玩家控制器）
                OwnerActor,         // 伤害来源（谁打了这一下）
                UDamageType::StaticClass()  // 伤害类型（基础类型）
            );

            UE_LOG(LogTemp, Log,
                TEXT("攻击命中：[%s]，伤害=%.1f"),
                *HitTarget->GetName(),
                DamageAmount
            );
        }
    }

    // ===== 第5步：调试可视化（开发时打开）=====
    if (bDrawDebug)
    {
        // 在检测位置绘制一个红色球体，持续2秒
        DrawDebugSphere(
            World,
            CheckCenter,
            DamageRadius,
            12,             // 球体的分段数（越高越圆）
            FColor::Red,    // 红色
            false,          // 不持久（2秒后消失）
            2.0f            // 显示2秒
        );
    }
}
```

> **关键理解**：`UGameplayStatics::ApplyDamage` 是UE内置的伤害系统入口。它会在目标Actor上调用 `TakeDamage` 函数。如果你的角色覆写了 `TakeDamage`，就能接收伤害。这里的使用方式与GAS（Gameplay Ability System）兼容。

## 第三步：创建UMyAnimInstance（核心动画逻辑）

这是整个动画系统的大脑——所有动画参数都在这里计算。

### 头文件

```cpp
// MyAnimInstance.h
#pragma once

#include "CoreMinimal.h"
#include "Animation/AnimInstance.h"      // 动画实例基类
#include "WeaponTypes.h"                  // 我们的武器状态枚举
#include "MyAnimInstance.generated.h"     // ⚠️必须最后！

// 前置声明（避免不必要的include，加快编译速度）
class AMyCharacter;
class UCharacterMovementComponent;

/**
 * 自定义动画实例
 * 负责计算所有动画参数，供动画蓝图使用
 */
UCLASS()
class MYPROJECT_API UMyAnimInstance : public UAnimInstance
{
    GENERATED_BODY()

public:
    // ===== 生命周期 =====

    // 初始化：动画蓝图首次创建时调用一次
    virtual void NativeInitializeAnimation() override;

    // 更新：每帧调用，计算最新的动画参数
    virtual void NativeUpdateAnimation(float DeltaSeconds) override;

protected:
    // ========================================
    // 第一组：基础移动参数
    // ========================================

    // 水平移动速度（cm/s）
    // 用于驱动 Blend Space 的 X 轴
    UPROPERTY(BlueprintReadOnly, Category = "动画|移动")
    float Speed;

    // 移动方向角度（度，-180~180）
    // 0=前方，90=右方，-90=左方，±180=后方
    UPROPERTY(BlueprintReadOnly, Category = "动画|移动")
    float Direction;

    // 是否在空中（跳跃/坠落）
    UPROPERTY(BlueprintReadOnly, Category = "动画|移动")
    bool bIsInAir;

    // 是否在蹲伏
    UPROPERTY(BlueprintReadOnly, Category = "动画|移动")
    bool bIsCrouching;

    // 是否在冲刺（奔跑速度 + 按下冲刺键）
    UPROPERTY(BlueprintReadOnly, Category = "动画|移动")
    bool bIsSprinting;

    // 垂直速度（cm/s，正=上升，负=下落）
    UPROPERTY(BlueprintReadOnly, Category = "动画|移动")
    float VerticalVelocity;

    // 移动输入值（-1到1，用于Start/Stop过渡动画）
    // >0 = 有移动输入，=0 = 无移动输入
    UPROPERTY(BlueprintReadOnly, Category = "动画|移动")
    float MoveInputAmount;

    // ========================================
    // 第二组：武器/战斗参数
    // ========================================

    // 当前武器持有状态
    UPROPERTY(BlueprintReadOnly, Category = "动画|战斗")
    EWeaponState CurrentWeaponState;

    // 是否正在攻击
    UPROPERTY(BlueprintReadOnly, Category = "动画|战斗")
    bool bIsAttacking;

    // ========================================
    // 第三组：瞄准参数（用于AimOffset混合）
    // ========================================

    // 上下瞄准角度（俯仰，-90~90）
    UPROPERTY(BlueprintReadOnly, Category = "动画|瞄准")
    float AimPitch;

    // 左右瞄准角度（偏航，-180~180）
    UPROPERTY(BlueprintReadOnly, Category = "动画|瞄准")
    float AimYaw;

    // ========================================
    // 第四组：缓存的指针（避免每帧重复获取）
    // ========================================

    // 缓存拥有者角色指针，初始化时获取一次
    // Transient = 不序列化到存档（运行时引用，存档后会失效）
    UPROPERTY(Transient)
    TObjectPtr<AMyCharacter> CachedCharacter;

    // 缓存移动组件指针
    UPROPERTY(Transient)
    TObjectPtr<UCharacterMovementComponent> CachedMovement;
};
```

### 源文件

```cpp
// MyAnimInstance.cpp
#include "MyAnimInstance.h"
#include "MyCharacter.h"                      // 我们的角色类
#include "GameFramework/Character.h"
#include "GameFramework/CharacterMovementComponent.h"
#include "Kismet/KismetMathLibrary.h"         // 数学工具

// ===== 初始化 =====
void UMyAnimInstance::NativeInitializeAnimation()
{
    // 调用父类初始化——永远不要忘记！
    Super::NativeInitializeAnimation();

    // 缓存角色指针，避免在每帧Update中反复Cast
    // 这是非常重要的性能优化技巧
    APawn* OwnerPawn = TryGetPawnOwner();
    if (OwnerPawn)
    {
        // 转换为我们的自定义角色类型
        CachedCharacter = Cast<AMyCharacter>(OwnerPawn);

        // 缓存移动组件——这也是每帧都会用到的
        if (CachedCharacter)
        {
            CachedMovement = CachedCharacter->GetCharacterMovement();
        }
    }

    // 初始化所有参数为默认值
    Speed = 0.0f;
    Direction = 0.0f;
    bIsInAir = false;
    bIsCrouching = false;
    bIsSprinting = false;
    VerticalVelocity = 0.0f;
    MoveInputAmount = 0.0f;
    CurrentWeaponState = EWeaponState::Unarmed;
    bIsAttacking = false;
    AimPitch = 0.0f;
    AimYaw = 0.0f;
}

// ===== 每帧更新 =====
void UMyAnimInstance::NativeUpdateAnimation(float DeltaSeconds)
{
    // 调用父类——引擎依赖这个调用来更新内部状态
    Super::NativeUpdateAnimation(DeltaSeconds);

    // 安全检查：角色指针是否有效
    if (!CachedCharacter || !CachedMovement)
    {
        return;  // 无效则跳过本帧，使用上一帧的参数值
    }

    // ==================================================================
    // 模块1：基础移动参数计算
    // ==================================================================

    // --- 获取速度 ---
    FVector Velocity = CachedCharacter->GetVelocity();

    // Size2D 只计算水平速度（忽略Z轴），确保跳跃时不会误判为高速移动
    Speed = Velocity.Size2D();

    // Z轴速度：用于区分"正在跳起"和"正在坠落"
    VerticalVelocity = Velocity.Z;

    // --- 状态判断 ---
    // IsFalling() 覆盖了所有离地情况：跳跃上升、跳跃下降、掉落、被击飞
    bIsInAir = CachedMovement->IsFalling();

    // IsCrouching() 判断角色当前是否在蹲伏状态
    bIsCrouching = CachedMovement->IsCrouching();

    // 冲刺判断：速度超过阈值 + 在地面上（空中不可能冲刺）
    // 600cm/s 约等于角色默认的跑步速度上限
    bIsSprinting = (Speed > 600.0f && !bIsInAir);

    // --- 移动输入量 ---
    // GetLastInputVector() 返回玩家最后一次的移动输入方向向量
    // Size() > 0 表示玩家正在按移动键（WASD或摇杆）
    FVector InputVector = CachedCharacter->GetLastMovementInputVector();
    MoveInputAmount = InputVector.Size();  // 0~1，表示输入强度

    // ==================================================================
    // 模块2：移动方向计算
    // ==================================================================

    if (Speed > 10.0f)  // 只有在实际移动时才计算方向
    {
        // 速度方向转成旋转角
        FRotator VelocityRotation = Velocity.ToOrientationRotator();
        // 角色当前的朝向
        FRotator ActorRotation = CachedCharacter->GetActorRotation();

        // 计算两个旋转之间的差值
        // NormalizedDeltaRotator 保证结果在 [-180, 180] 范围内
        FRotator DeltaRot = UKismetMathLibrary::NormalizedDeltaRotator(
            VelocityRotation,   // 实际移动方向
            ActorRotation       // 角色面朝方向
        );

        Direction = DeltaRot.Yaw;  // 取Yaw（偏航角）
        // 结果：0°=向前，90°=向右，-90°=向左，±180°=向后
    }
    else
    {
        // 静止时方向归零，避免动画闪烁
        Direction = 0.0f;
    }

    // ==================================================================
    // 模块3：武器状态（从角色类读取）
    // ==================================================================

    // 假设 AMyCharacter 中有一个 GetWeaponState() 方法
    // CurrentWeaponState = CachedCharacter->GetWeaponState();

    // 是否正在攻击
    // bIsAttacking = CachedCharacter->IsAttacking();

    // ==================================================================
    // 模块4：瞄准偏移
    // ==================================================================

    // 获取控制器的旋转（玩家的视角朝向）
    FRotator ControlRotation = CachedCharacter->GetControlRotation();
    // 获取角色的身体朝向
    FRotator ActorBodyRotation = CachedCharacter->GetActorRotation();

    // 计算视角相对于身体的偏移
    FRotator AimDelta = UKismetMathLibrary::NormalizedDeltaRotator(
        ControlRotation,     // 视角（玩家看向哪里）
        ActorBodyRotation    // 身体（角色面朝哪里）
    );

    AimPitch = FMath::Clamp(AimDelta.Pitch, -90.0f, 90.0f);  // 限制上下角度
    AimYaw = FMath::Clamp(AimDelta.Yaw, -90.0f, 90.0f);      // 限制左右角度
    // Clamp限制是为了防止脖子扭断（真实的人也不可能360度转头）
}
```

## 第四步：创建AMyCharacter（角色类）

角色类负责处理输入、播放蒙太奇、管理武器状态。

### 头文件

```cpp
// MyCharacter.h
#pragma once

#include "CoreMinimal.h"
#include "GameFramework/Character.h"
#include "WeaponTypes.h"          // 武器状态枚举
#include "MyCharacter.generated.h"

// 前置声明
class UAnimMontage;

/**
 * 自定义角色类
 * 管理输入响应、蒙太奇播放和武器状态
 */
UCLASS()
class MYPROJECT_API AMyCharacter : public ACharacter
{
    GENERATED_BODY()

public:
    AMyCharacter();

protected:
    // ===== 输入绑定 =====
    virtual void SetupPlayerInputComponent(class UInputComponent* PlayerInputComponent) override;

    // ===== 动作函数（绑定到输入）=====

    // 攻击（轻攻击/重攻击）
    void OnLightAttack();   // 鼠标左键
    void OnHeavyAttack();   // 鼠标右键

    // 冲刺
    void OnSprintStart();   // 左Shift按下
    void OnSprintStop();    // 左Shift松开

    // 切换武器
    void OnSwitchWeapon();  // 按1/2/3切换武器状态

    // ===== 蒙太奇播放 =====

    // 播放攻击蒙太奇（内部根据攻击类型和武器状态选择蒙太奇）
    void PlayAttackMontage(EAttackType AttackType);

    // 播放受击蒙太奇（根据受击方向）
    void PlayHitReactionMontage(FVector HitDirection);

    // 停止当前蒙太奇
    void StopCurrentMontage(float BlendOutTime = 0.25f);

    // ===== 武器状态 =====

    // 获取当前武器状态
    UFUNCTION(BlueprintPure, Category = "战斗")
    EWeaponState GetWeaponState() const { return CurrentWeaponState; }

    // 设置武器状态
    void SetWeaponState(EWeaponState NewState);

    // 是否正在攻击
    UFUNCTION(BlueprintPure, Category = "战斗")
    bool IsAttacking() const;

public:
    // ===== 受击接口 =====

    // 覆写TakeDamage，让这个角色能"受伤"
    virtual float TakeDamage(
        float DamageAmount,
        struct FDamageEvent const& DamageEvent,
        class AController* EventInstigator,
        AActor* DamageCauser
    ) override;

protected:
    // ========================================
    // 蒙太奇资源（在蓝图中配置具体资源）
    // ========================================

    // 轻攻击蒙太奇（按武器状态分开）
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "动画|攻击")
    TObjectPtr<UAnimMontage> LightAttackMontage_Unarmed;   // 空手轻攻击

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "动画|攻击")
    TObjectPtr<UAnimMontage> LightAttackMontage_OneHanded;  // 单手轻攻击

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "动画|攻击")
    TObjectPtr<UAnimMontage> LightAttackMontage_TwoHanded;  // 双手轻攻击

    // 重攻击蒙太奇
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "动画|攻击")
    TObjectPtr<UAnimMontage> HeavyAttackMontage_Unarmed;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "动画|攻击")
    TObjectPtr<UAnimMontage> HeavyAttackMontage_OneHanded;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "动画|攻击")
    TObjectPtr<UAnimMontage> HeavyAttackMontage_TwoHanded;

    // 受击蒙太奇
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "动画|受击")
    TObjectPtr<UAnimMontage> HitReaction_Front;  // 正面受击

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "动画|受击")
    TObjectPtr<UAnimMontage> HitReaction_Back;   // 背面受击

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "动画|受击")
    TObjectPtr<UAnimMontage> HitReaction_Left;   // 左侧受击

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "动画|受击")
    TObjectPtr<UAnimMontage> HitReaction_Right;  // 右侧受击

    // ========================================
    // 武器状态
    // ========================================

    // 当前武器状态
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "战斗")
    EWeaponState CurrentWeaponState = EWeaponState::Unarmed;

    // ========================================
    // 内部状态
    // ========================================

    // 是否正在冲刺
    bool bIsSprinting = false;

    // 攻击冷却计时器
```

```cpp
// ===== 播放攻击蒙太奇 =====
void AMyCharacter::PlayAttackMontage(EAttackType AttackType)
{
    // 第一步：根据武器状态和攻击类型选择合适的蒙太奇
    UAnimMontage* SelectedMontage = nullptr;

    if (AttackType == EAttackType::Light)
    {
        // 根据当前武器状态选择对应的轻攻击蒙太奇
        switch (CurrentWeaponState)
        {
        case EWeaponState::Unarmed:
            SelectedMontage = LightAttackMontage_Unarmed;
            break;
        case EWeaponState::OneHanded:
            SelectedMontage = LightAttackMontage_OneHanded;
            break;
        case EWeaponState::TwoHanded:
            SelectedMontage = LightAttackMontage_TwoHanded;
            break;
        }
    }
    else  // 重攻击
    {
        switch (CurrentWeaponState)
        {
        case EWeaponState::Unarmed:
            SelectedMontage = HeavyAttackMontage_Unarmed;
            break;
        case EWeaponState::OneHanded:
            SelectedMontage = HeavyAttackMontage_OneHanded;
            break;
        case EWeaponState::TwoHanded:
            SelectedMontage = HeavyAttackMontage_TwoHanded;
            break;
        }
    }

    // 第二步：检查蒙太奇是否有效
    if (!SelectedMontage)
    {
        UE_LOG(LogTemp, Warning,
            TEXT("PlayAttackMontage：没有为 [武器=%s, 攻击=%s] 配置蒙太奇！"),
            *UEnum::GetValueAsString(CurrentWeaponState),
            *UEnum::GetValueAsString(AttackType)
        );
        return;  // 资源缺失，安全退出
    }

    // 第三步：获取动画实例
    UAnimInstance* AnimInstance = GetMesh()->GetAnimInstance();
    if (!AnimInstance)
    {
        UE_LOG(LogTemp, Error, TEXT("PlayAttackMontage：AnimInstance为空！"));
        return;
    }

    // 第四步：检查是否已经同一蒙太奇在播放（连击处理）
    if (AnimInstance->Montage_IsPlaying(SelectedMontage))
    {
        // 正在播放同一攻击蒙太奇 → 不重复播放
        // 如果实现了连击系统，这里应该跳Section而不是重新播放
        UE_LOG(LogTemp, Log, TEXT("攻击蒙太奇已在播放中，忽略"));
        return;
    }

    // 第五步：播放蒙太奇
    float Duration = AnimInstance->Montage_Play(SelectedMontage, 1.0f);

    if (Duration > 0.0f)
    {
        UE_LOG(LogTemp, Log,
            TEXT("播放 [%s] [%s]，时长=%.2f秒"),
            *UEnum::GetValueAsString(CurrentWeaponState),
            *UEnum::GetValueAsString(AttackType),
            Duration
        );

        // 设置攻击冷却（防止连按）
        AttackCooldownRemaining = Duration * 0.6f;  // 冷却 = 动画时长的60%
        // 注意：这不是严格意义上的"冷却时间"，而是连续触发的防止间隔
    }
    else
    {
        UE_LOG(LogTemp, Warning, TEXT("播放攻击蒙太奇失败！"));
    }
}

// ===== 受击蒙太奇 =====
void AMyCharacter::PlayHitReactionMontage(FVector HitDirection)
{
    // 第一步：判断受击方向
    // 将世界空间中的受击方向转换到角色的本地空间
    FVector LocalHitDirection = GetActorTransform().InverseTransformVector(HitDirection);
    // InverseTransformVector 把世界方向转为本地方向
    // 结果：X=前方受击, -X=后方受击, Y=右方受击, -Y=左方受击

    // 第二步：根据方向选择蒙太奇
    UAnimMontage* SelectedHitMontage = nullptr;

    // 判断主要受击方向（哪个轴的绝对值最大）
    if (FMath::Abs(LocalHitDirection.X) > FMath::Abs(LocalHitDirection.Y))
    {
        // 前后方向为主
        if (LocalHitDirection.X > 0)
        {
            SelectedHitMontage = HitReaction_Back;   // 前方来的攻击 = 向后仰
        }
        else
        {
            SelectedHitMontage = HitReaction_Front;  // 后方来的攻击 = 向前倾
        }
    }
    else
    {
        // 左右方向为主
        if (LocalHitDirection.Y > 0)
        {
            SelectedHitMontage = HitReaction_Right;  // 右侧来的攻击 = 向左倾
        }
        else
        {
            SelectedHitMontage = HitReaction_Left;   // 左侧来的攻击 = 向右倾
        }
    }

    // 第三步：播放
    if (!SelectedHitMontage)
    {
        UE_LOG(LogTemp, Warning, TEXT("PlayHitReaction：没有配置对应方向的受击蒙太奇！"));
        return;
    }

    UAnimInstance* AnimInstance = GetMesh()->GetAnimInstance();
    if (!AnimInstance) return;

    // 受击蒙太奇的优先级更高（可以打断攻击动画）
    // 所以先停止当前蒙太奇，再播放受击
    AnimInstance->Montage_Stop(0.1f);  // 快速淡出当前动画

    // 播放受击蒙太奇
    AnimInstance->Montage_Play(SelectedHitMontage, 1.0f);

    UE_LOG(LogTemp, Log, TEXT("播放受击蒙太奇：%s"), *SelectedHitMontage->GetName());
}

// ===== 受击（覆写TakeDamage）=====
float AMyCharacter::TakeDamage(
    float DamageAmount,
    FDamageEvent const& DamageEvent,
    AController* EventInstigator,
    AActor* DamageCauser)
{
    // 第一步：调用父类（引擎内部可能有一些基础处理）
    float ActualDamage = Super::TakeDamage(DamageAmount, DamageEvent, EventInstigator, DamageCauser);

    // 第二步：如果实际伤害大于0，则播放受击动画
    if (ActualDamage > 0.0f)
    {
        UE_LOG(LogTemp, Log, TEXT("%s 受到 %.1f 点伤害"), *GetName(), ActualDamage);

        // 计算受击方向
        FVector HitDirection = FVector::ZeroVector;

        // 从伤害事件中提取信息
        if (DamageEvent.IsOfType(FPointDamageEvent::ClassID))
        {
            // 点伤害（如子弹）——有精确的方向信息
            const FPointDamageEvent* PointEvent = (const FPointDamageEvent*)&DamageEvent;
            HitDirection = PointEvent->ShotDirection;
        }
        else if (DamageCauser)
        {
            // 从伤害来源的位置推算受击方向
            HitDirection = GetActorLocation() - DamageCauser->GetActorLocation();
        }

        // 只有方向有效时才播放受击动画
        if (!HitDirection.IsNearlyZero())
        {
            PlayHitReactionMontage(HitDirection);
        }

        // TODO：扣除生命值、更新UI等
    }

    return ActualDamage;
}

// ===== Tick（冷却刷新）=====
void AMyCharacter::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    // 刷新攻击冷却
    if (AttackCooldownRemaining > 0.0f)
    {
        AttackCooldownRemaining -= DeltaTime;  // 每帧减少
        if (AttackCooldownRemaining < 0.0f)
        {
            AttackCooldownRemaining = 0.0f;  // 归零，可以再次攻击
        }
    }
}
```

### 5.4 创建动画图表（AnimGraph）——状态机

```
操作步骤：
1. 切换到"动画图表"标签
2. 移除默认的"Output Animation Pose"节点
3. 右键 → 搜索 "State Machine" → 添加

5. 将状态机的输出连接到 "Output Animation Pose"（最终输出）

双击进入状态机内部，创建以下状态：
```

#### 状态列表

| 状态名     | 动画内容                      | 说明                  |
| ---------- | ----------------------------- | --------------------- |
| Idle       | 待机动画                      | 角色不动时的动画      |
| Locomotion | Blend Space（走/跑/冲刺混合） | 根据Speed参数自动混合 |
| JumpStart  | 起跳动画                      | 从地面跳起的瞬间      |
| JumpLoop   | 空中循环动画                  | 在空中的持续动作      |
| JumpEnd    | 落地动画                      | 接触地面的瞬间        |

#### 各状态的内部配置

```
【Idle 状态】：
  → 拖入一个 Sequence Player 节点（或直接连 Pose 节点）
  → 选择 Idle 动画资源（如 A_Idle）
  → 输出到状态输出引脚

【Locomotion 状态】：
  → 拖入 Blend Space Player 节点
  → Asset 选择你创建的 Blend Space（如 BS_Locomotion）
  → 将 X 轴参数连接到 Speed 变量
  → 将 Y 轴参数连接到 Direction 变量
  → 输出到状态输出引脚

【JumpStart/JumpLoop/JumpEnd】：
  → 各拖入 Sequence Player 节点
  → 选择对应的动画资源
```

#### 转换规则设置

```
【Idle → Locomotion】：
  条件：Speed > 10.0
  Duration：0.2秒

【Locomotion → Idle】：
  条件：Speed <= 10.0
  Duration：0.25秒

【任意状态 → JumpStart】：
  条件：bIsInAir == true  AND  VerticalVelocity > 50
  Duration：0.1秒（快速）

【JumpStart → JumpLoop】：
  条件：JumpStart动画剩余时间 <= 0.1秒
  Duration：0.1秒

【JumpLoop → JumpEnd】：
  条件：bIsInAir == false
  Duration：0.1秒

【JumpEnd → Idle/Locomotion】：
  条件：JumpEnd动画剩余时间 <= 0.1秒
  Duration：0.15秒
```

## 常见错误和调试技巧

### ❌ 代码编译错误

```
❌ "GENERATED_BODY" 报错 → 检查 .generated.h 是否在最后一行include
❌ "UCLASS" 未定义 → 检查是否include了正确的头文件
❌ "BlueprintReadOnly" 未定义 → 检查 UPORPERTY 拼写（是U不是O）
```

### ❌ 运行时问题

```
❌ 角色摆T-Pose不动 → 动画蓝图没有正确连线到Output Pose
❌ 动画参数一直为0 → 检查 CachedCharacter 是否为null（TryGetPawnOwner失败）
❌ 蒙太奇播不出来 → 检查：
    - 蒙太奇资源是否在蓝图中赋值
    - Slot名称是否匹配（蒙太奇中设置的和AnimBP中创建的）
    - AnimInstance 是否获取成功
❌ 攻击无伤害 → 检查 AnimNotify 是否加到了蒙太奇时间轴上
❌ 上下半身混合无效 → 检查 Layered Blend Per Bone 的 Branch Filter 骨骼名是否正确
```

### ✅ 调试技巧

```
1. 在蓝图的EventGraph中加 Print String 节点打印 Speed/BIsInAir
   → 运行时屏幕左上方会显示数值

2. 使用 Gameplay Debugger（按 ' 键）查看角色状态
   → 显示碰撞体、AI信息、动画状态等

3. 打开 AnimBP 后进入 PIE（Play in Editor）
   → 可以实时看到动画蓝图中变量的变化和状态机的切换

4. 在 Anfractuosity > Console 中输入：
   ShowDebug ANIMATION
   → 显示当前所有动画参数和状态

5. 使用 UE_LOG 打印关键节点
   → 在 NativeUpdateAnimation 中加日志
```
