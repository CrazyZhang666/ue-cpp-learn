# 9.3 动画通知与蒙太奇

> **目标**：掌握动画通知（AnimNotify/AnimNotifyState）的C++自定义，以及动画蒙太奇（AnimMontage）的完整使用流程——播放、分段、跳段、混合。

---

## 第一部分：动画通知（AnimNotify）

### 什么是AnimNotify

动画通知是动画序列中**特定时间点**触发的事件。你可以把它理解为"动画时间轴上的书签"。

```
一个"攻击"动画序列的时间轴：

  0s      0.3s         0.5s       0.8s     1.2s
  │ 蓄力阶段 │ 挥砍阶段   │ 收招阶段  │
  │         │           │          │
  │    [通知：刀光出现]  │ [通知：伤害检测开始]...[通知：伤害检测结束]
  │                     │
  └─────────────────────┴───────────────────────────▶

每个通知标记都对应动画中的一个关键时刻
```

### AnimNotify vs AnimNotifyState

| 类型 | 特点 | 适用场景 |
|------|------|---------|
| **AnimNotify** | 瞬间触发，只调用一次 | 脚步音效、特效生成、装备切换 |
| **AnimNotifyState** | 持续一段时间，有Begin/Tick/End三个阶段 | 伤害检测窗口、无敌帧、蓄力判定 |

```
AnimNotify：           AnimNotifyState：
     │                    ┌── NotifyBegin（开始时）
  ●──触发                 │
     │                    ├── NotifyTick（每一帧）
                          │
                          └── NotifyEnd（结束时）
```

## 在C++中创建自定义AnimNotify

### 创建AnimNotify类（头文件）

```cpp
// MyAttackHitNotify.h
#pragma once

#include "CoreMinimal.h"
#include "Animation/AnimNotifies/AnimNotify.h"  // AnimNotify的基类
#include "MyAttackHitNotify.generated.h"        // ⚠️ 必须最后！

/**
 * 自定义动画通知：攻击命中检测
 * 在攻击动画的"伤害窗口"时间内被调用
 */
UCLASS()  // 标记为UE反射类
class MYPROJECT_API UMyAttackHitNotify : public UAnimNotify
{
    GENERATED_BODY()  // 生成反射代码

public:
    // 覆写这个函数！当动画播放到这个通知的时间点时，引擎自动调用
    // 参数：
    //   MeshComp   = 播放动画的骨骼网格体组件
    //   Animation  = 正在播放的动画序列
    // 返回值：true = 通知执行成功
    virtual void Notify(USkeletalMeshComponent* MeshComp, UAnimSequenceBase* Animation) override;

    // 你可以在这里定义通知的配置参数（在编辑器中可编辑）
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "伤害检测")
    float DamageAmount = 10.0f;  // 伤害值

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "伤害检测")
    float DamageRadius = 150.0f;  // 伤害半径（AOE攻击用）
};
```

### 创建AnimNotify类（源文件）

```cpp
// MyAttackHitNotify.cpp
#include "MyAttackHitNotify.h"
#include "GameFramework/Character.h"  // 获取角色
#include "Kismet/GameplayStatics.h"   // 球形检测等工具函数
#include "Engine/World.h"             // 获取世界

void UMyAttackHitNotify::Notify(USkeletalMeshComponent* MeshComp, UAnimSequenceBase* Animation)
{
    // 第一步：安全判空
    // MeshComp 是播放这个动画的骨骼网格体组件
    if (!MeshComp)
    {
        UE_LOG(LogTemp, Warning, TEXT("攻击通知：MeshComp为空，跳过"));
        return;
    }

    // 第二步：获取骨骼网格体的拥有者Actor（通常是ACharacter）
    AActor* OwnerActor = MeshComp->GetOwner();
    if (!OwnerActor)
    {
        UE_LOG(LogTemp, Warning, TEXT("攻击通知：OwnerActor为空，跳过"));
        return;
    }

    // 第三步：获取世界对象
    UWorld* World = OwnerActor->GetWorld();
    if (!World) return;

    // 第四步：构建检测位置
    // 从角色的位置向前方偏移一段距离作为检测中心
    FVector StartLocation = OwnerActor->GetActorLocation();
    FVector ForwardVector = OwnerActor->GetActorForwardVector();
    FVector CheckLocation = StartLocation + ForwardVector * 100.0f;  // 前方100cm处

    // 第五步：执行球形碰撞检测（检测攻击范围内的目标）
    TArray<FHitResult> HitResults;  // 存放检测结果的数组

    // 碰撞查询参数
    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(OwnerActor);  // 忽略自己（不要打到自己）
    QueryParams.bTraceComplex = false;         // 使用简单碰撞

    // 球形检测
    bool bHit = World->SweepMultiByChannel(
        HitResults,           // 输出：命中的结果
        CheckLocation,        // 检测起始位置
        CheckLocation,        // 检测结束位置（相同表示原地球形检测）
        FQuat::Identity,      // 球形不需要旋转
        ECC_Pawn,             // 碰撞通道：只检测Pawn类型
        FCollisionShape::MakeSphere(DamageRadius),  // 球形，半径=配置值
        QueryParams           // 查询参数（忽略自己）
    );

    // 第六步：对每个命中的目标施加伤害
    if (bHit)
    {
        for (const FHitResult& Hit : HitResults)
        {
            AActor* HitActor = Hit.GetActor();
            if (HitActor)
            {
                // 这里调用你的伤害系统
                // 实际项目中通常用 UGameplayStatics::ApplyDamage()
                UE_LOG(LogTemp, Log,
                    TEXT("攻击命中：[%s]，伤害=%.1f，命中位置=%s"),
                    *HitActor->GetName(),
                    DamageAmount,
                    *Hit.ImpactPoint.ToString()
                );

                // 施加伤害（需要项目中有伤害系统）
                // UGameplayStatics::ApplyDamage(HitActor, DamageAmount, nullptr, OwnerActor, UDamageType::StaticClass());
            }
        }
    }
}
```

## 创建自定义AnimNotifyState

AnimNotifyState有三个阶段，适合需要"持续"一段时间的效果。

### AnimNotifyState头文件

```cpp
// MyDamageWindowNotifyState.h
#pragma once

#include "CoreMinimal.h"
#include "Animation/AnimNotifies/AnimNotifyState.h"  // AnimNotifyState的基类
#include "MyDamageWindowNotifyState.generated.h"

/**
 * 自定义动画通知状态：伤害检测窗口
 * 在攻击动画的伤害帧区间内持续检测是否有目标进入范围
 */
UCLASS()
class MYPROJECT_API UMyDamageWindowNotifyState : public UAnimNotifyState
{
    GENERATED_BODY()

public:
    // ===== 阶段1：通知开始时调用 =====
    // 只在进入通知区间时执行一次
    virtual void NotifyBegin(
        USkeletalMeshComponent* MeshComp,     // 骨骼网格体组件
        UAnimSequenceBase* Animation,          // 动画序列
        float TotalDuration,                   // 通知的总持续时间（秒）
        const FAnimNotifyEventReference& EventReference  // 事件引用
    ) override;

    // ===== 阶段2：通知持续期间每帧调用 =====
    // 在通知区间内，每一帧都会执行
    virtual void NotifyTick(
        USkeletalMeshComponent* MeshComp,
        UAnimSequenceBase* Animation,
        float FrameDeltaTime,                  // 距离上一帧的时间
        const FAnimNotifyEventReference& EventReference
    ) override;

    // ===== 阶段3：通知结束时调用 =====
    // 只在退出通知区间时执行一次
    virtual void NotifyEnd(
        USkeletalMeshComponent* MeshComp,
        UAnimSequenceBase* Animation,
        const FAnimNotifyEventReference& EventReference
    ) override;

    // ---- 配置参数 ----

    // 伤害检测的碰撞通道
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "伤害窗口")
    float TraceRadius = 100.0f;

    // 检测起点相对于角色位置的偏移
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "伤害窗口")
    FVector TraceOffset = FVector(100.0f, 0.0f, 0.0f);  // 前方100cm

    // 是否启用调试绘制（开发时可视化检测范围）
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "调试")
    bool bDrawDebug = false;
};
```

### AnimNotifyState源文件

```cpp
// MyDamageWindowNotifyState.cpp
#include "MyDamageWindowNotifyState.h"
#include "Engine/World.h"
#include "DrawDebugHelpers.h"     // 调试绘制

void UMyDamageWindowNotifyState::NotifyBegin(
    USkeletalMeshComponent* MeshComp,
    UAnimSequenceBase* Animation,
    float TotalDuration,
    const FAnimNotifyEventReference& EventReference)
{
    // ---- 伤害窗口开启 ----
    // 这里做一些初始化工作

    if (!MeshComp) return;

    UE_LOG(LogTemp, Log,
        TEXT("伤害窗口开启！持续%.2f秒，检测半径%.1fcm"),
        TotalDuration,
        TraceRadius
    );

    // 可以在这里显示武器特效、开启碰撞等
    // 例如：激活武器的碰撞体
}
// 注意：AnimNotifyState 的 NotifyTick 在每帧调用（见下文）
// 这是它与普通 AnimNotify 的最大区别

void UMyDamageWindowNotifyState::NotifyTick(
    USkeletalMeshComponent* MeshComp,
    UAnimSequenceBase* Animation,
    float FrameDeltaTime,
    const FAnimNotifyEventReference& EventReference)
{
    // ---- 每帧检测 ----
    // 在伤害窗口打开的每一帧都执行
    // 用于持续检测碰撞、更新特效位置等

    if (!MeshComp) return;

    AActor* OwnerActor = MeshComp->GetOwner();
    if (!OwnerActor) return;

    UWorld* World = OwnerActor->GetWorld();
    if (!World) return;

    // 构建检测位置
    FVector OwnerLocation = OwnerActor->GetActorLocation();
    FRotator OwnerRotation = OwnerActor->GetActorRotation();
    FVector CheckLocation = OwnerLocation + OwnerRotation.RotateVector(TraceOffset);

    // 持续执行球形检测
    TArray<FHitResult> HitResults;
    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(OwnerActor);

    bool bHit = World->SweepMultiByChannel(
        HitResults,
        CheckLocation,
        CheckLocation,
        FQuat::Identity,
        ECC_Pawn,
        FCollisionShape::MakeSphere(TraceRadius),
        QueryParams
    );

    // 处理每个命中的目标
    for (const FHitResult& Hit : HitResults)
    {
        AActor* HitActor = Hit.GetActor();
        if (HitActor)
        {
            // 避免重复对同一目标造成伤害（需要额外记录）
            UE_LOG(LogTemp, Log, TEXT("伤害窗口内命中：[%s]"), *HitActor->GetName());
        }
    }

    // 调试绘制（开发用）
    if (bDrawDebug)
    {
        DrawDebugSphere(World, CheckLocation, TraceRadius, 12, FColor::Red, false, 0.05f);
    }
}

void UMyDamageWindowNotifyState::NotifyEnd(
    USkeletalMeshComponent* MeshComp,
    UAnimSequenceBase* Animation,
    const FAnimNotifyEventReference& EventReference)
{
    // ---- 伤害窗口关闭 ----
    // 做一些清理工作

    if (!MeshComp) return;

    UE_LOG(LogTemp, Log, TEXT("伤害窗口关闭"));

    // 关闭武器碰撞体、隐藏特效等
    // 例如：停用武器的碰撞体，防止在收招时误伤
}
```

## 第二部分：动画蒙太奇（AnimMontage）

### 什么是动画蒙太奇

动画蒙太奇是一种**特殊的动画资源**，专门用于"一次性播放"的动作——比如攻击、受伤、翻滚、交互。和普通动画序列不同，蒙太奇可以：

- 被代码控制播放/停止
- 分成多个Section（分段），可以跳转
- 通过Slot节点接入动画蓝图
- 支持混合（通知混合）

```
             AnimMontage
            ┌──────────────────────────────┐
            │  Section A    Section B      │
            │  [起手]  →  [挥砍]  →  [收招] │
            │                              │
            │ Slot: UpperBody (上半身插槽)   │
            └──────────────────────────────┘
```

### 和普通动画序列的区别

| 特性 | Animation Sequence | AnimMontage |
|------|--------------------|-------------|
| 用途 | 循环播放（走、跑、待机） | 一次性播放（攻击、受击） |
| 分段（Section） | 无 | 有，可以跳转 |
| 播放控制 | 由状态机自动控制 | 由代码手动控制 |
| 循环 | 通常是 | 通常否 |
| Slot绑定 | 不需要（状态机直接输出） | 需要Slot节点接入 |

### 在Character中封装攻击蒙太奇

```cpp
// MyCharacter.h
#pragma once

#include "CoreMinimal.h"
#include "GameFramework/Character.h"
#include "MyCharacter.generated.h"

UCLASS()
class MYPROJECT_API AMyCharacter : public ACharacter
{
    GENERATED_BODY()

public:
    // 播放攻击蒙太奇
    // 返回值：播放成功则返回true，失败返回false
    UFUNCTION(BlueprintCallable, Category = "战斗")
    bool PlayAttackMontage();

    // 播放受击蒙太奇
    UFUNCTION(BlueprintCallable, Category = "战斗")
    bool PlayHitMontage(FVector HitDirection);  // HitDirection = 受击方向

    // 停止当前蒙太奇
    UFUNCTION(BlueprintCallable, Category = "战斗")
    void StopCurrentMontage();

    // 跳转到蒙太奇的指定Section
    UFUNCTION(BlueprintCallable, Category = "战斗")
    void JumpToAttackSection(FName SectionName);

protected:
    // ---- 蒙太奇资源引用（在蓝图中指定具体资源）----

    // 攻击蒙太奇
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "动画|蒙太奇")
    UAnimMontage* AttackMontage;

    // 受击蒙太奇
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "动画|蒙太奇")
    UAnimMontage* HitMontage;
};
```

```cpp
// MyCharacter.cpp
#include "MyCharacter.h"
#include "Animation/AnimInstance.h"    // PlayAnimMontage 在这里定义
#include "Animation/AnimMontage.h"     // UAnimMontage 的头文件
#include "Components/SkeletalMeshComponent.h"  // 获取骨骼网格体

bool AMyCharacter::PlayAttackMontage()
{
    // 第一步：检查蒙太奇资源是否有效
    if (!AttackMontage)
    {
        UE_LOG(LogTemp, Warning, TEXT("PlayAttackMontage：AttackMontage为空！请在蓝图中指定"));
        return false;  // 资源为空，返回失败
    }

    // 第二步：获取动画实例
    // PlayAnimMontage 需要通过 AnimInstance 来播放
    UAnimInstance* AnimInstance = GetMesh()->GetAnimInstance();
    //  GetMesh() 返回角色的骨骼网格体组件
    //  GetAnimInstance() 返回绑定到该组件的动画实例（就是你的UMyAnimInstance）
    if (!AnimInstance)
    {
        UE_LOG(LogTemp, Warning, TEXT("PlayAttackMontage：AnimInstance为空！"));
        return false;
    }

    // 第三步：播放蒙太奇
    // 1.0f = 正常速度   NAME_None = 从头开始播放
    float PlayDuration = AnimInstance->Montage_Play(AttackMontage, 1.0f);
    //                                                  资源      播放速度
    // Montage_Play 返回蒙太奇的总时长（秒），可以用于设置冷却等

    if (PlayDuration > 0.0f)
    {
        UE_LOG(LogTemp, Log, TEXT("攻击蒙太奇开始播放，总时长=%.2f秒"), PlayDuration);
        return true;  // 播放成功
    }

    // 播放失败（例如角色已经在播放一段不可打断的蒙太奇）
    return false;
}

bool AMyCharacter::PlayHitMontage(FVector HitDirection)
{
    // 受击蒙太奇的播放逻辑类似攻击蒙太奇
    if (!HitMontage)
    {
        UE_LOG(LogTemp, Warning, TEXT("PlayHitMontage：HitMontage为空！"));
        return false;
    }

    UAnimInstance* AnimInstance = GetMesh()->GetAnimInstance();
    if (!AnimInstance) return false;

    float PlayDuration = AnimInstance->Montage_Play(HitMontage, 1.0f);
    return PlayDuration > 0.0f;
}

void AMyCharacter::StopCurrentMontage()
{
    UAnimInstance* AnimInstance = GetMesh()->GetAnimInstance();
    if (!AnimInstance) return;

    // 停止当前正在播放的所有蒙太奇
    // 0.25f 是混合退出时间（秒）—— 动画平滑过渡到待机
    AnimInstance->Montage_Stop(0.25f);
    //                          ↑ 淡出时间，不要设0（会瞬间切换，很突兀）
}

void AMyCharacter::JumpToAttackSection(FName SectionName)
{
    UAnimInstance* AnimInstance = GetMesh()->GetAnimInstance();
    if (!AnimInstance) return;

    // 跳转到蒙太奇的指定 Section
    // 前提：当前正在播放的蒙太奇中存在这个 Section
    AnimInstance->Montage_JumpToSection(SectionName, AttackMontage);
    //                                    ↑ Section名称  ↑ 在哪个蒙太奇中
}
```

## 蒙太奇的分段（Section）

### 什么是Section

Section是蒙太奇内部的时间段标记，让你可以把一段长动画分成几个逻辑段：

```
一个"连击"蒙太奇：

  Section: Attack1    Section: Attack2    Section: Attack3
  [起手 → 挥砍 → 收招][转身 → 横扫 → 回位][上挑 → 斩杀 → 收刀]
  0s             0.8s  0.8s          1.6s  1.6s           2.4s

好处：
- 玩家按攻击键时，可以从 Attack1 → Attack2 → Attack3 无缝衔接
- 如果玩家在 Attack2 收招时不按键，动画就自然结束
```

### 在编辑器中创建Section

```
操作步骤：
1. 打开 AnimMontage 资源
2. 在时间轴上找到你想要的位置（如挥砍动作结束点）
3. 右键 → "Add Montage Section" → 命名（如 "Attack1"）
4. 重复添加 Attack2、Attack3 等
5. Section会自动标记在时间轴上
```

### 在C++中控制Section跳转

```cpp
// 连击系统示例
void AMyCharacter::OnAttackInput()
{
    // 玩家按下攻击键

    UAnimInstance* AnimInstance = GetMesh()->GetAnimInstance();
    if (!AnimInstance) return;

    // 检查当前是否正在播放攻击蒙太奇
    if (AnimInstance->Montage_IsPlaying(AttackMontage))
    {
        // 正在播放中 → 尝试触发下一段连击
        // 获取当前正在播放的Section名称
        FName CurrentSection = AnimInstance->Montage_GetCurrentSection(AttackMontage);

        if (CurrentSection == FName("Attack1"))
        {
            // 从第一段跳转到第二段
            AnimInstance->Montage_JumpToSection(FName("Attack2"), AttackMontage);
            UE_LOG(LogTemp, Log, TEXT("连击！Attack1 → Attack2"));
        }
        else if (CurrentSection == FName("Attack2"))
        {
            // 从第二段跳转到第三段
            AnimInstance->Montage_JumpToSection(FName("Attack3"), AttackMontage);
            UE_LOG(LogTemp, Log, TEXT("连击！Attack2 → Attack3"));
        }
        // 如果是Attack3，不再跳转（连击最多3段）
    }
    else
    {
        // 没有在播放 → 从头开始播放第一段
        // 指定从 "Attack1" Section 开始
        AnimInstance->Montage_Play(AttackMontage, 1.0f);
        UE_LOG(LogTemp, Log, TEXT("开始连击：Attack1"));
    }
}
```

## 蒙太奇混合（Montage Blending）

### 为什么需要混合

如果不用混合，攻击动画会硬生生切断行走动画，产生"突然闪跳"。通过混合，可以让上半身播放攻击动画，下半身继续走路。

### Slot节点

```
动画蓝图 AnimGraph 中的典型布局：

  [状态机：下半身移动]  ──▶ [Slot: DefaultSlot]  ──▶ [输出Pose]
                                    │
                          蒙太奇在这里接入
                                    │
                    ┌───────────────┴───────────────┐
                    │  Slot: DefaultSlot            │
                    │  (行走/跑步动画在这里渲染)     │
                    │                               │
                    │  Slot: UpperBody              │
                    │  (攻击动画在这里渲染上半身)    │
                    └───────────────────────────────┘
```

### 蒙太奇中的Slot设置

```
操作步骤（在蒙太奇编辑器中）：
1. 打开 AnimMontage 资源
2. 在左侧 "Asset Details" 面板找到 "Slot" 设置
3. Slot设置决定了这个蒙太奇从动画蓝图的哪个Slot节点接入

常用Slot配置：
- DefaultGroup.DefaultSlot → 覆盖全身（适合翻滚、死亡等全身动画）
- UpperBody → 只覆盖上半身（适合攻击、换弹等）
```

### 混合选项

蒙太奇资源中的混合参数：

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| Blend In Time | 开始播放时的混合时间 | 0.1 ~ 0.2秒 |
| Blend Out Time | 结束时的混合时间 | 0.2 ~ 0.3秒 |
| Blend Option | 混合方式 | Inertialization（惯性混合，UE5推荐） |
| Blend In Curve | 混合曲线（控制混合速度变化） | 可选，默认线性 |

## 动画层（Layered Blend Per Bone）

### 概念

`Layered blend per bone` 是动画蓝图中的一个节点，允许**按骨骼名称分层混合**——某些骨骼播动画A，其余骨骼播动画B。

```
Layered Blend Per Bone 工作原理：

  输入1（基础Pose）    输入2（覆盖Pose）
  [全身走路动画]      [上半身攻击动画]
        │                   │
        └─────────┬─────────┘
                  ▼
        ┌─────────────────────┐
        │ Layered Blend Per   │
        │ Bone 节点            │
        │                     │
        │ 规则：              │
        │ spine_01 及以上骨骼 │ ← 使用 输入2（攻击动画）
        │ spine_01 以下骨骼   │ ← 使用 输入1（走路动画）
        └─────────────────────┘
                  │
                  ▼
           最终混合Pose
        [下半身在走路，上半身在攻击]
```

### 在动画蓝图中设置

```
操作步骤：
1. 在 AnimGraph 中右键 → 搜索 "Layered blend per bone"
2. 连接 Base Pose 输入（一般是移动状态机的输出）
3. 连接 Blend Pose 输入（一般是蒙太奇Slot的输出）
4. 选中节点，在细节面板设置：
   - Branch Filter：添加骨骼名（如 "spine_01"）
   - Blend Depth：从该骨骼起向上多少层骨骼要被替换（-1=全部向上）
   - 例如：Blend Depth = -1 → spine_01 及以上所有骨骼（头、手臂）都用输入2
```

### 典型配置表

| 场景 | 过滤骨骼 | Blend Depth | 效果 |
|------|---------|-------------|------|
| 上半身攻击 | spine_01 | -1 | 上半身播攻击，下半身播移动 |
| 头部注视 | head | -1 | 只有头朝向目标，身体自由 |
| 左手持盾 | clavicle_l | 2 | 左臂和左手播持盾动画 |
| 右手持枪 | clavicle_r | -1 | 右臂播持枪动画 |

## AnimNotify / AnimNotifyState 的选择决策树

```
需要实现的效果
│
├── 在动画某一帧触发一次事件？
│   └── 使用 AnimNotify
│       例如：脚步音效、特效生成、弹壳弹出
│
├── 在动画某一时间段内持续生效？
│   └── 使用 AnimNotifyState
│       例如：伤害检测窗口、无敌帧、蓄力能量累积
│
├── 需要每一帧更新？（如追踪目标位置）
│   └── 使用 AnimNotifyState 并覆写 NotifyTick
│
└── 需要在通知中访问角色数据做复杂计算？
    └── 使用 C++ AnimNotify / AnimNotifyState
        （蓝图Notify也能做，但复杂逻辑在C++中性能更好）
```

## 常见错误

### ❌ 蒙太奇相关

```cpp
// ❌ 错误1：在 Tick 中调用 PlayAnimMontage
void AMyCharacter::Tick(float DeltaTime)
{
    PlayAttackMontage();  // 每帧都在尝试播放！动画永远在第一帧
}

// ❌ 错误2：没有检查蒙太奇是否正在播放就重复播放
void AMyCharacter::Attack()
{
    PlayAnimMontage();  // 按一下攻击键就从头开始播，连击无法衔接
}

// ❌ 错误3：Montage_Stop 的 BlendOut 设成 0
AnimInstance->Montage_Stop(0.0f);  // 瞬间切换，画面闪烁
```

### ✅ 正确做法

```cpp
// ✅ 正确1：在输入回调中播放（只触发一次）
void AMyCharacter::SetupPlayerInputComponent(UInputComponent* InputComp)
{
    InputComp->BindAction("Attack", IE_Pressed, this, &AMyCharacter::OnAttackInput);
}

// ✅ 正确2：先检查是否正在播放，决定是连击还是重新开始
void AMyCharacter::OnAttackInput()
{
    if (AnimInstance->Montage_IsPlaying(AttackMontage))
    {
        JumpToNextComboSection();  // 正在打 → 连击
    }
    else
    {
        PlayAttackMontage();  // 没在打 → 开始第一段
    }
}

// ✅ 正确3：BlendOut 时间至少 0.15 秒
AnimInstance->Montage_Stop(0.25f);  // 平滑淡出
```
