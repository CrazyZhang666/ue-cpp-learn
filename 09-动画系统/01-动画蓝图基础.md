# 9.1 动画蓝图基础

> **目标**：理解动画蓝图（Animation Blueprint）的本质——它是骨骼动画的"大脑"，决定了角色每一帧播放什么动画、怎么混合动画。

---

## 什么是动画蓝图？

想象一下：你已经导入了一个角色的骨骼网格体（Skeletal Mesh），也导入了走、跑、跳、攻击等一堆动画资源。但这些动画**互不相干**——谁来决定"角色走路时播放走路动画，跑步时播放跑步动画"？

这就是动画蓝图的工作。

```
┌──────────────────────────────────────────────────┐
│                  动画蓝图（AnimBP）                │
│                                                  │
│  ┌─────────────┐    ┌──────────────────────┐     │
│  │  事件图表     │    │     动画图表           │     │
│  │  (EventGraph)│───▶│   (AnimGraph)         │     │
│  │             │    │                      │     │
│  │ 每帧更新变量  │    │ 状态机/混合空间        │     │
│  │ Speed=?     │    │ 根据变量选动画          │     │
│  │ bJumping=?  │    │ 输出最终Pose          │     │
│  └─────────────┘    └──────────┬───────────┘     │
│                                ▼                 │
│                         最终骨骼姿势（Pose）       │
└──────────────────────────────────────────────────┘
```

### 核心类比

| 概念                            | 类比         | 说明                           |
| ------------------------------- | ------------ | ------------------------------ |
| 骨骼网格体（Skeletal Mesh）     | 木偶的骨架   | 定义了"有哪些骨头，骨头怎么连" |
| 动画资源（Animation Sequence）  | 一段视频     | 一段预录的动作，如"走3步"      |
| 动画蓝图（Animation Blueprint） | 木偶的操控师 | 决定"放哪段视频、怎么混合"     |
| 最终Pose                        | 屏幕上的画面 | 角色渲染出来的姿态             |

## 动画蓝图的两个核心图表

每一份动画蓝图打开后，你都会看到两个图表：

### 事件图表（EventGraph）

**作用**：计算"动画参数"，每帧调用一次。

```
事件图表每帧做的事情：
1. 获取角色当前的移动速度 → 存入 Speed 变量
2. 判断角色是否在空中 → 存入 bIsInAir 变量
3. 判断角色是否蹲伏 → 存入 bIsCrouching 变量
4. ...更多计算
```

> **关键理解**：事件图表**只计算数值**，不直接播放动画。它像一个传感器系统，把角色的状态变成数字。

### 动画图表（AnimGraph）

**作用**：根据事件图表计算出的参数，**决定播放什么动画**。

```
动画图表做的事情：
1. 读 Speed 变量 → Speed > 600？播跑步动画，否则播走路动画
2. 读 bIsInAir 变量 → true？播跳跃动画覆盖一切
3. 读 bIsCrouching 变量 → true？播蹲伏动画
4. 把选中的动画混合（Blend）输出最终Pose
```

> **关键理解**：动画图表**只读不写**。它从事件图表拿数据，决定动画输出。你永远不应该在动画图表里计算数据。

## 动画蓝图和C++ AnimInstance的关系

很多新手会困惑：有了蓝图为什么还需要C++？

### 继承关系

```
UAnimInstance（C++基类，引擎内置）
    │
    └── UMyAnimInstance（你的C++子类）
            │
            └── ABP_MyCharacter（蓝图，继承自UMyAnimInstance）
```

### 分工原则

| 工作内容            | 在哪里做                 | 原因                 |
| ------------------- | ------------------------ | -------------------- |
| 计算动画参数        | C++（UAnimInstance子类） | 快、可复用、类型安全 |
| 搭建状态机/混合空间 | 蓝图（AnimBP）           | 可视化拖拽，直观     |
| 播放蒙太奇          | C++（Character中）       | 游戏逻辑控制         |
| 动画通知            | C++或蓝图                | 都行，C++性能更好    |
| 调试动画            | 蓝图编辑器               | 实时预览，所见即所得 |

### 为什么用C++计算参数

```cpp
// ✅ 在C++中计算速度——纯粹、高效、类型安全
void UMyAnimInstance::NativeUpdateAnimation(float DeltaSeconds)
{
    Super::NativeUpdateAnimation(DeltaSeconds);

    ACharacter* Owner = Cast<ACharacter>(TryGetPawnOwner());
    if (Owner)
    {
        Speed = Owner->GetVelocity().Size2D();  // 一行搞定
    }
}
```

对比在蓝图里用节点连线的做法：C++只需要一行代码，蓝图中需要5-6个节点连线，而且性能更差。当游戏中有上百个角色时，这个差距会非常明显。

> **原则**：能算的都用C++算好暴露给蓝图，蓝图中只负责"选动画"和"混合动画"。

## State Machine（状态机）的基本概念

### 什么是状态机

状态机是一种**有限状态模型**：角色在任意时刻，只能处于**一个状态**中，播放该状态对应的动画。

```
最简单的移动状态机：

       [开始走路]         [加速到跑步]
   ┌─────────┐           ┌─────────┐
   │  Idle   │──────────▶│  Walk   │──────────▶│  Run   │
   │ (待机)  │◀──────────│ (走路)  │◀──────────│ (跑步)  │
   └─────────┘  [停下]   └─────────┘  [减速]   └─────────┘
        ▲                                            │
        │              [落地]                         │
        │         ┌──────────┐                       │
        └─────────│  Jump    │◀──────────────────────┘
                  │ (跳跃)   │    [按空格]
                  └──────────┘
```

### 状态机的核心要素

| 要素                            | 说明                          | 示例                        |
| ------------------------------- | ----------------------------- | --------------------------- |
| **状态（State）**               | 角色的一个动作                | Idle、Walk、Run、Jump       |
| **转换规则（Transition Rule）** | 什么条件下从A状态切换到B状态  | Speed > 100 → 进入Walk      |
| **入口状态**                    | 状态机启动时进入的第一个状态  | 通常是Idle                  |
| **每层一个状态机**              | 一个AnimGraph可以有多个状态机 | 上半身状态机 + 下半身状态机 |

### 在动画蓝图中创建状态机

```
操作步骤（蓝图编辑器内）：
1. 打开动画蓝图 → 切换到"动画图表"（AnimGraph）标签
2. 右键 → 搜索 "State Machine" → 添加
3. 双击状态机节点进入内部
4. 右键 → "Add State" 添加状态
5. 从一个状态拖线到另一个状态 → 创建转换
6. 点击转换线 → 在右侧细节面板编辑转换条件
```

## 状态转换（Transition Rule）

### 转换规则是什么

转换规则是一个**布尔表达式**。当它为 `true` 时，状态机从当前状态切换到目标状态。

```
Idle ──────[转换规则：Speed > 10]──────▶ Walk

意思是：当 Speed 大于 10 时，从待机切换到走路
```

### 转换规则的常见设置

| 设置项                                      | 说明                  | 推荐值            |
| ------------------------------------------- | --------------------- | ----------------- |
| **Duration**                                | 转换在多少秒内完成    | 0.1 ~ 0.2         |
| **Mode**                                    | 转换模式（平滑/立即） | Blend（平滑混合） |
| **Blend Profile**                           | 使用哪个混合配置文件  | 无（默认）        |
| **Automatic Rule Based On Sequence Player** | 自动基于Sequence时长  | 一般不勾选        |

### 转换条件示例

```
场景：Idle → Walk 的转换规则

蓝图节点连线：
  Try Get Pawn Owner → Get Velocity → Vector Length 2D → [>] → [输出布尔]
                                                     ↑
                                                  Speed阈值（如10.0）

更简单的做法（如果用C++计算好了Speed参数）：
  在AnimBP的事件图表中获取 Speed 变量 → [> Speed > 10.0] → [Can Enter Transition]
```

### ❌ 常见错误

```cpp
// ❌ 错误：在转换规则中做复杂计算
// 转换规则应该只做简单的布尔判断，复杂计算放在事件图表

// ❌ 错误：状态转换设置Duration=0
// 会导致动画瞬间切换，看起来"闪跳"

// ❌ 错误：忘记处理"反向转换"
// 只设置了Idle→Walk，没设置Walk→Idle
```

### ✅ 正确做法

```cpp
// ✅ 转换规则保持简单：只判断一个布尔值或比较一个数值
// ✅ Duration一般设置0.1~0.2秒，保证平滑过渡
// ✅ 每个转换都要配对的"反向转换"
// ✅ 动画参数用C++计算好，蓝图只做判断
```

## Blend Space（混合空间）

### 什么是混合空间

当你有走、慢跑、快跑三种动画时，不用创建三个状态和三组转换规则——用一个**混合空间**搞定。

```
Blend Space一维示例（X轴 = 速度）：

    Walk(150)    Jog(400)    Sprint(800)
       │           │           │
    ───┼───────────┼───────────┼───▶ X轴 = Speed
       │           │           │
       ├───────────┼───────────┤
       │  慢走区    │  跑步区    │  ← 自动在相邻动画之间插值

    当Speed=275时：输出 = Walk × 0.5 + Jog × 0.5（中间混合结果）
```

### 混合空间的类型

| 类型           | 维度         | 用途                         |
| -------------- | ------------ | ---------------------------- |
| Blend Space 1D | 一维（单轴） | 速度→走跑混合、血量→受伤程度 |
| Blend Space    | 二维（双轴） | 移动方向+速度 → 八方向移动   |

### 二维混合空间示例

```
          Y轴: Direction（方向）
              90°(右)
              │
    Walk_Right│Sprint_Right
    180°(后)──┼──0°(前)──▶ X轴: Speed
    Walk_Left │Sprint_Left
              │
            -90°(左)

根据 (Speed, Direction) 坐标自动在四个角动画之间混合：
- Speed=300, Direction=45° → 在前右动画和前跑动画之间插值
- Speed=0, Direction=0°   → 直接播放Idle
```

### 在动画蓝图中使用Blend Space

```
操作步骤：
1. 在状态机的某个状态中，右键 → 搜索 "Blend Space Player"
2. 选择你创建的Blend Space资源
3. 连接 Blend Space Player 的 X 和 Y 输入到你的动画参数
4. 动画蓝图会自动根据参数值在动画之间混合
```

### Blend Space的关键参数

| 参数                            | 说明                          |
| ------------------------------- | ----------------------------- |
| **X Axis (Horizontal)**         | 横轴参数（通常绑定Speed）     |
| **Y Axis (Vertical)**           | 纵轴参数（通常绑定Direction） |
| **Interpolation Time**          | 混合过渡时间                  |
| **Target Weight Interpolation** | 目标权重插值速度              |

## 动画蓝图中的关键节点

### 常用动画节点一览

| 节点名称                   | 功能             | 使用场景               |
| -------------------------- | ---------------- | ---------------------- |
| **State Machine**          | 状态机容器       | 管理一套互相排斥的状态 |
| **Blend Space Player**     | 混合空间播放器   | 根据参数在多动画间混合 |
| **Sequence Player**        | 播放单个动画序列 | 播放固定动画（如攻击） |
| **Blend Poses by bool**    | 根据布尔值选Pose | 蹲伏/站立切换          |
| **Blend Poses by int**     | 根据整数选Pose   | 多武器切换             |
| **Layered blend per bone** | 按骨骼分层混合   | 上半身射击+下半身移动  |
| **Slot**                   | 蒙太奇插槽       | 播放蒙太奇动画         |
| **Apply Additive**         | 应用叠加动画     | 呼吸、受伤晃动         |
| **IK Rig**                 | IK绑定           | 脚部贴地、手部持枪     |

## 事件图表 vs 动画图表：深入对比

| 对比维度         | 事件图表（EventGraph）      | 动画图表（AnimGraph） |
| ---------------- | --------------------------- | --------------------- |
| **调用频率**     | 每帧一次（行为类似于Tick）  | 每帧一次              |
| **职责**         | 计算/更新参数               | 根据参数输出Pose      |
| **能修改参数吗** | ✅ 能（写入）               | ❌ 不能（只读）       |
| **能播放动画吗** | ❌ 不能                     | ✅ 能                 |
| **蓝图节点类型** | 普通蓝图节点                | 动画专用节点          |
| **是否有返回值** | 无（更新变量）              | 有（输出Final Pose）  |
| **访问角色数据** | 可以（通过TryGetPawnOwner） | 一般不直接访问        |

### 事件图表的执行流程

```
每帧执行顺序：
1. Event Blueprint Update Animation（引擎自动调用）
2. └── 计算 Speed
3.     └── 计算 Direction
4.         └── 计算 bIsInAir
5.             └── 计算 bIsCrouching
6.                 └── 计算其他自定义参数
                        │
                        ▼ 所有参数就绪
7. 动画图表读取参数 → 输出Pose
```

## 创建动画蓝图的实际步骤

### 步骤1：准备资源

```
你需要：
✅ 一个骨骼网格体（Skeletal Mesh，比如 SK_Mannequin）
✅ 一个骨架（Skeleton，和骨骼网格体绑定）
✅ 若干动画序列（Idle、Walk、Run、Jump等）
```

### 步骤2：在编辑器中创建动画蓝图

```
1. Content Browser 右键 → Animation → Animation Blueprint
2. 选择目标骨架（Target Skeleton）：选 SK_Mannequin_Skeleton
3. 父类（Parent Class）：选 AnimInstance（或你的自定义C++类）
4. 命名：ABP_MyCharacter
5. 双击打开
```

### 步骤3：如果选择了C++父类

```
如果你已经创建了 UMyAnimInstance C++类：
1. 创建动画蓝图时 Parent Class 选 MyAnimInstance
2. 这样动画蓝图就能直接读取C++中 UPROPERTY 暴露的动画参数
3. 事件图表中可以使用这些参数作为输入
```

### ❌ 常见操作错误

```
❌ 选错骨架 → 动画蓝图无法使用该骨架的动画
❌ 在动画图表中计算参数 → 应该放在事件图表
❌ 忘记在C++ UPROPERTY 中加 BlueprintReadOnly → 动画蓝图读不到变量
❌ 状态机没有入口状态 → 角色摆成T-Pose（默认姿势）
```
